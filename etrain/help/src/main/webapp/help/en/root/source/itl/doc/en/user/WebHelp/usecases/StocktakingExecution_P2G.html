Person-To-Goods Stocktaking Execution (Offline)

Description

The Person-To-Goods stocktaking execution process in Offline Mode is performed on the Counting Orders screen. After a worker executes the offline Counting process using the Manual Counting List, the counting results are confirmed on the Report Counting Results screen.

      
Triggers and Actors

This starts when:

  • a Counting Order in a Person-To-Goods Zone is available and released for Offline Counting.
      
Prerequisites
  • At least one Counting Order of counting type "Person-To-Goods Offline" is available.

 INFO 

See "Creation of Stocktaking Order" for Counting Order creation.
     
Required Resources
  • Counting Orders
  • Report Counting Results
  • Report Counting Result.
      
Work Flow

Step

Action

1.

Open the Counting Orders screen and release one Counting Order, where Stocktaking type is Person-To-Goods Offline with use of the button
.

2.

Print the appropriate counting list. Usually the Manual Counting List supports the initial counting operation with use of the function

 INFO 

See the button description of Counting Orders for details.

3.

Perform the counting process using the printed counting list.

4.

Enter the result for the Counting Order Lines:

  1. Click "Report Counting Result" for a whole Counting Order or
  2. for a single Counting Order Line.

4a).

Enter the Counting result by editing a grid for all active, valid Counting Order Lines.
The current user and current time display at the top as default for subsequent confirmation and can be changed. Click OK to apply values to all changed Counting Order Lines.

 INFO 

See Report Counting Results for details.

4b).

Enter a single Counting Order Line result in "active" status by opening the Report Counting Result screen and entering the executor and counted quantity.

5.

After entering valid Counting results, the status of the related Counting Order Lines changes to "complete".

6.

When all Counting Order Lines are confirmed (or cancelled), the Counting Order status proceeds to "to confirm".

7.

Initiate a recount for a Counting Order Line in "complete" status with use of the function
(1).

8.

Confirm a single Counting Order Line  in "complete" status for a Counting Order in "to confirm" status with use of the function
 (2). 
Only possible and required if the number of repeated counts exceeds the configured Maximum Number Of Recounts (configured in Stocktaking Zone Configurations).

 INFO 

Only possible with manually initiated recounts.

9.

Confirm a Counting Order in "to confirm" status with use of the function "Book Counting Order"  in the Counting Orders
.

With this action, all Inventory bookings are performed.
     
Result

The complete counting process for a Counting Order of type "Person-To-Goods Offline" executes successfully. Inventory was adjusted and booked according to the Counting results.
