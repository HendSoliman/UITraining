Manual Creation of a Shipping Order

Description

Shipping Orders are intended to remove inventory out of the warehouse (e.g. shipping to a customer or production) by pick certain amounts of SKU from stored Load Units to Shipping Units. This use case describes the manual creation of a new local Shipping Order  with possible selection criteria Stock Keeping Units or Load Units. The creation is done with use of the screen Orders.

With subsequently (manually or automatically) release of the Shipping Order the system starts the Order Fulfillment process.

      
Triggers and Actors

The use case is started by a local user when

  • a shipping process for one or multiple Inventory Units of specific Stock Keeping Units should be executed
  • a shipping process for one or multiple Load Units should be executed.
      
Prerequisites
  • The desired and required Inventory must be available in the warehouse.
      
Required Resources
  • Orders.
      
Work Flow

Step

Action

1.

Open the screen Orders and execute the button
("Inserts a new Entry") in the upper grid.

2.

Create of a new Shipping Order in the Order Detail Dialog with subsequent confirmation by executing the button "OK".
At least the following inputs must be made (in order of appearance within the tab "Order"):

  • Client (accordant to given selection)
  • Type ("shipping")
  • Priority (value > 0)
  • Configuration Status remains "incomplete"
  • Destination (tab "Handling". The destination must be a location of the usage type "transport point" which is accessible by the selected Load Units or respectively the Pick Containers in case of picking is required.)
  • Delivery Time (tab "Times") (The required time the Order should arrive at the Customers unloading address. Be aware of the Delivery Time must consider the defined "Time Restrictions of an Order" and must ensure that the calculated Latest Staging Time will be in the future.)

 INFO 

In case of any specialties are required refer to the further fields as described in Order Detail Dialog.

3.

Creation of new Order Lines.

The Order Lines could be created with respect to the possible selection criteria Stock Keeping Unit or Load Unit. One Order could have Order Lines of each of these criteria types.

Press the button  ("Inserts a new Entry") in the appropriate frame for the desired selection criterion.

 INFO 

See also Order Line Detail Dialog.

4.

Create of a new Shipping Order Line with subsequent confirmation by executing the button "OK".

 INFO 

Repeat this step for each desired criterion (Order Line).
4a. Selection Criterion "Inventory":

All Inventory Units will be selected for shipping process that matches the selected criteria regarding SKU attributes.

At least the following inputs must be made:

  • ID (free identification string that must be unique within the Order, recommended is a unique running number)
  • SKU (A Chooser dialog "SKU" is displayed with use of the chooser button in the SKU field.
    The displayed filter could be used for restriction of the desired selection. Select only one desired entry and confirm with button "OK".)
  • Quantity Sales (The required quantity for this new Order Line)
  • Quantity Unit Sales (The accordant quantity unit for this new Order Line is initialized by default)

 INFO 

In case of any specialties are required refer to the further fields as described in Order Line Detail Dialog.

4b. Selection Criterion  Load Unit:

All Inventory Units will be selected for shipping process that resides on selected Load Units.

At least the following inputs must be made:

  • ID (free identification string that must be unique within the Order, recommended is a unique running number)
  • Load Unit (A Chooser dialog "Load Unit" is displayed with use of the chooser button in the Load Unit field.
    The displayed filter could be used for restriction of the desired selection. Select only one desired entry and confirm with button "OK".)

 INFO 

In case of any specialties are required refer to the further fields as described in Order Line Detail Dialog.

5. Select the new created Order in the upper grid by execution of the appropriate filter and press the button ("Edit selected Entry").
6. For finalization of Shipping Order change the
  • Configuration Status to "complete".
7. The Shipping Order is available for manual or automatic release. The release mode is configured in the Task Types for the related Task Type.
     
Result

A Shipping Order for desired selection criteria Stock Keeping Units or Load Units is created and available for release. With release of this Order all relevant Inventory Units according to the given selection criteria will be identified by the system and considered for the subsequent order fulfillment process.
