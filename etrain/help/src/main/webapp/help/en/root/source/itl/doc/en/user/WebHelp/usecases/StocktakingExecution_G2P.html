Goods-To-Person Stocktaking Execution

Description

For performing a Goods-To-Person stocktaking execution process, the Counting Operation screen is used at a defined Workstation. This describes how a valid Counting Order within a Goods-To-Person Stocktaking Zone is processed on this screen.

 INFO 

 If no Workstation is defined for the desired terminal, the screen informs about a missing Workstation configuration and opens in standby mode (no actions are possible in this state).
     
Triggers and Actors

This starts when:

  • an available, released Counting Order within a Goods-To-Person Stocktaking Zone should be executed
  • the Counting screen is opened at a valid Workstation.
      
Prerequisites
  • at least one Counting Order of counting type "Goods-To-Person" is available
  • at least one Workstation is configured to fulfill counting operations within the desired Stocktaking Zone
  • at least one Counting Order is assigned to the Workstation.

 INFO 

For Goods-To-Person Counting the Workstation Workload Management for assigning the Counting Order to single Workstations is used (see Workstation Queue Entries).
This is either done by the "FeedWorkstationDemon" (see Cron Jobs screen) which must be up and running or based on events during counting at the Workstation or when the Workstation is activated.

 INFO 

It is not possible to execute Stocktaking operations at an active Pickstation.

 INFO 

See "Creation of Stocktaking Order" for Counting Order creation.
     
Required Resources
  • Cron Jobs
  • Workstation Zones
  • Counting Orders
  • Counting Operation.
      
Work Flow

Step

Action

1.

Open the Counting Orders screen and release the desired number of Counting Orders, where Stocktaking type is Goods-to-Person with use of the button
.

 INFO 

Alternatively the assigned Task type for Counting Orders is configured with a release mode "automatically".

2.

Open the Counting Operation screen at the configured Workstation. All Load Units of the activated Counting Order are transported to the related Location of the Workstation.

 INFO 

If no Counting Order is currently available the following message appears and then the Standby screen displays.

3.

The Scan Load Unit screen starts. All requested Source Load Units for Counting display in a list on the left.

4.

Scan the first Load Unit. The Count/Recount screen displays.

 INFO 

If scan of the SKU is required (Stocktaking Zone configuration "SKU Scan Required") the Scan SKU screen appears before.

5.

Enter the counted quantity in the selected field. Press Enter or click "OK".

If a quantity difference is detected, the worker is requested to recount the selected Insert of the scanned Load Unit

 INFO 

See Stocktaking Zone Configurations screen for number of recounts and recount tolerance.

6.

After all Inserts of a Load Unit are counted successfully, the Push Load Unit screen displays.

7.

When all Counting Order Lines are confirmed (or cancelled), the Counting Order status proceeds to "to confirm".
Confirm these Counting Orders in the Counting Orders screen using the button "Book Counting Order"
.

With this action, all Inventory bookings are performed.
     
Result

The complete Counting process for a Counting Order of type "Goods-To-Person" executes successfully. Inventory was adjusted and booked according to the Counting results.
