Velocity Class Analysis

Description

Velocity Class Analysis occurs for configured velocity classes (Inventory turnover ratio) of the SKUs. For each velocity class, the minimum number of accesses required is defined to reach the appropriate velocity class. Single access is defined by one, singular shipment booking of each Inventory Unit.
Velocity Class Analysis can consider up to 3 time periods with its own weightage for calculation of velocity classes, i.e. the Inventory turnover ratio during one time period is considered with assigned weightage for calculation of the overall Inventory turnover ratio. The result represents the basis for calculation of the velocity classes.
Analysis is time-controlled by batch program execution.
Changing the SKUs' velocity classes based on the current Velocity Class Analysis must be initiated separately by execution of a different batch program. The affected changes are logged for a later manual check.

      
Triggers and Actors

This is started when:

  • Velocity Class Analysis should be activated
  • Recalculation of the velocity classes should be executed.
      
Prerequisites
  • Base configuration for the calculation time periods of the Velocity Class Analysis must be done
  • Configuration of the appropriate number of accesses for velocity class definition must be done.
      
Required Resources
  • Velocity Class Analysis Configuration
  • Basic Velocity Class Configuration
  • Velocity Class Analysis Journal.
      
Work Flow

Step

Action

1.

Define considered time periods and weightages:

Configure the considered time periods and assigned weightages on the Basic Velocity Class Configuration. Here, entries can be changed and stored due to the desired configuration.

2.

Define limits for the appropriate number of accesses:

Configure the number of accesses of the velocity classes on the Velocity Class Analysis Configuration. Here, the limits of the desired number of accesses for the available velocity classes can be configured.

3.

Configure a batch program for Velocity Class Analysis:

Open Cron Jobs screen to activate the appropriate batch "AggregateSKUAccessesBatch".

A daily run after end of operational work is recommended.

4.

Check Velocity Class Analysis results:

Check the current Velocity Class Analysis results on the Velocity Class Analysis Journal.

The number of accesses and the overall access quantity per analysis day can be checked by filtering a specific SKU ID.

5.

Configure a batch program to change SKU velocity classes:

To assign the Velocity Class Analysis results to adapt the SKU velocity classes, a special batch must be activated: "VelocityClassAnalysisBatch".

A manual start is recommended, which can be initiated in desired (sporadic) intervals.

 INFO 

Changes to SKU velocity classes cannot be revoked.

 INFO 

Logging changes occurs only with a standard audit trail of appropriate SKU attributes. The audit trail must be separately configured.
     
Result
  • Complete configuration of the Velocity Class Analysis
  • Periodically executed Velocity Class Analysis
  • Calculation of the SKUs velocity classes and storage of the results.
