Receiving of Full Load Units failed

Symptom

With use of the Full Load Unit Receiving dialog an error message rises after scanning a Load Unit.

      
Brief Description

With the scan of a Load Unit in order to confirm that a new Load Unit has arrived and can be stored subsequently, LMS has to do some checks that may lead to error messages. 

      
Possible Reasons

The reason for these error messages may be

  • missing configuration in LMS
  • Unavailable advice data for the scanned Load Unit
  • insufficient state of the Load Unit.
      
Trouble Shooting

Determine what exactly happened

Step

Action

1

Find the error message that has been displayed list of reasons below.

 

Reason

Description

1

Message: "There is no location configured for your terminal ... " There has to be a LMS workstation configured for your computer and this workstation has to be configured for your receiving location. This is missing.

2

Message: "There is no Advice data available for the entered Load Unit". The Advice is not yet known to the LMS.

3

Message: "Load Units cannot be created for incompletely configured Advices!" This means the Advice is not in configuration status "complete".

4

Message: "The task of the Advice is not released!"

5

Message: "The Advice Line Load Unit does not define a Load Unit Type and the Load Unit Type of receiving location has no assigned Load Unit Type".

 

Reason 1: No workstation configured configured incorrectly

Step

Action

1

Open the "Dialog Workstation Administration"

2

Insert a new workstation for your computer (terminal id has to be the hostname of your computer).

 

Reason 2: No Advice data available

Step

Action

1

When the Advice enters the LMS receiving of the Load Unit could proceed. Continue with other Load Units.

 

Reason 3: Advice not complete

Step

Action

1

Open the Dialog Advice Administration.

2

Check the complete specification and data of the Advice, Advice Lines and Advice Line Load Units and set the configuration status to "complete".

 

Reason 4: Task not released

Step

Action

1

Open the Dialog Task Administration.

2

Filter with the reference "Advice ID" the assigned Task and release it.

 

Reason 5: Load Unit Type not configured

Step

Action

1

Determine the Location Type of the assigned Workstation Location with use of the Dialog Location Administration.

2

Open the Dialog Location Type Administration.

  • Check the Load Unit Types that are assigned to the Location Type.
  • If there is no Load Unit Type assigned, add an assignment for the appropriate Load Unit Type.
  • If more than one Load Unit Types are assigned to the Location Type, the Load Unit Type has to be defined in the Advice.
