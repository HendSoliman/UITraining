Detail Dialog Pick Walk

Purpose

This screen opens when viewing a Pick Walk.

      
Screen

      
Fields

Field

Description

ID

Unique ID of the Pick Walk.

Zone

Pick Zone of the Pick Walk.
Status

Current execution state of the Pick Walk:

  • released (Pick Walk automatically time controlled or manually released for execution)
  • active (Pick Walk execution has started)
  • complete (All Pick Missions of the Pick Walk are complete)
  • cancelled (Pick Walk was cancelled without the defined execution)
  • finished (Pick Missions and all required tasks for the Pick Walk are finished).
Offline Picking

If enabled, Pick Walk was started through Pick Walk Screen and thus not processed with any online picking process.
Terminal ID Unique ID of the Terminal that the Pick Walk is assigned to.

Handling
Urgency

Set on behalf the Pick Order. Defines the processing sequence. Lower value indicates a previous execution.
Service Qualifier

Set on behalf of the Pick Order to request a specific processing quality. May be project specific. Only Pickstations with the configured Service Qualifier can execute these Pick Missions.

  • default (No specific handling deviation is required)
  • rush-order (Rush Orders can be clustered in a specific way for rapid processing)
  • repacking (Repacking Orders are picked in a specific way)
  • block (Picks of different Orders are picked into the same Pick Container. Picks of multiple Pick Orders are clustered to a Pick Mission. This Service overrides the original one which could be launched through the Pick Order anyway).

Times
Start Time

Actual start time of the Pick Walk.
End Time

Actual end time of the Pick Walk.
