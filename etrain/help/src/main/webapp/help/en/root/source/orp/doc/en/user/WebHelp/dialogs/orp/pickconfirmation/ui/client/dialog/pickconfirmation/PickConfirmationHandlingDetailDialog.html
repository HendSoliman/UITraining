Screen |  Fields

Detail Dialog Pick Confirmation Handling

Purpose

This screen opens when creating, viewing, or modifying a Pick Confirmation Handling configuration.

 INFO 

For configuration purposes, see the Warehouse Administration Guide.
     
Screen

      
Fields

Field

Description

Pick Controller Zone

Pick Controller for which this configuration entry is valid. (see Detail Dialog Zone).

Error Code

Error codes for which this configuration entry is valid:

  • OK (Pick Order Line successfully completed)
  • missing quantity (Pick Order Line not fully picked due to Inventory shortage)
  • load unit missing (Pick Order Line not fully picked because no Load Unit found)
  • wrong SKU (Pick Order Line not fully picked because wrong SKU found)
  • wrong load unit (Pick Order Line not fully picked because wrong Load Unit found)
  • wrong best before date (Pick Order Line not fully picked because wrong best before date found on Inventory)
  • wrong packaging level (Pick Order Line not fully picked due to wrong packaging level)
  • breakage (Pick Order Line not fully picked because broken Inventory found)
  • blocked (Pick Order Line not fully picked because Location to be picked from is blocked and inaccessible)
  • underdelivery (Pick Order Line not fully picked because explicitly picking of less than target quantity)
  • overdelivery (Pick Order Line not fully picked because explicitly picking of more than target quantity)
  • split (Pick Order Line was split because Pick Container was full or required use of different Quantity Unit)

Logistic Error Code

Set within the Load Unit. Specifies errors related to logistic aspects of a Load Unit. All values are defined in the appropriate field descriptions of the Detail Dialog Load Unit. Picking related values are:

  • wrong inventory (wrong Inventory on the Load Unit which must be clarified (at a Clarification Station))
  • wrong load unit (Load Unit was not expected during Picking process and must be clarified (at Clarification Station))
  • picking clarification (Load Unit was sent to a Clarification Station by any Picking process)

Logistic Error Reason

This configured logistic error reason is set within the Load Unit and provides a free text to explain the reason of the logistic error for the involved Load Unit.

Movement Type

For Inventory Booking Operation of the Pick Confirmation. Values are configured in the screen Inventory Movement Types.

Movement Reason

For Inventory Booking Operation of the Pick Confirmation. Values are configured in the screen Inventory Movement Reasons.

Alarm Type

If set, the related alarm is raised during this Pick Confirmation.

Class Name/Handling Plug-in

Class that implements error handling; invoked in addition to the handling already defined by this Pick Confirmation Configuration.

Configuration/Handling Plug-in

The configuration related to the Handling Plug-in Class.

Inventory Correction required

If enabled, inventory on the Source Load Unit is corrected to a zero quantity.

Lock required

If enabled, the Source Location of the related Pick Order Line is locked. In case of Goods-to-Person picking the Source Load Unit will be locked instead of the Location.

Replacement Reservation required

If enabled, a replacement reservation is initiated for the missed quantity of the related Pick execution. If disabled, no replacement reservation is initiated and the missing quantity is cancelled.
