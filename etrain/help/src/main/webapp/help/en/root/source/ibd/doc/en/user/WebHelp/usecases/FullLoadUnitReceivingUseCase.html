Full Load Unit Receiving Use Case

Brief Description

Full Load Unit Receiving, in automated or manual Areas, provides all functionality for receiving fully advised Load Units and initiating concluding inbound transports.

      
Triggers and Actors

This is started when:

  • delivered goods are available for goods receipt and
  • delivery receipts are available and the receiving process of the delivered goods on full and advised Load Units should be started.
      
Prerequisites
  • Advice must be successfully transmitted from Host to Local system. It is available in "released" or "active" status for processing and can be checked in Advices screen.
  • Load Unit should be received as a Full Load Unit and is advised properly.
      
Required Resources
  • Full Load Unit Receiving.
      
Work Flow

Step

Description

1.

Start the Full Load Unit Receiving screen.

2a.

Scan (Enter) a valid barcode

A barcode includes, without limitation, load unit information. It is scanned and Advice Line data fills the appropriate fields.

 INFO 

 It's possible to receive a Load Unit Stack. If the Load Unit equals a Load Unit Stack, the Base Load Unit must be scanned (entered).

2b.

Enter (Scan) a valid Load Unit ID

The operator enters the Load Unit ID in the "Load Unit" field by entering the load unit code on the load unit label.

 INFO 

 It's possible to receive a Load Unit Stack. If the Load Unit equals a Load Unit Stack, the Base Load Unit must be entered (scanned).

3.

The screen displays the most important information about load unit content according to the given Advice.
Load Unit ID, Load Unit Type, Height class and Weight from the load unit also display.

4a.

Operators can change some data: (F4 key: Modify Data)

  • Default Destination
  • Load Unit Type
  • Height Classification
  • Weight.

4b.

Operators can reprint the Advice Line LU Label. (F9 key: Reprint Label)

5a.

Operators can complete the receiving process of the Load Unit related Advice and the Advice status proceeds to "finished". (F2 key: Complete Advice)

5b.

Operators can complete the receiving process of the Load Unit related Advice Line and the Advice Line status proceeds to "complete". (F3 key: Complete Advice Line)

6.

Clarification is required
The load unit is rejected and the system creates a transport request to the clarification location and sends it to the underlying transport system (MFC). This is necessary if any problems with inventory or load unit data occurred and must be clarified at the Clarification Station. (F8 key: To Clarification)

7.

Confirmation
The operator checks the Load Unit content and confirms by pressing "OK" in the status "Complete Receiving".
This creates/updates information about the load unit and its contents (inventory) in the system.
The Advice proceeds to "active" status with receiving of the first pallet.

8.

Putaway Transport
The system creates a transport request and sends it to the underlying transport system (MFC).

9.

Update the Advice Line Load Unit
The system sets the Advice Line Load Unit to "receiving complete" or "complete" status if the global confirmation level is not on "confirmation on storage complete event".

  • If all Advice Line Load Units are set to "complete" and announced inventory is completely received, the Advice Line is set to "complete" if the global parameter for automatic close of Advice Lines is set.
  • If all Advice Line Load Units are set to "receiving complete" and announced inventory is completely received, the Advice Line is set to "receiving complete" if the global parameter for automatic close of Advice Lines is set.

10.

The Transport is executed and finished by the underlying transport system (MFC) and finally confirmed.

11.

DCD sets the Advice to "to confirm" status if all of its Advice Lines are "complete" and initiates the host confirmation process.

 

Exceptional Cases:

  • Load Unit with the identified ID already in the warehouse.
  • Screen shows a corresponding message.
  • Operator can clarify the situation directly or scan another pallet.
  • Operator detects no consistency of the physical Load Unit with the Advice data.
  • Operator can clarify the situation directly or scan another pallet.
     
Result

Successful processing of goods receipt due to one/multiple Advice Lines of one/multiple Advices occurs. The received Load Units with deposited goods have an Inbound Transport Request and are in transit to their Storage Locations. After arrival and Advice finalization, the Inventory is available for disposition processes.
