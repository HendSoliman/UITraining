Overview

Supervised Zone

The Replenishment Supervised Zone in that this Replenishment Threshold has to be checked.
Active Only in case of enabled this Threshold Configuration entry will be considered. Could be temporarily disabled with use of this flag.
Level

The Level specifies how to fulfill the Threshold based Replenishment. Possible values are:

  • Aggregated (The replenishment calculation considers the inventory of the complete Replenishment Supervised Zone).
  • Dedicated Pick Face (The replenishment calculation considers inventory on the Dedicated Pick Faces of the SKU in the Replenishment Supervised Zone. The quantity definition of the Replenishment Threshold are related to one Dedicated Pick Face).

 INFO 

The level is not used for the Replenishment Type "lookahead".

 INFO 

If a Dedicated Pick Face is supplied from the same Dispo Zone, the Dedicated Pick Face is belonging to, the Replenishment process will create directly Transport Requests instead of Replenishment Orders!

Replenishment Type

The replenishment type defines the kind of replenishment to be executed. Possible values are:

  • fix quantity (Replenishment of a fix quantity (the delivery quantity) when the total quantity (inventory) in the supervised zone or the dedicated pick face falls below the threshold).
  • fix quantity effective (Replenishment of a fix quantity (the delivery quantity) when the free quantity (inventory) in the supervised zone or the dedicated pick face falls below the threshold).
  • total quantity (Replenishment to a final total quantity (quantity total) when the total quantity (inventory) in the supervised zone or the dedicated pick face falls below the threshold).
  • total number of load units (Replenishment to a total number of Load Units (number of Load Units total) when the total number of Load Units (of the SKU) in the supervised zone or the dedicated pick face falls below the Number of Load Units Threshold. This Replenishment Type is only supported, when no repacking or depalletizing is involved in the replenishment order fulfillment.).
  • lookahead (The replenishment demand is calculated according to the comparison of the available quantity in the Replenishment Supervised Zone and estimated future demand in the Replenishment Supervised Zone by Orders with an estimated start time in the look ahead period. The replenishment calculation for this replenishment type is done on Disposition Zone level and must be configured per Replenishment Supervised Zone).

SKU

Client

The Client is the owner of the Inventory of this Stock Keeping Unit (SKU).
SKU The Stock Keeping Unit (SKU) that is configured by this Replenishment Threshold.

Description

A textual description of the SKU.

Quantity Based Replenishment

Quantity Threshold

When the replenishment type is fix quantity, fix quantity effective or total quantity and when the quantity (or the free quantity) in the replenishment supervised zone (or on a dedicated pick face) falls below this threshold replenishment is triggered.

Quantity Delivery

When the replenishment type is fix quantity or fix quantity effective and when a replenishment demand has been determined a Replenishment Order with a quantity of quantity delivery is created.

Quantity Total Inventory in the replenishment supervised zone or in a dedicated pick face is filled up to quantity total, if the replenishment type is total quantity. Inventory is filled up at least to quantity total, if lookahead replenishment is configured for this replenishment supervised zone.
Base Quantity Unit The base quantity unit of the SKU.
Minimum Replenishment Quantity When the calculated quantity to be replenished is lower than the minimum Replenishment Quantity no Replenishment Order will be created.
Maximum Replenishment Quantity When the calculated quantity to be replenished is higher than the Maximum Replenishment Quantity, a Replenishment Order with the Maximum Replenishment Quantity is created. When the Maximum Replenishment Quantity is not defined an unlimited replenishment quantity is allowed.

Load Unit Based Replenishment

Number of  Load Units Threshold

When the replenishment type is total number of load units and when the number of load unit with this SKU in the replenishment supervised zone (or on a dedicated pick face) falls below this threshold, replenishment is triggered.

Number of Load Units Total

When the replenishment type is total number of load units and when replenishment demand has been determined the number of load units of the SKU in the replenishment supervised zone (or the dedicated pick face) is filled up to the number of load units total.

Advised Inventory Lookahead Advised inventory is considered, when this field is defined. The Advised Inventory Lookahead specifies the number of hours that the expectation date of advised inventory may lie in the future to be still considered by replenishment. Advised inventory without expectation date is considered as well. When the advice or advice line has a default destination or the SKU has a preferred destination there must be a bin in the supervised zone that is covered by the default destination or the preferred destination.
Replenishment Running This flag indicates a currently running Replenishment Order.
Generated This flag indicates whether the threshold was automatically generated by the system based on definitions in the replenishment supervised zone and the SKU load unit capacity.

Handling - Order Template

Priority

Relative urgency of the created Replenishment Orders. Usually an integer value between 1- 99. Lower the value, the higher the urgency. Default priority for Replenishment Orders is 7.

Service Qualifier

The Service Qualifier of the created Replenishment Orders describes how the Replenishment Order must be fulfilled. May be project-specific.

Preferred Load Unit Type

The Preferred Load Unit Type of the created Replenishment Orders. Replenishment is trying to deliver the Inventory preferably in this Load Unit Type. Other Load Unit Types are considered only if this Load Unit Type is not available.

Earliest Offset (min)

The Earliest Offset (min) of the created Replenishment Orders defines the delay (in minutes) the Replenishment Order is started at the earliest.

Latest Offset (min)

The Latest Offset (min) of the created Replenishment Orders defines the delay (in minutes) the Replenishment Order is started at the latest.

Overdelivery Tolerance

The Overdelivery Tolerance of the created Replenishment Order Lines. Factor, in percent, to which overdelivery is allowed.

Underdelivery Tolerance

The Underdelivery Tolerance of the created Replenishment Order Lines. Factor, in percent, to which underdelivery is allowed. If this limit will be undershot, the related Replenishment Order Line is cancelled.

Unlimited Overdelivery Allowed

This flag defines if unlimited overdelivery is allowed in created Replenishment Orders.
Minimum Rest Quantity The Minimum Rest Quantity of the created Replenishment Order Lines. If less than the Minimum Rest Quantity would remain free after the reservation on a source inventory unit the complete free quantity is reserved within the boundaries of over delivery and under delivery tolerance.

Inventory Rounding Method

The Inventory Rounding Method of the created Replenishment Order Lines. How Inventory Units will be rounded during reservation:

  • none (no special rounding method)
  • at least full inventory unit (if the quantity to be reserved is completely available on the first matching Inventory Unit, the complete free quantity on the Inventory Unit is reserved. If this would exceed the over delivery tolerance, the maximum quantity within the overdelivery tolerance is reserved.)
  • complete packaging unit (only complete packaging units are reserved within the boundaries of over delivery and under delivery tolerance. The quantity to be reserved on Inventory Units with a packaging level is rounded up to an integer quantity related to the packaging level. The quantity to be reserved on Inventory Units without a packaging level is rounded up to an integer quantity related to the Base Quantity Unit of the SKU. If the rounded up quantity exceeds over delivery tolerance, there is no rounding.)

Are Whole LUs Requested

If this flag of the created Replenishment Order Lines is enabled, the whole Load Unit is delivered regardless of the quantity to be delivered but with respect to the configured Overdelivery Tolerance.
