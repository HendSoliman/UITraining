Complete Receiving

Purpose

This state provides the functions to:  complete receiving (OK key), cancel this state (ESCAPE key), complete the Advice (F2 key), complete the Advice Line (F3 key), Modify Data (F4 key), send the Load Unit to Clarification State (F8 key) and Reprint Advice Line LU Label (F9 key).

      
Screen

      
Fields

Full Load Unit Receiving.

      
Special Buttons

Button

Description

Confirms the Receiving of Load Unit data. Creates the Load Unit and specified Inventory in the system.
If the barcode includes batch and/or best before date, the existing Advice is modified with new data from the scanned barcode.
The Advice Status switches to "active" if not already active.
The system creates a Transport Request and sends it to the underlying transport system (MFC).
The system sets the Advice Line Load Unit State to "receiving complete" or "complete".
If all Advice Line Load Units are set to "complete", the Advice Line is set to "complete" or "receiving complete".
If all Advice Lines are set to "complete", the Advice is also set to "complete". The status "receiving complete" is needed when Load Unit storage triggers Advice data confirmation and when Inventory is not available as long as it is not stored. This process depends on the system parameters described above.

 INFO 

If parameter 'closeAutomatically' is set to false value in config/dcd/ibd-receiving.xml, then following comfirmation dialog will be shown when total quantity of current advice line is received.

The Scan step will be activated.

Confirms Receiving of Load Unit data. Creates the Load Unit and specified Inventory in the system.
If the barcode includes batch and/or best before date, the existing Advice is modified with new data from the scanned barcode.
The Advice Status switches to "active" if not already active.
The system creates a Transport Request and sends it to the underlying transport system (MFC).
The system sets the Advice Line Load Unit State to "receiving complete" or "complete".
The Advice is set to "complete".
Corresponding Advice Lines are set to "complete" or "receiving complete".
If the Advice Line includes further Advice Line Load Units with state "pending", the system doesn't receive the corresponding Advice Line Load Units and sets the state to "complete".

Confirms Receiving of Load Unit data. Creates the Load Unit and specified Inventory in the system.
If the barcode includes batch and/or best before date, the existing Advice is modified with the new data from the scanned barcode.
The Advice status switches to "active" if not already active.
The system creates a Transport Request and sends it to the underlying transport system (MFC).
The system sets the Advice Line load unit state to "receiving complete" or "complete".
The Advice Line is set to "complete" or "receiving complete".
If the Advice Line includes further Advice Line Load Units with state "pending", the system doesn't receive the corresponding Advice Line Load Units and sets the state to "complete".

Releases Load Unit data (displayed in upper frame "Load Unit") for modification. Changeable fields:

  • Load Unit Type
  • Default Destination
  • Height Classification
  • Weight (kg).

Rejects Load Unit. System creates a Transport Request to the Clarification Location and sends it to the underlying transport system (MFC).

Necessary if any problems with Inventory or Load Unit data occurred and must be clarified at the Clarification Station.

Reprints Advice Line LU Label.

Enabled only if the Product Code of the SKU is available.
