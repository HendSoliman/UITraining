Overview

Lookahead Calculation

Purpose

This section describes the replenishment calculation for the Lookahead Replenishment; the evaluation of future demand in a specific Replenishment Destination Disposition Zone based on pending Orders.

      
Overview

Prerequisites:

  • Configured Replenishment Supervised Zone (Disposition Zone where Replenishment is heading to)
  • Area Access entries with
  • the flag replenishment set and
  • with a destination area that matches the area of the destination of the Replenishment Supervised Zone (to define possible sources for replenishment)
  • Replenishment Threshold configurations for single SKUs are optional.

Step

Description

1.

In this calculation, a "number of units" is used to decide if the defined lookahead workload will be reached and is initially set to 0.

2.

Get the next (ordered by the urgency of the Order) SKU Quantity Unit, quantity per unit and quantity information for order lines with the following criteria:

  •  "Preferred Fulfillment Handling type" must be defined for the SKU quantity unit.
  • The quantity unit must be requested by the order line.
  • Destination Disposition Zone must be assigned to a Location with usage type BIN within the Preferred Fulfillment Zone of the SKU.
  • Order configuration status must be complete, and order type must be shipping. With service qualifier Rapid Pal the order status must be released or active. Otherwise the order status must be pending.
  • Latest Start Time or Earliest Start Time must be defined.
  • (Latest Start Time + Earliest Start Time)/2 must be earlier than (now + replenishment lookahead period) configured in the Replenishment Supervised Zone. If one time is not defined, the considered time period must be earlier than the end of the replenishment lookahead period.
When the SKU Quantity Unit is the base quantity unit the quantity per unit is calculated as the maximum factor to base of a SKU Quantity Unit of the SKU with a Preferred Fulfillment Zone whose bins are assigned to the Destination Disposition Zone.

3.

Total the quantity per SKU Quantity Unit in a list.

4.

For each SKU Quantity Unit, the following quantities are considered:

  •  available quantity (quantity free of the Aggregated Inventory of the Disposition Zone) in the Replenishment Destination Disposition Zone (will be subtracted)
  • quantity already ordered by Replenishment to this Disposition Zone (existing open Orders of type replenishment) (will be subtracted)
  • minimum quantity at destination, if a Replenishment Threshold exists for the SKU (will be added)

5.

Add (quantity/quantity per unit) to the "number of units".

6.

If the "number of units" is smaller than the lookahead workload defined in the Replenishment Supervised Zone, continue with step 2.

7.

If a SKU-specific Replenishment Threshold exists, the following conditions must be fulfilled. (Otherwise, each calculated quantity greater zero creates a new Replenishment Order Line)

  • calculated quantity for the SKU must be bigger than Minimum Replenishment Quantity.
  • "sum of quantity free in the Disposition Zone" + "already ordered replenishment to this zone" - "Shipping Orders from that zone" must be smaller than the threshold quantity.

8.

The list of SKU Quantity Units to replenish is sorted by Client and destination, so that only one Order with many Order Lines for every Client and destination is created.
