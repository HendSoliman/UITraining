Shipping General

Description

Overall description of the required prerequisites and steps for using the shipping processes with the basic functionalities only.

      
Triggers and Actors

The use case is started when

  • A basic shipping process needs to be executed with use of the basic functionalities only.
      
Prerequisites
  • Prepared warehouse infrastructure regarding Locations, Marshalling Grids, Docks, Area Access.
  • The following specific configurations must be available
  • For use of the Shipping subsystem the configuration topic "obd-load:LoadManagement.isShippingModuleEnabled" has to be set to true.
  • If Routes are used the configuration topic "obd-route:RouteManagement.routesUsed" has to be set to true.
  • The system has to be configured to use Dispatch Load Units by the configuration topic "orp-orderfulfillment:OrderFulfillmentServiceQualifier.pickToDispatchLoadUnits".
      
Required Resources
  • Routes
  • Orders
  • Shipping Area
  • Marshalling Grids
  • Loads
  • RF-Terminal Loading.
      
Work Flow Using a Route

Step

Description

1.

Amend the Client by a default address and create a Carrier and a Customer including address data.

2.

Create a Route in status 'planned' that should be used for the Shipping Order with the four required time attributes, Dock, Carrier and Truck.

3.

Create a Shipping Order with the following specific settings:

  • The destination must set to the Area of the selected Dock for the loading process (DK01)
  • The (previously created) Route
  • If intermediate storage is required the intermediate destination must be set to an appropriate storing area
  • The (previously created) Customer
  • The Delivery Time.

4.

Create a Load for the Order.
With use the function "Create Load" in the Orders screen the system creates a new Load referencing the Route when configured or assigns an existing and not yet loaded Load of the Route.

5.

Assign the Load to the selected Marshalling Grid with use of the function "Assign Loads to Marshalling Grid" if it has not been done automatically.

6.

Release the Order with use of the related function in the Orders screen.

 INFO 

The Order could only be released in case of the Load of the Order is the currently active within the Marshalling Grid.

7.

Fulfill the required execution steps of the Order.

After picking into the order assigned Dispatch Load Units the systems transports them to the intermediate destination or Marshalling Grid.

8.

When the earliest staging time of the Route is reached the "AuthorizeForStagingDemon" will change the status of the Load to 'authorized for staging'.

9.

Load the Dispatch Load Units with use of the "RF-Terminal Loading" with the final closing of the Load and confirmation of the truck departure.

 INFO 

The "Assembling Status" of the Load changes (in the standard) automatically to "completed" when the Load was fully loaded.
     
Work Flow without a Route

Step

Description

1.

Amend the Client by a default address and create a Carrier and a Customer including address data.

2.

Create a Shipping Order with the following specific settings:

  • The destination must be set to the Area of the selected Dock for the loading process (DK01)
  • If intermediate storage is required the intermediate destination must be set to an appropriate storing area
  • The (previously created) Customer
  • The Delivery Time and Earliest Staging Time.

3.

Create a Load for the Order.
With use the function "Create Load" in the Orders screen the system creates a new Load.

4.

Assign Dock, Truck and Carrier with use of the functions in the Loads screen.

5.

Assign the Load to the selected Marshalling Grid with use of the function "Assign Loads to Marshalling Grid" if it has not been done automatically.

6.

Release the Order with use of the related function in the Orders screen.

 INFO 

The Order could only be released in case of the Load is the currently active within the Marshalling Grid.

7.

Fulfill the required execution steps of the Order.

After picking into the order assigned Dispatch Load Units the systems transports them to the intermediate destination or Marshalling Grid.

8.

If an intermediate destination has been specified the order has to be requested in the Retrieval Administration screen, the system transports the Dispatch Load Units to the Marshalling Grid.

9.

Load the Dispatch Load Units with use of the "RF-Terminal Loading" with the final closing of the Load and confirmation of the truck departure.
     
Result

The result of this use case is the successful processing of Shipping Order and the assigned Route.
The loading was finished, the loaded Dispatch Load Units are 'shipped' and the Load is 'departed'.
