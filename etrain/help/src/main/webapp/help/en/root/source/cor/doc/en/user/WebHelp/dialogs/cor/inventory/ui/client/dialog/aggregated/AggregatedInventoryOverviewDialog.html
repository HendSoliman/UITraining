Aggregated Inventory

Purpose

This dialog shows Aggregated Inventory and assigned Reservations (of Aggregated Inventory only). For term definitions, see Inventory Management - Terms and definitions.

 INFO 

Management of Aggregated Inventory is basically done only for SKU with the usage type 'default'.
In order to increase the booking performance the overall management of Aggregated Inventory could be disabled per Dispo Zone with configuration topic "cor-inventory:AggregatedInventory.aggregatedInventoryManaged".

 INFO 

Setting or removing Inventory holds on Level of single Inventory Units, the whole Inventory of a SKU or of a SKU/Batch combination is done within the Inventory Hold Assignment screen which could be launched from the Inventory Units screen.

 INFO 

The Unit of the displayed dimensions are given by the appropriate Base Quantity Units defined by the Dialog Unit of Measure.

Fields (Aggregated Inventory Reservations)

 INFO 

Only the reservations on the Aggregated Inventory is mentioned here. Concrete reservations on the level of Inventory Units are assigned to the Inventory Units screen.

Field

Description

Client

Client of the Reservation on Aggregated Inventory level.

Quantity

Portion of Aggregated Inventory reserved by this Aggregated Inventory Reservation. Quantity expressed in the Base Quantity Unit of the SKU.

Base Quantity Unit

Base Quantity Unit of the Reservation.

Order

The Order, Order Line and reservation reason reference the object that requested the reservation. Reserved inventory must not be used for another purpose as specified by the Order, Order Line, and reservation reason.

Order Line

The Order, Order Line and reservation reason reference the object that requested the reservation. Reserved inventory must not be used for another purpose as specified by the Order, Order Line, and reservation reason.

Priority

Importance of the reservation. Higher value indicates higher priority. Reservations with lower priority are cancelled earlier than those with a higher priority.

Reservation Reason

Business process type that caused the reservation (ex: Replenishment, Shipping). The Order, Order Line and reservation reason reference the object that requested the reservation. Reserved inventory must not be used for another purpose as specified by the Order, Order Line, and reservation reason.
