Starting SAP connector failed

Symptom

One or more of the connectors failed while starting up.

      
Brief description

The connnector application(s) went to state failed and are shown as red node in the Runtime State Administration dialog.

      
Categories
  • EAI
  • Host
      
Possible reasons

In case of a failed start, the following reasons are possible:

  • The SAP host system is not reachable.
  • The Database connection is not available.
  • The SAP Libraries are not available.
      
Detail description

The Virtual Machine (VM) or one of the connector applications has been started within the Runtime State Administration dialog or on system level.

The SC and the VM are up and in state running, but one of the applications stopped in state failed.

      
Trouble shooting

Determine what exactly happened

Step

Action

1

Look in the Syslog Dialog for the error message.

2

Look in the application trace files for a detailed error description.

3

Lookup the reason below

 

Reason

Description

1

If there is an error which states a class starting with com.sap.* can't be loaded or can't be found.

2

If there is an error which states a specific shared library (DLL under Windows) can't be found

3

If there is an error which states the supplied shared libraries have the wrong version.

4

Ii there is an error which states there is no connection to the database.

5

There are error messages about missing Microsoft Runtime Librarys

(msvcr71.dll, msvcp71.dll, usw.)

 

Reason 1:

Step

Action

1

Ensure the sapjco.jar file from your download from SAP is in the {path_to_the_eai_bim_folder}/lib directory.

2

Restart the connector.

 

Reason 2:

Step

Action

1

Ensure the librarys from your download from SAP may be found by the operating system.

Windows: The librarys must be on the search path of the operating system. This means a directory defined in your PATH variable or the working directory the System Controller is started with. Consult your local system administrator when you have problems here. This can be a tricky thing on Windows depending on your system setup.

Unix/Linux: Put the directory you copied the libraries to in the appropiate environment variable. (LIBPATH (AIX) / SHLIBPATH (HPUX) / LD_LIBRARY_PATH (Solaris/Linux)). Ask your local system administrator for help if needed.

 

Reason 3:

Step

Action

1

There has to be another SAP Client installation on your system. The library from that system is used by the operating system.

Ensure the correct library is found. There may be a conflict. Please ask your local administrator for help here. The search paths need to be set up so the correct libraries corresponding to sapjco.jar you are using are found.

2

Restart the connector.

 

Reason 4

Step

Action

1

Go to {path_to_the_eai_bim_folder}/build.properties.

Check the parameters for the database connection.

2

Restart the connector.

 

Reason 5

Step

Action

1

You are using a version of JCo which is linked to Visual Studio .NET 2003 or greater. Typically there is no other SAP Software >= 6.40 installed.

Please get the missing runtime DLLs from Microsoft.

See SAP Advices 684106 and 549268 for help.

2

Restart the connector.
