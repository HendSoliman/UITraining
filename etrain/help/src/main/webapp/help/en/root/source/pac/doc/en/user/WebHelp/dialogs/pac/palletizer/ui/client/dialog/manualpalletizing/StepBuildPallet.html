Manual Build Step

Purpose

This step supply necessary functionalities to complete the current pallet and bring the system into a stabile state when an error happened. It also can create a new pallet for the Open Packages when a pallet is full.

The dialog in this step can be refreshed automatically when a Load Unit in an active PickOrderLine is sent to the current RapidPick Workstation.

     
Prerequisites
  • Some error happened
  •  The current pallet is full

 

      
Screen

Manual Mode

 

      
Procedure

Step

Description

It books a new package in a PickOrderLiner and retrieve it to the RapidPick Workstation,then put the package into pallet manually

When a pallet is full, after pressing this button, the status of the relative Pick Container is change to picked, a new Transport Order is create.

It also will generate a new Pick Container for the left Active PickOrderLine.

User can switch to Auto Mode by press this button when all errors are solved.

This option is similar to the “Pallet Split” with the only difference that the current pallet is not send to the destination but send to The Default Location of the Workstation Destination Location.

The dialog like below:

The operator can use the slider to switch to another already booked packages on the pallet, e.g. in the sequence backward marking the package blue.
When the operator press the button “Remove Item” the Pick Order Line will be cancelled and the package is shown in black.
After all destroyed items have been logically and physically removed from the pallet, the operator continues palletizing manually until the pallet is completed.
