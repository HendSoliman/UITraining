Workstation

Purpose

This topic describes Workstation configuration.

      
Configuration

A Workstation is a working place for a user, which is provided with a terminal and assigned to a Warehouse Location.
The following Workstation attributes control business logic. They are not predefined by the system and must be configured:

Attribute

Mandatory

Hint

Workstation Type

yes

Control standard business logic and can control project specific logic.

Active

yes

Must be set to active.

Location

no

See user guide screen for details.

Clarification Point

no

See user guide screen for details.

Load Unit Type

no

See user guide screen for details.

Load Unit Grid Orientation

yes

See user guide screen for details.

Queue Limit

yes

See user guide screen for details. The Queue Entries will be created automatically.
Ensure that the Queue Type is aligned with the Workstation Type. 

Request Size

yes

See user guide screen for details.

Request Threshold Counter

yes

See user guide screen for details.

Capability

no

See user guide screen for details.

The following relations must be configured:

Relation

Mandatory

Hint

Terminal

yes

Must be assigned to exactly one workstation. See user guide screen for details.

Locations

no

See user guide screen for details.

Pickstations require at least each type once: source, work (as Put Locations), destination.

Printer

no

All printer that should be used within an operational screen must be assigned to the terminal of the Workstation.

Zone

no

See user guide screen for details.
