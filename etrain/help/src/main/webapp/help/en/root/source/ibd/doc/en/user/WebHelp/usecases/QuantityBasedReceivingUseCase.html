Quantity based Receiving

Brief Description

Quantity-based Receiving, in automated or manual Areas at a Receiving Workstation ordered to given Advices, provides all functionality for deposit of delivered Advice Inventory in single Inserts onto Load Units and initiates the concluding Inbound Transports.

      
Triggers and Actors

This is started when:

  • delivered goods are available for goods receipt and
  • delivery receipts are available and the Receiving process of the delivered goods should be started.
      
Prerequisites
  • Advice was successfully transmitted from host to local system. It is available in "released" or "active" status for processing and can be checked in Advices screen.
      
Required Resources
  • Receiving.
      
Work Flow

Step

Action

1.

Start the Receiving screen.

2a.

Enter (Scan) a valid barcode
The barcode is scanned and Advice Line data, according to this barcode, fills the appropriate fields. If the Advice Line cannot be uniquely determined, the desired Advice Line must be selected from a subsequently displayed list.

2b.

Enter (Scan / Choose) an Advice and Advice Line

Select an Advice and according Advice Line by directly entering an appropriate ID or using the list.

Scan / enter / choose ("F3: Choose Advice" button) an Advice/Advice Line in "released" or "active" status (or a message appears).

3.

Enter a Load Unit
Scan the desired Destination Load Unit where goods are to be deposited. The Load Unit can be unknown to the system or already exist. In later case, it must be located on the Location of the current Workstation.
Click "OK" ("Enter" key or automatically with scan) to display the Inserts possibly located on this Load Unit.

 INFO 

If the Load Unit does not exist in the system the Load Unit Type must be chosen and could be modified later on with use of the button "F10: Choose Load Unit Type".

4.

Select an Insert

Only if the Load Unit has Inserts an Insert for deposition can be selected by "arrow keys" or using a mouse. It will be marked in red. Initially, the first free Insert is also marked in red.

5.

Input received quantity

Change the quantity only if the quantity that should be deposited onto the displayed Load Unit (Insert respectively) should differ from displayed quantity remaining.
The quantity is initialized with the outstanding quantity and can be changed using the "F9: Change Quantity" button.

6.

Check / Change Destination

The desired Destination Area for subsequent Putaway transport of the Load Unit initializes with the accordant value of the Advice of the displayed Advice Line. It can be changed after Load Unit entry regarding reachable destinations of the current Workstation with respect to the Load Unit Type of the selected Load Unit.

If desired, choose another destination from the "Destination" field menu using the "F9: Change Destination" button.

 INFO 

Ensure that the right Destination Area is selected if the default of the Advice Line shall be changed.

7.

Print EAN-Labels

Press the "F5: Print Label" button. Enter the number of Labels to print in the "Label Quantity" field. Change the automatically calculated Inventory quantity per label (if required) and click "OK" to confirm the print request.
Optional step. Can be processed at another time.

8a.

Load Unit ready
Press the "OK" button if the Load Unit should be finalized and is not needed for further Receiving steps. The received and displayed quantity will be booked onto the Load Unit (respectively Insert) and the outstanding Advice Line quantity is decreased accordingly. The Load Unit is finished and receives an inbound Transport Request for Putaway into the displayed Destination Area. Additional receiving steps are not possible for this Load Unit. The screen proceeds to initial state to enter (scan/choose) the next Advice. (step 2)

8b.

Quantity ready / same Advice Line
Press the "F2: Quantity Ready" button if the Load Unit should stay on the dedicated Location for additional receiving operation. The received and displayed quantity will be booked onto the Load Unit (respectively Insert) and the outstanding Advice Line quantity is decreased accordingly.
Screen proceeds with same state with Enter of the next Load Unit (step 3.)

8c.

Quantity ready / further Advice Line
Press the "F2: Quantity Ready" button if the Load Unit should stay on the dedicated Location for additional receiving operation. The received and displayed quantity will be booked onto the Load Unit (respectively Insert) and the outstanding Advice Line quantity is decreased accordingly. Press the "Esc: Cancel" button (key Esc) for processing another Advice.
The screen proceeds in initial state to enter (scan/choose) the next Advice. (step 2)

9.

Restock (additional deposit on yet stored Load Units)

Press the "F6: Restock" button for additional deposit on yet stored Load Units. Select Load Unit(s) from the list. Click "OK". All selected Load Units directly receive an Outbound Transport Request to current Workstation. As soon as they arrive at the Workstation, they are available for additional deposit and you can proceed with step 2.

 INFO 

If the Advice Type is "Return", stored Load Unit selection is restricted to Load Units with the same SKU and batch. The appropriate function is provided by the "F7: Choose Load Unit" button.

10.

Modify Data

Press the "F4: Modify Data" button if changing or completing Advice Line data (displayed in upper frame "Advice Data"). All allowed fields are now released for changing. Press the "F11: Modify Best Before Date" button to change of the best before date. Click "OK" to save all changes. All subsequently created Inventory obtains the changed data. The screen proceeds to the previous state.

 INFO 

This is restricted by special authorization.
     
Result

Successful processing of goods receipt, due to one or multiple Advice Lines of one or multiple Advices, occurs. Received Load Units with deposited goods have an Inbound Transport Request and are in transit to their Storage Locations. After arrival and Advice finalization, the Inventory is available for disposition processes. Alternatively, Load Units reside at the Workstation for further processing.
