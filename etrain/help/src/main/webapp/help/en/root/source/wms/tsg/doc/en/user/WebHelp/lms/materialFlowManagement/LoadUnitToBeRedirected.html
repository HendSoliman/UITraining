Restart of an Inbound Transport

Symptom

Load Units which shall be stored in any desired storage location report at an unusual transport point, e.g. at a Clarification Workstation. In addition the Load Unit may have a suboptimal current planned destination which might be unsuitable from the supervisors point of view. This destination was automatically determined by the system for example in case of the optimal and desired area was temporarily full.

      
Brief Description

This topic explains the clarification handling of Load Units with a currently active inbound transport which did not encounter any "standard transport error" like e.g. "bin occupied". In general this is how to restart the transport (from the Clarification Workstation) and if required how the System will assign a new destination to the Load Unit.

Possible Reasons

When the originally planned destination of a Load Unit is not reachable for various reasons like temporarily blocked aisles or other temporarily faults of the transport system, the Load Unit will be transported automatically to the configured Clarification Station (NIO).

Additionally in case of no free Location can be found in the intended destination area the Load Unit could be directed previously to a different area which is configured as a possible destination area. In this case the Load Unit has an unsuitable planned destination.

      
Detailed Description
  1. The Load Unit is automatically transported to the NIO-Location which is assigned to the transport point where the problem was detected.
  2. In case there is no possible transport path from this location to the desired destination the Load Unit has to be transferred manually to a different location where the inbound transport is executable (for example the related Clarification Station or the next I-Point).
  3. The Load Unit has to be analyzed at the Clarification Station and according to the result the transport has to be continued or restarted.
  4. In case of the currently planned destination of the Load Unit is not appropriate or desired, the destination selection for the Load Unit can be repeated.
      
Trouble Shooting

The basic idea of this clarification handling is to restart the transport from the current location. After a detailed analysis of the situation it might also be required to let the System assign a new destination to the currently active transport.

Analysis and clarification handling can be done with the following dialogs:

  • Load Unit Guide Dialog
  • Dialog Transports (MFC)
  • Load Units

For a more detailed analysis the following dialogs might be required as well:

  • Dialog Transport Administration
  • Business Journal (Transports).

Step

Action

1.

Transport the Load Unit manually (i.e. without system support) to the next Clarification Station (in case the Load Unit is not already at a Clarification Station).

2.

Scan Load Unit in the Load Unit Guide Dialog and analyze the displayed data (refer to the "Problem Analysis").

3.

 In case of the displayed Location of the Load Unit is not equal to the actual current Location of the Load Unit:
Change the current Location of the Load Unit to the actual Location, e.g. Clarification Station or I-Point (Function "F6").

4.

The destination of the Transport Request is suitable and shall not be changed.

5.

Clarify the Load Unit (i.e. all transport errors will be removed) in case of this function is enabled (Function "F2").

6.

Restart the Transport (if the Transport is not continued automatically) (Function "F3").

Alternate Flows:

4a1.

The destination of the Transport Request is not suitable.

4a2.

Cancel the Transport in the Load Unit Guide Dialog. (Function "F4").

4a3.

 Refresh the data in the Load Unit Guide Dialog (Function "F5").

4a4.

In case of the cancellation of the Transport Request was successful continue with step 5.

In case of the cancellation of the Transport Request was not successful continue with step 5a.

 

5a.

Check if a Transport Order exists in Dialog Transports (MFC).

In case of yes, delete the Transport Order (context menu).

6a.

Check if the Transport Unit exists in Load Units.

In case of yes, delete the Transport Unit.

7a.

Check in the appropriate PLC screen if there is an active transport for this Transport Unit.

In case of yes, delete this PLC transport.

Continue with step 4a2.
