Unloading with RF-Terminal

Description

The Receipt based Receiving in automated or manual Areas ordered to given Shipments provides all required functionalities for deposit the delivered Inventory of a Delivery in single Inserts onto Load Units and initiate the concluding Inbound Transports using a RF-Terminal.

      
Triggers and Actors

The use case is started when

  • a Shipment is available for goods receipt and
  • the Receipts are available and
  • the Receiving process of the delivered goods should be started with support of a RF-Terminal.
      
Prerequisites
  • The Shipment must be confirmed at a Dock
  • It is available in status "released" or "active" for processing (which could be checked in the Dialog Shipment Administration)
  • The Load Unit respectively the product is labeled with an appropriate GS1-Label.
      
Required Resources
  • RF-Terminal Unloading.
      
Work Flow

Step

Description

1.

Open the dialog for RF-Terminal Unloading from the main menu.

2.

The dialog proceeds to step "Scan Dock.

3.

Scan the Dock.
Key "Enter" is activated implicitly with scan.

3a.

In case of an invalid Dock is scanned, an error message appears. Proceed with step 3.

4.

The dialog proceeds to step Scan Shipment.

5.

Scan the Shipment.
Key "Enter" is activated implicitly with scan.

5a.

In case of an invalid Shipment is scanned an error message appears. Proceed with step 4.

5b.

In case the current Dock do not match the Dock of the Shipment, use the function "F2 : This Dock" to shift the Shipment to the current Dock.

6.

The dialog proceeds to step Scan Load Unit.

7.

Scan the appropriate barcode of the Load Unit where the Inventory to be received is deposited.

8.

The dialog proceeds to step Check LU Type.

9.

Check whether the Load Unit Type matches the advised type and confirm with "Enter: OK".

9a.

If the Load Unit Type is not correct press button "F2: Change LU Type", select the correct Load Unit Type from the list and confirm with "Enter: OK".
Cancel the selection without any changes with "ESC: Cancel". Proceed with step 9.

10a.

If no Receipt could be uniquely identified the dialog proceeds to step Identify Product.

10a.1

Scan the product (GS1 - GTIN) or enter the SKU, batch and quantity manually to identify a Receipt.

11.

The dialog proceeds to step Check Product.

12.

Enter the quantity with button "F4: Quantity" to receive a different quantity as displayed or just press "Enter" to acknowledge the advised quantity.
If a batch needs to be defined, press button "F3: Batch".

13.

The dialog proceeds with step 6 for processing the next Load Unit.
     
Result

The result of this use case is the successful processing of goods receipt due to one Receipt.
The received Load Units with deposited goods have an Inbound Transport Request and are on the way to their Storage Locations.
After arrival and finalization of the Receipts the inventory is available for disposition processes.
