Load Unit Guide

Purpose

This screen provides different, mainly project-specific, Load Unit information and clarification functions for use at Clarification Stations. It provides specific handlers to deal with the Load Unit and its specific situation; which depends on various predefined criteria.
Every handler defines a specific information data page, a simple help page, and a variable number of related functions.

      
Overview

Frame

Description

Info

Displays related, relevant Load Unit data in HTML format. Any data can display. Display style is variable and free (any HTML format).

Help

Each handler has its own help page, located in the help package of the Load Unit Guide screen. Help page name is similar to handler name.

Load Unit

"Load Unit" field allows scanning any Load Unit in the system. Buttons allow loading, reloading and removing of the suitable handler.
Actions Provides all functions of the specific handler.

Available Handlers

List of available handlers, in the appropriate order:

  1. Quality Assurance Handler
  2. Clarification Handler
  3. Default Handler

 INFO 

Handler order should be from the most specific conditions and handlings to more common use. If no specific handler is applicable, one is provided.
     
Description of Buttons

The specific handlers provide further buttons.

Button / Function

Description

Enter

Validates scanned Load Unit and initiates loading of the suitable handler. If a valid handler is determined, all related data fills in.
If a Load Unit is already loaded and displayed, the Refresh function executes.

 INFO 

If the scanned Load Unit is still in transport, the operator can confirm the running transport:

Click "OK" for the system to send the request to the underlying transport system (MFC). The result displays immediately.

Escape

Removes the loaded handler and steps back to initial state.

F5

Validates scanned Load Unit and initiates reloading of a suitable handler.
