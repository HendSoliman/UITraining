Brief Description Load Unit

Definition Load Unit

The Load Unit (LU) is a central term in a Warehouse Management System.

Goods identified by Client, SKU and other attributes are mainly located on a Load Unit. This Load Unit is transported inside the warehouse and the Storage Location of the Load Unit is evaluated with respect to the properties of the Load Unit and the Inventory on the Load Unit.

     
Managing Load Units

To manage the Location of the LU with respect to the objectives concerned with its Inventory (Storing, Picking, Shipping, etc.) is one main task of the Logistics Manager. The LU provides different properties regarding the transport status, reservation status and overall usage. A detailed description is given in the Field Descriptions of Detail Dialog Load Unit.

A Load Unit has a type which reflects physical properties, which must fit to the allowed Storing Locations or to the allowed transport connections.

As Inventory is usually bound to a Load Unit and the Load Unit is bound to a Location, virtual Load Units exist when the stock is physically bound direct to a Location.

Also, it is possible to stack Load Units. A parent or child Load Unit reflects this relationship (see Inserts / Compartments)

      
Properties of Load Units

Some Load Unit properties are given by the loading device (e.g. euro-pallet, specific tote or tray) represented by the Load Unit Type. Others are given by the Load Unit as a whole (e.g. height, height classification, measured values such as width, length, weight or the scanned orientation). Some properties are given by the Inventory of the Load Unit (e.g. empty, last cycle count).

      
Inserts / Compartments

Load Units which are stacked on Base Load Units in a specifically defined geometrical raster (of small rectangles) on well defined positions are used as Inserts or Compartments. The related Base Load Unit type must provide raster sizes for X and Y greater than 0.
The graphical view of the Inserts is given system wide as follows:

  • white: not yet stacked Compartments on the raster of the Base Load Unit
  • red: Base Load Unit or Compartment is currently selected
  • green: occupied with a not selected empty Compartment which is still removable
  • yellow: occupied with a not empty (and not selected) Compartment (inventory inside!) and thus not removable.
