Person-To-Goods Stocktaking Execution (Online)

Description

Person-To-Goods Stocktaking execution process in Online Mode is performed with support of the RF-Counting screen with direct confirmation of the counting results.

      
Prerequisites
  • at least one Counting Order in a Person-To-Goods Zone is available and released for Online Counting or
  • an already active Counting Order is assigned to the Worker (and the execution continues).

 INFO 

See "Creation of Stocktaking Order" for Counting Order creation.

 INFO 

If the Worker has an assigned workload of any different reference type, counting will not be possible until this task will be finished or released from the worker.
     
Required Resources
  • Counting Orders
  • RF-Counting.
      
Work Flow

Step

Action

1.

Open the Counting Orders screen and release one Counting Order of stocktaking type "Person-To-Goods Online" with use of the function
.

2.

Open the screen for "RF-Counting" from the main menu.

The dialog loads the next released Counting Order of stocktaking type Person-To-Goods Online.
In case a Counting Order for the used terminal that is already in status active is available it will be reloaded for further execution.

In case of no appropriate Counting Order is available an information message is shown.

3.

The dialog starts with "Scan Location".

4.

Scan the Location.
Key "Enter" is activated implicitly with scan.

4a.

In case of an invalid Location is scanned an error message appears. Proceed with step 4.

4b.

In case of the Location is temporarily not accessible use the function "Skip" and
the dialog proceeds with step 3. for the next Location (the last one must be counted anyway).

5.

In case of the Stocktaking Zone configuration requires a check of the number of stored Load Units at the scanned location ("Check #LU at Locations" is enabled) and the subsequent scan of the Load Unit to be counted (defined by "Scan Load Unit Handling") is required
the dialog proceeds to "Check number of Load Unit at Location".

Otherwise proceed with step 6.

5.1

Enter the number of stored Load Units at the current Location.

5.2

The dialog proceeds to "Check number of Load Unit Scan".

5.3

Scan the first Base Load Unit.

 INFO 

If a Load Unit is scanned, which is not booked on this Location, the operator is requested to deliver it to the Clarification Station for further investigation.

5.4

Repeat step 5.3 for each available Base Load Unit on the Location.

6.

In case of the Stocktaking Zone configuration requires a scan of the Load Unit to be counted (defined by "Scan Load Unit Handling")
the dialog proceeds to "Scan Load Unit".
Otherwise proceed with step 7.

6.1

Scan the displayed Base Load Unit.

6.1a

In case of an invalid Load Unit is scanned an error message appears. Proceed with step 6.1.

7.

In case of the Stocktaking Zone configuration requires a scan of the SKU to be counted (defined by "SKU Scan Needed")
the dialog proceeds to "Scan SKU".
Otherwise proceed with step 8.

7.1

Scan the displayed SKU.

7.1a

In case of an invalid SKU is scanned an error message appears. Proceed with step 7.1.

8.

The dialog proceeds to "Counting / Recounting.

9.

Enter the counted quantity of the displayed Stock Keeping Unit and confirm with "OK".

9a.

In case of the Load Unit is missing use the related function for confirmation.
The current position will be completed and the dialog proceeds with the next position.

 INFO 

If "Inventory Correction is not Required" for the related Counting Confirmation Handling entry, the missing Load Unit is not deleted immediately but booked on a fix Location for further investigation.
The ID of the Location must be configured in the configuration topic "cor-inventory:InventoryUnit.lostAndFoundLocation" for each affected Area.

9b.

In case of the displayed Quantity Unit is not appropriate for counting use the related function for changing it.
The dialog changes the displayed Quantity Unit and repeats step 9.

10.

The dialog checks the configured tolerances.
In case of the counting result exceeds the tolerance and the "creation of Recount Counting Order Lines" is allowed a new recount position is created immediately.
The dialog proceeds with step 9 for this new position and allows for the direct recount operation.

11.

Repeat step 9. until no further counting positions of the active Counting Order are displayed .

12.

After the Counting Order is completely executed and all counting results are confirmed the dialog directly proceed with step 2.

13.

After all Counting Order Lines are confirmed (or cancelled), the Counting Order status proceeds to "to confirm".
Confirm the results of these Counting Orders using the "Book Counting Order" button in the Counting Orders
.
     
Result

The counting operation for a Counting Order of type "Person-To-Goods" (Online) is executed and the counting results are reported. The related Inventory Units were adjusted and booked according to the confirmed counting results.
