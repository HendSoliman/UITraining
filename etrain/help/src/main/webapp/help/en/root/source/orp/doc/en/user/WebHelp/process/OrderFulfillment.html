Order Fulfillment

Overview

Within the overall Order Processing the Order Fulfillment supports all kind of execution processes for the fulfillment of orders with coordination of the different Task execution subsystems involved in Order Fulfillment like Picking, Palletizing and Transportation subsystems.

With (manual, automatic or triggered) Order Release the Retrieval Strategies are used to allocate the optimal required Inventory for the given Order Line Criteria which are Inventory, Load Unit and Location Criteria. As a result the related Execution Tasks are created which can be groups of Full Load Unit Retrievals and Pick Orders for subsequent Order Fulfillment.

Full Load Unit Retrieval Management

Full Load Unit Retrievals represents whole Load Units which are retrieved from storage directly to the marshalling destination without Picking. All Full Load Unit Retrievals belonging to one Order are grouped together by a Full Load Unit Retrieval Task. When a Task of this type is being released (possibly independent from order release) all related Full Load Unit Retrievals will be released and Transport Requests will be created. Full Load Unit Retrievals are confirmed when the related transports are confirmed by the underlying transport system.

Picking

Pick Orders represent the parts of an Order which could not be served by whole Load Units and thus need to be picked. Each Pick Order is represented by a specific Task as well. When a Task of this type is being released the Picking request is sent to an underlying Picking subsystem for execution. The picking subsystem creates its own picking directions on the basis of the corresponding Picking requests which are represented by the single Pick Order Lines. The Pick Orders are confirmed when the related Picking requests are confirmed by the Picking subsystem. The Pick Confirmation Handling could be specific for different Picking Zones and thus this could be configured with respect to the Inventory Bookings and further operations which are dependant on the specific Pick Confirmation.

In case the related Order provides an intermediate destination the Pick Container will be temporarily buffered before finally shipped.

In case of any shortages within the system (e.g. due to any locks of resources or transport cancellations) Order Fulfillment initiates a replacement inventory allocation with subsequent specific processing of the new Full Load Unit Retrievals or Pick Order Lines. In case of the underdelivery tolerance of any Order Line will be violated the Order will be configurable cancelled, set to "paused" (with need of later manual cancellation or further release) or simply leave untouched without any action.

 INFO 

Manual palletizing (e.g. at ErgoPal Station) is processed as a Picking task.

 INFO 

The overall basics of the supply of Workstations with empty Load Units are described in the section Empty Container Supply of the Retrieval Overview.

Palletizing

Pick Orders represent as well the parts of an Order which need to be palletized with the specific RapidPal system. The palletizing subsystem uses for palletizing directions the Pick Order Lines where a Pick Order represents one single pallet.

Order Completion

Order Fulfillment notifies Order Management about the completion of an Order when all related Load Units arrived at the given final Destination Location. After the host confirmation Order Fulfillment initiates the deletion of the Order related Inventory.
Optionally the fulfillment can be done by use of Dispatch Load Units which are the Order related Shipping equivalent of the Load Units for support of the Shipping processes and for representing the shipped units within the Orders even if the related Load Units and the Inventory are deleted already.

Order Fulfillment Topics

The main responsibilities are:

  • Global general Lifecycle Management of Pick Orders and Full Load Unit Retrievals
  • Fulfillment of Orders with providing the appropriate specific execution management
  • Coordination of all involved Task Execution Subsystems which are
  • Pick Controller (PIC - for Picking tasks)
  • Pack Controller (PAC - for Palletizing tasks)
  • Material Flow Controller (MFC - for Transportation tasks)
  • Cancellation and Completion of Pick Orders (for Picking and Palletizing as well) and Full Load Unit Retrievals

Retrieval Administration

The Retrieval Administration screen is a special service dialog with functionalities to fix various problems in the warehouse. This dialog supports the analysis of possible reasons for the cancellation of Order Lines.

Pick Orders

Pick Orders represents the parts of an Order which could not be served by whole Load Units and thus need to be picked. One Pick Order specifies the part of an Order which need to be picked within one Picking Zone.

Pick Confirmation Handlings

Description of the Pick Confirmation Handling with configuration of the possible bookings and every type of handling on the basis of a Pick Confirmation by the underlying Pick Controller.
