Workstations

Purpose

A Workstation is a working place for a user, which is provided with a terminal and assigned to a Warehouse Location. Examples:

  • a Pickstation where Goods-to-Person Picking operations are processed
  • a location where Goods-to-Person Counting operations for stocktaking are processed
  • a location in the Receiving Area, where the Receiving based on Advices or Shipments of Purchase Orders is done
  • a location in the Shipping Area where some Load Unit Labels must be applied.

Workstation properties are grouped into:

  • Locations - Locations of the Workstation with specific operational roles
  • Printer - assigned printer are used by global print functions of the Workstation
  • Queue Entries - the preloaded workload is represented by sequenced and assigned work which is defined by a reference (e.g. a Pick Mission)
  • Zones - only workload of the assigned Zones is allowed to get into the queue
  • Capabilities - allowed/preferred workload type defined by the service qualifier.

 INFO 

For configuration purposes refer to the related topic in the Warehouse Administration Guide.

 INFO 

The overall basics of the supply of Workstations with empty Load Units are described in the section Empty Container Supply of the Retrieval Overview.

 INFO 

If any transport error occur, the destination for the required and initiated clarification transport is set to the Default Destination of the Location.
