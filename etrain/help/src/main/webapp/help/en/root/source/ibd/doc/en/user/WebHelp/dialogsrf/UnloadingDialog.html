RF-Terminal Receiving

Purpose

The dialog covers the user interface for Quantity and Advice based Receiving using a RF-Terminal.

      
Main Functions

The "RF-Terminal Receiving" dialog is used to perform the following tasks:

  • Quantity based receiving in automated or manual Areas ordered to given and selected Advice
  • Final storage of Load Units of goods receipt
  • Support of GS1 barcodes
  • In more detail
  • Selection of a Dock, Client, Supplier and Advice
  • Identification of the product and the Load Unit
  • Receiving of a certain quantity of the product

A detailed description of this use case is given in the use case "Quantity based Receiving with RF-Terminal".

      
Main Dialog Overview

Dialog Step

Description

Scan Dock

Step "Scan Dock" appears as the initial screen where the Id of the Dock needs to be scanned.

Select Client

Step "Select Client" is next after "Scan Dock" if there are is more than one client in the system.

Identify Advice

Step "Identify Advice" selects a supplier and is next after Dock and Client have been selected.

Select Advice

Step "Select Advice" is next after the selection criteria have been entered and matching advices are available.

Scan Load Unit

Step "Scan Load Unit" is next after an Advice was selected. A Load Unit has to be scanned where the Inventory to be received is deposited.

Check LU-Type

Step "Check LU-Type" is next after a Load Unit has been scanned. The Load Unit type could be initially set or changed.

Identify Product

Step "Identify Product" is next after a Load Unit and its type has been defined and more than one valid Advice Line is available.

Check Product

With this confirmation the determined product will be received with the specified quantity against the uniquely identified Advice Line.
