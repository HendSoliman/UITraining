Inventory Management - Terms and Definitions

Purpose

This section defines basic terms introduced by Inventory Management.

     
Business Entities

Business Entity

Description

Aggregated Inventory

Aggregated Inventory represents the sum of all Inventory Units within a Dispo Zone grouped by the Inventory separating attributes besides Load Unit. Dispositive processes may reserve Dispo Zone related Aggregated Inventory. These reservations are represented by Inventory reservations.
Inventory Management supports only an aggregation within Dispo Zones.

 INFO 

Management of Aggregated Inventory is basically done only for SKU with the usage type 'default'.
In order to increase the booking performance the overall management of Aggregated Inventory could be disabled per Dispo Zone with configuration topic "cor-inventory:AggregatedInventory.aggregatedInventoryManaged".

Inventory Reservation

Inventory reservations reserve a part of Aggregated Inventory for a defined purpose. The purpose of the reservation is defined by the reservation reason, the order ID and the order line ID. Reserved Inventory may only be used for its defined purpose.

Inventory Unit Reservation

Inventory unit reservations reserve a part of an Inventory Unit for a defined purpose. The purpose of the reservation is defined by the reservation reason, the order ID and the order line ID.
Reserved Inventory may only be used for its defined purpose.

Inventory Unit

An Inventory Unit is the smallest unit of Inventory managed by the LMS. It specifies the SKU, the quantity, the Load Unit and additional Inventory attributes.

An Inventory Unit is always booked onto a Load Unit, even if there is no physical representation of the Load Unit. When there is no physical representation of the Load Unit, a virtual Load Unit is used.
