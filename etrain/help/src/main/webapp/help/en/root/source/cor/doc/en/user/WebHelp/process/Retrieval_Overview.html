Retrieval Process Overview

Introduction

Order related Inventory allocation occurs within the Retrieval process with disposition, selection and reservation, of optimal Inventory Units or respectively Load Units. For selection of appropriate Inventory Units, obligatory aspects are given by the:

  • overall actual condition of the Inventory and directly related further business objects and
  • the match between the required aspects of the Order Line and the Inventory Units.

Optional optimization aspects of allocated Inventory can be configured using the provided Retrieval Strategies. Strategy details can be configured in the screen Retrieval Strategies.

 INFO 

For description of the different Strategies and use of the specific configuration topics refer to the related topic in the Warehouse Administration Guide.
     
Retrieval Strategies - Basic Conditions

The basic conditions that must be fulfilled for the selection process of appropriate and optimized Inventory Units, or respectively Load Units, are:

  • Aspects given by the Order Line to be processed are defined by Order Line Criteria.
  • Inventory Criteria (Requirements to Inventory aspects (e.g. SKU, batch, best before date, etc.) lead to selection of parts of Inventory Units or complete Load Units)
  • Load Unit Criteria (Requirements to specific and concrete Load Units, always lead to selection of complete Load Units)
  • Location Criteria (Requirements to specific and concrete Locations (e.g. the whole Facility, specific Areas, Aisles or concrete Storage Locations) always lead to selection of complete Load Units).

  • If Full Load Unit Retrievals are to be preferred the selection occurs on Inventory Unit level, but the complete Load Unit is to be retrieved instead of partial quantities that lead to Picking tasks.

  • Three disposition types are provided (defined by the reservation type of the Disposition Zone):
  • aggregated (Disposition is based on Aggregated Inventory only without concretion to specific Inventory Units. In this case, the aggregated reservation is consumed directly during operating processes (e.g. Picking). This is preferred for Picking on Dedicated Pick Faces)
  • per inventory unit (Disposition is based on the Inventory Units and Load Units, as described below. This is the standard type for most processes,)
  • two-Phase (Disposition occurs in two phases; first based on Aggregated Inventory with later concretion to specific Inventory Units. This is preferred for specific transport processes within a Multishuttle where e.g. late binding for selection of concrete Load Units for outbound transport issues is required).

  • Inventory Unit aspects:
  • Availability Status is "free" (only if Full Load Unit Retrieval processing) or "reserved"
  • Free Quantity > 0
  • Lock Status not "on hold".

  • Load Unit aspects:
  • Without any Error Code
  • Not virtual
  • Lock Status is "available"
  • Not empty
  • Reservation Status is "free" (only if Full Load Unit Retrieval processing) or "Inventory reserved" or "Child reserved"
  • Is single and is full (only if Full Load Unit Retrieval processing).

 

  •  Location aspects:
  • Without any error code
  • Not virtual
  • Lock Status has no part such as, "locked for outbound"
  • Related Area is connected via Area Access to the Destination Area of the Order (Line)
  • Related Area is of type, "stock", "buffer" or "transport"
  • Not a Dedicated Pick Face (only if Full Load Unit Retrieval and Goods-to-Person Picking processing).

 

  • Disposition Zone aspects (only if Full Load Unit Retrieval processing):
  • Full Load Unit Retrieval allowed
  • Related area is connected via Area Access to the Destination Area of the Order (Line) and no Unpacking Area is assigned.

 

  •  Disposition Zone aspects (only if Picking processing):
  • Picking allowed
  • Reservation Type is "per inventory unit" for standard Picking processes with single phase (direct) disposition
  • Reservation Type is "two-phase" for Multishuttle areas with late binding principles due to transport processes
  • Reservation Type is "aggregated" for manual Person-to-Goods picking processes with Dedicated Pick Faces
  • Related area is connected via Area Access to the Destination Area of the Order (Line)
  • Related Picking Zone is active and of type "Pick Controller".

 INFO 

A short lock (technical lock) of a Location is not considered, but consideration could be enabled through the configuration topic "cor-retrieval:Retrieval.isLmsRtvDispShortLock".

 INFO 

If Dedicated Pick Faces are used and multiple Pick Faces are available the selection is done randomly.

 INFO 

Dynamic Slotting does not affect the Retrieval Strategies at all.
     
Retrieval Strategies - Basic Process Flow

Step

Description

1.

For each order line, the order management component calls directly one retrieval method depending on the Order Line Retrieval Mode:

  • Retrieval by Inventory Criteria
  • Retrieval by Load Unit Criteria
  • Retrieval by Location Criteria.

2.

After evaluation and based on the retrieval mode, a sorted list is created with support of the associated configuration tables (dynamic database view).

3.

Within a loop, each matched Inventory Unit or respectively Load Unit is checked for accessibility and availability.

4.

Positive checked units should be reserved using reservation component.

5.

After reserving the requested quantity, a collection of Inventory Units or load units is returned.

6.

If an error (e.g. no possible Inventory Units are found or tolerance limits are violated) possible reservations are deleted and nothing is returned.
     
Empty Container Supply

Automatic Goods-to-Person Pick Stations, or all automatically supplied Workstations, often must be supplied with "Empty Load Units" for usage as e.g. Pick Container. Those Load Units have an Inventory Unit of a special SKU with usage type, "EMPTY LU" booked to them and thus can be requested using retrieval strategies. This SKU must belong to the single Client of type, "INTERNAL". The related Load Unit Type is to be configured as the preferred Load Unit Type of this SKU.

When the Workstation requests new Empty Load Units, the system determines the configured default Load Unit Type of the Workstation and allocates the requested amount of the related SKU. The inventory reservation will be consumed on arrival at the Workstation.

If any booking of any Inventory onto the Empty Load Unit, the Inventory representing the "empty" SKU is deleted automatically.

 INFO 

Creation and storage processes for this type of Empty Load Unit must be project-specific. This is also valid for reuse of Load Units, e.g. those that became empty during picking.
