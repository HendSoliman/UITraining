Fields

Field

Description

Order Line

Order
Client

Owner of the Order. Thus, owner of the Inventory which can be allocated with this Order.
Order ID

Order ID. Unique within a Client.
Status

Current execution state of the Order:

  • created (Order was created and not ready for pending, but ready for start of preparation phase (project specific use))
  • prepared (Order preparation phase is finished, but not ready for pending (project specific use))
  • pending (Order is awaiting release for fulfillment)
  • released (Order is released for fulfillment)
  • active (Order execution already started; Full Load Unit Retrievals and/or Pick Orders are in progress)
  • complete (Order execution is finished and the Order is fully completed; all Order Lines are completed or cancelled)
  • to confirm (Order is ready for confirmation message to the host system)
  • finished (All required tasks for the Order are finished. If required, the Order was confirmed to the host)
Urgency

Sequence of automatic fulfillment tasks related to this Order. Lower value indicates a previous task execution. Value is calculated by the system. Factors that influence calculation are project specific and can be changed by using the appropriate policy. Usually, priority and Delivery Time are considered.
Primary Key

System wide unique identification of a business object. Key for the current Order.

Order Line
Order Line ID

The identification of the Order Line is unique within an Order and is usually represented by a running number.
Status

Current execution state of the Order Line:

  • pending (Order is awaiting release for fulfillment and all Order Lines proceed to this status)
  • released (Order is released for fulfillment and all Order Lines proceed to this status)
  • active (Order execution already started; Full Load Unit Retrievals and/or Pick Orders are in progress and all Order Lines proceed to this status)
  • cancelled ((single) Order Line was cancelled without the defined execution)
  • complete (single Order Line execution is finished and the Order Line is fully completed)
Primary Key

System wide unique identification of a business object. Key for the current Order Line.

Pick Order Line

Pick Order
ID

Pick Order ID. Unique within a Client. Usually represented by a running number.
Status

Current execution state of the Pick Order:

  • pending (Pick Order is awaiting release for fulfillment)
  • released (Pick Order is released for fulfillment)
  • active (Pick Order execution started; the first Source Load Unit was requested or the first Pick was confirmed)
  • complete (Pick Order execution is finished and the Pick Order is fully completed; all Pick Order Lines are completed or cancelled)

Pick Zone

Pick Controller Zone (a Zone of type "Pick Controller") where the Pick Order must be executed. One Pick Order represents the part of an Order which
must be picked within one Pick Controller Zone.

Primary Key

System wide unique identification of a business object. Key for the current Pick Order.

Pick Order Line
Pick Order Line ID

The Pick Order Line ID is unique within a Pick Order. Usually represented by a running number. Each Pick Order Line that is split up is denoted by an added "-digit" where "digit" is started with "1". ("1-1" and "1-2" are the first two split Pick Order Lines of Pick Order Line "1".)

 INFO 

Creation of a split Pick Order Line occurs after Pick Confirmation by the Pick Controller for each different combination of Error Code and Error Reason.
The "-digit" Pick Order Lines represents always the already finished part while the original one represents the still open portion. 

Status

Current execution state of the Pick Order Line:

  • pending (Pick Order is awaiting release for fulfillment and all Pick Order Lines proceed to this status)
  • released (Pick Order is released for fulfillment and all Pick Order Lines proceed to this status)
  • active (Pick Order execution started and all Pick Order Lines proceed to this status)
  • cancelled ((single) Pick Order Line was cancelled without the defined execution)
  • complete (Pick Order Line execution finished and the Pick Order Line is fully completed)

Primary Key

System wide unique identification of a business object. Key for the current Pick Order Line.
Sequence Number

Sequence of Picking Operation of all Pick Order Lines within one Pick Order. If Picking with transports to Pick Stations, represents the order in which Load Units of the assigned group should arrive at the final destination.
Error Code

Last error that occurred with this Pick Order Line:

  • OK (successfully completed)
  • missing quantity (not fully picked due to Inventory shortage)
  • load unit missing (not fully picked because no Load Unit found)
  • wrong SKU (not fully picked because wrong SKU found)
  • wrong load unit (not fully picked because wrong Load Unit found)
  • wrong best before date (not fully picked because wrong Best Before Date for Inventory found)
  • wrong packaging level (not fully picked due to wrong packaging level)
  • breakage (not fully picked because broken Inventory found)
  • blocked (not fully picked because Location to be picked from is blocked and inaccessible)
  • underdelivery (not fully picked because explicitly Picking of less than target quantity)
  • overdelivery (not fully picked because explicitly Picking of more than target quantity)
  • split (was split because Pick Container was full or required use of different Quantity Unit).
Error Reason

Text reason of the confirmation. Configured within the Pick Controller. Described in Detail Dialog Confirmation Reason for "Reason".
Source Base Load Unit ID

Base Load Unit ID that the Load Unit is stacked onto.
Source Load Unit ID

Load Unit ID to be picked from.

Inventory

Requested Stock Keeping Unit.
Description

Description of the SKU.
Pick Quantity

Requested quantity in Pick Quantity Units for this Pick Order Line.
Unit

SKU Quantity Unit ID. Used for display purposes within Pick operations.
Quantity Base Target

Requested quantity in Base Quantity Units for this Pick Order Line.
Unit

Base Quantity Unit of the SKU. Represents the relevant Quantity Unit for the quantity base target, quantity base target host, quantity base dispo and quantity base current  as set by the related Order Line.
Quantity Base Current

Already fulfilled quantity. Set with status progress of the Pick Order Line to "complete".
