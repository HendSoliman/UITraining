Pick Controller - Introduction

Introduction

The PIC subsystem includes further specialized components. The combination of all components provides the required functionality for an entire Pick Controller.

 INFO 

The specific processes of Goods-to-Person Picking and Route Based Picking are described separately.
     
Specialized Components

Name

Description

Picking Generals

Provides operative objects and services for picking. For general configurations and parts of planning and execution of picking processes.

Picking Goods-to-Person

Supported for many Load Unit types and many configurations of material handling equipment. Deals with all picking processes special to automated areas where goods are transported to fixed Pickstations for pick operation and subsequently conveyed back to the storage area.

The Pick Controller for Goods-to-Person Picking includes a specific collaboration with transport management concerning supply with source and empty pallets, as well as pallet release at a stationary Pickstation when a pick is completed. Use of containers, cases or cartons instead of pallets is varies mainly with transport handling. The different main Picking screens support external trigger for automatic process flow with signaling of the conveyed Load Units by the PLC and with external signals for pick confirmations (e.g. by additional Put-to-Light application).

Picking Route based

Supported through Person-to-Goods picking processes. Deals with all picking processes special to manually operated areas where Pickers are guided to storage locations for pick operations from the Pick Face into the manually transported Pick Containers.

Person-to-Goods Picking can occur with online or offline picking mode. Online picking mode is supported by guiding the Picker with a web based RF-Terminal and direct online pick confirmation. Offline picking is paper based with subsequent pick confirmation for whole Pick Walks by screen.
     
Basic Entities

Each single pick in the system is represented by the entity Pick. To perform the primary action of picking inventory from Pick faces to Pick Containers, each Pick must be associated with exactly one Pick Mission, which could be associated with exactly one Pick Walk. The picking process can start only when Pick Missions are available.

The following basic entities represent picks within the Pick Controller:

Object (Entity)

Description

Pick

Item of a Pick Mission. Defines the primary action of picking inventory from Pick face to Pick Container. Provides additional data to one Pick Order Line specific for picking. One Pick Order Line could be completed by multiple Picks.

Pick Mission

Parts of one or more Pick Orders assigned to one Pick Controller for fulfillment. Represents the workload of all Picks to be picked into exactly one Pick Container. Provides additional data to Pick Orders specific for picking.

Pick Walk

One or more Pick Missions that must be processed within one entire Pick Walk in case of Person-To-Goods Picking only.

Following entities are general in PIC and used by all Pick Controllers:

Entity

Description

Pick Zone Config

For configuration purposes which are based on the Pick Controller or the subordinated Picking Zone of the Pick Controller.

Pick Confirmation Reason

Represents different Confirmation Reasons for selected confirmation of the pick operation. Each confirmation is associated with one Confirmation Code and one Pick Error Code for confirmation within Order Fulfillment.
