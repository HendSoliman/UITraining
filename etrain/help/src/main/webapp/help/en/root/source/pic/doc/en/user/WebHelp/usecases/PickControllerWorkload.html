Pick Controller Workload

Description

The Pick Controller Workload is committed by the Order Processing subsystem of the Dematic iQ® and initiated by the release of a Pick Order.
Based on the Pick Order Lines new Picks are created and directly grouped to Pick Missions. Some Picks are intended for deferred grouping and will be handled time based later on.
Pick Missions will be released for assignment to picking resources directly with creation.
Goods-To-Person Pick Controller uses the Workstation Workload Management for assigning the Pick Missions to single Workstations.
Person-To-Goods Pick Controller provide a pulling mechanism for assignment. When a Picker (Worker) requests new work a defined number of Pick Missions are grouped to a Pick Walk which then gets assigned for execution.

      
Triggers and Actors
  • A Pick Order is released by the Order Processing subsystem and handed over to the configured Pick Controller.
      
Prerequisites
  • Fully configured Pick Controller
  • Order and related Pick Orders assigned to the mentioned Pick Controller Zone are released successfully.
      
Required Resources
  • Cron Jobs.
  • Workstations.
      
Work Flow

Step

Description

1.

Pick Controller creates Picks based on the Pick Order Lines of the released Pick Order with respect to the Pick Zone Configuration.

2a.

Pick Controller directly groups the created Picks to Pick Missions.
All defined types of Picks which are defined to be intended for deferred grouping are excluded.

 INFO 

In the standard these are Picks with Service Qualifier "BLOCK".

2b.

Picks that meet the condition for deferring are excluded from direct grouping to Pick Missions. This is done periodically by the "HandleDeferredPicksDemon" which must be up and running.

 INFO 

To be checked in Cron Jobs.

3.

The Pick Missions in status "RELEASED" are provided for assignment to Workstations or Workers.

3a.

For Goods-To-Person Pick Controller the Workstation Workload Management for assigning the Pick Missions to single Workstations is used (see Workstation Queue Entries).
This is either done by the "FeedWorkstationDemon" which must be up and running or based on events during picking at the Workstation or when the Workstation is activated.

 INFO 

The assignment is done only if the Workstations "Request Size" number of Workstation Queue Entries are free. If so exactly "Request Size" number of Pick Missions will be assigned. Less Pick Missions will only be assigned if less are available.
If any already assigned entry exists (if not enough Pick Missions could be assigned in any former run) the number will be filled up.

3b.

For Person-To-Goods Pick Controller the assignment is done directly on request.
When a Picker requests new work a defined number of Pick Missions are grouped to a Pick Walk which then gets assigned to the Picker.
     
Result

Picks are created and grouped to Pick Missions. The Pick Missions are released for assignment to Workstations or for grouping to Pick Walks for assignment to Worker.
