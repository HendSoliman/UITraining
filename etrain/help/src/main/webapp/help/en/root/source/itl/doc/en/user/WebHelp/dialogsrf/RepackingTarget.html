Unpacking Target

Short Description

In dialog state "Unpacking Target" the Target Load Unit for the unpacking operation is determined.

If the Target Load Unit is unknown a Load Unit Type has to be selected as well.

      
Screen

After entering a Target Load Unit:

      
Fields

Field

Description

Source-LU

Id of the already scanned Source Load Unit.

Location

Current Location of the Base Load Unit.

Client

The Client which is the owner of the Source Load Unit and of the Inventory inside.

Batch

The batch of the selected Inventory Unit.

SKU

The Stock Keeping Unit of the selected Inventory Unit.

Source Quantity

The available quantity of the selected Inventory Unit.

Unit

The accordant Quantity Unit.

Target-LU

Scanned Load Unit that will be the target for the inventory to be unpacked.

LU-Type

Load Unit Type of the scanned Target Load Unit.

 INFO 

Unpacking of a Dispatch Load Unit requests the creation of a new Dispatch Load Unit. The type will be defined by the type of the new Load Unit. In case of multiple DLU Types exists for the used Load Unit Type the system applies any without providing a specific selection logic. This is done for example if a Pick Container that is represented as a DLU as well is manually repacked in a new Load Unit.
     
Buttons

Function/Button

Description

Ok

Acknowledge the scanned load unit id

Canc.

Abort the dialog and return to the dialog state "Unpacking Source"

LU-Type

Change the type of the new yet unknown Target Load Unit
