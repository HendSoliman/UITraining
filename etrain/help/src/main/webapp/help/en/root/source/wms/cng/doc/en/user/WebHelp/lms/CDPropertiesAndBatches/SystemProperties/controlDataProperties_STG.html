This topic describes the configuration parameters of Strategy.
Located in demo\target\configuration\cor-strategy.xml.

Default

Description

Strategy.isStrategyMultishuttle

boolean

area

false

This Parameter defines whether the area uses the put-away and retrieval strategies for multi-shuttle.
Strategy.isPutawayStrategyConfigurationCached boolean area false Enable/disable caching of strategy configuration (Dialog Putaway Strategies) for Multishuttle in application scope.
If true, changing settings requires application server restart.
Recommended setting for production environment: true.
Strategy.isAMCAPStrategyNeeded boolean area false Activates AMCAP Strategies additionally to configured Multishuttle Strategies.
This strategy distributes the items to be stored in the Multishuttle according to the demand of the orders to be palletized.
Inter-aisle transfers are avoided, if possible according to the demand. When the PLC requests a destination for two items the destinations of the items are selected with the same Y-coordinate.
When inter-aisle transfer is necessary for both items the strategy avoids an inter-aisle transfer into to same direction.
Strategy.freeShuffleSlotCapacity int area 1 Specifiy an amount of slots in each level to be reserved for shuffle moves in putaway strategy. These slots will not be available for putaway.
If the amount of free slots is under the configured value the strategy uses the concrete slot width to calculate precise buffer capacities.
Only active in Multishuttle Flex storages.
Strategy.isIATStorage boolean area false
 Defines if an area comes with the Inter-Aisle Transfer technology.
This technology enables optimized putaway strategies (see Putaway Strategy).
Only available for Multishuttle storages.
Strategy.areaDimension.area string area  

Specifies area to apply StrategySwitch parameters to.

This Element is optional. If not specified these values are applied to all areas.

Example: <areaDimension><area>WT01</area></areaDimension>
