Booking & ERP

Booking Operation Type

Booking operation represented by this Inventory Journal entry:

  • add (Inventory Unit is created and the host is notified, if the host is the SKU owner)
  • merge (two Inventory Units are merged into one, representing the quantities of the two Inventory Unit)
  • split (Inventory Unit A is split into two Inventory Units. Total quantity of the two Inventory Units equals the quantity of A)
  • pack (Inventory Unit is moved from one Load Unit to another)
  • reserve unit (Inventory Unit or part of the Inventory Unit is reserved)
  • cancel unit reservation (Inventory Unit reservation or part of the Inventory Unit Reservation is cancelled)
  • create unit hold (Inventory Unit hold is created)
  • delete unit hold (Inventory Unit hold is removed)
  • requalify (properties of the Inventory Unit are changed)
  • remove (Inventory Unit or part of the Inventory Unit is removed and the host is notified, if the host is the SKU owner)
  • consume unit reservation (Inventory Unit Reservation or part of the Inventory Unit Reservation is consumed. If the Inventory Unit is reserved for a Shipping Inventory Reservation Reason, the Destination Inventory Unit has the availability status picked and the order ID and order line ID of the reservation are set directly in the Destination Inventory Unit. If the Inventory Unit is not reserved for a Shipping Inventory Unit Reservation Reason, the Destination Inventory Unit is free.)
  • finish non-shipping order (finishing of a Non-Shipping Order is booked for an Inventory Unit. Order assignment is taken from the Inventory Unit and the Inventory Unit is booked to free Inventory)
  • cancel order (cancellation of an Order is booked for an Inventory Unit. Order assignment is taken from the Inventory Unit and the Inventory Unit is booked to free Inventory)

Booking Service Type

Booking service used to create this specific booking operation. May be fulfilled by a sequence of booking operations.

  • add (Inventory is added on the specified load unit)
  • repack (Inventory Unit or a part of an Inventory Unit is moved from one Load Unit to another and the bookings connected to this repacking are performed. Can be the consumption of an Inventory Unit Reservation, the transition of the availability status, a split operation and a merge operation)
  • reserve unit (Inventory Unit or a part of an Inventory Unit is reserved)
  • reserve (Aggregated Inventory is reserved on Disposition Zone level)
  • cancel unit reservation (an Inventory Unit Reservation of a part of the Inventory Unit Reservation is cancelled)
  • cancel reservation (a reservation or a part of a reservation of Aggregated Inventory on Disposition Zone level is cancelled)
  • prepare unit reservation (reservation type in case of two phase reservations is changed from aggregated to concrete unit)
  • consume unit reservation (an Inventory Unit Reservation or a part of the Inventory Unit Reservation is consumed. For details, see the booking operation consume unit reservation)
  • correct quantity (quantity of an Inventory Unit changed)
  • set hold (hold for the complete inventory of a SKU or a batch is created)
  • set unit hold (Inventory Unit hold is created)
  • remove hold ()
  • remove unit hold ()
  • requalify ()
  • move ()
  • rebook.()
  • notify order finished (notification that Order processing is finished for an Inventory Unit Order Load Unit is booked. If a Shipping Order, Inventory Unit is removed. Otherwise, Inventory Unit is booked to free Inventory)
  • notify order cancelled (notification that an Order was cancelled, the Inventory Unit is booked to free Inventory)

Booking Time

Date / time of the booking.

Booking Comment

If a user initiated the booking, comment is recorded here.

Correction Booking

If the booking is caused by the discovery of a difference between the physical reality and the Inventory in the LMS. Can be an unexpected loss or appearance of Inventory or due to cycle counting.

SKU Description

Description of the Source SKU.
Availability Status

Status of the Inventory Unit at the beginning of the booking operation:

  • in receiving
  • free
  • reserved
  • picked
  • packed
  • loaded
  • leave confirmed
  • shipped
Batch

Identification of the batch, in which the Inventory was produced.
Quantity

Total source quantity represented by the Inventory Unit expressed in the Base Quantity Unit of the SKU.
Quantity Free

Part of the source quantity represented by the Inventory Unit expressed in the Base Quantity Unit of the SKU, which is not reserved by an Inventory Reservation.
Quantity Missing

Quantity identified as missing (e.g. by a near zero stocktaking during Picking). In some cases, a missing quantity is required, until all bookings connected to the missing quantity are finished. When reserved inventory is identified as missing, the missing quantity is managed in LMS until the Inventory Unit Reservations on the missing quantity are canceled. Source missing quantity is not part of the total quantity. When the missing quantity and the total quantity are zero, an Inventory Unit is removed.
