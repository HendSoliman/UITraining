Pick Orders

Purpose

Pick Orders represent the parts of an Order which cannot be served by whole Load Units, and must be picked. One Pick Order specifies the part of an Order which must be picked within one Picking Zone. When a Pick Order Task is being released, a Picking Request is sent to an underlying picking subsystem (Pick Controller) for execution. The Picking subsystem creates its own Picking directions based on related Picking Requests which are represented by single Pick Order Lines. Pick Orders are confirmed when related Picking Requests are confirmed by the Picking subsystem, which occurs for each single Pick separately and directly after execution. Pick Containers represent the Order related Load Units where the Inventory of a Pick Order Line is put into.

Pick Orders represent as well the parts of an Order which need to be palletized with the specific RapidPal system. The palletizing subsystem uses for palletizing directions the Pick Order Lines where a Pick Order represents one single pallet.

Special Buttons

Button

Description

Release Pick Orders for start of fulfillment. Only possible for Pick Orders in "pending" status. With Pick Order Release, the Pick Order is delegated to the related Pick Controller for execution. Pick Execution within provided Pick Controller is described in the Pick Controller Overview.

Cancel Pick Orders with underdelivery. Only possible for Pick Orders that are not yet "complete". A Request for Cancellation is sent to the related Pick Controller.

  • Cancellation will be refused if the first Pick Order Line was (partly) picked and confirmed by the Pick Controller.

 INFO 

If refused, Pick Order Cancellation it is possible to cancel single Pick Order Lines (as described in the screen Pick Order Lines).

 INFO 

With manual cancellation of a Pick Order no replacement inventory allocation will be initiated!
