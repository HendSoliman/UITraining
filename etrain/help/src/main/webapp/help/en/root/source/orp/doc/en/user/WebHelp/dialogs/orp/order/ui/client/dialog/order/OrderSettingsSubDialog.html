Order Settings

Purpose

This screen opens when modifying one or multiple attributes of all selected Orders to the same target properties.

      
Screen

 INFO 

Only attributes enabled by the check box in front of them are considered for change. Changes affect all selected Orders for the selected attributes (within the screen Orders).

     
Fields

Field

Description
Priority

Influences the urgency. Detailed information about factors influencing the priority is only available within the Host system as the owner of the (Shipping) Orders. Internal Orders get a default value.
Use Modify specific settings button to manually override in the screen Orders.

Configuration Status

If Order configuration was accomplished and the Order can be processed. Allows users to interrupt their work and continue Order creation later.
Route

Orders could be grouped to Routes which define the same transport for Collective Shipping. The Route is a simple grouping property without its own features.

Location Settings

Destination

Destination for all Dispatch Load Units of the Order. Can be a Facility, Area, Aisle, Zone or Location. Used by Order Management as a proposal. Can be changed manually or by any business process specific decisions (project specific).
Intermediate Destination

Location which all Load Units for this Order must be transported to before arrival at the final destination (e.g. marshalling grid). Used only if intermediate destination is not specified in an Order Line.
Host Shipping Point

For informational and organizational purposes to support shipment operations. Set by the Host.

Time Settings
Start Picking Deadline

Planned deadline to start Picking (project specific use only).
Paused

An Order can be held if it should not be processed for any reason at this moment (for further details see description of the underdelivery tolerance in Order Line).
Earliest Staging Time

Earliest time to deliver to the Staging Area. (calculated as configured fix offset to the Latest Staging Time).
Latest Staging Time

Latest time to deliver to the Staging Area. (If shipping transports are planned with "Pick-By-Time Requirements", is usually the Time after the required shipment is no longer possible). (Set by configured default (configuration topic) if not specified directly, or calculated with configured "Staging Time Determination Policy", or calculated as configured fix offset to the target shipping time).
Target Shipping Time

Planned date when Shipping operations should be done (calculation possible with configured "Shipping Time Determination Policy").
Delivery Time

Required time the Order should arrive at the Customer's unloading address.

 INFO 

 This is the only time that must be set during manual Order creation. All other Times depend on the desired Delivery Time and are calculated based on global configurations or Order Management policies.
