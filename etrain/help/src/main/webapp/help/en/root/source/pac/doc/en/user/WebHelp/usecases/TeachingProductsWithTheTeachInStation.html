Teaching Products with the Teach-In Station

Description

This topic describes how to teach products with the Teach-In Station.

 

      
Triggers and Actors
  • All packaging variants must be teached before they first enter the AMCAP area.
  • Teaching is typically done when a new packaging variant is received for the first time within a warehouse.
  • Packaging variants must be teached if characteristic dimensions deviate from those of previously teached variants.
  • Items which are systematically rejected after the dimension check must be teached in order to verify their characteristics.
  • If an AMCAP system is set up for an existing warehouse, all products which are actually stored in the warehouse must be teached.

 

      
Prerequisites
  • SKU data must already have been transferred from the commercial (ERP) system.
  • Load Unit Capacities for layers as well as for pallets must have been transferred from the ERP system.
  • The Length/Width meter on the Teach-In workstation is calibrated.
  • The Height meter on the Teach-In workstation is calibrated.
  • The scale display shows 0 while the measuring area is empty.
  • Teach-In Station Dialog is started with the specific TEACHIN terminal.

 

      
Required Resources
  • At least one item of the packaging variant to be teached.
  • The item must be uniquely identifiable, typically through a GS1 barcode including the GTIN.
  • A full source pallet, preferably from the supplier, should be available.
  • Teach-In workstation with connection to the Dematic iQ® server.

 

      
Work Flow

Step

Teach-In Step

Action

1.

Select Client

If step "Select Client" is shown, select the client for which products shall be teached.

Press <Next> to proceed to step Scan Product Code.

2.

Scan Product Code

With the cursor in field Barcode, scan the barcode containing the GTIN.
(see Barcodes for Identification)

3.

Scan Product Code

If a GTIN could be extracted from the barcode, this will be shown in field GTIN.

4.

Scan Product Code

If a batch could be extracted from the barcode, this will be shown in field Batch.

5.a

Scan Product Code

If the GTIN is found for one single SKU of the active client, the relevant SKU data will be shown for verification.

The <Next> button will be active
Pushing the button will activate step Select Packaging Variant.
Continue with step 8.

5.b

Scan Product Code

If no GTIN was found or if the same GTIN exists for multiple SKUs,
no SKU data will be shown.

The <Next> button will be active.
Pushing the button will activate step Select SKU.

6.

Select SKU

Look for a product description on the packaging.

If necessary, open the packaging and inspect the content.

Use the product description to filter with respect to SKU description

7.

Select SKU

Select the correct SKU and press <Next>

8.

Select Packaging Variant

The screen will show different Trade Items (SKU Quantity Unit) for the active SKU.

  • Trade items are shown with their GTIN to allow comparison with the barcode on the teach-item.
  • For every Trade Item, the name of the quantity unit as well as the number of contained base units are shown.
  • Only those trade items are shown for which the number of contained base units is equal to or larger than the number of contained base items for the scanned trade item.
    This is required for the case that the  scanned trade item was a contained item and not the outer packaging.
  • If at least one packaging variant has already been teached for a trade item, the ID of the packaging variant will be shown in brackets, e.g. "[C2A]"
    In addition, also the image of the packaging variant will be shown.
    (see Representation of Packaging Variant Entries)

9.

Select Packaging Variant

Choose the correct variant or trade item using barcode information, number of contained bas units as well as image.

10.

Select Packaging Variant

If no trade item exists for the selected entry (no "[...]" and no image) or if the teach-item is a new variant for the same trade item, press <New> to create a new packaging variant.

  • A separate screen will open for specification of the ID of the variant.
  • Naming of packaging variants is arbitrary.
    A recommended scheme is concatenation of the first character of the ID of the SKU Quantity Unit, the number of contained base units, and a letter (A, B, C, ...)
  • Push <Save>  to use the selected ID.

11.

Select Packaging Variant

If a valid Packaging Variant entry is selected, the <Next> button will be activated.

Push the <Next> button to continue

12.

Pallet Structure

If a supplier pallet is not available (e.g. teaching of single items),

  • select "No Source LU"
  • Push <Next>

Continue with step 18.

13.

Pallet Structure

Select the Load Unit Type of the source pallet.

Dimensions of the Load Unit Type will be shown as well as the number of base units per layer and the number of layers per pallet.

14.

Pallet Structure

Check the number of base units per layer.

If the number shown is wrong, correct it.

Check the number of layers per pallet.

If the number shown is wrong, correct it.

15.

Pallet Structure

If there is an overhang of the load on the pallet,

  • measure length and width of the load
  • enter the measured values in the corresponding fields
  • This will cause a value for the overhang to be calculated.
    This is the larger value of the overhang for length and width.

 INFO 

Overhang is used for the layer simulation for verification of dimensions. If this is not required, measurement of length and width of the load can be skipped.

16.

Pallet Structure
  • Measure the height of the source pallet from the floor (including the empty pallet).
  • Enter the height in field Total Height.
  • Count the number of slip sheets, if any, and enter the value in field # Slip sheets.

While changing values, the height if a single layer is calculated which can later be compared with the measured height of an item.

17

Pallet Structure

Push <Next> to continue

18.

Manage Pictures

Adjust the camera so that the live picture shows the characteristics of thepackaging

19.

Manage Pictures

Push the <Take Picture> button.

The view will show the taken image instead of the live view

20.

Manage Pictures

Push <Next> to continue.

21.

Dimensions & Weight

On the Teach-In workstation, place the teach-item upright on the ground plate with the length edge oriented from left to right:

22.

Dimensions & Weight
  • Push the Length/Width Meter to the left until the teach-item is firmly pressed
  • On the Teach-In screen, push the <Length> button.
    The measured length is displayed in the field below.

23.

Dimensions & Weight
  • Push the Height Meter down until it firmly presses on the teach item.
  • If the teach item has stacking lugs, make sure that they are bent down or removed in the area of the Height Meter.
    The height to be measured shall not include the stacking lugs.
  • On the Teach-In screen, push the <Height> button.
    The measured height is displayed in the field below.
  • If a layer height is shown from the Pallet Structure step, compare the measured height with the calculated layer height.
    In case of differences larger than 10 mm, clarify the reason
  • If the calculated layer height is smaller than the measured height by less than 5 mm, consider to use the calculated layer height instead of the measured height.
    The reason for the difference may be the compressibility of the teach item.

24.

Dimensions & Weight
  • Make sure that no other items are on the measuring device
  • On the Teach-In screen, push the <Weight> button.
    The measured weight is displayed in the field below.

25.

Dimensions & Weight

Turn the teach item by 90°:

 

 

26.

Dimensions & Weight
  • Push the Length/Width Meter to the left until the teach-item is firmly pressed
  • On the Teach-In screen, push the <Width> button.
    The measured width is displayed in the field below.

27.

Dimensions & Weight

Place the teach item on the Toppling Plate with the length edge oriented from left to right:

28.

Dimensions & Weight
  • With the Toppling Handle increase the angle of the Toppling Plate until the teach item starts toppling.
  • On the Teach-In screen, push the <Toppling Length> button.
    The measured angle is displayed in the field below.

29.

Dimensions & Weight

Decrease the angle of the Toppling Plate and turn the teach item by 90°:

30.

Dimensions & Weight
  • With the Toppling Handle increase the angle of the Toppling Plate until the teach item starts toppling.
  • On the Teach-In screen, push the <Toppling Width> button.
    The measured angle is displayed in the field below.

31.

Dimensions & Weight
  • Verify that the AMCAP compliance fields show all "OK".
  • In case that the Length/Width ration is shown in red verify that the value is >=1 and <= 4 (4 is the maximum allowed value)

32

Dimensions & Weight

If the item dimensions are outside the acceptable tolerance in comparison to historic measurements,
the <Next> button will be inactive and the <New Variant> button will be active, indicating that the item must be teached as new Packaging Variant.

For more details, see working with historical teach data

33.a

Dimensions & Weight
  • In case that all compliance checks are OK, and if there are no other contradictions against AMCAP processing,
    choose option <Yes> for "Process with AMCAP".
  • Push <Next> to continue
  • with step 34 in case that a specific source Load Unit Type has been selected in step Pallet Structure.
  • with step 36 in case that step Pallet Structure has been skipped.

33.b

Dimensions & Weight
  • In case that AMCAP compliance is not granted, only option <No> can be selected for  "Process with AMCAP"
  • Pushing <Next> will request a confirmation and book the packaging variant as "non-AMCAP" (see Result)
  • The dialog will be reset and return to step 2.

34.

Layer Simulation

The screen will show

  • the dimensions of the pallet (including overhang, if present)
  • the pack dimensions of the teach item  which have been increased from the measured dimensions according to a specific formula (For details see Teach Parameters Overview)
  • the number of items per layer.
    (this number has been received from the ERP system and is stored in SKU Load Unit Capacities for Load Unit Type *_LAYER
  • the number of items per layer simulated with the given pack dimensions.

35.a

Layer Simulation

If Items/Layer and Items/Layer (simulated) are identical, proceed with step 36.

35.b

Layer Simulation

If Items/Layer and Items/Layer (simulated) are dfferent

  • Check the pack dimensions.
    If these are wrong, press the <Prev> button to return to Dimensions & Weight where the faulty dimension can be corrected (step 21)
  • Recount the number of items in a layer on the source pallet
    If the number is different from Items/Layer,
    Press <Cancel> and repeat teaching starting with step 2.
    In step 14, enter the correct number.

36.

Select Packaging Group

In this step, packaging group of the teach item is selected.

For a list of characteristics see the list of Packaging Types.

When a group has been selected, characteristics are shown in the help area to support selection.

After selection push <Next> to continue.

37.

Select Packaging Type

This step appears if the Packaging Types are sub-grouped.

When a subgroup has been selected, characteristics are shown in the help area to support selection.

After selection push <Next> to continue.

38.

Select Content Group

In this step, the content group of the teach item is selected.

For a list of characteristics see the list Content Types.

After selection push <Next> to continue.

39.

Select Content Type

This step appears if the Content Types are subgrouped.

When a subgroup has been selected, characteristics are shown in the help area to support selection.

After selection push <Next> to continue.

40.

Cover Restriction

In this step, the teach item is classified with respect restrictions related to coverage by other items within a pallet structure.

After selection push <Next> to continue.

41.

Basic Parameters 1

This page requires exclusively boolean decisions.

Some of them require further information which will be queried on the next screen.

After all decisions have been made, push <Next> to continue.

42.

Basic Parameters 2
  • Rate the Strength Class of the teach item
  • If the teach item is suitable for pre-stacking  select the pre-stacking type allowed for the packaging variant.
  • Determine the de-palletizing program according to the specified procedure.
  • For all other measures requested on the screen, use the digital measuring slide for measurement and press the associated measuring button.

43.

Edge Reduction

This step is only necessary if the formulas for edge reduction indicate that edge reduction may occur for the selected packaging type and content type.

(See Teach Parameters Overview)

The initial values are only suggestions which are not necessarily the real values.

The edge reduction values are measured using the digital measuring slide.

After all values have been specified, press <Next> to continue.

44.

Ready To Finish

This is the final step for the normal workflow.

The screen allows revisiting previous steps for checks or modifications.

In addition, it is also possible to check and modify the advanced settings.

Finally, press the <Finish> button to store all data gathered during the teach session.

This will return to step 2.

45.

Advanced Settings

The set of Advanced Settings screens contain parameters which are automatically determined using predefined formulas (See Teach Parameters Overview).

Under normal circumstances, especially when a packaging variant is teached for the first time, all parameters can be kept.

If it turns out during operation that the behavior of tha packaging variant in the system is not as expected, selected parameters may be modified.

 

      
Result
  • If the teach-Item has been classified as AMCAP compliant
  • The "Preferred Fulfillment Handling Type" of the SKU Quantity Unit has been set to "AMCAP".
  • If a change of the quantity per layer had been confirmed, the SKU Load Unit Capacity for the Load Unit Type of the source pallet has been updated.
    Eventually, a change message has been sent to the ERP system.
  • A Packaging Variant (Palletizing Quantity Unit) has been created or updated with the values specified in the teach process.
  • If the Packaging Variant has been specified as default for the SKU Quantity Unit, the dimensions of the SKU Quantity Unit have been updated with those of the default Packaging Variant (Palletizing Quantity Unit)
    In case of a change with respect to the previous values, eventually a message has been sent to the ERP system.
  • If the teach-Item has been classified as AMCAP non-compliant
  • The "Preferred Fulfillment Handling Type" of the SKU Quantity Unit has been set to a project default other than "AMCAP"
