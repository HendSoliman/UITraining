Overview

Execution of Stocktaking

Purpose

For Stocktaking execution, the process of Perpetual stocktaking with an explicit planned Counting Order is described. It consists of a Stocktaking Order creation, release of this Order, automatic creation of related Counting Orders, execution of the counting process, and a final booking of counting results.

      
Overview

Chapter

Description

Creation of a Stocktaking Order

For execution of Perpetual stocktaking with an explicit planned Counting Order, a Stocktaking Order must be created. This describes a new Stocktaking Order creation with SKU, Load Units or Locations. With (manual or automatic) release of the Stocktaking Order, the system creates Counting Orders separately per Stocktaking Zone. In automated Areas, a Counting Order is retrieved on demand for execution of Counting operations. In manually operated Areas, manual execution is based on counting lists and final booking of counting results on the administration screen.

Goods-To-Person Stocktaking Execution

For performing a Goods-To-Person Stocktaking execution process, the Counting Operation screen is used at a defined Workstation. This describes how a valid Counting Order within a Goods-To-Person Zone is processed on this screen.

Person-To-Goods Stocktaking Execution (Offline)

Person-To-Goods Stocktaking execution processes in Offline Mode is performed with support of the Counting Orders. After a worker executes the offline Counting process using the Manual Counting List, counting results are confirmed through the Report Counting Results.

Person-To-Goods Stocktaking Execution (Online)

Person-To-Goods Stocktaking execution process in Online Mode is performed with support of the RF-Counting screens with direct confirmation of the counting results.
