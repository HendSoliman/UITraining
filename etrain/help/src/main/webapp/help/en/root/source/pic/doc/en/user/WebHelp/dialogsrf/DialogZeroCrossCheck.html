Zero-Cross Check

Purpose

"Zero-Cross Check" appears in case of zero-cross or nearly zero-cross check of Source Load Units Inventory Unit is required because remaining quantity falls below configured limit of Stock Keeping Unit's Cycle Counting Threshold and zero cross check is allowed in the related Picking Zone Configuration.

      
Screen

In state "Zero-Cross Check" the screen is displayed as follows in the first figure.
In case of counted quantity violates the configured deviation limit the following state of this dialog appears as mentioned in second figure.

      
Prerequisites

For entering the "Zero-Cross Check" state the following prerequisites have to be fulfilled:

  • The current pick operation of Inventory Unit is finished.
  • The remaining quantity falls below configured limit of stock keeping unit's Cycle Counting Threshold.
  • The Pick Zone is configured to allow zero-cross checks.
  • In case of demanded recounting the confirmed quantity violates the configured deviation limit first time.

Special Buttons

Function/Button

Description

Enter "* Quantity Remaining"

With launching this dialog the field "Quantity Remaining" is focused for entering the counted quantity remaining on Source Load Unit in configured pick quantity unit.

Cancel

key "ESC"

Cancel this dialog and return to proceed with picking operation without confirmation of counted remaining quantity on Source Load Unit.

Confirm count result

key "Enter":

The entered quantity is validated against violation of configured deviation limit. In case of first violation recounting is requested. In case no violation of deviation limit or in case of second input the entered quantity is finally confirmed and as the case may be corrected against present quantity of Inventory Unit.

Enter "* Quantity Remaining" for recounting

With first entering a quantity that violates the configured deviation limit the operator is requested to recount and the field "Quantity Remaining" is focused for entering the counted quantity remaining on Source Load Unit again.
