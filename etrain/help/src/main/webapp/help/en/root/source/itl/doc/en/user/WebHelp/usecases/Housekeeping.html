Execution of Housekeeping

Brief Description

The mislocation level of all stored Load Units is calculated periodically using an appropriate batch program only. It is not calculated directly with the storage operation of a Load Unit.

For required relocation transports, Housekeeping periodically creates Transport Requests for the Load Units with the highest mislocation levels using an appropriate batch program. Transport Request execution can be started automatically or by manual release based on the related configuration.

      
Overview

For more information, see the Warehouse Optimization.
The maximum number of relocation transports to be executed within one batch run is configurable per area to avoid any overload of the conveying systems.
The sequence for identification of Load Units to be relocated is defined by the default Housekeeping Configuration and depends mainly on the configured mislocation levels of the different mislocation issues (see Housekeeping Configuration).

      
Triggers and Actors

This is started when:

  • Relocation of not optimal stored Load Units should be initiated once or
  • Relocation of not optimal stored Load Units should be executed periodically.
      
Prerequisites
  • Base configurations for Housekeeping are finished (see Housekeeping Configuration).
      
Required Resources
  • Housekeeping Configuration
  • Cron Jobs.
      
Work Flow
Step

Action

1.

Configure maximum number of relocation transports to execute per automatic Area with the

configuration topic "itl-hsk:HousekeepingMovement.MoveNumberLimit".

2.

Configure batch program for calculation of mislocation levels of the Load Units:

Open Cron Jobs screen to activate the appropriate batch "CalculateMislocationLevelsIteratorBatch".

A daily run at the end of each working day is recommended.

3.

Configure batch program for execution of optimization transports:

Open Cron Jobs screen to activate the appropriate batch "CreateHousekeepingTransportsBatch".

A daily run at the end of each working day is recommended after (!) the batch run described in 2.

 INFO 

Created relocation transports are executed automatically until the configured maximum number of transports is exceeded.

     
Result
  • Transport Requests created for relocation transports of not optimally stored Load Units for automatic Areas
  • Optimized storage utilization after execution
