Demons

COR - Common Services

Demon Class Name

Description

com.dematic.wms.cor.putaway.control.

SwitchDoubleDeepPutawayStrategyDemon

This demon checks for all Classic areas with existing Putaway-Bin-Strategy "Prefer Empty" or "Prefer Partial" and existing Control Data (see below), if there is an aisle with a number of empty bins below a threshold (control data  Putaway.StrategySwitch.ThresholdPreferPartialBins and if there is an aisle with a number of empty bins above a threshold (control data Putaway.StrategySwitch.ThresholdPreferEmptyBins). In case both is true for an area, the strategy for this area is left unchanged.

When the number is below the ThresholdPreferPartialBins, the strategy "Prefer Partial" will be activated, so that Bins that are partially occupied will be chosen first.

When the number is above the ThresholdPreferEmptyBins, the strategy "Prefer Empty" will be activated, so that empty Bins will be chosen first.

If the strategy to be activated does not exist, the demon will create it automatically.

For Multishuttle areas the business logic is similar. Since there dont exist an adequate strategy the switch is implemented on base of configuration topic "cor-putaway:StrategySwitch.isPreferPartialBins".

Precondition: "cor-putaway:StrategySwitch" must be configured for the concrete Areas defining the "cor-putaway:StrategySwitch.AreaDimension".

ORP - Order Processing

Demon Class Name

Description

com.dematic.wms.orp.fullu.control.

ReleaseFLRGroupsDemon

This demon polls the database for pending FLR groups and releases them. This is required in cases where the first try to release a FLR group failed. A FLR group release can fail for example, because a destination of an existing transport request cannot be changed.

com.dematic.wms.orp.orderfulfillment.control.

FillAreaTransportWorkloadDemon

This demon is instantiated once per Area, if the area is related to a Dispo Zone of Reservation Type two phases. If the current number of active outbound transports falls below Area.outboundTransportsMinimalThreshold the demon starts the second phase Disposition until Area.fulfillmentLoadUnitRefillThreshold new Load Units are planned for this Area.

 INFO 

If the configuration topic "orp-orderfulfillment:OrderFulfillmentGeneral.avoidParallelAccessInSecondPhase" is set, Orders that causes a concurrent access exception are skipped temporarily.

 

com.dematic.wms.orp.precubing.control.

DLUQuantityEstimationDemon

This demon performs the pre-cubing method DLU quantity estimation for orders specified by the view V_ORD_READY_4_DLU_QTY_EST. By default the view selects orders in order planning status PRE_CUBING. When a DLU Quantity estimation cannot be executed for an order, the planning status of the order is set to PRE_CUBING_FAILED. Note that the order planning status sequence can be configured by the CD parameter Order.OrderPlanningStatusSequence.
