Pickstation Analysis

Purpose

This screen shows the current situation for Pickstations. All Picks and related Source Base Load Units are displayed in ascending order regarding the current transport to the selected Pickstation

 INFO 

If failed initiation of the Transport Requests for the Source Load Units of a Pick Mission the Pickstation will not be supplied with new assignable Pick Missions. In this case, a specific alarm is raised because not all Load Units of a Pick Mission could be requested (due to exceptional transport situation of the Load Unit). Be aware of these specific Pick Controller Alarms (PIC-0002/PIC-0007).

 INFO 

Once a Pick is booked, the related entry no longer displays. The list must be refreshed to display the current situation.

 INFO 

For specific configuration purposes refer to the related topic in the Warehouse Administration Guide.

Fields

For upper section field descriptions, see the Detail Dialog Workstation.

Field

Description

Load Unit

Unique ID of the Load Unit the inventory to be picked is booked on.

Sequence/Workstation Queue

Sequences assigned Pick Missions. Each removal of a Pick Mission shifts the following entries one step forward.

Group Sequence/Transport Request

Order between whole groups for arriving at the final destination.

Sequence Number/Transport Request

Order in which Load Units of the assigned group should arrive at the final destination.

Sequence/Pick

Processing order of Picks within the Pick Mission or Pick Walk.

Detailed Location

Location the Load Unit is currently booked on.

Source Detailed Location

Location the transport of the Load Unit was started.

Destination Detailed Location

Location the transport of the Load Unit ends.

Pick Mission

Parts of one or more Pick Orders assigned to one Pick Controller for fulfillment. Represents the workload of all Picks to be picked into exactly one Pick Container. Provides additional data to Pick Orders specific for picking.

ID/Pick

Item of a Pick Mission. Defines the primary action of picking inventory from Pick Face to Pick Container. Provides additional data to one Pick Order Line which is specific for picking. One Pick Order Line could be completed by multiple Picks.

Status/Pick

Current execution state of the Pick:

  • active (Pick Mission execution already started and all Picks of the Pick Mission proceed to this status)
  • complete (Pick was executed by the Picker, but not yet confirmed to the Order Fulfillment System)

 INFO 

Additional statuses (created, pending, released, confirmed, cancelled, finished) do not display.

Primary Key/Pick Order

Parts of customer or Shipping Orders assigned to a Picking subsystem for fulfillment.

Primary Key/Pick Order Line

Item of a Pick Order.

Primary Key/Order

Unique Order ID of the Pick Order related Order.

Primary Key/Order Line

Order Line assigned to the Pick Order related Order.

Status/Transport Request

See Detail Dialog Transport Order.

Process Status/Transport Order

See Detail Dialog Transport Order.
