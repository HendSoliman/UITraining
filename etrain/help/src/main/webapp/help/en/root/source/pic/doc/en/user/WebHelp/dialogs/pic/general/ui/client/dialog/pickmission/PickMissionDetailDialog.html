Pick Mission Detail

Fields

Field

Description

ID

Unique ID of the Pick Mission.

Zone

Pick Zone of the Pick Mission.

Status

Current execution state of the Pick Mission:

  • released (Pick Mission was automated time controlled released for execution)
  • queued (Pick Mission is assigned to a Workstation (Goods-to-Person only))
  • active (Pick Mission execution has started, in case of Goods-to-Person the Source Load Units are requested for transport to the Workstation)
  • complete (All Picks of the Pick Mission are complete (i.e. confirmed to the Order Fulfillment System or cancelled)
  • cancelled (Pick Mission was cancelled without the defined execution)
  • finished (related Pick Container was delivered and all required Pick Mission tasks are finished).

Serial

Defines whether the Pick Mission represents a Pick Container that is processed in serial picking mode. In this mode the Pick Order is not split up to multiple Pick Zones within the Pick Controller Zone. The Pick Order is planned in a way that only one Picker is able to operate the whole Pick Order. The processing sequence of the (Customer) Order across all Pick Controllers (represented by different Pick Orders) is given by the sequence number of the Zone and ordered by serialization of related Tasks.

Pick Missions with parallel processing must not be merged with Pick Missions with required serial processing.

 INFO 

In case of serial picking the Pick Container is handed over from one Pick Zone to the next one within the defined chain. The destination for the transport of the Pick Container is determined project specifically with use of a policy which is defined by the configuration topic "orp-pickconfirmation:PickConfirmationGeneral.SerialPickingDestinationPolicy". With arrival of the Pick Container the related Pick Mission of the assigned Pick Controller (PIC) switches from status "created" to "pending" and is assigned to that Pick Container and thus ready for further picking.

Pick Container

Associated Pick Container. All Picks of this Pick Mission are collected in this Pick Container.
The Pick Container groups all Picks of one Pick Mission and contains the already picked Inventory Units.

Pick Container Alias

After assignment of Pick Container to a Put Location, is usually the Alias of Put Location. If Person-to-Goods Picking, used to verify the correct Pick Container while picking.

Handling

Urgency

Set on behalf the Pick Order. Defines the processing sequence. Lower value indicates a previous execution.

Blocked

If set, Pick Mission is blocked due to concurrent use of resources by other Pick Missions and cannot be processed until unblocked (with respect to the activation of the "Blocking Management" of the related Pick Zone).

 INFO 

 To avoid concurrent transports of the same Source Load Units to different Pick Stations, those Pick Missions that require Load Units which are currently not stored get blocked until all required Load Units are stored and thus able to be retrieved. The execution of blocking and unblocking occurs on the Pick Mission level where all required Source Load Units are checked.  The Pick Mission is unblocked only if each single Load Unit is actually stored. For optimization purposes, blocked Pick Missions can be assigned and started if all required blocking Source Load Units are already during transport to the required destination and this kind of optimization is requested.  Further optimization can be configured on the Pickstation level to start blocked Pick Missions as well.

Service Qualifier

Set on behalf of the Pick Order to request a specific processing quality. May be project specific. Only Pickstations with the configured Service Qualifier can execute these Pick Missions.

  • default (No specific handling deviation is required)
  • rush-order (Rush Orders can be clustered in a specific way for rapid processing)
  • repacking (Repacking Orders are picked in a specific way)
  • block (Picks of different Orders are picked into the same Pick Container. Picks of multiple Pick Orders are clustered to a Pick Mission. This Service overrides the original one which could be launched through the Pick Order anyway.
    Be aware of a special demon (HandleDeferedPicksDemon) which groups these Picks to a Pick Mission separately)
  • RAPIDPAL (Orders for automated robot palletizing on specific AMCAP Station).
  • ERGOPAL (Orders for palletizing on specific ErgoPal Station).

Last Assigned Terminal

ID of the last assigned Terminal where the Pick Mission is or was processed.

Failed Request

Indicates whether the last request of Source Load Units for this Pick Mission failed for any temporarily reason. This flag will exclude the Pick Mission from further handling until the 'ResetMissionWithFailedRequest' demon removes it periodically.

Failed Request Time

Point in time of the last failed request.

Times

Start Time

Actual Start time of the Pick Mission.

End Time

Actual End time of the Pick Mission.
