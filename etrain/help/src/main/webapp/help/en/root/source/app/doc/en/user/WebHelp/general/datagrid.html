Data Grid

Purpose

This chapter describes the usage of data grid based dialogs.

 

      
Overview

The data grid is one of the central controls of the UI. It encompasses the following elements from top to bottom:

Element

Description

Filter row

The filter row allows the specification of selection criteria which limits the displayed datasets to only those which matches the specified selection criteria.

The selection criteria for a table column are always directly above the column, even if the column layout is resorted.

Column header

The column header displays the name of the column.

By clicking on the header area the user can change the sorting settings of the corresponding column.

Tabular data

The data area displays the actual data. In order to increase readability, a zebra coloring schema is used.

The user can select one or more line via left click in order to trigger an action on them. This can be achieved by:

  • pressing an enabled button in the button bar
  • opening the context menu via right click and choosing the appropriate menu item therefore.
  • double clicking on a line. This triggers the default action (in general edit/view) of the context menu.

Personalization

The user is able to customize a data grid to optimal fit his personal preferences and use cases he is working on. The following properties of a data grid can be customized:

  • Column Layout
  • predefined Layout Sets
  • predefined Filter sets

 

The first column of a data grid on the right hand side acts as a row header. It contains the following functionality:

  • For the filter row (): resetting all currently applied selection criteria
  • Column header (): resetting all currently applied sortation settings
  • Data row: just the index of the row beginning with 1 is displayed.
      
Filtering

For each column in a data grid a selection criteria can be specified which limits the set amount of data sets being displayed to only those matching the selection criteria. The selection criteria is always entered in the text field directly above the column header. This is still true even when the column ordering has been adapted by the user. Selection criteria applying to different columns are logically combined by and AND relation, i.e. a data set must match all specified selection criteria in order to show up in the grid.

Specifying or changing a selection criterion does not implicitly trigger a refresh of the grid. In order to minimize the load on the database, the refresh must be explicitly triggered by the user either by pressing the refresh button of by pressing ENTER. As long as the selection criteria and the data sets being displayed in the grid are not in sync, the grid is grayed out. In this status the grid is still fully usable, i.e. any actions can be triggered for s data set.

The exact behavior of a filter field depends on the data type of the column:

Data Type

Description

String

The wildcard * can be used in the selection criterion to match any string.

 

Suppose you want to search for all order names beginning with 110. In this case, you enter the string 110* in the order number selection field. If you now click the Refresh () button, in order to refresh the record display, only orders whose order numbers begin with 110 are displayed.

 

The following picture shows how logical operators can be used to combine several selection criteria. The shown popup dialog appears as soon as text is entered in the filter field.

Choose operator by clicking on it, shown to left of text field

The following logical operators are available:

  • = Equals-Operator (default, not shown left)

  • ! NOT-Operator: filter data sets that do not match criteria

  • | OR-Operator: filter data sets matching one or more filter criteria

  • !| NOR-Operator: filter data sets that do not match at least or more filter criteria

  • Ø Null-Operator: filter data sets with empty values (not yet set)

  • !Ø NOT-NULL-Operator: filter data sets with not empty values (values set)

The example show in the picture above means: select all data sets having a column "User Name" which starts either with "S" or "a".

Numerical

Only numbers are allowed as filter criteria for numeric values

Additional logical operators:

  • < less than operator: filter data sets with values less than criteria

  • <= less or equal than: filter data sets with values less or equal than criteria

  • > greater than operator: filter data sets with values greater than criteria

  • >= greater or equal than: filter data sets with values greater or equal than criteria

Domain values

Domain values may be selected by a combo box  which allows to select multiple filter criteria

Sorting

The grid data may be sorted on each column. The sort order can be changed from ascending to descending by simply clicking on the column header label. The sorting direction is indicated by an arrow. Sorted columns are consecutively numbered to indicate the sort order.

Layout

The context menu offers the Layout sub menu:

The data grid Layout encompasses multiple aspects as described below:

Layout

Description

Column Width

 To adapt the width of a column drag & drop the right hand column separator.

Column Visibility (Change Layout)

 Select a line in the data grid and open the context menu. Choose "Layout ... --> Change Layout"


The right hand side entitled "available" shows the columns which are currently not visible. The left hand side shows the columns which are visible. The user can transfer columns from one side to the other by using the horizontal arrow icons or by drag & drop. The buttons showing two arrows transfer all columns.

Column Ordering (Change Layout)

 Proceed as described above but use the vertical arrows to change change the ordering.
Auto Resize Columns By this options the available data grid space will be spread over all columns. No horizontal scroll bar will be available. The option makes sense for Data Grids having only few columns. By default this option is turned off, column widths are set to their needed size and the horizontal scrollbar becomes available.
Layout-Sets The current selected Layout can be saved as a Layout-Set. A detail dialog is opened to maintain Layout-Sets.
Filter-Sets Current Filter and sorting settings can be saved as a Filter-Set. A detail dialog is opened to maintain Filter-Sets.
Reset Layout All column layout options (see below) will be reset to default values.
Reset Column Order Reset only the changed column order and order columns by default.
Reset Column Widths Reset changed column width and apply default dimensions.
Reset Column Visibility Reset changed hidden or shown columns and apply default column visibility
