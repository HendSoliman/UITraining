Overview

RF-Counting

Functions
  • Counting for Stocktaking purposes using a RF-Terminal in manual guided areas
  • with positions are ordered by the location sequence
  • with activated or deactivated required additional check of the stored Load Units on a Location
  • with activated or deactivated required scan of the Load Units
  • configurable bookings for different confirmations (Counting Confirmation Handling)
  • Counting confirmation for a missing Load Unit
  • Use of different Quantity Units and counting of cracked quantity as well
  • Skip a Location for later processing
  • Automatic loading of available Counting Orders
  • Automatic creating and executing of positions for recounting dependent on the configuration.

A detailed description of the operational steps is given in the use case "Person-To-Goods Stocktaking Execution (Online)".

      
Overview

Screen

Description

No Counting Order available

This screen indicates that currently no Counting Orders are available.

Scan Location

The counting operation always starts with the scan of the requested Location.

Check number of Load Units at Location

In case of required by configuration the number of stored Load Units on this Location is to be checked.

Check number of Load Unit Scan

In case of required check of the number of Load Units each Load Unit is to be verified by a single scan. The number of to be scanned Load Units is defined by the previously entered number of located Load Units.

Scan Load Unit

In case of required by configuration each new Base Load Unit to be counted must be verified by a scan.

Scan SKU

In case of required by configuration each inventory to be counted must be verified by a scan.

Counting / Recounting

The counting and recounting screens provide all counting relevant information and various functions for different kind of confirmation of the counting result.
