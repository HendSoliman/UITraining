Screen |  Fields

Detail Dialog Counting Order

Purpose

This screen opens when creating, viewing, or modifying a Counting Order.

 INFO 

For configuration purposes refer to the related topic in the Warehouse Administration Guide.
     
Screen

      
Fields

Field

Description

Order

Stocktaking Order of this Counting Order.

ID

Unique ID of the Counting Order.

Client

The Client of the Inventory counted by this Counting Order is the owner of the Counting Order.

Stocktaking Zone

Stocktaking Zone of this Counting Order.

Stocktaking Type

Type of stocktaking execution. Used in the related Stocktaking Zone. Defines basic fulfillment procedures for Counting Order execution.

  • Goods-to-Person (Counting is executed using the screen for Counting. Load Units will be requested to a Workstation)
  • Person-to-Goods Offline (Counting is executed using printed Counting Lists. Requires manual entry of counting results on the Counting Orders)
  • Person-to-Goods Online (Counting is executed using mobile terminals. Operator is guided to locations with requested counting operation).

Status

Status

Current execution state of the Counting Order:

  • pending (Counting Order is awaiting release for execution)
  • released (Counting Order is released for execution)
  • active (Counting Order execution already started)
  • to confirm (Counting Order was counted completely and must be confirmed against the Stocktaking Order)
  • finished (Counting Order execution finished and the Counting Order is fully completed).

Configuration Status

If Order configuration was accomplished and the order can be processed. Allows users to interrupt their work and continue Order creation later.

Handling

Default Destination

Used for Goods-to-Person Stocktaking processes as Default Destination Location for outbound transports of the Load Units.

This is the source location of the workstation where the counting order was processed or is currently running.
Terminal Terminal ID where the Counting Order was processed or is currently running.

List already printed

Enabled with the print of the Manual Counting Order List.

Times

Counting Start Time

First confirmation time of any counting result of the Counting Order.

Counting End Time

Confirmation time of the last counting result of the Counting Order.

Planned Earliest Fulfillment Time

If configured, scheduled automatic release of Counting Orders should be the earliest possible time for Counting Order release.

Planned Latest Fulfillment Time

If configured, scheduled automatic release of Counting Orders should be the latest possible time for Counting Order release.

Owner

Source of the Counting Order. Directly linked to the owner of the Stocktaking Order:

  • host (initiated from an external Host using the host interface)
  • operator (based on a manually created Stocktaking Order)
  • system (created by the system based on the Stocktaking Order; which was created by the system based on Stocktaking configurations).

Task

Counting Orders are handled as Tasks. The assigned Task contains additional attributes of the Counting Order (see Tasks).
