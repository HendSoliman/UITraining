Released order doesn't start

Symptom

A released order doesn't continue. The order still stays in status "RELEASED" and doesn't run.  

      
Brief Description

A released order should have a group of corresponding full load unit retrieval objects. These objects have a common task. It is necessary to release this task if the automatic release processing is not active in the current configuration.

Most important dialogs for analysis and trouble shooting support:

  • Dialog Retrieval Administration.
      
Possible Reasons

The user forgot to release the task of the corresponding full load unit retrieval objects. 

      
Detail description

When a new order is submitted into the order management it is required to release the order for processing. A new order has status "PENDING". After an order is released it changes it's status from PENDING to RELEASED. There are three possibilities to release a pending order:

release the order via the release button in the order administration GUI.

release the order via the release button in the task administration GUI.

the order is released automatically by the task management in background

The third possibility needs an appropriate configuration of the task management.

While the release is running the order management instructs the retrieval strategy to compute proper load units necessary for the fulfillment of the given criteria in the order lines. The result of this computation is represented by full load unit retrieval object. All resulting full load unit retrieval objects have one common corresponding task. This task has status pending. To start the physical retrieval of the computed load units it is necessary to release the corresponding task. There are two possibilities to release this pending task of the full load unit retrieval objects:

release the corresponding task via the release button in the task administration GUI.

the corresponding task is released automatically by the task management in background

The second possibility needs an appropriate configuration of the task management.

When the task is successfully released the status of the task is "RELEASED". The status of the order and the corresponding order lines should be "ACTIVE"
