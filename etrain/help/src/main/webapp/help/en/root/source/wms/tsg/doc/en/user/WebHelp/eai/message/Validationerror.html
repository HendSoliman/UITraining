Validation error

Symptom

A message can't be processed. It has an error message of type EAIValidationFailedException.

 

      
Brief description

In the Transfer Tables dialog a message went into state E (Error). Furthermore the error text indicates that an EAIValidatonFailedException has been occurred.

 

      
Categories
  • EAI
  • Mapping

 

      
Possible errors

In case of validation failed, following reasons are possible:

  • A mandatory field has no value
  • Inconsistence data type
  • Inconsistence format
  • Wrong values

 

      
Detail description

A message in state N (New) is available in the Transfer Tables dialog. After processing this message it went into state E (Error) and the error text contains an EAIValidationFailedException.

 

      
Trouble shooting

Determine what exactly happened

Step

Action

1

Look in the Transfer Tables dialog for the error message.

2

Look in the application trace files for a detailed error description.

 

In all EAI Validaton error cases you have to find out, which source value is being mapped into the target field, where the validation exception occurred. For this purpose proceed as follows:

Step

Action

1

Go to the Transfer Tables dialog

2

Select the suffix of the table, which contains the erroneous message.

For example: R01 for all from SAP received messages. (see: EAI Translator Configuration Dialog[1])

3

Select the erroneous message and press Detail Dialog button.

4

Switch to the tab error and find out in which field the exception occurred.

5

Go to the Mapping Dialog[1] and load the mapping which was used for this message.

6

Within mapping dialog: Find the field (where the exception occurred) in the target tree and find out which source value is beeing mapped in this field.

7

Go back to Transfer Tables dialog, choose again the erroneous message and opens the detail dialog.

 

If the message is a SAP IDOC, because only SAP messages are editable.

Step

Action

1

Find out the source field whose value is being mapped.

2

Open the Transfer Tables dialog, select the erroneous message and open the detail dialog.

3

Find in the detail dialog the source field.

4

Modify the value of the selected field and press the save button

5

Close the detail dialog

6

Reset the erroneous message

Additional information

The modification of source message within the detail dialog of the Transfer Tables dialog is a very helpful feature, but in order to received the message with correct values the host system should be checked.
