Management Information System

Overview

The Management Information System (MIS) as a subsystem of the Dematics Dematic iQ® is covering the strategically relevant management information based on long term data within a warehouse. It collects and displays trend information by providing reports on specifically defined Key Performance Indicators (KPIs).

The KPIs are based on archived data which is periodically or event triggered transmitted from productive system to a separate MIS database. The data is time based aggregated into the time slices according to the KPI specific configuration.

The following list gives an overview of all available KPI reports.

      
Reports

KPIs - External Interfaces

Description (Dimensions)

Host Document Movements

Number of all document movements to and from the connected external systems (host / message type / direction / status).

 

KPIs - Transport

Description (Dimensions)

I-Point Information

Amount of I-Point movements (location / transport unit type / contour error).

Transport Orders Over Connection

Number of transport orders per connection ().

Active Transport Orders

Average amount of active transport orders in the processing queue (subsystem).

 

KPIs - Storage

Description (Dimensions)

Load Units per Storage Area

Average number of load units in each storage area/aisle (client / SKU / best before date).

Storage Utilization

Percentage of space and storage utilization (area / area type / aisle / multiple status).

Relocations

Number of completed transport requests (client / transport request type / source area / destination area).

 

KPIs - Device

Description (Dimensions)

Connection Error-Duration

Time spent for a MTS id in error/no error states (connection number).

Connection Locked-Duration

Time spent for a MTS id in released/locked states (connection number).

 

KPIs - Order Fulfillment

Description (Dimensions)

Announced Shipping Orders

Number of created shipping orders (client / order type).

Finished Shipping Orders

Amount shipping orders finished with average amount of units (client / order type).

Full Load Units Handled

Number of completed full load unit retrievals (client / load unit type / source location / destination location).

 

KPIs - Receiving

Description (Dimensions)

Advices

Number of advices averaged advised pallets (advice type / status).

Advice Lines

Number advice lines (client / advice type / supplier / host advice / mode advice).

Advice Finished

Number of completed advices and some sum information (client / advice type / supplier / host advice / mode advice).

Load Units Received

Information concerning receiving performed during the time slice (client / advice type / supplier / host advice / mode advice / load unit type / sku).
