Detail Dialog Location Type

Purpose

This screen opens when creating, viewing, or modifying Location Types.

 INFO 

For configuration purposes refer to the related topic in the Warehouse Administration Guide.

 INFO 

The Unit of the displayed dimensions are given by the appropriate Base Quantity Units defined by the "Dialog Unit of Measure".
     
Screen

      
Fields

Field

Description

ID

Id of the Location Type for identification.

Capacity

Maximum number base units of Load Units that can be stored on a Location of this type. Usually one Load Unit consumes one base unit on a Location. If the Location is a 2-3 Location that can load 2 bigger or 3 smaller Load Units, it is useful to assign the capacity 6 to the Location Type so that the smaller Load Unit consumes 2 and the bigger Load Unit consumes 3 base units. This capacity is mainly used to book Load Units on Bins in Areas with automatic disposition (the system selects Location and position for a Load Unit). Recommended capacity value for Location Types of other Locations (e.g. transport points) is 1.

Booking Type

How to determine and book a position of a location of this type:

  • default (applies to all regular bins and transport points and considers the configured layout)
  • throughput (applies to unidimensional bins with flowing throughput positions e.g. bins of a flow rack).

LU Types assigned

If the Location's Type has Load Unit Types assigned.

Dimension

Height

Height (in mm) must fit to the height of Load Units (only in case of exceeding 0).

Width

Width (in mm) must fit to the width of Load Units (only in case of exceeding 0).

Depth

Depth (in mm) must fit to the depth of Load Units (only in case of exceeding 0).

 INFO 

The value represents the depth of each single deepness level.

Deepness

The discrete deepness defines the number of Load Units which could be stored in depth direction.

  • 1 (single deep)
  • 2 (double deep)
  • 4 (quad deep).
