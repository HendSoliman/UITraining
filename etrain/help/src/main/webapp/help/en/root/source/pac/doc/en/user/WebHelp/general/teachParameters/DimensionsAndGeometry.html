Teach Dimensions

Parameter Name:
  • teachLength
  • teachWidth
  • teachHeight

Display Name:
  • Teach Length
  • Teach Width
  • Teach Height

Description:

Dimensions of tight bounding box of a representative item of a packaging variant.

The following schema must be used for orientation:

  • Items are always oriented in their natural vertical orientation.
  • Height is the vertical dimension
  • Length is the longer one of the horizontal dimensions
  • Width is the shorter one of the horizontal dimensions 

Unit:

mm

Example:

200

Limitations:

For AMCAP Compliance, dimensions must comply with the following restrictions:

  • 150 mm ≤ teachLength ≤ 650 mm
  • 100 mm ≤ teachWidth ≤ 650 mm
  • 50 mm ≤ teachHeight ≤ 450 mm
  • 1 ≤ teachLength/teachWidth  ≤ 4

Data Source:

Measurement at Teach-In Station.

Step Dimensions & Weight.

Measuring Method:

The dimensions are measured mechanically:

  1. Place item between two plates.
  2. Measured dimension is vertical to the plates.
  3. Apply moderate pressure to the plates
  4. Measure dimension using automatic measuring button

Used for:
  • Virtual Load Unit for single item
  • Depalletizer
  • Case Conveyor
  • Dimension Check
  • Multishuttle

Usage for Depalletizer:

All dimension are transferred to the Depalletizer System (DPS) in the Depalletizing Mission.

The Depalletizer actually uses only the height of an item as layer height for positioning of the depalletizing head.

Usage for Case Conveyor:

Teach Dimensions are transferred to the Case Conveyor System (CCS) in the Transport Unit Mission (AMCAP TUMI, see Message Trace) for a layer (type=LYCS) at de-palletizing.

The CCS uses the teach dimensions for the following purposes:

  • Tracking
  • Determination of actual orientation.
    Extensions in x- and y- orientation (conveyor coordinates) are compared with teach dimensions under consideration of Dimension Tolerances.
  • Pairing of items at pick station
    The item length is used to decide how many items fit on the lift.

Usage for Dimension Check:

At the Dimension Check Station, the measured dimensions are compared with the Teach Dimensions under consideration of Dimension Tolerances.

If all measured dimensions can uniquely assigned to one Teach Dimension under consideration of its dimension tolerance, the item is rated OK.

Otherwise, it is rated non OK and sent to the reject lane.

Pack Dimensions

Parameter Name:
  • length
  • width
  • height

Display Name:
  • Length
  • Width
  • Height

Description:

The pack dimensions are used as dimension base for pallet planning.

For Length and Width, they are derived from the Teach Dimensions by adding a small "security distance".
This accounts for variances in real dimensions and is required to avoid collisions during palletizing.

The pack height is taken identical to the teach height in order to avoid the accumulation of height differences with increasing palletizing height.

Unit:

mm

Example:

254

Limitations:

When increasing the teach length and teach width, module measures are regarded.

Module measures are specific length values which divide a pallet length and/or a pallet width into an integer number of portions.

By default, the "security distance" to be added, is a system parameter.

If, by adding this parameter, the next module measure is exceeded, the respective dimension is set to this module measure.

Toppling Angles

Parameter Name:
  • topplingLengthWifth
  • topplingWidthLength

Display Name:
  • Toppling Length
  • Toppling Width 

Description:

The toppling angles are used as criterion whether or not a packaging variant can be used for AMCAP or not.

The Product Specification for AMCAP defines a critical toppling angle below which packaging variants cannot be used for AMCAP.

This angle applies to the most stable orientation
