Counting Operation

Purpose

This screen is used to perform counting operations for a specific Counting Order. It provides all functionality for counting purposes at Goods-to-Person Workstations for any Load Unit Type. Features include detailed instructions and information of the Counting operation, additional information regarding ordered Source Load Units, and confirmation of the Counting operation.
All operations will be done in single steps as described below.

      
Screen

After the operator has select an Error Code for confirmation this will be displayed in the screen:

      
Fields

The left frame provides information about the requested Load Units whose inventory must be counted for the current Counting Order.

Field

Description

Load Unit

Unique ID of the Base Load Unit.

Number of Counts

Number of remaining counts within this Load Unit.

The upper, middle frame provides information about the current Counting Order and Source Load Unit and current operating instructions regarding this Source Load Unit.

Field

Description

Counting Order

Unique ID of the current Counting Order in process.

Load Unit

Unique ID of the current Source Load Unit.

The lower, middle frame provides instructions for the current count operation and information about the SKU.

Field

Description

SKU

Uniquely identifies a Stock Keeping Unit within a Client.

Description

Text description of the SKU.

Client

Unique ID of the Client as the owner of the SKU and Inventory Units to count.

Batch

Batch of the Inventory Unit to count.

Best Before Date

Best before date of the Inventory Unit to count.

Counting Method

The counting method of the SKU indicates the activity: counting, metering, weighting, estimating, etc.
Error Code

Counting Error Code defines the confirmation of the counting operation:

  • OK (successfully completed)

  • load unit missing (counting not possible, because no Load Unit found)

  • load unit not available (counting not possible, because the mentioned Load Unit is currently not available for counting)

  • inventory not available (counting not possible, because the mentioned Inventory Unit is currently not available for counting).

Counted Quantity

Counted quantity within the current Load Unit for current SKU.

Quantity Unit

Quantity Unit of the displayed quantities.

The Inserts (geometrical stacked Load Units) of the current Load Unit are marked as follows:

  • Red: selected Insert to be counted
  • Yellow: not selected, occupied Insert
  • Green: not selected, empty Insert.
      
Special Buttons

Button

Function

Confirm input within the current state, e.g. scan of the current SKU, push away of Load Unit or counting result.

Cancel current action and change to "Count" or "Recount" state.

This function allow to change the Error Code. The selected Error Code will cause a different confirmation handling.

Select another Quantity Unit for current SKU if incorrect for the Counting operation.
     
Steps
  • Standby
  • Wait for Load Unit (optional)
  • Scan Load Unit (optional)
  • Scan SKU (optional)
  • Count/Recount
  • Push Load Unit.
