Packaging Variants

Purpose

This topic describes how Packaging Variants are handled in Dematic iQ®.

 

      
Structure and Dependencies

 

      
Terms

In order to handle different packaging types for products handled in a warehouse, Dematic iQ® uses three different levels:

Stock Keeping Unit
  • The Stock Keeping Unit (SKU) uniquely identifies a specific product for which inventory can be managed within a warehouse
  • Stock Keeping Units are identified through a unique SKU ID.
  • For a Stock Keeping Unit, there may exist multiple Quantity Units.
    One of these is the Base Quantity Unit

For management of SKUs, see screen Stock Keeping Units.

SKU Quantity Units
  • SKU Quantity Units distinguish the different packaging types which may exist for an SKU.
  • In the sense of GS1 (see Reference 1), an SKU Quantity Unit is considered as a Trade Item.
  • Within Dematic iQ®, an SKU Quantity Unit has a unique ID
  • An SKU Quantity Unit can have a Product Code.
    The GTIN is usually taken as Product Code
  • According to GS1, Trade Items shall be assigned a unique GTIN
  • Dematic iQ® can also support scenarios where the same Product Code is used for SKU Quantity Units of different SKUs.
    (It has been found that this is sometimes used within restricted distribution within a company)
  • If the Product Code does not uniquely identify an SKU, user intervention is required whenever items need to be identified exclusively through their barcode.
  • In multi-client systems, Dematic iQ® supports, of course, scenarios where the same products are traded by different clients.

For management of Trade Units in Dematic iQ®, see screen SKU Quantity Units.

Packaging Variants
  • It is possible that Trade Items exist in different variants which are equivalent from a trading perspective (same GTIN) but which are different from a logistics handling perspective.
  • Differences may originate from variations in packaging material and form or from other reasons.
  • Often these differences occur when the same Trade Item (SKU Quantity Unit) is produced by different suppliers.
  • Different Packaging Variants have the same Product Code but differences in physical properties which are significant for handling these items in an automated system, especially an AMCAP system.
  • Packaging Variants are managed in Dematic iQ® as "Palletizing Quantity Units"
  • They are distinguished through a unique ID which is typically specified in the Teach-Process.
    This ID is usually used only internally within Dematic iQ® and not known within the commercial system (ERP system)
  • Often ERP systems have knowledge about the dimensions and weight of Trade Items.
    In order to be able to use data captured within the Teach Process within the logistic system, it is possible to define specific Packaging Variants as default variants.
    In this case, an SKU Quantity Unit will use the dimensions measured for its default Packaging Variant.

For management of Packaging Variants in Dematic iQ®, see screen Palletizing Quantity Units.
