Detail Screen Counting Confirmation Handling

Purpose

This screen opens when creating, viewing or modifying a Counting Confirmation Handling configuration.

      
Screen

      
Fields

Field

Description

Zone

Zone for which this configuration entry is valid (see Detail Dialog Zone).

Error Code

Counting Error Code for which this configuration is valid:

  • OK (successfully completed)

  • load unit missing (counting not possible, because no Load Unit found)

  • load unit not available (counting not possible, because the mentioned Load Unit is currently not available for counting)

  • inventory not available (counting not possible, because the mentioned Inventory Unit is currently not available for counting).

 INFO 

If "Inventory Correction is not Required" for "load unit missing", the missing Load Unit is not deleted immediately but booked on a fix Location for further investigation. The ID of the Location must be configured in the
configuration topic "cor-inventory:InventoryUnit.lostAndFoundLocation" for each affected Area.

Logistic Error Code

Set within the Load Unit. Specifies errors related to logistics aspects of a Load Unit. All values are defined in the appropriate field descriptions of the Detail Dialog Load Unit.

Logistic Error Reason

This configured logistic error reason is set within the Load Unit and provides a free text to explain the reason of the logistic error for the involved Load Unit.

Movement Type

For inventory booking operation. Values are configured in Inventory Movement Types.

Movement Reason

For inventory booking operation. Values are configured in Inventory Movement Reasons.

Alarm Type

If set, the related alarm is raised during this Counting Confirmation.

Handling Plug In

Class that implements error handling, invoked in addition to the handling already defined by this Counting Confirmation Handling.

Inventory Correction Required

If enabled, inventory on the Source Load Unit is corrected.

Lock Required

If enabled, the Base Load Unit of the related Counting Order Line is locked.
