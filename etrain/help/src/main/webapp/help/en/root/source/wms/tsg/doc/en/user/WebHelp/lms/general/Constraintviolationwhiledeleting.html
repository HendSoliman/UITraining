Constraint violation while deleting

Symptom

One or more entitles can not be deleted, caused by a violation of a database constraint. 

      
Brief Description

Database constraints ensure referential consistency, i.e. no database entry can be deleted when it is referenced by another entry.  

      
Possible Reasons
  • The deleted entity is referenced by one or more another entities.
      
Trouble Shooting

Determine what exactly happened

Step

Description

1

Look for valid objects referencing the deleted entity

2

Look for invalid objects referencing the deleted entity

 

Reason

Description

1

The deleted entity is referenced by one or more valid objects

2

The deleted entity is referenced by one ore more invalid objects. An invalid object indicates a general problem and can be the consequence of an inconsistent system state.
Before continue with troubleshooting: Contact your WCS system administrator!

 

Reason 1:

Step

Action

1

Open the administration dialog for the referencing object (The referencing object is shown in the error message dialog)

2

Delete the referencing entity (be aware of consequences!)

3

If step 2 can not be performed cause of constraint violation, repeat step 1 and 2 with the referencing object

 

Reason 2 (Before continue: Contact your WCS system administrator!):

Step

Action

1

Open the database

2

Identify the referencing object (The referencing object is shown in the error message dialog)

3

Delete the referencing entity from database (be aware of consequences!).

4

If step 3 can not be performed cause of constraint violation, repeat step 2 and 3 with the referencing object
