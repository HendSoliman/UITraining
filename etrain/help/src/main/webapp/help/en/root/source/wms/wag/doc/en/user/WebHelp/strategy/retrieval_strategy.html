Retrieval Strategy

Configuration

Retrieval Strategies can be configured in terms of assigning order sequences to strategy aspects.

 INFO 

It is recommended to load the strategy configuration of the demo project. This configuration should then be adapted to the requirements of the project.

 INFO 

Configuration topic "cor-retrieval:Retrieval.ispreferFullLoadUnits" determines if the retrieval strategy tries to select Full Load Unit Retrievals before it tries to select Picks. A strict FIFO can only be achieved when this parameter is set to false.

 INFO 

All retrieval strategies prefer by default partially free bins as obligatory lowest 'order by' criteria. This empties bins whenever possible and allowed by all configured retrieval strategies.
This is done with respect to the location fill level (expressed by the reservation capacity of the location).

 INFO 

For palletizing orders Retrieval executes an implicit strategy to avoid consecutive requests being retrieved from same subgroup to optimize lift workload. See "cor:retrieval.Retrieval.lastRetrievedPalletizingSubgroupsListMaxCapacity".

For palletizing orders other strategies are considered secondary.

Retrieval Strategies

Order By

Description

Batch

Matching inventory is sorted by the Batch of the Inventory Unit.

Best Before Date Retrieval

Matching inventory is sorted by the Best Before Date with preference to the oldest date.

First In First Out Retrieval

Matching inventory is sorted by the FIFO date of the Inventory Unit with preference to the oldest date.

Group Fill Level

Consideration of the fill level of the Groups (aisles) with preference to the aisle with the highest fill level.

Group Inventory Balance

Balancing of SKUs over the Groups (aisles). A specific discriminating aspect of Inventory is also considered (e.g. batch). If yet allocated and order related Inventory, this aspect is the Order ID. For Inventory Balance principles, see the Inventory Balance Dialog.

Group Transports

Prefer Groups (aisles) with lower number of currently planned outbound transports.

Hot Zone

Prefers Inventory Units from locations that are flagged as Hot. For more information, see Detail Dialog Location.

Pick Quantity Optimization

Matching inventory is sorted by the free quantity on the Inventory Unit.

Quantity Match

Prefers Inventory Units that have exactly the free quantity requested. Then Inventory Units with more quantity free than requested with a lower free quantity are preferred. For the remaining inventory units, a higher free quantity is preferred, e.g. Requested Quantity = 100,
Quantities of Existing Inventory Units: 80, 90, 99, 100, 101, 105, 110.
This leads to the following sortation: 100, 101, 105, 110, 99, 90, 80

Reservation Count

Matching inventory is sorted by the number of reservations on the Base Load Unit with preference to fewer reservations. This helps to reduce concurrent accesses to Inventory Units and thus, Load Units.

Subgroup Fill Level Balance

Consideration of the fill level of the Subgroups (Levels) with preference to the Level with the highest fill level.

Subgroup Inventory Balance

Balancing of SKUs over the Subgroups (Levels). Furthermore a specific discriminating aspect of Inventory is considered (e.g. batch). If yet allocated and order related Inventory, this aspect is the Order ID.  For Inventory Balance principles, see the Inventory Balance Dialog.

Subgroup Transports

Prefer Subgroups (Levels) with lower number of currently planned outbound transports.

 INFO 

Project specific Retrieval strategies can be introduced by extending the Retrieval database views. If this is insufficient, project specific Retrieval strategies must be implemented.
