Inventory Movement Type

Purpose

This topic describes the configuration of the Inventory Movement Type.

      
Configuration

Inventory Movement Types represent a classification of reasons for host-relevant inventory changes.

The following inventory movement types are required and automatically created by system initialization:

Inventory Movement Type

Description

BZCHANGE

Default movement type for booking zone changes.

INTREC

Default movement type for internal (not host advised) receiving of inventory.

RECTOFREE

Default movement type for receiving of free (not set on hold) inventory.

RECTOLOCK

Default movement type for receiving of locked (set on hold) inventory.

INTSHIP

Default movement type for internal (not host requested) shipping of inventory.

 

Inventory movement types with the following properties must be configured for exactly two entries:

  • Direction = in and the flag Default for Correction is set.
  • Direction = out and the flag Default for Correction is set.

In the demo warehouse, these movement type IDs are:  INVPLUS and INVMINUS.
