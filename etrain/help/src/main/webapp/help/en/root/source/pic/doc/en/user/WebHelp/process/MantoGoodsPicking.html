Basic Processes Person-to-Goods Picking

Introduction

The specialized Pick Controller for Route Based Picking Processes supports Person-to-Goods Picking with:

  • Offline picking paper based
  • Online picking with web based RF-Terminals.
      
Configuration

In addition to the basic configuration of a Pick Zone only Worker Groups must be specifically configured to allow for picking operations. Each Picker, who should be assigned to Person-to-Goods picking tasks must be configured as Worker.
If the workload should be assigned to special terminals instead of requesting Picker a specific configuration for Terminal Zones must be provided.

      
Picking with RF-Terminals

Picking in Person-to-Goods Pick Zones is done using the RF-Picking screens which provide all functionality for picking purposes with Person-to-Goods picking strategies. Single and multi-order picking are supported. For multi-order picking, it is assumed that the Pick Containers are transported on special Pick Cart.
The Pick operation with all available confirmation codes, Reverse Picking execution, and Pick operation splitting is supported by detailed instructions and information by separate states for each operational step.

The Workload is assigned to the Picker by direct request only. Up to a configured limit single Pick Missions are grouped to a new Pick Walk.

      
Offline Picking

Offline Picking in Person-to-Goods Pick Zones is done using the Pick Walk where multiple Pick Walks can be activated and separately finalized. With activation, the configured Pick List prints on the Pick Zone's default list printer to support the Pick Walk.
Finalization occurs for a single Pick Walk with confirmation of each single Pick with support of different Confirmation Codes and preparation of Pick Container.

      
Picking Documents

A pick list contains a list of all Picks of one Pick Walk. Pick lists can print for such Pick Walks, where the associated Zone is used for Person-to-Goods picking. At the activation time a Pick list prints automatically (see "Print Pick list(s)").
One pick label contains all Pick related information, such as SKU, requested and picked quantity, etc. (see "Print Pick label(s)").
