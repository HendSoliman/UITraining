Unloading General

Description

Overall description of the unloading processes based on Deliveries with the basic steps only.

      
Triggers and Actors

The use case is started when

  • A basic unloading process needs to be executed with use of the basic steps only.
      
Prerequisites
  • Prepared warehouse infrastructure regarding Locations, Docks, Area Access
  • A Purchase Order is available
  • An Advice has been transmitted by the supplier (optional)
  • A truck (optional with a trailer) has been arrived at the gatehouse
  • The truck driver has handed over the shipping documents
  • An operator in the gatehouse is responsible for creating the Shipment.
      
Required Resources
  • Dialog Purchase Order Administration
  • Dialog Shipment Administration
  • Dialog Delivery Administration
  • Dialog Receipt Administration
  • RF-Terminal Unloading
  • Dialog Shipment Claim Administration.
      
Work Flow

Step

Description

1.

Start the Dialog Shipment Administration and create a Shipment based on the provided information of the shipment documents.

Set license plate truck, trailer identification, number of container, Carrier, number of pallets and weight.

2.

Assign a Dock to the Shipment in the Dialog Dock Assignment with selecting a Dock and confirm with "OK". With use of the function "Unassign" an assigned Dock could be unassigned.

3.

Assign a Delivery to the Shipment based on the provided information of the delivery notes in the Dialog Delivery Assignment with use of the function "Add a Delivery to the Shipment".

Set the Client and the Supplier.

3a.

If an Advice has been transmitted by the Supplier, this Advice has to be selected by the according chooser dialog. If an Advice has been assigned, the Advice Line information are used for automatic creation of the associated Receipts of the Delivery.

3b.

If no Advice is available (step 3a.) an open Purchase Order could be selected by the according chooser dialog. If a Purchase Order has been assigned, the Purchase Order Line information are used for automatic creation of the associated Receipts of the Delivery.

3c.

If no Advice and no Purchase Order are available, the Receipts have to be created as described in step 5.

3d.

By clicking the OK button the Dialog Delivery Administration opens automatically.

4.

Modify the Delivery as desired.

5.

If Receipts have not been created automatically in a previous step, add Receipts to the Delivery with use of the function "create" in the Dialog Receipts according to the delivery note.

5a.

If Receipts have been created automatically in a previous step, compare the Receipts with information from the delivery note. In case of differences modify the data and create missing Receipts accordingly.

6.

Authorize the Shipment for unloading with use of the function "Authorize the Shipment for receiving" in the Dialog Shipment Administration.

A report is printed that contains all information on the Shipment with a barcode, that allows scanning the Shipment ID.

7.

Unload the Shipment with use of the "RF-Terminal Unloading" as described in use case "Unloading with RF-Terminal".

8.

Purchase Order and (if an Advice has been transmitted before) the Advice are booked accordingly.

8a.

If less Receipt quantity has been unloaded the Receipts needs to be closed with use of the function "Close Receipt" in the Dialog Receipt Administration.
If less Load Units than expected have been unloaded the Delivery needs to be closed with use of the function "Close Delivery" in the Dialog Delivery Administration.

9.

If a Shipment Claim occurred within the receiving operations, the operator can create a Shipment Claim for the according Shipment, Delivery, Receipt or single Load Unit.
     
Result

The result of this use case is the successful receiving of at least one Delivery of a Shipment based on a Purchase Order (or Advice).
The Deliveries and the Shipment have been completed or finished.
The received Load Units have an Inbound Transport Request and are on the way to their Storage Locations.
