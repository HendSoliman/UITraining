Detail Dialog Wave

Purpose

This screen opens when viewing or modifying a Wave.

      
Screen

      
Fields

Field

Description

ID

Unique ID number of the Wave.

Status

 Current execution state of the Wave:
  • created (Wave was created and no Order is assigned yet)
  • pending (Wave is awaiting release for execution)
  • released (Wave is released for execution)
  • active (Wave execution started; first Order of the Wave is switched to active)
  • complete (Wave execution finished and the Wave is fully complete; all Wave Orders are completed or finished)
  • cancelled (Wave was cancelled without defined execution).

User Limit

By default, start/end times are recalculated if an Order is assigned/removed to or from a Wave. If set, the user modified the start and/or end times and this recalculation is not done.

Phase

The phase to which the wave settings belong. If multi phase operations are required this refers to a specific phase. Phase 0 always refers to the overall wave.

Multi Phase

If the phase is part of a multi phase operation (e.g. 2-phase picking). Only if enabled the phase could have a value greater than 0.

Facility

The facility where the wave need to be run.

Owner

Source of the Wave:

  • host (initiated from an external Host by the host interface)
  • operator (manually created by a local user)
  • system (created automatically by the system within the related business process).

Type

The type of the Wave (for project specific use only).

Assembly Status

The current assembly status indicates whether all required Orders have been added to the Wave:

  • incomplete (Not all required Orders are yet assigned)
  • complete (All required Orders are yet assigned).

Overlap

If the wave is allowed to start before the previous wave was completely finished.

Pre-Cubing Method

Which method is to be used for the determination and the pre-cubing of the Dispatch Load Units for the Orders of the Wave (see Process: Order Pre-Cubing for all available methods).

Earliest Start Time

Earliest time that Wave execution should begin. Can be considered if the Release Mode of the related Task Type is "scheduled". Set automatically while Order assignment depends on the appropriate Order attributes.

Latest Start Time

Latest time that Wave execution should begin. Can be considered if the Release Mode of the related Task Type is "scheduled". Set automatically while Order assignment depends on the appropriate Order attributes.

Earliest End Time

Earliest time that Wave execution should be finished. Can be considered if the Release Mode of the related Task Type is "scheduled". Set automatically while Order assignment depends on the appropriate Order attributes.

Latest End Time

Latest time that Wave execution should be finished. Can be considered if the Release Mode of the related Task Type is "scheduled". Set automatically while Order assignment depends on the appropriate Order attributes.

Planned Start Time

Planned start time when the Wave shall be started.

Actual Start Time

The actual start time when the wave was really started.

Planned End Time

Planned end time when the Wave shall be finished.

Actual End Time

The actual end time when the wave was really finished.

Task

Waves are handled as Tasks. The assigned Task contains additional Wave attributes (see Tasks).
