Teach-In Step: Scan Product Code

Purpose

This step is used to identify the trade item by scanning the product barcode.
The barcode may be, for example, a GS1-13 or a GS1-128 including Application identifier 01 for the GTIN.

Usually, the barcode of the outer package shall be scanned.
If the outer package has no barcode, it is also possible to scan the barcode of a contained unit.

Special Buttons

Button

Description

 

 

This button allows changing the active client for future teach operations.
Step Select Client is activated

 

The button toggles between two states depending on the Prepared Fulfillment Handling Type of the Trade Item (See screen SKU Quantity Unit)

  • If Prepared Fulfillment Handling Type is AMCAP,
    the button is labeled with "No AMCAP".
    Pressing the button excludes the Trade Item from the AMCAP process after an additional confirmation.
    Teaching for this Trade Unit is immediately terminated.
  • If Prepared Fulfillment Handling Type is not AMCAP,
    the button is labeled with "AMCAP".
    Pressing the button enables the Trade Item from the AMCAP process.
    Teaching of the Trade Item can be continued.

 

  • If the Trade Item (SKU and Quantity Unit) is uniquely determined by the GTIN, the dialog proceeds to step Select Packaging Variant
  • If multiple SKUs exist for the same GTIN, or if no GTIN has been specified, the dialog proceeds to step Select SKU

 

      
Barcodes for Identification

In order to choose a suitable barcode for identification, proceed as follows:

Alternative

Description

1.

Inspect the outer packaging

2.

If there is a GS1 barcode with Application Identifier (01), scan this barcode 

3.

If there is an ITF-14 barcode, scan this barcode

4.

If there is a GS1-Datamatrix Barcode with Application Identifier (01), scan this code

5.

If there is an EAN/UPC barcode, scan this barcode

6.

If none of alternatives 2-5 applies, check the contained items
and continue with step 2

7.

If no valid barcode has been found,
push the <Next> button and try to identify the product through its name.

 

When a barcode is read, the system will search the Product Codes of the SKU Quantity Units for the active client for the code.

In case that a GS1-128 code is found, the system will parse the code and try to identify application identifiers (01) (GTIN) and (10) (batch).
If a GTIN has been found, the Product Codes of the SKU Quantity Units for the active client will be searched for a match.
