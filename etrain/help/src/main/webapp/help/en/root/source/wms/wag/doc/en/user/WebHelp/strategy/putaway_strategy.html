Putaway Strategy

Configuration

Putaway strategies can be configured in terms of the distribution of order sequences or weights to single strategy aspects.

 INFO 

It is recommended to load the strategy configuration of the demo project. This configuration should be then adapted to the requirements of the project.

 INFO 

For Multishuttle Areas specific default strategies are affected. The decision for the appropriate strategy depends on the configuration topic "cor-strategy:Putaway.isStrategyMultishuttle" for each Area.

 INFO 

A Maximum Fill Level of Aisles/Levels can be set via the configuration topic "cor-putaway:PutawayAreaConfigurations.maximumFillLevel" per Area. The default is set to 98%. The Default Strategies for choosing an aisle in classic Warehouses and the default strategies for choosing a bin in Multishuttles will then ignore aisles/levels with a fill Level higher or equal to this value, if the current Location of the given load unit is not within this aisle/level.

 INFO 

To optimize bin usage in double deep Classic areas, use SwitchDoubleDeepPutawayStrategyDemon.

 INFO 

In case of a particular area has at least one specified entry all strategies that should apply for this area need to be set with the area value as well.

 INFO 

Dual Cycle lift configurations: Levels with missing Inbound conveyors will be not considered from the strategies and thus stay unused.

Classic Putaway Strategies

Order By

Description

Group Fill Level

Expressed as a percentage and represents the fill level of the group (aisle). This is the relation from full capacity to currently occupied capacity in this aisle. The fill level will be recalculated when reservation for one certain identified bin Location is made or one Load Unit is leaving certain bin Location by retrieval start.

Group Last Dispo Date

Every time the group (aisle) was chosen, the property "Last Dispo Date" of that aisle updates to the current date / time. Use of this aspect in the aisle selection strategy in the first sequence (as sorting property) within an area provide the effect of an Aisle ROUND ROBIN (regardless of the SKU) in the strongest form.

Group Balance

Balancing of SKUs over the groups (aisles). A specific discriminating Inventory aspect is also considered (e.g. batch). If yet allocated and order related Inventory, this aspect is the Order ID. For Inventory Balance principles, see the Inventory Balance Dialog.

Group Balanced AS/RS Workload

Prefer a group (aisle) (or respectively a storage bin) with the lowest number of currently active inbound transports. Thus the possible destinations are sorted by the current inbound workload of the affected AS/RS.

Height Classification

Possible destinations are sorted by the height classification of the Locations. The height classification of Locations must always be equal or greater than the height classification of the Load Unit. This strategy prefers lower height classifications.

Hot Zone

Prefer storage bins that match the Hot Zone requirement of the Load Unit. A Load Unit shall preferably be stored on a Location of the Hot Zone, if the number of Load Units with the same Discriminating Key planned for hot zoning or stored in the Hot Zone is smaller than the limit defined by the Hot Zone Configuration. For Hot Zone principles, see the Hot Zone Configuration.

Multi Deep

Prefer a Location with free and unblocked positions that store Load Unit(s) with the same Discriminating Key. In case of this strategy is enabled and the Load Unit contains only one inventory unit all other strategies will not be considered anymore in case an appropriate Location could be found.

Please note that enabling Coordinates Ascending decreases the possibility for Multi Deep matches significantly.

Empty Bins

Prefer empty Bins. Possible destinations are sorted by residual capacity of the Location, starting with maximum value.

 INFO 

This strategy may be enabled or disabled by SwitchDoubleDeepPutawayStrategyDemon!

Partial Bins

Prefer partial filled Bins. Possible destinations are sorted by residual capacity of the Location, starting with minimum value.

 INFO 

This strategy may be enabled or disabled by SwitchDoubleDeepPutawayStrategyDemon!

Velocity Classification

Possible destinations are sorted by the velocity classification of the Locations. The order of velocity classifications depends on the slowest velocity classification of the SKUs on the Load Unit to be stored:
SKU velocity classification=A: A -> B -> C
SKU velocity classification=B: B -> C -> A.
SKU velocity classification=C: C -> B -> A
If the configuration topic "cor-putaway:PutawayGeneral.isLMSDispoHigherVelocityClass" is false, Locations with a faster velocity classification than the slowest velocity classification of the SKUs on the Load Unit to be stored are not selected.

Weight Limit

Possible destinations are sorted by the currently residual weight limit of the Location (only locally with consideration of the single Location).

 INFO 

The standard strategy does not optimize in terms of weight distribution with respect to configured Weight Limiters. If needed this must be done by extending the Putaway database views.

Coordinates

Possible destinations are sorted by x- and y-coordinate. That means from the front to the back and from the bottom to the top of a rack.

 INFO 

General fallback strategy. Is executed implicitly and can not be configured.

 INFO 

Project specific Putaway strategies can be introduced by extending the Putaway database views. If this is insufficient, project specific Putaway strategies must be implemented.
     
Multishuttle Putaway Strategies

Multishuttle Putaway Group Strategies (for selection of an Aisle)

 

Order By

Description

Balance

Balancing of Stock Keeping Units over the aisles. Furthermore a specific discriminating aspect of Inventory is considered (e.g. batch). If yet allocated and order related Inventory this aspect is the Order ID.
For Inventory Balance principles, see Inventory Balance.

Dual Cycle

Prefer an Aisle having a released but still inactive outbound request.

Group Fill Level

Prefer Aisles with lower fill level.

Group Transports

Prefer Aisles with lower number of currently planned inbound transports.

Multi Deep

Prefer a Location with free and unblocked positions that store Load Unit(s) with the same Discriminating Key.
If this strategy is enabled and the Load Unit contains only one inventory unit all other strategies will not be considered anymore if an appropriate Location could be found.

Please note that enabling cor-putaway:StrategySwitch.isPreferPartialBins decreases the possibility for Multi Deep matches significantly.

Velocity Classification

Prefer an Aisle with capacity that matches the velocity classification and Hot Zone storage requirements of the Load Unit to be put away. A Load Unit shall preferably be stored on a Location of the Hot Zone, if the number of Load Units with the same Discriminating Key planned for hot zoning or stored in the Hot Zone is smaller than the limit defined by the Hot Zone Configuration. For Hot Zone principles, see the Hot Zone Configuration.

 INFO 

Only Aisles with an appropriate and available Bin respectively Slot are selected in principle (checked directly on aisle selection).

Multishuttle Putaway Subgroup Strategies (for selection of a Level)

Order By

Description

Balance

Balancing of SKUs over the levels of the aisle. Furthermore a specific discriminating aspect of Inventory is considered (e.g. batch). If yet allocated and order related Inventory this aspect is the Order ID.
For Inventory Balance principles, see Inventory Balance.

Dual Cycle

Prefer a Level having a released but still inactive outbound request.

Number of Shuttles

Prefer the level that has currently no shuttles in order to avoid unnecessary empty shuttle movements in a multishuttle in configuration roaming

 INFO 

Activate this strategy only for Multishuttle Roaming storages.

Fill Level

Prefer Levels with lower fill level.

Subgroup Transports

Prefer Levels with lower number of currently planned inbound transports.

Subgroup Multi Deep

Prefer a Location within a given aisle with free and unblocked positions that store Load Unit(s) with the same Discriminating Key.
If this strategy is enabled and the Load Unit contains only one inventory unit all other strategies will not be considered anymore in case an appropriate Location could be found.

Please note that enabling cor-putaway:StrategySwitch.isPreferPartialBins decreases the possibility for Multi Deep matches significantly.

 INFO 

In case of dynamic slotting Multi Deep considers the "slot best fit" strategy implicitly.

Velocity Classification

Prefer a Level with capacity that matches the velocity classification and Hot Zone storage requirements of the Load Unit to be put away. A Load Unit shall preferably be stored on a Location of the Hot Zone, if the number of Load Units with the same Discriminating Key planned for hot zoning or stored in the Hot Zone is smaller than the limit defined by the Hot Zone Configuration. For Hot Zone principles, see the Hot Zone Configuration.

Height Classification

Prefer levels with lower height.

Bin Multi Deep

Prefer a Location within a given subgroup with free and unblocked positions that store Load Unit(s) with the same Discriminating Key.

Only takes effect if "Subgroup Multi Deep" is disabled.

Lift Stops

Prefer a Location within a given subgroup with free and unblocked positions, with focus on lift performance (minimizing number of lift stops).
The strategy implementation is based on current Rack Conveyor allocations and optimized for lift pairing.

 INFO 

Activate this strategy only for Multishuttle storages with Tandem Lift configurations.

Interaisle Transfer (IAT)

Prefer a Level with lowest IAT load.
IAT load means the number of active or planned transports via IAT of the load units current aisle.

 INFO 

IAT transports may cross several groups to reach their destination. The strategy does not consider target or transit groups.

 INFO 

Level and Bin (or respectively Slot) are selected at once.

Multishuttle Putaway Bin/Slot Selection

Bin/Slot Strategies in Multishuttle Areas are not configurable.

In case of discrete raster Layouts (defined through the positioning method of the related Area), the Location within a Level is selected by following ordered strategies

  1. Velocity Classification: if configured in Multishuttle Putaway Subgroup Strategies Velocity Classification settings are considered first.

  2. Coordinates / Prefer Empty: sorting the Locations by depth (z-coordinate or respectively the position within the Location) and x-coordinate ascending.
    Sorting Locations by the x-coordinate is done by determine the closest x-coordinate to the current location of the Load Unit.
    This means empty bins close to current location are preferred. See description of Multi Deep Strategy above.

    This default behavior can be switched by enabling the configuration aspect cor-putaway:StrategySwitch.isPreferPartialBins. See cor-putaway.

In case of dynamic slotting, the Slot within a Level is selected by the combination of the following ordered strategies:

  1. Velocity Classification: if configured in Multishuttle Putaway Group Strategies Velocity Classification settings are considered first.

  2. Coordinates / Prefer Empty: prefer empty slots and prefer nearby X coordinates (details as described above).

  3. Shuffle Move Buffer: Slot selection is done with respect to Shuffle Move Buffer configuration (configuration topic: cor-strategy:Strategy.freeShuffleSlotCapacity).

  4. Slot Best Fit: This calculation assures that the chosen Slot has a width which is equal to or larger than but closest to the required width.
    This supports building Slots with cartons of similar size rather than storing small cartons in the same Slot with large cartons.
    To assure that small bays are not preferred in situations where a subgroup is almost empty this strategy considers the area property maximum considered width (described in Area Detail Dialog). All slots having this width or larger are treated as equivalent.
    Storing in an occupied Slot is done with respect to the Extended Mixed Size Slotting (as described in Slotting Definitions).

Multishuttle Inter-Aisle Transfer (iAT) Strategies

The Inter-aisle transfer technology allows optimization of routes and efficiency in multishuttle storages by providing the ability of horizontal movements between aisles without the use of lifts.

Therefore, these strategies consider the horizontal aligned subgroups (levels) - in contrast to all other subgroups strategies that always consider the vertical (all subgroups of an aisle).

 INFO 

This feature is only active if inter-aisle transfer is enabled on this storage area (configuration topic "cor-strategy:Strategy.isIATStorage").

  1. Shuffle Moves via iAT

    If a load unit A is requested for retrieval, which is blocked by another load unit B, then B will automatically be relocated within a Shuffle Move. For this transport a destination is selected in the same subgroup or in any other subgroup of same level.

    The current subgroup of the load unit is preferred most. Rating of other subgroups is based on their distance (nearest first).

     INFO 

    Fill Levels and Subgroup Locks for inbound or outbound are ignored in case of Shuffle Moves.
