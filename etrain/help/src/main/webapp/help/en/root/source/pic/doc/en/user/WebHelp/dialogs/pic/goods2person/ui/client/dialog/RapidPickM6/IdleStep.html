Standby

Purpose

The "Standby" state appears in exceptional cases that make any operation impossible. Single cases are listed below.

      
Screen

      
Prerequisites
  • Pickstation currently has no assigned Pick Missions. Supervisor must prepare the workload.
  • Pickstation has no Put Locations.
  • Pickstation has no Source Location.
  • Wrong number of Put Locations (e.g. not 6 for A6/M6 and not 1 for A1 station)
      
Functions / Buttons

The screen must be restarted when an exception appears during start up. If the reason for this state is a missing workload, the screen changes the state automatically when workload is available again. In this state, no operation is permitted and thus no function or button is available.
