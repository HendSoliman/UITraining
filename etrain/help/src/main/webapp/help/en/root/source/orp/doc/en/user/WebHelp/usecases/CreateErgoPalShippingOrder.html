Creation and Processing of an ErgoPal Shipping Order

Description

Shipping Orders of this type are manually palletized (picked) at a special ErgoPal (Ergonomic Palletizing) Workstation.

During Order creation, the Service Qualifier must be set to, "ERGOPAL". For all other aspects of Order creation, see "Manual Creation of an Shipping Order".

After the required pre-cubing planning step and the release of the Shipping Order, the system starts the specific Order Fulfillment process.
The required configuration for the special Pre-Cubing planning step is described in the Prerequisites of this map.

      
Triggers and Actors

This is started by a local user when:

  • a shipping process for one or more Inventory Units of specific SKUs should be executed at an ErgoPal Workstation.
      
Prerequisites
  • Desired and required Inventory must be available in the warehouse with exactly one case per Load Unit.
  • A Workstation was configured with:
    • Service set to "ERGOPAL".
    • the Request Size of the Workstation is set to 1.
    • Two virtual Load Units, with identical IDs to those of the Source Locations of the Workstation, are booked on these locations.
  • Two configuration topics must be configured (CL100 is the ID of the client used as an example).
    Configuration Topic Dimension Value
    orp-order:OrderClientOrderType.planning.orderPlanningStatusSequence CLIENT_ORDERTYPE:
    CL100 SHIPPING    		 PRE_CUBING
    orp-orderfulfillment:OrderFulfillmentServiceQualifier.pickToDispatchLoadUnits SERVICE_QUALIFIER: ERGOPAL true
  • SKU, Load Unit Type dimensions, preferred Fulfillment Zone and Preferred Fulfillment Handling Type must be configured correctly.
  • A Dispatch Load Unit Type representing an Euro pallet must be configured.
  • A Fulfillment Zone must be configured, as required by the planning step. Especially assign the needed Dispatch Load Unit Type
  • The used Disposition Zone must not allow full load unit retrievals.
  • An appropriate Area Access has to be configured.
      
Required Resources
  • Orders.
      
Work Flow

Step

Action

1.

Open the Orders screen and create an Order using the Manual Creation of an Shipping Order.

 INFO 

Service Qualifier must be "ERGOPAL".
2. Order Management performs the Pre-Cubing planning step. View planning results in the screen Order Pre-Cubing Results.

3.

 The operator releases the Order manually on screen.
If appropriately configured, the Order is released automatically after all required planning steps are finished.
4.

 Order Fulfillment starts allocating the optimal required Inventory and creates Pick Orders accordingly.

 INFO 

Considering Pre-Cubing results, one Pick Order is created for each planned pallet.

5.

 The operator releases one or all Pick Orders of the Order manually on the Pick Orders.
If appropriately configured, Pick Orders are released automatically after creation.
6.

The assigned Pick Controller performs Pick Order execution with:

  • Creation of one Pick Job for each Pick Order
  • Assignment of Pick Jobs to available Pickstation(s), if free capacity.
  • Requesting required Source Load Units in correct sequence.
  • Automatic booking of Picks when the Source Load Unit arrives at the induction point of the source buffer of the station.

 INFO 

Details operational handlings at the station (e.g. conveyors handling and resulting pallets are project specific.

     
Result

A Shipping Order with Inventory criteria is created and released.
The subsequent order fulfillment process is executed at ErgoPal Pickstations.
All planned result pallets of the Order are palletized and provided for further shipping processes.
