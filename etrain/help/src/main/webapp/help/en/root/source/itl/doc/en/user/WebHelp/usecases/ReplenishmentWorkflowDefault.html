Lookahead Replenishment Planning Workflow

Description

Recurring checks for Lookahead Replenishment are based on one demon per configured Replenishment Supervised Zone (Disposition Zone) with Replenishment Type lookahead. One demon for all configured Replenishment Supervised Zones should exist.

The started demon uses the configured Replenishment Supervised Zones for the Disposition Zone. Calculation starts if an appropriate entry exists. In the calculation phase, configuration values are checked by the Lookahead Calculation.

Lookahead Replenishment is created if open Orders for SKUs with a Preferred Fulfillment Zone in the given Disposition Zone exist, within the lookahead period. Orders are created until the lookahead workload is reached.

Replenishment initiation occurs by creating Replenishment Orders that contain configured parameters from the Replenishment Supervised Zone or from the Threshold Configuration, if existing. 

      
Triggers and Actors

This is started by the Control Center Supervisor when:

  • Lookahead Replenishment must be executed periodically.
      
Prerequisites
  • Replenishment Default Configuration for the Disposition Zone is set
  • Area Access configuration done.
      
Required Resources
  • Replenishment Supervised Zones
  • Area Access
  • Cron Jobs.
      
Work Flow for Demons

Step

Description

1.

Open the Cron Jobs screen to activate Lookahead Replenishment demon functionality.

2.

Select a Demon by filtering the "Demon Class Name" for "CheckLookaheadReplenishmentDemon".

One demon for all configured Replenishment Supervised Zones should exist. Additionally Replenishment Supervised Zone specific demons can be configured.

4.

Start the desired demon.
     
Result

Replenishment Orders are created for SKUs where future orders in the lookahead period exist for the given Disposition Zone are ordered by urgency.
