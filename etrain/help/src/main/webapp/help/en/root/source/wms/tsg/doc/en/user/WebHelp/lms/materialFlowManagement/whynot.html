Why not?

Symptom

This topic explains a general approach to find out why a Load Unit does not move in the way it is supposed to. Along with the analysis of the issue the typical fault recovery actions are mentioned.

The techniques described here are not meant to be a world formula which can explain all problems conceivable in a warehouse. It is intended to be a starting point for a project specific troubleshooting guide which can address the special peculiarities of a certain installation.

 INFO 

It is important to start the first steps of troubleshooting within the underlying transport execution system (MFC) before further steps are considered thereafter.

 CAUTION 

In any case, please make sure that you have a clear understanding of the cause of the problem prior to carrying out and fault recovery actions.

Otherwise the situation might get even worse.

Not Moving

The trigger of this scenario is that there is somewhere a transport unit in the warehouse which is supposed to move but doesn't. In order to get an idea of the possible reasons and further hints for troubleshooting this situation it is recommended to start always within the underlying transport execution system MFC.

The description of the required steps is given in the chapter "Why not?" and and overview for supporting the analysis is given in  Introduction of MFC Trouble Shootings. 

      
Transport Request failed

In case the transport of the Load Unit is in any exceptional situation and the situation within the MFC is completely clear the analysis could proceed with the analysis and clarification of the related Transport Request.

The main dialog related to the troubleshooting of transports - in case the Transport Request of the LMS is relevant - is the Dialog Transport Administration where a combination of transports of the following involved levels is provided:

  • LMS (Logistics Manager) requests the execution of a transport at the underlying transport execution system (MFC) using Transport Requests and
  • MFC (Material Flow Controller) executes the transports represented by Transport Orders.

 

The further analysis strongly depends on the status and the type of the Transport Request and should start with filtering the mentioned Load Unit in the superior filter of the Dialog Transport Administration.

 CAUTION 

The troubleshooting of the Transport Request in the LMS must always consider the related business process which will be fulfilled with the execution of the transport. Be aware in case the Transport will not be fulfilled at the planned destination this could have huge impact to the process.

For example: In case the outbound transport of a Pick Container to a Pickstation is cancelled manually the Pick Order will never be finished anyway.
