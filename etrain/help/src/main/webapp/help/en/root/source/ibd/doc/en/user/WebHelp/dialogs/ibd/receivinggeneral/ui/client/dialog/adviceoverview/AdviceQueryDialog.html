Advice Overview

Purpose

Each row represents one Advice Line, respectively a specific Load Unit of an Advice Line if one or multiple Load Units are already assigned.

 INFO 

Any operations on Advices will be provided by Advices screen.
     
Screen

      
Fields

Field

Description

Client

Owner of the advice within that new inventory shall be managed.
Advice ID

Identifies an Advice uniquely within a client and an Advice type.
Advice Line Advice line uniquely within an Advice.
Load Unit Code Load Unit ID that contains the advised inventory.
Base Load Unit Code ID of the Base Load Unit where the Load Unit is stacked onto.
Advice Type

Origin of the inventory:

  • Supplier Advice
  • Production Advice
  • Return
  • Unexpected Advice (for manual Advice creation within this screen).
Status/Advice

How far the Advice was processed:

  • pending (Advice is completely prepared for processing and available for release)
  • released (Advice is released for processing and available for operational receiving)
  • active (operational receiving process of at least one Advice Line has started)
  • receiving complete (receiving process of all Advice Lines finished, i.e. inventory creation was completed for all Advice Lines)
  • complete (Advice is completely processed and all related Load Units arrived at final storage locations)
  • to confirm (Advice is ready for confirmation message to the host system)
  • finished (All required tasks for the Advice are finished. If required, the advice was confirmed to the host)

Status/Advice Line

How far the Advice Line was processed:

  • pending (Advice Line is completely prepared for processing and available for release)
  • active (operational receiving process of the Advice Line has started)
  • to be clarified (Advice Line must be clarified, e.g. if the SKU of the Advice Line is marked for deletion) (project specific use only)
  • receiving complete (receiving process of the Advice Line finished, i.e. inventory creation was completed)
  • complete (Advice Line is completely processed and all related Load Units arrived at final storage locations)
  • rejected (complete Advice Line cannot be processed for some reason and was rejected internally) (project specific use only)
  • to confirm (Advice Line is ready for confirmation message to the host system)
  • finished (All required tasks for the Advice Line are finished. If required, the advice was confirmed to the host)

Status/Advice Line Load Unit

How far the Advice Line LU was processed:

  • pending (Advice Line Load Unit is completely prepared for processing and available for release)
  • arrived (operational receiving process of the Advice Line Load Unit has started) (project specific use only)
  • to be clarified (Advice Line Load Unit must be clarified, e.g. if the SKU of the related Advice Line is marked for deletion) (project specific use only)
  • receiving complete (receiving process of the Advice Line Load Unit finished, i.e. inventory creation was completed)
  • complete (Advice Line Load Unit is completely processed and the related Load Unit arrived at the final storage locations)
  • rejected (complete Advice Line Load Unit cannot be processed for some reason and was rejected internally) (project specific use only)
  • to confirm (Advice Line Load Unit is ready for confirmation message to the host system)
  • finished (All required tasks for the Advice Line Load Unit are finished. If required, advice was confirmed to the host)

SKU

Stock Keeping Unit ID of the announced inventory specified by this Advice Line.

Quantity Current

Quantity of the so far created inventory for this Advice Line. Expressed in relation to the 'receiving quantity unit' of the Advice Line.

Quantity Target

Total quantity of inventory announced by this Advice Line. Expressed in relation to the 'receiving quantity unit' of the Advice Line.

Quantity Target (LU)

Total quantity of inventory announced by this Advice Line Load Unit. Expressed in relation to the 'receiving quantity unit' of the Advice Line.

Batch

Batch in which inventory was produced.

Best Before Date

Date before it is recommended to consume the inventory. Used as FIFO criterion when goods are selected for an order.
