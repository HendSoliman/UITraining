Detail Dialog Area

Purpose

This screen opens when creating, viewing, or modifying Areas.

For an overview of all elements used to group locations in a warehouse, see the Warehouse Structure Description.

 INFO 

After creating or deleting an Area, the Materialized View that relates Areas and Disposition Zones must be refreshed.

Fields

Field

Description

Facility

Selected Facility. Read only field.

ID

Unique identifier of the Area. Can only be specified when creating a new Area.

Area

Name

Unique name. For display purposes.

Description

Detailed description of the Area. For display purposes.

Configuration Status

If Area configuration was accomplished and the Area can therefore be used by the system. Can only be modified when changing properties of an existing Area.

Virtual Location

The Virtual Location represents the Area as a Location.

Handling

Type
Area Type

Overall use:

  • receiving (Area for operative Receiving processes)
  • shipping (Area for operative Shipping processes)
  • storage (Normal Storage Area, e.g.(High Bay Warehouse or Multishuttle))
  • buffer (Area for intermediate buffering of Load Units subsequent to specific business processes (e.g. Picking))
  • clearing (Area for Clearing purposes: Clarification Stations are typically located within a Transport Area)
  • virtual (Logical Areas necessary for logical booking operations only (e.g. for Inventory Bookings))
  • transport (Area for transport purposes of Load Units only).
Location Type

Default Location Type during mass generation of Locations in this Area.

Load
Operation Mode

If an Area operates in manual or automatic mode.

  • manual (Within this Area no automatic material handling equipment is available or managed)
  • automatic (This Area is controlled by automatic material handling equipment).
Granularity

Granularity level (area, aisle, bin) a destination of a Transport Request for Load Units from this Area must be requested.
Keep Empty Load Units

If Load Units should not be automatically deleted in this Area if the Load Unit became empty.

 INFO 

Load Unit Type must be configured appropriately as well.
Needs Leave Source Confirmation

If a Transport Request needs confirmation when the Load Unit leaves its source. If defined, a notification should be sent from the underlying subsystem (MFC) when the Load Unit leaves its source.

Inventory

Auto Dispo

If automatic disposition is allowed in this Area. Automatic disposition means the automatic allocation of Destination Locations for Putaway operations by the system.

Inventory Balanced

If collection of Inventory Balancing data is enabled. If enabled, the Inventory data will be collected within this Area based on the defined discriminating key.

See the Inventory Balance screen.

 INFO 

In order to use this for balancing the inventory the related putaway strategy must be enabled as well.

Multishuttle

Workload

Outbound Transports Minimal Threshold

Minimal number of active outbound transports of this Area. If the current number of active outbound transports falls below this threshold, new pick orders are released until the number of active Fulfillment Load Units specified in "Load Unit Refill Threshold" is reached.

 INFO 

Only valid for Multishuttle areas if two-phase reservation is set for the assigned Disposition Zone.

Fulfillment Load Unit Refill Threshold

Minimal number of not completed Fulfillment Load Units. This number of outbound transports should be reached if the current outbound transport workload pool of a Multishuttle falls below the configured "Min Outbound Transport Threshold" level.

 INFO 

Only valid for Multishuttle areas if two-phase reservation is set for the assigned Disposition Zone.

Dynamic Slotting

Positioning Method

How positions of Load Units within a location are specified in general.

  • discrete (Use the discrete Raster Layout without dynamic slotting.)
  • dynamic (Dynamic Slotting with use of Slots define the positioning method in this Area).

Bay Width

Width of a Location (Bay) within this area (in base units of measure). The width of a Bay could be individualized for single bays by the Location.

Width Gap

The gap between two Load Units along the X-axis required for the Multishuttle telescope (in base units of measure).

Depth Gap

The gap between two Load Units in Z-direction required for the Multishuttle finger engagement (in base units of measure).

Height Classification

The height classification of all slots of the area.

Raster Unit

The raster unit (in base unit of measure) assumed for specification of X-coordinates.

Min Slot Width

The minimal width of all Slots in this Area.

Max Slots

The maximum number of Slots within a Location (bay) in this Area.

Number of Unused Slots

The number of unused Slots to be generated for a Location (bay) in order to improve the performance.
Max Considered Width The maximal slot width in raster units considered by the "slot best fit" strategy. All slots whose width is equal or greater than this value are equally handled by this strategy.
This allows other Order By criteria to be more taken into account.

 INFO 

This value could be set either to the the width of the smallest bay in this area or to the width of a shuttle.

Formatting Policies
 

Defines how IDs, Names, Aliases, Descriptions, external names and external positions of the Locations and Levels assigned to this Area automatically format during the generation process. Used during creation of Locations or Levels. Changes do not impact existing Locations and Levels.

  • f: facility
  • a: area
  • i: aisle
  • s: side
  • x: x-coordinate
  • y: y-coordinate
  • p: position

For Levels only-- f, a, i, y have an effect!
