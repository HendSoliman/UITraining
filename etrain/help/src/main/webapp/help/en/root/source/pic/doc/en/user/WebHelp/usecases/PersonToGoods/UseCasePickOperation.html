Use Case Pick Operation

Description

The Source Location must be scanned to indicate the right location for picking after arriving at the next required Pick Face. The subsequent pick operation includes the standard case for single pick confirmation:

  • without variation
  • with handling of inventory shortage
  • with additional split pick operation
  • with advised overdelivery
  • with blocked Pick Face
  • with reverse picking functionality.
      
Triggers and Actors

The following persons must perform this for all active Picks of current Pick Walk after all required Pick Containers are put onto the Pick Cart by executing use case: New Pick Container.

  • Picker operating in Person-to-Goods Pick Zone.
      
Prerequisites
  • All required Pick Containers are put on the correct Put Locations on the Pick Cart.
      
Work Flow

"OK": standard case

Step

Description

1.

Open the Dialog RF-Picking and perform Use Case: New Pick Container.

2.

The screen Scan Source Location displays.

3.

Scan a valid Source Location and press "Enter".

3a.

If an invalid Source Location is scanned, an error message appears. Proceed with step 3.

4.

The screen Picking displays.

5.

Pick the required quantity from Source Load Unit and put it into the Pick Container at required Put Location.

6.

Press "Enter".

Required scan of Product Label

4a.1

Scan a valid Product Label and press "Enter".
Proceed with step 5.

Required scan of Pick Container (Put Location Alias)

7.

If a Pick Container scan is required, the screen Verify Pick Container displays.

8.

Scan the required Put Location
"Enter" key is activated implicitly with scan.

9.

If an invalid Put Location is scanned, an error message appears. Proceed with step 8.

 

"Short"

Step

Description

1.

Perform all steps up to "Picking" (see standard case while considering special steps, "Required scan of Pick Container").

2.

Not enough inventory is available on Source Load Unit.
Press "F2" key.

3.

The screen Short displays.

4.

Pick the remaining quantity from Source Load Unit and put it into the Pick Container at required Put Location.

5.

Enter the picked quantity.

6.

Press "Enter".

Change Function Reason

6a.1

Press "F9" key to change the displayed Function Reason.

6a.2

Select another Function Reason. Press "Enter" or discard changes with "F12" key.

6a.3

The new, selected Function Reason displays for later confirmation. Proceed with original step.

Cancel

6b.

If a miscount and enough inventory is available, press "F12" key.

 INFO 

In case of any shortage the shorted Pick will be re-planned by Order Fulfillment in the short term and, if possible, reassigned to the current Pick Walk.
Replenishment of the shorted Pick Face is done separately based on the appropriate Replenishment configuration.

"Over"

Step

Description

1.

Perform all steps up to Picking (see standard case while considering special steps, "Required scan of Pick Container").

2.

Operator decides to pick more than requested quantity.
Press "F3" key.

3.

The screen Over displays.

4.

Pick the desired quantity from Source Load Unit and put it into the Pick Container at required Put Location without violating the overdelivery tolerance.

5.

Enter the picked quantity.

6.

Press "Enter".

Change Function Reason

6a.1

Press "F9" key to change the displayed Function Reason.

6a.2

Select another Function Reason and press "Enter". Or discard changes with "F12" key.

6a.3

The new, selected Function Reason displays for later confirmation. Proceed with original step.

Cancel

6b.

If a mistake and no advised overdelivery is desired, press "F12" key.

 

"Skip"

Step

Description

1.

Perform all steps up to Picking (see standard case).

2.

The Pick Face is temporarily blocked and currently unreachable.
Press "F4" key.

3.

The screen Confirm Skip displays.

4.

Press "Enter" to confirm the desired skip of current Pick and it's rescheduling to the end of current Pick Walk for later picking.

Cancel

4a.

If a Pick Face becomes reachable, press "F12" key.

 

"Blocked"

Step

Description

1.

Perform all steps up to Picking (see standard case).

2.

The Pick Face is permanently blocked and unreachable.
Press "F5" key.

3.

The screen Confirm Blocked displays.

4.

Press "Enter" to confirm the desired skip of current Pick without appending it to the end of current Pick Walk.

Cancel

4a.

If a Pick Face becomes reachable, press "F12" key.

 

"Full"

Step

Description

1.

Perform all steps up to Dialog State "Picking" (see standard case while considering special steps, "Required scan of Pick Container").

2.

The Pick Container is full and not all of required quantity fits into the current Pick Container.
Press "F6" key.

3.

The screen Full displays.

4.

Pick less than the required quantity from Source Load Unit and put it into the Pick Container at the required Put Location.

5.

Enter the picked quantity.

6.

Press "Enter".

7.

Proceed with "Use Case: New Pick Container" to provide another Pick Container for remaining Picks of this closed Pick Container.

Change Function Reason

6a.1

Press "F9" key to change the displayed Function Reason.

6a.2

Select another Function Reason. Press "Enter" or discard changes with "F12" key.

6a.3

The new, selected Function Reason displays for later confirmation. Proceed with original step.

Cancel

6b.

If a mistake and all required quantity fits into the Pick Container, press "F12" key.

 

"Reverse"

Step

Description

1.

Perform all steps up to Picking (see standard case while considering special steps, "Required scan of Pick Container").

2.

Decide to perform a reverse picking operation with exchange of Source Load Unit and Pick Container, e.g. if reverse picking is recommended by the system.
Press "F7" key.

3.

The screen Reverse displays.

4.

Leave the required quantity on the Source Load Unit and pick the remaining quantity into an empty Pick Container at the required Put Location.

Note: Empty the Pick Container if it is not empty.

5.

Enter the picked and thus, reverse quantity.

6.

Press "Enter".

7.

Perform the exchange of the Source Load Unit and Pick Container manually.

Shortage

4a.1

Not enough inventory is available on Source Load Unit.
Press "F2" key.

4a.2

Perform "Short" as mentioned from step 5.

4a.3

Proceed with step 7.

Cancel

6b.

If a mistake and reverse picking is impossible at present, press "F12" key.
     
Result
  • In standard case, the Pick proceeds to "confirmed" status and the related Inventory Unit is booked into the Pick Container. Additionally, if "Over" an advised overdelivery is booked.
  • If "Short", only the remaining quantity is booked into the Container. The missing quantity is booked related to its deletion and the shortage of related Pick Order is handled appropriate to its configuration.
  • If "Full", the Pick is split and a new Pick is available for picking the remaining quantity into a new Pick Container.
  • If "Skip", the Pick is rescheduled to the end of current Pick Walk and is processed later. If "Blocked", the Pick is not processed and removed from current Pick Walk.
  • If "Reverse", an additional exchange of Source Load Unit and Pick Container is booked by the system.
