Description

ID

The ID of the related Pick Order.

Pick Order Line ID

The Pick Order Line ID is unique within a Pick Order. Usually represented by a running number. Each Pick Order Line that is split up is denoted by an added "-digit" where "digit" is started with "1". ("1-1" and "1-2" are the first two split Pick Order Lines of Pick Order Line "1".)

 INFO 

Creation of a split Pick Order Line occurs after Pick Confirmation by the Pick Controller for each different combination of Error Code and Error Reason.
The "-digit" Pick Order Lines represents always the already finished part while the original one represents the still open portion. 

Status

Current execution state of the Pick Order Line:

  • pending (Pick Order is awaiting release for fulfillment and all Pick Order Lines proceed to this status)
  • released (Pick Order is released for fulfillment and all Pick Order Lines proceed to this status)
  • active (Pick Order execution started and all Pick Order Lines proceed to this status)
  • cancelled ((single) Pick Order Line was cancelled without the defined execution)
  • complete (Pick Order Line execution finished and the Pick Order Line is fully completed).

Overview

Service Qualifier

Set by the related Pick Order. Influences concrete fulfillment of the Pick Order Line within specific execution processes. May be project specific. Can limit the list of available service qualifiers using the configuration topic "orp-order:OrderOrderType.allowedServiceQualifiers".

Reservation Type

How inventory is reserved on inventory selection (allocation, reservation, disposition):

  • none: Inventory will not be reserved.
  • per load unit: Inventory Unit is reserved directly and immediately.
  • aggregate: Inventory is reserved on the Aggregated Inventory level. The reservation on Aggregated Inventory is consumed directly.
  • two-phase: Inventory is reserved on the Aggregated Inventory level. Before the reservation is consumed, it is concretized to a Inventory Unit Reservation for later use.

Destination Location

Destination Location for all Pick Containers of the Pick Order Line where the Pick Containers must be put after Pick operation is finished.

Reference

Order Line

Related Order Line based on which this Pick Order Line was created.

Previous Phase Pick Order Line

If Two Phase Picking is processed and this Pick Order Line represents the second phase, this denotes the Pick Order Line ID of the consolidated Pick Order Line of the first phase.

Sequence

Group Sequence

Order between whole groups for arriving at the final destination.

Sequence Number

Sequence of Picking Operation of all Pick Order Lines within one Pick Order. If Picking with transports to Pick Stations, this represents the order in which the Load Units of the assigned group should arrive at the final destination.

Source

Source Location

Location ID of the Source Load Unit.

 INFO 

In case of the picking is done with use of Dedicated Pick Faces this defines the selected one (if multiple Pick Faces are available the selection is done randomly).

Source Base Load Unit ID

Base Load Unit ID that the Load Unit is stacked onto.

Source Load Unit ID

Load Unit ID to be picked from.

Pick Container

Variation

Error Code

Last confirmation of the related underlying task execution system which occurred with this Pick Order Line leads to an Error Code based on the configuration:

  • OK (successfully completed)
  • missing quantity (not fully picked due to Inventory shortage)
  • load unit missing (not fully picked because no Load Unit found)
  • wrong SKU (not fully picked because wrong SKU found)
  • wrong load unit (not fully picked because wrong Load Unit found)
  • wrong best before date (not fully picked because wrong Best Before Date on Inventory found)
  • wrong packaging level (not fully picked due to wrong packaging level)
  • breakage (not fully picked because broken Inventory found)
  • blocked (not fully picked because Location to be picked from is blocked and inaccessible)
  • underdelivery (not fully picked because explicitly Picking of less than target quantity)
  • overdelivery (not fully picked because explicitly Picking of more than target quantity)
  • split (was split because Pick Container was full).

Error Reason

The reason for the Error Code is a description of the confirmation for further classification and additional display purposes. Only separately defined values are available as described in Confirmation Reason for "Reason".

Cancel Reason

Reason for cancellation of this Pick Order Line, only set in case of status cancelled:

  • underdelivery tolerance (cancelled due to violated underdelivery tolerances)
  • overdelivery tolerance (cancelled due to violated overdelivery tolerances)
  • operator (cancelled manually by an operator)
  • previous phase line cancelled (cancelled because the related Previous Phase Pick Order Line was cancelled).

Quantity

Pick Quantity

Requested quantity in Pick Quantity Units for this Pick Order Line.

Unit

ID of the Quantity Unit of the SKU. Used for display purposes within Pick operations.

 INFO 

This Unit is the Default Pick Quantity Unit of the SKU only if exactly one Quantity Unit is configured as "Default Pick". Otherwise always the Base Quantity Unit is used for picking operations.

Quantity Base Target

Requested quantity in Base Quantity Units for this Pick Order Line.

Unit

Base Quantity Unit of the SKU. Represents the relevant Quantity Unit for the quantity base target, quantity base target host, quantity base dispo and quantity base current, as set by the related Order Line.

Factor To Base

Factor from the Base Quantity Unit of the SKU to the Target Quantity Unit, as set by the related Order Line.

Quantity Base Dispo

Quantity Base Dispo from the related Order Line. Set only if the Pick Order is directly created by the Host system in Order Fulfillment scenario (without having any related Order).

 INFO 

Not used in the standard scenario.

Quantity Base Current

Already fulfilled quantity. Set with status progress of the Pick Order Line to "complete".

Number of Load Units Base

The number of load units required for this inventory criteria.

 INFO 

Only used if the inventory rounding method is "number of load units" or "at most number of load units".

Number of Load Units Dispo

The number of load units currently reserved for this inventory criteria.

 INFO 

Only used if the inventory rounding method is "number of load units" or "at most number of load units".

Tolerance

Overdelivery Tolerance

Factor, in percent, up to which overdelivery is allowed.
Example: Required target quantity is 100. Overdelivery tolerance is 8. Delivery up to 108 is allowed.
Underdelivery Tolerance

Factor, in percent, up to which underdelivery is allowed. If set to 100%, an Order Line is set to completed even nothing was delivered.

 INFO 

If the underdelivery tolerance of any Order Line will be violated while Order Release the further actions on the Order will be configured by the configuration topic "orp-orderfulfillment:OrderFulfillmentClientOrderType.underdeliveryAction". Possible actions are:

  • DEFAULT (Cancel the underdelivered Order Lines only)
  • PAUSE (Set the Order to "paused" and stop further processing until either manual cancellation or further order release. Pause functionality is not supported during the replacement reservation process.)
  • CANCEL (Cancel the Order)
  • NONE (Proceed all actions on the Order Line with yet available quantity and ignore the underdelivered quantity. Further inventory reservation is possible until the status is not yet complete.)
Unlimited Overdelivery Allowed

If enabled, overdelivery is allowed without any limit. Useful if only Full Load Units are requested, regardless of quantity.
Request Whole LUs

If enabled, only Full Load Unit Retrievals are allowed for this Order Line. Picking operations are not initiated. If set, full load unit retrievals are selected from all disposition zones regardless of the flag full load unit retrieval in the disposition zone.
Rounding Method

The Inventory Rounding Method of the created Replenishment Order Lines. How Inventory Units will be rounded during reservation:

  • none (no special rounding method)
  • at least full inventory unit (if the quantity to be reserved is completely available on the first matching Inventory Unit, the complete free quantity on the Inventory Unit is reserved. If this would exceed the over delivery tolerance, the maximum quantity within the overdelivery tolerance is reserved)
  • complete packaging unit (only complete packaging units are reserved within the boundaries of overdelivery and underdelivery tolerance. The quantity to be reserved on Inventory Units with a packaging level is rounded up to an integer quantity related to the packaging level. The quantity to be reserved on Inventory Units without a packaging level is rounded up to an integer quantity related to the Base Quantity Unit of the SKU. If the rounded up quantity exceeds over delivery tolerance, there is no rounding)
  • whole layers (only complete layer (on the pallet) are reserved within the boundaries of overdelivery and underdelivery tolerance. The quantity to be reserved on Inventory Units is rounded up to the next full number that fills a full layer of the SKU. If the rounded up quantity exceeds over delivery tolerance, there is no rounding)
  • number of load units (if possible, exactly the specified number of load units are reserved)
  • at most number of load units (the specified quantity base target is reserved but not more than the the specified number of load units).
Minimum Quantity Per LU The minimum quantity of the specified inventory that must be reserved per load unit. Inventory is not selected, if there is less available quantity on one base load unit.
Minimum Rest Quantity The Minimum Rest Quantity of the created Replenishment Order Lines. If less than the Minimum Rest Quantity would remain free after the reservation on a source inventory unit the complete free quantity is reserved within the boundaries of over delivery and under delivery tolerance.
