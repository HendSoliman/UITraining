Picking

Brief Description

The "Picking" screen appears after a Source Load Unit scan. It provides instructions for the current pick operation, and information about the SKU, Source Load Unit and the required Pick Container.

This function is the standard procedure for this screen. The requested pick operation is confirmed without any variation regarding displayed data.

 INFO 

 Reverse Picking is only allowed if the Source Load Unit is not stacked and of the same type as the Pick Container.
     
Screen

If the appropriate configuration in pick zone configuration(attribute: Limit Reverse Picking) and correct pick quantity, this state displays with a reverse picking recommendation:

      
Prerequisites
  • All required Pick Containers are put on correct Put Locations.
  • Required Source Load Unit with finished transport to inbound buffer is scanned.
  • Previous Pick is confirmed and additional Pick on same Source Load Unit is available.
  • "Reverse Pick recommended" display requires the appropriate Pick Zone configuration (see Detail Dialog Pick Zone).
      
Fields

For field descriptions, see Picking Screen Details.

Function/Button

Description

Change to "Short" state

Related Button or "F2" key:

In "Short" state, operator can confirm inventory shortage due to several configured shortage reasons.

See Short.

Change to "Over" state

Related Button or "F3" key:

In "Over" state, operator can confirm more than requested quantity due to several configured over reasons.

Enabled only if a sufficient overdelivery tolerance of the related Order exists.

See Over.

Change to "Full" state

Related Button or "F6" key:

In "Full" state, operator can confirm that not all required quantity can be picked into the current Pick Container.

See Full.

Related Button or "F7" key:

In "Reverse" state, operator can confirm reverse picking operation with exchange of Source Load Unit and Pick Container.

See Reverse Picking.
