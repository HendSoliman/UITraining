Teach-In Step: Select Packaging Variant

Purpose

In this step, the Packaging Variant is identified for which teaching shall be done.

For every Trade Item identified by a GTIN (SKU Quantity Unit), there may be one or more Packaging Variants.

(For more details, see the topic on Packaging Variants)

The screen will show all Packaging Variants, compatible with the scanned Product Code, which have already been teached.

In addition, all Trade Items will be shown for which no Packaging Variants have been teached so far.

From this overview, the user may either select an existing Packaging Variant for re-teaching or he may create a new variant in case that the actual variant does not fit to an existing one.

Representation of Packaging Variant Entries

The representation of entries in the list of Packaging Variants varies depending on the available data:

Representation

Description

SKU Quantity Unit for which Packaging Variants are not yet defined.

Information shown is the GTIN (Product Code), the ID of the SKU Quantity Unit.

Factor and ID of Base Quantity Unit show the number of contained base quantity units.

Default Packaging variant (C12A in the example) for an SKU Quantity Unit.

Non-default Packaging variant (C12B in the example) for an SKU Quantity Unit.

 

 

      
Usage
  1. If only one Packaging Variant exists which fits to the scanned Product Code, this will automatically be selected.
  2. If multiple entries exist, choose the one which fits best to the item to be teached
  3. If the selection does not include information on an existing Packaging variant (see Packaging Variant Entries, only button <New> active), Press <New> and specify the new Packaging Variant ID.
    After return, the new entry will automatically be selected.
  4. If the new entry is not the default, it may be set as default with button <Default>.
  5. Proceed with the <Next> button.

 

      
Special Buttons

Button

Description

 

The button is activated when an existing Packaging Variant is selected which is not the default for an SKU Quantity Unit.

Pressing the button sets the Packaging Variant as default.

This causes the dimensions of the Packaging Variant to be transferred to the Palletizing Quantity Unit.

 

This button is activated when an existing Packaging Variant or an SKU Quantity Unit without Packaging Variants has been selected.

Pressing the button initiates creation of a new Packaging Variant through a New Packaging Variant ID
