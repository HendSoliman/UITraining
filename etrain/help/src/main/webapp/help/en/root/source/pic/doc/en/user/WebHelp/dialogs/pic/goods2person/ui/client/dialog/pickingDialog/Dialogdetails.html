Picking Screen Details

Description of the frame "Source"

The upper middle frame provides information about the current Pick Mission and/or Source Load Unit, with additional operating instructions regarding the Source Load Unit.

Field

Description

Source Load Unit

Unique ID of the current Source Load Unit.

Remaining Picks

Number of remaining active Picks within this Load Unit.

Icon with text

Icon displaying a short instruction hint for the current Source Load Unit.
     
Description of the frame "Pick"

The middle frame provides instructions for the current pick operation and information about the SKU. The Quantity value is initialized with the Quantity Target value.
Main information for current operation is highlighted with dark color.

Field

Description

SKU

Uniquely identifies a Stock Keeping Unit within a Client.

Description

Textual description of the SKU.

Batch

Batch of the Inventory Unit to be picked.

Best before date

Best before date of the Inventory Unit to be picked.

Quantity Target

Target quantity to be picked into the Pick Container.

Quantity Current

Quantity currently in the Pick Container.

Quantity Reverse

Picked and counted quantity for subsequent Putaway and restoring.

Quantity Unit

Quantity unit of the pick operation. This is the "Pick Quantity Unit" if exactly one Default Pick Quantity Unit is set for the quantity units of the SKU. Otherwise, the Base Quantity Unit displays.

Confirmation Reason

Reason for the Confirmation Code.
     
Description of the frame "Destination"

The lower, middle frame provides information about the destination of the current pick operation, with additional operating instructions regarding the Pick Container.

Field

Description

Pick Container

Collects all Picks of one Pick Mission. Contains Inventory Units that are already picked.

Put Location

Location alias with the role "pick" represents the name of the Put Location for easy identification.

Customer

Unique name per Client. Used for display purposes.

Service

How the Pick Order must be fulfilled.

Icon with text

Icon displaying a short instruction hint for the current Pick Container.
     
Buttons

Button

Function

Operator can confirm inventory shortage due to several configured shortage reasons.

Operator can confirm more than requested quantity due to several configured over reasons.

Enabled only if a sufficient overdelivery tolerance of the related Order exists.

Operator can confirm that not all required quantity can be picked into current Pick Container.

Operator can confirm the operation of a reverse Pick.

Operation is confirmed. Next screen state appears to proceed with next picking operation. Precise action is contextual and described within the section specific to the screen state.
