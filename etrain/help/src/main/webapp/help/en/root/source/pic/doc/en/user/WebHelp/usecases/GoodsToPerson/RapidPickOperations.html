Rapid Pick Operations

Description

Pick operations include the standard case for confirmation of the single pick:

  • without variation
  • with handling of inventory shortage
  • with further split operation due to a full Pick Container
  • with advised overdelivery.
      
Triggers and Actors

This must be performed by a picker after a Source Load Unit is available and scanned at the station. 

      
Prerequisites
  • Pick Container for current pick is located at the correct Put Location.
  • Required Source Load Unit with finished transport is scanned at the station.
  • At least one active Pick from the scanned Source Load Unit is available.
  • Function Reasons for Function Codes Short, Full and Over are available for the used Client.
      
Required Resources
  • RapidPick Screen
  • Picking step
  • Exceptional screen.
      
Work Flow

"OK": standard case

Step

Action

1.

Open RapidPick screen and perform "Put new Pick Container at RapidPick Station"

 INFO 

Not needed for automatic station.

2.

The screen Picking displays.

3.

Touch the Pick Container which shows the required quantity.

 

"Short"

Step

Action

1.

Open RapidPick screen and perform "Put new Pick Container at RapidPick Station"

 INFO 

Not needed for automatic station.

2.

Not enough inventory is available in the Source Load Unit.
Press "F3" or the Exception button on the screen.
3. The screen Exception displays.
4.

Pick the remaining quantity from the Source Load Unit and put it into the Pick Container at required Put Location.
5. Reduce the quantity to the actual picked quantity.
6. Press the Function Reason where the Function Code Short is assigned.
Cancel
6a.

If a miscount and enough inventory is available press "Esc" or click "Cancel".

 

"Full"

Step

Action

1.

Open RapidPick screen and perform, "Put new Pick Container at RapidPick Station"

 INFO 

Not needed for automatic station.

2.

The Pick Container is full and not all of the required quantity fits into the current Pick Container.
Press the "F3" or the Exception button on the screen.
3. The screen Exception displays.
4.

Pick less than required quantity from the Source Load Unit and put it into the Pick Container at required Put Location.
5. Reduce the quantity to the actual picked quantity.
6. Press the Function Reason where the Function Code Full is assigned.
Cancel
6a.

If a miscount and enough inventory is available, press "Esc" or click "Cancel".

 

"Over"

Step

Action

1.

Open RapidPick screen and perform "Put new Pick Container at RapidPick Station"

 INFO 

Not needed for automatic station.

2.

Picker desires to pick more than requested quantity.
Press "F3" or the Exception button on the screen.
3. The screen Exception displays.
4.

Pick the desired quantity from the Source Load Unit and put it into the Pick Container at required Put Location.
5. Increase the quantity to the actual picked quantity.
6. Press the Function Reason where the Function Code Over is assigned.
Cancel
6a.

If a miscount and enough inventory is available, press "Esc" or click "Cancel".
     
Result

Typically, the Pick proceeds to "confirmed" status and the related Inventory Unit is booked into the Pick Container.

If "Short", only the remaining quantity is booked into the Container. The missing quantity is deleted and the shortage of related Pick Orders are handled appropriate to the configuration.

If "Full" the Pick is split and a new Pick is available for picking the remaining quantity.

If "Over", the over delivered quantity is booked into the Container.
