Length

The length is calculated form the teachLength, which is the measured length on the Teach-In station.

length = techLength + addLength, where addLength is a system parameter, e.g. 1cm.

if teachLength < moduleLength(teachLength) < length then length = moduleLength(teachLength)

moduleLength is the next module length larger than the teachLength. Following moduleLengths are defined:

100mm, 120 mm, 150mm, 160mm, 200mm, 240mm, 300mm, 400mm, 600mm

 

This calculation prevents that due to inexact measurement the pallet planning is bad. For example if the packages have normally the length of 600mm and thus 2 packages on the euro pallet with length 1200mm can be placed, but the measurement was:

teachLength = 591mm then we have length = 601 (teachLengtt+addLength) and only one package can be put on the euro pallet. The second calculation correct this: teachLength=591 < moduleLength(591) = 600 < length = 601 --> use length=600mm.

 INFO 

This is used for pallet planning.

Use Minimum slot Width This attribute has a totally different meaning as the area attribute minSlotWidth.

Cover Restriction

Defines the borders of the package that always needs to be covered by the overlying packages.

  • None (package has no restrictions in terms of covering)
  • Full Length (full length of the package must be covered due to only the edges of the width side are stable)
  • Full Width (full width of the package must be covered due to only the edges of the length side are stable)
  • Full Length or Width (full width OR full length of the package must be covered due to all edges are stable)
  • Full (the whole upper side of the package must be covered due to only the corners are stable).

Edge Radius

This value gives the rounding of the package in the corners. This parameter is used to consider the rounding during mixed-size slots.

Necessary in order to cover the package with the Multishuttle fingers.

Pack Overhang Reduce

Default a package can have overhang of 25% (1/4), e.g. 1/4 of the package is not directly on another package. This parameter reduced this overhanging in percentage.

Support Edge Extension

The Packbuilder has two control parameters "minSupportEdgeBottom" and "minSupportEdgeTop". These parameters are used to indicate how much of the package edge must be used from the overlaying package.

For example if minSupportEdgeTop is set to 15mm, then minimum 15mm of the edge must be used from the overlaying package.

With the SupportEdgeExtension these two control parameters are adjusted per order line. This is a value between 0 and 100%.

Depalletizing

Auto Depalletize

True, if the item can be de-palletized automatically. If this value is set to false, the complete pallet containing this item must be de-palletized manually.

There are also other cases in which the pallets are manually de-palletized although this flag is set. These are for example:

  • No of the automatic de-palletizer is available.
  • The first layer is not complete and therefore automatic de-palletizer cannot be used.
  • The target aisle is not connected with an automatic de-palletizer.

Disallow Roundup

Indicates if round-up is disallowed for de-palletizing. This flag can be set if a layer consists of a large number of pieces and if it is not desired to depalletize whole layers, e.g. due to low velocity classification.
De-Palletizer Program Test Pick

During teach-in the operator can set a test program for the SKU. This is general for all load unit types (pallet types).
When the SKU is de-palletized (automatic) for first time, the program configured here is used for the "Test Pick". If nothing is set here as default program 10 is used. This is configured in mfc-core.xml.

 INFO 

The de-palletizing program should be a numeric value.

Common

Accept New Default

Default the value of this attribute is set to true. As soon the attribute parameterModified changed to true, the value of this attribute is set to false. Afterwards this attribute can be set to true only manually.

If true, the complete rule engine call will update all parameters for which rules are defined in PalletizingQuantityUnitInitializer.drl.

If false, none of the parameters are updated, e.g. such palletizing quantity units are ignored during rule engine call.

Conveying Orientation

The orientation in which the package is to store in the Multishuttle. Depending on this orientation and the current dimensions PLC will turn and tilt the package and bring it into the correct orientation.

The possible values are:

  • LW (Length x Width)
  • LH (Length x Height)
  • WL (Width x Length)
  • WH (Width x Height)
  • HL (Height x Length)
  • HW (Height x Width)

Vertical Shift

Some packages have handles or any other over standing parts which can obstruct the pusher plate of the palletizer. Therefore for such packages a vertical shift is defined which is the height of this over standing part (or handle). Thereby for the overlaying package the pallet is pulled down with the length + vertical shift of the underlying package.

Raster X

Number of item on the package in length direction.

Raster Y

Number of items on the package in width direction.

Strength Class

The load capacity of a package defines how many weight / packages it can carry without breakage. During Teach-In the operator can set a strength class which influence the load capacity.

  • Minor Strength (The package has minor strength and therefore the calculated load capacity is multiply with 0,25)
  • Less Strength (The package has less strength than normal and therefore the calculated load capacity is multiply with 0,50)
  • Normal (The package has normal strength and therefore it is multiply with 1)
  • Much stronger (The package is much stronger than normally and therefore the calculated load capacity is multiply with 1,50).
