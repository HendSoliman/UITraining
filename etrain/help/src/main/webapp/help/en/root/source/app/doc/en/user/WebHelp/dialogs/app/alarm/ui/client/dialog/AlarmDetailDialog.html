Alarm Detail

Purpose

The detail dialog is opened when viewing, commentate or acknowledging Alarms by applying the respective buttons in the overview window.

 

      
Screen

 

      
Fields

Field

Description
General

Category

Alarm category of the alarm: Info, Warning, Error

Alarm Domain

 The Alarm Domain groups Alarm Types in a raw way  (Base, LMS, MFC, etc.)

Alarm Type

 The Alarm Type defines basic Alarm attributes e.g. Message, Lifecycle type

Status

 The current Alarm Status: Created, Up, Acknowledged, Gone

Count

  For counted Alarms the Counter is increased when a new Alarm is raised of the same type

Tag

 An Alarm Tag specializes an Alarm instance for a certain business object, e.g. Load Unit Id

Description

User language dependent Message of the Alarm

Acknowledge Time

Time of the acknowledgement of the alarm.

Acknowledger

Name of the acknowledger of this alarm
Comment

Comment

Comment for the Alarm. An entered Comment will be published to all subscribers of this Alarm
Changes
Each status change of an Alarm is shown in the Changes list		  
Alarm Status The Status of this Alarm Change
Comment Comment of this Alarm Change
Created At Date of Alarm Change
User Name Name of the User who caused the Alarm Change, e.g. Acknowledger
