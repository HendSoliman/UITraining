Active

Purpose

In this screen the actual de-palletizing process is done. For each package the bar code is received from the built-in scanner. It is also possible to enter the bar code manually or use a hand scanner.

Between the instructed and entire pallet mode the only different are the shown quantities. In instructed mode the quantity of the current SKU for de-palletizing is shown. As long the quantity to be de-palletized is not reached the same SKU has to put on the conveyor. In case of wrong scan an exception is shown and the operator is instructed to put the correct SKU on the conveyor.
In entire pallet mode the operator can put any package from the pallet on the conveyor.
It is not possible to change the physical position from this screen. If this is necessary the operator can switch to step “Special” .

      
Prerequisites
  • Click Next button in preparation step when no other active position exists.
  •  Activate position in pending step.
  • After current Active step finished. Current position is determined to be active.                                                                                                  :INFO: Determination rule: activate next pallet which has the older arrival date on one of the n source locations

 

      
Screen

Instructed Mode

If SKUQuantityUnit cannot be parsed according to the barcode scanned. Search / Same / Again buttons will be enabled.

 

 

Entire Pallet Mode

 

      
Procedure

Instructed Mode:

Step

Description

1.

Load the pick mission from the workstation queue.

2.

Load next pick of this pick mission considering the sequence.
3. Show the SKU, the total quantity to be palletized and the already de-palletized quantity in the dialog.
4. The operator puts a package from the pallet on the conveyor.
5.

The scanner sends a list of barcodes and a synchronization number
6. Determine the correct package using the received barcode list. If exact one packaging type (SKUQuantityUnit) can be determined use this. Otherwise the operator has to select the correct SKUQuantityUnit.
7. If SkuQuantityUnit cannot determined according to the barcode received from the scanner:
● If this is the same product as last shown, the operator can press the button “Same”.
● Otherwise he/she has to use the “Search” button to identify the correct packaging type.
8. Compare the determined/selected SKUQuantityUnit with the next expected. If both are same call DepalletizingHelper.bookPick with the current pick and quantity = 1. Confirmation code should be ConfirmationCode.PARTIAL.
9. After all picks are confirmed and the pick mission is completed:
● Proceed with idle step, when the pallet is empty or deleted .
Trigger the determination of the next to activate source location.
● Proceed with complete step, when the pallet is not empty

 

Entire Pallet Mode:

Step

Description

1.

Load the pick mission from the workstation queue.

2.

Show the number of packages on the pallet in “#Caces to Depal”.
3. The operator puts a package from the pallet on the conveyor.
4. The scanner sends a list of barcodes and a synchronization number.
5.

Determine the correct package using the received barcode list. If exact one packaging type (SKUQuantityUnit) can be determined use this. Otherwise the operator has to select the correct SKUQuantityUnit.

:INFO: Only the products on the pallets are to consider.
6.

If SkuQuantityUnit cannot determined according to the barcode received from the scanner:

●      If this is the same product as last shown, the operator can press the button “Same”.

Otherwise he/she has to use the “Search” button to identify the correct packaging type.
7. Determine the correct pick from the loaded pick mission for the deter-mined/selected product.
Call DepalletizingHelper.bookPick with this pick and quantity = 1 as parameter. Confirmation code should be ConfirmationCode.PARTIAL.
8. After all picks are confirmed and the pick mission is completed, proceed with idle step .
Trigger the determination of the next to activate source location.
