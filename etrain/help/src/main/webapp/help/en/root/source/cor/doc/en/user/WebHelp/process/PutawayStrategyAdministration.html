Putaway Strategies

Putaway Strategies define a complete set of all required Location Selection Strategies to find the optimal Storage Location during Putaway processes based on specific requirements, basic conditions and configurable parameters.

 INFO 

For configuration purposes, see the related topic in the Warehouse Administration Guide.

The process follows the described flow (for details, see Putaway Process Overview - Basic Process Flow). Putaway Configuration with use of configured values are used with prior consideration of the possible SKU-based Dedicated Pick Faces or Preferred Destinations. The different strategies are affected on the different levels Area, Aisle, Level (only Multishuttle), Bin and Slot (only Multishuttle Flex).

The used level depends on the Load Unit's current destination (defined by the related Transport Request) and defines the next higher (more detailed) Level as the current one.

 INFO 

Cases where strategies are used are described in Putaway Process Overview.

 INFO 

The assigned level detail of the Destination Location depends on the configured granularity of the Area of the current Location of the Load Unit. That means the granularity of all Areas where Putaway Strategies take place for automatic determination of the next transport destination should be aligned with the yet defined current destination of the Load Units, which ask for next transport destination.
