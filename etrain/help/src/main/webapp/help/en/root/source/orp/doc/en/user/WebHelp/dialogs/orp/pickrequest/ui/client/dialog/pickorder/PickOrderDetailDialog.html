Field

Description

Order

Unique Order ID of the Pick Order related Order.

ID

Pick Order ID. Unique within a Client. Usually represented by a running number.

Client

Owner of the Pick Order. Thus, owner of the Inventory to be picked by this Pick Order.

Overview

Pick Zone

Pick Controller Zone (Zone of type "Pick Controller") where the Pick Order must be executed. One Pick Order represents the part of an Order which must be picked within one Pick Controller Zone.

Previous Phase Pick Order

Denotes the Pick Order Id of the consolidated Pick Order of the first phase if Two Phase Picking is processed and this Pick Order represents the second phase.

Status

Status

Current execution state of the Pick Order:

  • pending (Pick Order is awaiting release for fulfillment)
  • released (Pick Order is released for fulfillment)
  • active (Pick Order execution started; the first Source Load Unit was requested or the first Pick was confirmed)
  • complete (Pick Order execution finished and the Pick Order is fully completed; all Pick Order Lines are completed or cancelled)
  • finished (Pick Order execution finished; Pick Order is confirmed to the host, if required).
Configuration Status

Configuration status:

  • complete (Pick Order configuration was accomplished)
  • incomplete (Pick Order can be modified).

Allows users to interrupt their work and continue manual Pick Order creation later.

 INFO 

Pick Orders with configuration status "incomplete" exist if required serial picking and any preceding Pick Zone is located in a Dispo Zone of reservation type "two phase" where the Pick Orders are not created until the second phase. Pick Orders in the subsequent Pick Zones are created directly. Each following Pick Order must be set on hold until the preceding Pick Order has finished. This occurs through the configuration status which switches when preceding Pick Orders of the serial picking process allow to proceed. Thus, the assigned Task has the release mode "triggered". See also the description of "Retrieval Sequence Type strict".

Owner

Source of the Pick Order:

  • host (initiated from an external Host through the host interface. If "finished" status is set, not until host confirmation was executed)
  • operator (manually created by a local user)
  • system (automatically created by the system within the related business process on basis of an Order).

Customer ID

Customer of the related Order. Must be related to an existing Customer of the same Client.

Customer Name

Name of the Customer of the related Order.

Handling

Service Qualifier

Influences concrete fulfillment of the Pick Order within specific execution processes. May be project specific. Can limit the list of available service qualifiers using the configuration topic "orp-order:OrderOrderType.allowedServiceQualifiers".

 INFO 

Can be changed by changing the accordant attribute of the related, unreleased Order.

Retrieval Sequence Type

Chaining type of the multiple Pick Orders of the same Order which must be processed in different Picking Zones:

  • none (different Pick Orders are completely independent without chaining)
  • non strict (different Pick Orders could be picked in parallel picking mode. The Pick Orders of different Picking Zones will be picked into different Pick Containers)
  • strict (different Pick Orders must be picked strictly chained in serial picking mode. All Pick Orders will be picked in serial and usually into the same Pick Container. The Pick Order is not split up to multiple Pick Zones within the Pick Controller Zone, so that it is possible that only one Picker operates the whole Pick Order. The processing sequence of the (Customer) Order across all Pick Controllers (represented by different Pick Orders) is given by the sequence number of the zone and ordered by serialization of related Tasks).

 INFO 

In case of serial picking the Pick Container is handed over from one Pick Zone to the next one within the defined chain. The destination for the transport of the Pick Container is determined project specifically with use of a policy which is defined by the configuration topic "orp-pickconfirmation:pickconfirmationGeneral.SerialPickingDestinationPolicy". With arrival of the Pick Container the related Pick Mission of the assigned Pick Controller (PIC) switches from status "created" to "pending" and is assigned to that Pick Container and thus ready for further picking.

Default Destination

Default destination for all Pick Containers of the Pick Order. Can be a Facility, Area, Aisle, Zone or Location. Used by Order Management as a proposal and can be changed manually or by any business process specific decisions (project specific).
Used only if the default destination is not specified in a Pick Order Line.

 INFO 

Can be changed by changing the accordant attribute of the related, unreleased Order.

Preferred LU Type

Preferred Load Unit Type of the related Order is considered as default proposal for the Pick Container types while appropriate Pick Operation.

 INFO 

Can be changed by changing the accordant attribute of the related Order in any status.

Wave ID

Wave of the related Order. Groups Orders for collective disposition and execution for Picking and Shipping purposes.

 INFO 

Can be changed by changing the assignment of the related Order in any status.

Times

Earliest End Time

Earliest time to finish Pick operations and deliver the last Pick Container of the Pick Order to the required destination.
Set by the earliest staging time of the related Order.

Latest End Time

Latest time to finish Pick operations and deliver the last Pick Container of the Pick Order to the required destination.
Set by the latest staging time of the related Order.

 INFO 

Can be changed by changing the accordant attribute of the related Order in any status.

Pick Start Time

Time the first Pick of the Pick Order proceeds to "active" status.

Pick End Time

Time the last Pick of the Pick Order proceeds to "complete" status.

Given Time

Previously calculated time period for complete fulfillment of the Pick Order. (project specific use only)

 INFO 

Can be changed by changing the accordant attribute of the related Order in any status.

Pallet

Manual Palletizing required

Indicates that manual palletizing is required for the assigned pallet of the Pick Order (due to any reason that prevents from automatic palletizing).

Stretch Program

The stretch program is used to inform the underlying PLC, how a palletized pallet shall be stretched:

  • 0 (no stretching required)
  • 1 (stretching required).

Allowed Label Positions

Positions where labels may be put on a stretched pallet. This information is set during the pallet planning step.

Task

Pick Orders are handled as tasks. The assigned Task leads the scheduling of all Tasks of this Pick Order (see Tasks).
