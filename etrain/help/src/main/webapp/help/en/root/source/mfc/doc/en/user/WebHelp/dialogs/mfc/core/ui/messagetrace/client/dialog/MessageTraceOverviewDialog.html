Message Trace

Purpose

The Message Trace window displays all logged DCI (Dematic Communication Interface) messages. The entries to be displayed can be filtered and organized according to the filter criteria.

This description assumes that the reader is familiar with the principles of DCI communication. Please refer to the DCI documentation for further details.

 

      
Screen

 

      
Sub-Screens

The following sub dialogs can be opened from this dialog:

  • Message Trace Detail

 

      
Fields

The Message Trace window shows the following information:

Field

Description

Trace ID

Unique Id identifying the trace record.

Date

Date of creation
Operation Id DCI operation ID
User Data The user data of the DCI telegram.
Flow Control Flag specifying whether the telegram has to be confirmed with an Acknowledge-Message.
Block Count Number of data blocks that the telegram includes.
Seq. No. DCI sequence number
Source Node DCI source node
Dest. Node DCI destination node
Source Service TIM source service (for DCI identical with the Node)
Dest. Service TIM destination service (for DCI identical with the Node)
Header Data The header data of the DCI telegram.
Message Manager The name of the message manager used for communication.

Connection

Name of the physical connection that will send the message.

 

      
Special Buttons

Button

Description
Toggle button which controls whether acknowledge telegrams shall be displayed or not (default).
Resend Resends the selected telegrams
