Detail Dialog Pick / Pick Mission / Pick Walk

Purpose

This screen opens when viewing the selected data. 

      
Screen

Pick

Pick Mission / Pick Walk

      
Fields

Field

Description

Zone

Pick Controller Zone (a Zone of type "Pick Controller") where the Pick must be executed.

Pick

ID

Unique ID of the Pick.
Status

Current execution state of the Pick:

  • created (Pick is currently not (yet) residing within a Picking Zone and thus not ready for release)
  • pending (Pick waiting for grouping to a Pick Mission)
  • released (Pick grouped within a Pick Mission)
  • active (Pick Mission execution started and all Picks of the Pick Mission proceed to this status)
  • complete (Pick executed by the Picker, but not yet confirmed to the Order Fulfillment System)
  • confirmed (Pick successfully confirmed to the Order Fulfillment System)
  • cancelled (Pick cancelled without the defined execution)
  • finished (Pick Mission finished and all Picks of the Pick Mission proceed to this status).

Reference

Order ID

Order ID. Unique within a Client.
Order Line ID

The identification of the Order Line is unique within an Order and is usually represented by a running number.
Pick Order

Pick Order ID Unique within a Client. Usually represented by a running number.
Pick Order Line

Unique Pick Order Line ID that the Pick is associated with.

Handling
Sequence

Processing order of Picks within the Pick Walk or the Pick Mission in case of sequencing is required for Goods To Person picking.
Urgency

Urgency of the Pick. Set on behalf of the Pick Order. Defines the processing sequence. Lower value indicates a previous execution.
Code Denotes the specific processing of the Pick. Defines the main actions within the Order Fulfillment System after Pick confirmation, as configured in Confirmation Reason screen for the Confirmation Code:
  • OK (target quantity and required services were fully executed)
  • Short (target quantity was not fully executed due to inventory shortage)
  • Full (Pick was split because target quantity does not fit into the Pick Container)
  • Crack (Pick was split because remaining cracked quantity could be picked only with another quantity unit)
  • Blocked (inventory of this Pick is currently blocked and could not be reached; the Pick was not processed further)
  • Skip (Pick is explicitly skipped temporarily and injected to the end of the current Pick Walk for later processing and confirmation)
  • Over (Explicit Picking of more than the target quantity, while observing overdelivery tolerance limits)
  • Under (Explicit Picking of less than the target quantity, while observing underdelivery tolerance limits).
Reason

Current Confirmation Reason of the Pick, which is associated with Function Code. Possible Reasons are configured in the Confirmation Reason screen and define the main actions within the Order Fulfillment System.

Inventory

Pick Mission / Pick Walk

Pick Walk
ID

Unique ID of the Pick Walk.
Status

Current execution state of the Pick Walk:

  • released (Pick Walk automatically time controlled or manually released for execution)
  • active (Pick Walk execution has started)
  • complete (All Pick Missions of the Pick Walk are complete)
  • cancelled (Pick Walk was cancelled without the defined execution)
  • finished (Pick Missions and all required tasks for the Pick Walk are finished).
Offline Picking

If enabled, the Pick Job was started using the Pick Walk screen and thus not processed with any online picking process.
Terminal ID

Unique ID of the Terminal that the Pick Walk is or was assigned to.

Pick Mission
ID

Unique ID of the Pick Mission.
Status

Current execution state of the Pick Mission:

  • pending (Pick Mission waiting for release for execution)
  • released (Pick Mission was automated time controlled released for execution)
  • queued (Pick Mission is assigned to a Workstation (Goods-to-Person only))
  • active (Pick Mission execution has started, in case of Goods-to-Person the Source Load Units are requested for transport to the Workstation)
  • complete (All Picks of the Pick Mission are complete (i.e. confirmed to the Order Fulfillment System or cancelled)
  • cancelled (Pick Mission was cancelled without the defined execution)
  • finished (related Pick Container was delivered and all required Pick Mission tasks are finished).
Blocked

Blocking flag of the Pick Mission. If set, the Pick Mission is blocked due to concurrent use of resources by other Pick Missions and cannot be processed until unblocked (with respect to the "Blocking Management" of the Pick Zone).

 INFO 

 To avoid concurrent transports of the same Source Load Units to different Pick Stations, Pick Missions that require Load Units which are currently not stored are blocked until all required Load Units are stored and thus able to be retrieved. The execution of blocking/unblocking occurs on the Pick Missions level where all required Source Load Units are checked.
The Pick Mission is unblocked only if each single Load Unit is actually stored.
Blocked Pick Missions can be preferably assigned and started if all required blocking Source Load Units are already in transport to the required destination and this optimization is requested. Optimization can also be configured on the Pick Zone level to start blocked Pick Missions.
Pick Container Alias

Usually the Alias of Put Location after a Pick Container assignment to a Put Location. Used to verify the correct Pick Container during picking in Person-to-Goods opeation.
Pick Container ID

Associated Pick Container. All Picks of this Pick Mission are collected in this Pick Container.
The Pick Container groups all Picks of one Pick Mission and contains the already picked Inventory Units.

Workstation
ID

Unique ID of the Workstation where the Pick Mission is or was assigned to.
