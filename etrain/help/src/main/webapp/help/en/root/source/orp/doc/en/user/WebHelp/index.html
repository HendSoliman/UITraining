Order Management

Overview

Within the overall Order Processing the Order Management supports many business processes for administration, maintenance, planning and fulfillment of Orders. All requests for disposition of existing warehouse resources related to Inventory, Load Units or Locations are assigned to the system with creation of appropriate Orders. For each type of Order planning and fulfillment, a particular order type is provided; the most common being a "Shipping Order". The order type controls the principle order planning and fulfillment processes of the Order.

Within order planning, Order Pre-Cubing estimates the number of Dispatch Load Units per Dispatch Load Unit Type.

Orders can be created, updated or cancelled by: an external (Host) system, an internal request within DCD, or manually by a local user.

The main step in the lifecycle of any Order is Order Release to start the required execution processes to fulfill Order related demands. Any preparation and planning steps (mainly for Shipping Orders which are highly project specific) must be finished before release for execution.

Order Release occurs in the Order Fulfillment within the Order Processing. It can be triggered by releasing the related Order Task. With Order Release, Retrieval Strategies are used to allocate the optimal required Inventory for the given Order Line criteria. Order Line criteria (Inventory, Load Unit and Location Criteria) are described in the Retrieval Strategies.

As a result, Full Load Unit Retrievals and Pick Orders are created for subsequent Order Fulfillment. After finishing the fulfillment of a Shipping Order, related Inventory is deleted and the Order may be confirmed to the Host. Shipping Orders can be pooled to Waves. Wave Management provides a basic Wave Release.

Main responsibilities of the Order Management:

  • Global, general Order Lifecycle Management of all supported Order Types.
  • Order Release Management and delegation to the Order Fulfillment for execution processes.
  • Cancellation and Completion of Orders with optional Host confirmation.

Dispatch Load Units

A Dispatch Load Unit is the Order related Shipping equivalent of the Load Unit to support Shipping processes and to represent a shipped unit within an Order.

Order Types

Order Type

Description

Shipping

Shipping Orders are intended to remove Inventory from a warehouse (e.g. Shipping to a Customer or production) by picking certain SKU amounts from stored Load Units to Shipping Units. Shipping Orders are usually imported by an external Host system or manually created by a local user.

Replenishment

Replenishment covers stock relocation in a warehouse to support Picking and delivery; having the required quantity of the required stock accessible at the optimal Location. Replenishment processes are described in the Replenishment overview. Replenishment Orders are usually created by the system during Replenishment planning processes.

Stocktaking

Stocktaking covers the determination of stock levels of stored Inventory within a warehouse. Typical methods include, Perpetual Stocktaking or End-of-Period Stocktaking. Stocktaking processes are described in the Stocktaking overview. Stocktaking Orders are usually created by the system during stocktaking planning processes, if initiated by the Host. Or, they are created manually by a local user.

Relocation

Relocation covers any relocation operations for Load Units within Storage Areas of a warehouse, unrelated to any process represented by another order type. The creation of Relocation Orders is project specific.

Operator Retrieval

Operator Retrieval Orders are intended to move Load Units inside any warehouse Areas for outbound transports. The source of such operator retrieval moves can be a single Load Unit, a Location, a complete Aisle, an Area or a Facility. The destination can be manually specified or calculated by the system using Putaway strategies. Operator Retrieval Orders are created by a local user only. If an operator wants to retrieve a specific Load Unit for manual inspection, an Operator Retrieval Order supports this request.

 INFO 

Since the Client is part of the reservation, the Client of the Order and the Client that the Inventory or Load Unit belongs to may differ.

Clearing

Clearing Orders are intended to initiate transports or Clear Trips for clarification purposes of locked Locations. Clarification processes are described in the Clarification overview. Clearing Orders are usually created by the system or by operators using the Dialog Clarification Bin.

 INFO 

Since the Client is part of the reservation, the Client of the Order and the Client that the Inventory or Load Unit belongs to may differ.
