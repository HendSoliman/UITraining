Fields

Field

Description

Advice ID

Identifies an Advice uniquely within a client.

Overview

Status

How far the Advice was processed:

  • pending (Advice is completely prepared for processing and available for release)
  • released (Advice is released for processing and available for operational receiving)
  • active (operational receiving process of at least one Advice Line has started)
  • receiving complete (receiving process of all Advice Lines finished, i.e. inventory creation was completed for all Advice Lines)
  • complete (Advice is completely processed and all related Load Units arrived at final Storage Locations)
  • to confirm (Advice is ready for confirmation message to the host system)
  • finished (All required tasks for the Advice are finished. If required, advice was confirmed to the host)

Configuration Status

If Advice configuration was accomplished and the Advice can be processed. Allows users to interrupt their work and continue Advice creation later.

Owner

Client

Owner of the advice within that new inventory shall be managed.

Owner

If Advice was created by a host or locally by the LMS. A locally created Advice is not confirmed to a host.

Handling

Advice Type

Origin of the inventory:

  • Supplier Advice
  • Production Advice
  • Return
  • Unexpected Advice (for manual Advice creation within this screen)

Confirmation Type

How the Advice shall be confirmed for host Advices:

  • With Handling Units (Host needs information about the stored Load Units)
  • Without Handling Units (Host don't need this information)

 INFO 

Clustering of host confirmation within an advice occurs dependant on the configuration topic "ibd-receivinggeneral:adviceConfirmationTopic.confirmationMode" (per finished Advice, Advice Line, or Advice Line Load Unit).

Default Destination

Default destination for announced inventory or Load Units. Can be a Facility, Aisle, Zone or Location. Must be interpreted as a proposal.

Number of Load Units

Number of Load Units announced with this Advice. Calculated once, after the Advice is completely entered by a user or a host system.

Detail

Expectation Date

When goods are expected to arrive in the warehouse.

Arrival Date

Set by receiving management when the first inventory is created for this Advice.

Reference

Supplier

Client within that new inventory shall be managed.

Purchase Order

PO or ASN (advanced shipping note) used to receive the inventory.

Delivery Text

Usually contains handling instructions for the announced inventory. Displays in receiving dialogs. Is default text that can be overwritten by the delivery text in the Advice Line.

Task

Every Advice with a complete status is attached to a Task for releasing. Based on the configuration, an operator must activate this Task or the Task is activated automatically (see Tasks).
