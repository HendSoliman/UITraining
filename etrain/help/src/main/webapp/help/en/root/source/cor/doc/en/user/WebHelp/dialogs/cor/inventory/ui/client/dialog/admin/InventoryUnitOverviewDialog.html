Inventory Units

Purpose

The identifying part of an Inventory Unit is composed of the Client, SKU and sometimes batch. It can be further defined by Host Plant and Host Storage Location. As an Inventory Unit is associated with a LU, the LU number belongs to the identifying part.

Thus, an Inventory Unit is an undistinguishable part of an article (SKU) related Inventory on a specific Load Unit with a given quantity and further configurable Inventory separating attributes.

 INFO 

The Unit of the displayed dimensions are given by the appropriate Base Quantity Units defined by the "Dialog Unit of Measure".

 INFO 

Setting or removing Inventory holds on Level of single Inventory Units, the whole Inventory of a SKU or of a SKU/Batch combination is done within the Inventory Hold Assignment screen. This can be launched from this screen.

 INFO 

Configuring separating attributes is done SKU Blueprint specific and described in detail in SKU Blueprint Configuration.
       
Screen

The lower grid tabs display the Inventory Units Holds and Reservations.

      
Inventory Unit Hold

Display of all holds made on the selected Inventory Unit. Inventory Unit Holds could be set or released with the Inventory Hold Assignment screen launched by the provided button.

      
Inventory Unit Reservation

Display of all reservations of the Inventory Unit. In addition to the reserved quantity the referencing Order Line and the reservation reason are shown

 INFO 

Only the reservations on the single Inventory Unit is mentioned here. Aggregated Inventory reservations are assigned to the Aggregated Inventory screen.

Field

Description

Quantity

Portion of Inventory Unit reserved by this Inventory Unit Reservation.

Quantity Unit

Quantity Unit of the Reservation.

Client

Client of the Reservation on Inventory Unit level.

Order

The Order, Order Line and reservation reason reference the object that requested the reservation. Reserved inventory must not be used for another purpose as specified by the Order, Order Line, and reservation reason.

Order Line

The Order, Order Line and reservation reason reference the object that requested the reservation. Reserved inventory must not be used for another purpose as specified by the Order, Order Line, and reservation reason.

Priority

Importance of the reservation. Higher value indicates higher priority. Reservations with lower priority are cancelled earlier than those with a higher priority.

Reservation Reason

Business process type that caused the reservation (ex: Replenishment, Shipping). The Order, Order Line and reservation reason reference the object that requested the reservation. Reserved inventory must not be used for another purpose as specified by the Order, Order Line, and reservation reason.

Special Buttons "Upper Grid"

Button

Description

Opens the Correct Quantity screen to change the quantity of the selected Inventory Unit.

Opens the Inventory Hold Assignment screen to hold the Inventory/SKU/batch.

Opens the Modify Special Inventory screen to set a special inventory mark and a special inventory reference (multi-selection possible).

Retrieve the Load Unit(s) of selected Inventory Units.
An available Workstation must be chosen first.

 INFO 

Access rights must be available for the Load Unit Types to the selectable Workstations.

Reprint the Packaging Labels
