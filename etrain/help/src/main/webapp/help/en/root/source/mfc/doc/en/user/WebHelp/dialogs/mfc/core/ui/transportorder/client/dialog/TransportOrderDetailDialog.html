Status

Status

Special execution states of the Transport Request:

  • created (created and not ready for pending. Project specific use only)
  • pending (awaiting release for execution)
  • released (released for execution and sent to the underlying transport system)
  • active (execution started)
  • complete (execution finished) (Transport Request is directly deleted)
  • cancelled (cancelled with Load Unit logically located at current Location) (Transport Request is directly deleted)
  • error (aborted due to error situation).

Reason

The transport reason classifies the order fulfillment or transport related motivation for the transport request:

  • Pick Container (Transport of a completed Pick Container)
  • Empty Load Unit (Empty Load Unit into a buffer)
  • Product Load Unit Request (Outbound transport of a product)
  • Product Load Unit Request without Load Unit ()
  • Product Load Unit Putaway (Inbound transport of a product)
  • Pick Container in Pick Zone ()
  • Full Load Unit Retrieval (Full Load Unit is requested by an Order)
  • Shuffle Move (Load Unit blocks and must be shuffled)
  • Storage ()
  • AMCAP Balancing ().

Type

Direction or purpose of a transport:

  • Inbound (Load Units must have a possible final destination and a Storage Location of type, "bin")
  • Outbound (Load Units must have a Source Storage Location of type, "bin")
  • Relocation (Relocate a Load Unit from one Location to another Location of any type)
  • Clearing (Initiated for Load Units that must be clarified and should have Final Destination Locations of type, "transport")

Special Trip

Representation of a transport for special business or transport issues:

  • none (standard entry)
  • clear trip (initialized as a transport for clearing of an error situation. Destination is usually a Clarification Station.)
  • clear trip UFO (usually initialized as a transport for clearing of a "bin full" error situation. Logically empty Storage Location must be clarified using a clear trip with an unknown transport object. The Load Unit is represented by a virtual "UFO-ID".)

External System

Underlying transport control system; responsible for execution of the transport of the Load Unit, e.g. MFC.

Processing States - Transport Order

The following diagrams shows the available state and related state transitions of a transport order:

State

Description

created

Initial state after creation of transport order.

active

Transport order is under control of transport subsystem e.g. vehicle, however the PLC has no mission.

sent

Transport order is under control of PLC or message manager. An active mission exists.

finished

The global destination is reached. The transport order will be confirmed to the host system.

rejected

Transport order cannot be processed due to invalid data or configuration problems.

queued

Transport order is waiting for execution in a sequencing Multishuttle scenario.

waiting

Transport order is waiting for execution (e.g. it is member of a transport group and is waiting for its predecessor to complete).

scheduled

Transport order is planned to be executed by a fork lift truck but the truck has not yet picked it up. This status exists only in the context of the FLS.
