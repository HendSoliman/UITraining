Location

Purpose

This topic describes Location configuration.

      
Configuration

Locations are a central resource for a Warehouse Management System. They are needed to represent storage and transport locations. A location is the basic spatial unit which can be addressed in a warehouse. For an overview of all elements used to group locations in a warehouse, see the Warehouse Structure description.

Not all transport locations that are known to the MFC or PLC must be represented in LMS. The LMS should represent exactly the locations that may be used to communicate between LMS and MFC.

A multi-deep bin in an area with automatic disposition is represented as one location. Load units are stored on different positions of the multi-deep location. A load unit is booked onto a multi-deep location using the layout assigned to the location type and the load unit type. The z-coordinate of the location is not used for the representation of a multi-deep bin in an area with automatic disposition. It is set to 1 in this case.

Configuration of Location Mapping

For locations that are relevant in an external system such as the MFC, an external location mapping must be defined.

For transport points, a location mapping with undefined position and undefined external position is usually required.

For bins the standard LMS supports DCI coordinates. The last 2 digits of a DCI coordinate contain raster number and position. Because the raster number can change with location allocation, the last 2 digits of the coordinate are appended automatically by LMS. Therefore, for bins, the DCI coordinates without the last 2 digits must be configured as external name for each possible position of the bin.

Bins in areas without automatic disposition usually require a location mapping with undefined position and external position 1.

      
Configuration of transport points

The following transport points must be configured:

Types

Description

Transport Start Points

Each location, where a transport request may start and which is not a bin, must be configured as transport point.

Transport End Points

Each location, where a transport request may end and which is not a bin, must be configured as transport point.

Aisle Decision Points

Each location, where the MFC may request the destination aisle for a load unit, must be configured as transport point.

Bin Decision Points

Each location, where the MFC may request the destination bin for a load unit, must be configured as transport point. If the bin decision point is an entry point of an aisle, it is recommended to assign this point to the destination aisle.

SRM Fork

For each miniload or highbay warehouse SRM, a transport point must be configured. Then if a bin full exception occurs, the load unit is booked onto the fork temporarily (when an alternative destination bin is selected).
