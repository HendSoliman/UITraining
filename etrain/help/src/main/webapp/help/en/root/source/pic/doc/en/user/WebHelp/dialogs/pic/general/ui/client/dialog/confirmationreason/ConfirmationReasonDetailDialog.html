Fields

Field

Description

Client

All combinations of Codes, Reasons, and Error Codes depend on Clients.
ID Technical ID of the Confirmation Reason. Must be unique together with the client.

Sequence

Ranking of the Confirmation Reasons of one Confirmation Code within screen selections.

Confirmation Code

Confirmation Code that the Confirmation Reason is assigned to.

  • OK (target quantity and required services were fully executed)
  • Short (target quantity was not fully executed due to inventory shortage)
  • Full (Pick was split because target quantity does not fit into the Pick Container)
  • Blocked (inventory of this Pick is currently blocked and could not be reached; the Pick was not processed further)
  • Skip (Pick is explicitly skipped temporarily and injected to the end of the current Pick Walk for later processing and confirmation)
  • Over (Explicit Picking of more than the target quantity, while observing overdelivery tolerance limits)
  • Under (Explicit Picking of less than the target quantity, while observing underdelivery tolerance limits).

Reason

The reason for the Confirmation Code is a description of the confirmation for further classification and additional display purposes. Only separately defined values are available.

Error Code

Pick error code assigned. Directs the superior Order Fulfillment System on how to book this entry. Also defines the main actions within the Order Fulfillment System after Pick confirmation.

  • OK (Pick Order Line successfully completed)
  • missing quantity (Pick Order Line not fully picked due to inventory shortage)
  • load unit missing (Pick Order Line not fully picked due to no Load Unit found)
  • wrong SKU (Pick Order Line not fully picked due to wrong SKU found)
  • wrong load unit (Pick Order Line not fully picked due to wrong Load Unit found)
  • wrong best before date (Pick Order Line not fully picked due to wrong Best Before Date of the inventory found)
  • wrong packaging level (Pick Order Line not fully picked due to wrong packaging level)
  • breakage (Pick Order Line not fully picked due to broken inventory)
  • blocked (Pick Order Line not fully picked because location to be picked from is blocked and inaccessible)
  • underdelivery (Pick Order Line not fully picked due to explicitly picking of less than target quantity)
  • overdelivery (Pick Order Line not fully picked due to explicitly picking of more than target quantity)
  • split (Pick Order Line was split because Pick Container was full).

Default

This reason is used as a default when no reason is set with confirmation. For each Client and Eror Code combination one default reason may be defined.
