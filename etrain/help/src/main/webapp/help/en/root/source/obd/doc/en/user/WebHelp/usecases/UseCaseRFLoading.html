Loading with RF-Terminal

Description

The Loading using a RF-Terminal provides all required functionalities for the whole loading process of Dispatch Load Units of a Load at a selected Dock.

      
Triggers and Actors

The use case is started when

  • All Dispatch Load Units of a Load are marshaled in the appropriate Shipping Area
  • This Load is authorized for loading and the loading process should be started with support of a RF-Terminal.
      
Prerequisites
  • The Load must be available in status "authorized for loading"
  • The Load has been assigned either to a Route or to Dock and truck
  • The Dispatch Load Units have been staged in the corresponding Marshalling Grid.
      
Required Resources
  • RF-Terminal Loading.
      
Work Flow

Step

Description

1.

Open the screen for "RF-Terminal Loading" from the main menu.

2.

The dialog starts with "Scan Dock".

3.

Scan the Dock.
Key "Enter" is activated implicitly with scan.

3a.

In case of an invalid Dock is scanned an error message appears. Proceed with step 3.

4.

The dialog proceeds to "Identify Truck".

4a.

In case of a Dock and a Truck are already assigned to the Load the dialog proceeds to "Load".
Proceed with step 6.

5.

Scan or choose a Route (with already specified Dock and truck).
Key "Enter" is activated implicitly with scan.

6.

Scan the Load Unit to be loaded.
Key "Enter" is activated implicitly with scan.

7.

Repeat step 6. for all Load Units to be loaded.
After loading the last one choose the function "complete".

8.

The dialog proceeds to "Close Load".

9.

Confirm the closure of the Load with "OK".
The dialog proceeds to "Confirm Departure".

10.

Confirm the departure of the truck with "OK" and thus release the Dock.
     
Result

The result of this use case is the successful processing of loading of a selected Load.
The loaded Dispatch Load Units are 'shipped' and the Load is 'departed'.
The Dock is available for further loading.
