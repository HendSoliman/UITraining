Retrieval Administration

Purpose

This a special service screen to fix various problems in the warehouse. It provides a detailed overview about all main retrieval data of Logistics Manager and Pick Controller and superior filters, which filter all data within all levels in the context of the specific filter.
For a fast overview, the number of result sets is provided for the open/finished Pick Order Lines, Full Load Unit Retrievals (FLRs), Pick Container (PC) and Dispatch Load Units (DLU).

Using the context menu of each grid, related data in the other grids can be selected.
The first grid displays the Pick Order Line with data from Order, Order Line and Pick Order. Orders which are not yet started display as well. The second grid provides data for FLRs, PCs and DLUs. The third grid provides Pick related data; Picks, Pick Missions and Pick Walks combined with Pick Orders and Orders.

 INFO 

To hide all provided actions in this dialog, there is a predefined User Group called "RetrievalInformationDialog", which can be assigned to Users that shall not have permission for these actions

Superior Filters

"Superior filters" allow data selection in all grids.

  1. Order:  valid Order ID is required.
    > Filter all grids with this Order ID.
  2. Load Unit:  valid Load Unit ID is required.
    Register Pick Order Line (first tab)
    > OR-Conjunction on attributes: Source Load Unit, Source Base Load Unit and Pick Container
    Register FLR/PC/DLU (second tab)
    > OR-Conjunction on attributes: FLR Load Unit ID, Pick Container and Dispatch Load Unit
    Register Pick (third tab)
    > OR-Conjunction on attributes: Source Load Unit, Source Base Load Unit and Pick Container 
  3. Pick Walk:  valid Pick Walk ID is required.
    Register Pick (third tab)
    > Filter with this Pick Walk ID
    Register Pick Order Line and FLR/PC/DLU (first and second tab)
    > all available Orders according to the selected Picks can be entered.

Special Buttons

Button

Description

Remove (delete) Pick Orders.

 INFO 

Only Pick Orders with configuration status "incomplete" can be removed.
Pick Orders of this kind only exist if serial picking required and any preceding Pick Zone is located in a Dispo Zone of reservation type "two phase" where Pick Orders are not created until the second phase. Pick Orders in subsequent Pick Zones are created directly. Thus, it is required to set each following Pick Order on hold until the preceding Pick Order has finished. This occurs through the configuration status which switches when the preceding Pick Orders of the serial picking process allows to proceed. Thus, the assigned Task has the release mode, "triggered".

Cancel Pick Order Actions

 

 Cancel Pick Orders. Only possible for Pick Orders that are "pending", "released" or "active".
The Replacement Dispo flag allows to trigger a new reservation for the canceled parts of the Pick Order. If not set, cancellation is performed with underdelivery without consideration of the underdelivery tolerance of the Order. 

 

Cancel Pick Order Lines. Only possible for Pick Order Lines that are "pending", "released" or "active".
The Replacement Dispo flag allows to trigger a new reservation for the canceled parts of the Pick Order Line. If not set, cancellation is performed with underdelivery without consideration of the underdelivery tolerance of the Order.

If multiple Inventory Units refer to the selected Pick Order Line, the first one is used for correction.

Tidy Actions

 

Tidy Orders related to selected lines tolerating inconsistent data and partially ignoring order states. All open activities belonging to the given orders are cancelled. If the orders are not yet in status FINISHED, they will be confirmed to the host system. 

 

Tidy Order Lines related to selected lines tolerating inconsistent data. All open activities belonging to the given order lines are cancelled. 

 

Tidy Pick Orders related to selected lines tolerating inconsistent data and partially ignoring pick order states. All open activities belonging to the given pick orders are cancelled. If the pick orders are not yet in status FINISHED, they will be confirmed to the host system. 

 

Tidy Pick Order Lines related to selected lines tolerating inconsistent data. All open activities belonging to the given pick order line are cancelled. 
Full Load Unit Retrievals / Pick Container / Dispatch Load Units (2nd tab)

Cancel Full Load Unit Retrievals. Only possible for Full Load Unit Retrievals that are "released". Requests for cancellation are sent to the related underlying transport system.

 INFO 

Cancellation for a single Full Load Unit Retrieval is refused if the Full Load Unit Retrieval Transport is already finished.

 INFO 

With manual cancellation of a Full Load Unit Retrieval, no Replacement Dispo is initiated and the underdelivery tolerance of the Order will not be considered!

Confirm transport of Load Units at the defined destination. Can be a Full Load Unit Retrieval (FLR), a Pick Container (PC) or a Dispatch Load Unit (DLU). Only available for "released" FLRs and not yet "completed" PCs and DLUs.

Only LUs that fulfill the following the conditions are considered:

  • No yet arrived at final destination and
  • Existent Transport Order in MFC or
  • Existent Transport Request in LMS

Tidy Actions

 

Tidy Full Load Unit Retrieval groups related to the selected lines tolerating inconsistent data. All open activities belonging to the given group are cancelled. (A Full Load Unit Retrieval group is defined by all Full Load Unit Retrievals of the related order.)

 

Tidy Full Load Unit Retrievals related to the selected lines tolerating inconsistent data. All open activities belonging to the given full load unit retrieval are cancelled. 

 

Tidy Pick Containers related to the selected lines.

Remove block flag of Pick Missions. Only available when this flag is set. Pick Missions then are available for processing.

 INFO 

Handles concurrent requests of Load Units and ensures that each Load Unit is requested only once at a time. Be sure that no Load Unit of the selected Pick Missions is used by another Pick Mission which is not blocked.

Remove Pick Mission from the Queue of the Workstation. Only possible for single selection and "queued" or "active" Pick Missions for Goods-to-Person Picking. Besides clearing the queue, the Pick Mission is set back to status "released".

Cancel Pick. Only possible for Picks in "active" status and assigned to a Person-To-Goods Pick Zone. Only one Pick can be cancelled at once (single selection).

 INFO 

Order Fulfillment System is informed about this cancellation without request of Replacement Dispo.

Tidies Picks belonging to the selected lines while tolerating inconsistent data.
