Field

Description

Order ID

Order ID is only unique within a Client.

Client

Owner of the Order. Thus, owner of the Inventory to allocate with this Order.

Type

Controls principle order planning and fulfillment processes. See Order Types in Order Management Overview for a listing.

Overview

Priority

Influences the urgency (see below). Detailed information about what influences the priority is only available within the Host system as the owner of the (Shipping) Orders. Internal Orders receive a default value. Order priorities can be manually overwritten, for multiple orders simultaneously, using the Modify specific settings button on the screen Orders. A priority entered by a user must be at least 1. System itself is allowed to set priority >= 0. There is no limit for a maximum of priority.

Next Order Planning Processor

The next order planning processor, which is to be applied after processing of the current one is completed (which need to be explicitly configured as successor).

Status
Status

Current execution state of the Order:

  • created (Order was created and not yet ready for pending, but ready for preparation phase (project specific use))
  • prepared (Order preparation phase is finished, but not ready for pending (project specific use))
  • pending (Order is awaiting release for fulfillment)
  • released (Order is released for fulfillment)
  • active (Order execution already started; Full Load Unit Retrievals and/or Pick Orders are in progress)
  • complete (Order execution is finished and Order is fully completed; all Order Lines are completed or cancelled)
  • to confirm (Order is ready for confirmation message to the host system)
  • cancelled (The Order is successfully canceled and waiting for confirmation to be sent to host)
  • finished (All required tasks for the Order are finished. If required, Order was confirmed to the host).
Paused

An Order can be held if it should not be processed for any reason at this moment.

This value could be changed by the Modify Specific Settings button for multiple Orders simultaneously.

 INFO 

If the underdelivery tolerance of any Order Line will be violated while Order Release the further actions on the Order will be configured by the configuration topic "orp-orderfulfillment:OrderFulfillmentClientOrderType.underdeliveryAction". Possible actions are:

  • DEFAULT (Cancel the underdelivered Order Lines only)
  • PAUSE (Set the Order to "paused" and stop further processing until either manual cancellation or further order release. Pause functionality is not supported during the replacement reservation process.)
  • CANCEL (Cancel the Order)
  • NONE (Proceed all actions on the Order Line with yet available quantity and ignore the underdelivered quantity. Further inventory reservation is possible until the status is not yet complete.)
Planning Status

Several order planning processing steps can be defined in a project specific way. Reflects the entry status for the next processing step:

  • to be cubed for ErgoPal (Determination of pallets for use with ErgoPal palletizing is required)
  • to be cubed for case picking (Determination of shipping cartons (DLUs) and pallets for consolidation is required)
  • cubed (Pallets and shipping cartons (DLUs) have been determined)
  • waiting for route assignment (The Order waits on assignment of a route by TMS)
  • route assigned (A Route has been assigned to the Order)
  • load planned (The Load has been planned)
  • planning completed (The order planning has been completed)
  • pre-cubing (Determination of Dispatch Load Units is required)
  • pre-cubing failed (Determination of Dispatch Load Units failed).
Configuration Status

If order configuration was accomplished and the order can be processed. Allows users to interrupt their work and continue order creation later.
Use Modify Specific Settings button to change it for multiple Orders simultaneously:

  • incomplete (Data creation not yet completed. Usage prohibited.)
  • complete (Data creation completed. Usage allowed.) 

Owner

Source of the Order:

  • host (initiated from an external Host by the host interface)
  • operator (manually created by a local user)
  • system (created automatically by the system within the related business process).

Parent Order

If an Order is split, the Parent Order groups all split Orders and represents the original or "root" Order. A root Order never has a parent Order.

Handling

Service Qualifier

Influences concrete fulfillment of the Order within specific order planning or execution processes. May be project specific. Can limit the list of available service qualifiers using the configuration topic "orp-order:OrderOrderType.allowedServiceQualifiers".

Retrieval Sequence Type

Chaining type of the multiple Pick Orders of the Order which must be processed in different Picking Zones:

  • none (different Pick Orders are completely independent without chaining)
  • non strict (different Pick Orders could be picked in parallel picking mode. The Pick Orders of different Picking Zones will be picked into different Pick Containers)
  • strict (different Pick Orders must be picked strictly chained in serial picking mode. All Pick Orders will be picked in serial and usually into the same Pick Container. The Pick Order is not split up to multiple Pick Zones within the Pick Controller Zone, so that it is possible that only one Picker operates the whole Pick Order. The processing sequence of the (Customer) Order across all Pick Controllers (represented by different Pick Orders) is given by the sequence number of the zone and ordered by serialization of related Tasks).

 INFO 

In case of serial picking the Pick Container is handed over from one Pick Zone to the next one within the defined chain. The destination for the transport of the Pick Container is determined project specifically with use of a policy which is defined by the configuration topic "orp-pickconfirmation:pickconfirmationGeneral.SerialPickingDestinationPolicy". With arrival of the Pick Container the related Pick Mission of the assigned Pick Controller (PIC) switches from status "created" to "pending" and is assigned to that Pick Container and thus ready for further picking.

Wave

Groups Orders for collective disposition and execution for picking and shipping purposes.

Wave Exclusion

If activated, the Order must not be included in a Wave.

Preferred LU Type

Considered while Retrieval Strategies determine the optimal Load Units to be delivered and as default proposal for the Pick Container types during pick operations.

Shipping
Calculated Volume

Calculated volume of inventory required to fulfill the order. Usually calculated during order preparation (e.g. while cubing) (project specific use only).
Calculated Weight

Calculated weight of inventory required to fulfill the order. Usually calculated during order preparation (e.g. while cubing) (project specific use only).
Route

The Order could be grouped to routes which defines the same transport for collective shipping. The Route is only a grouping property without its own features.
Use Modify Specific Settings button to change it for multiple Orders simultaneously.
Destination

Default destination for all Order Lines. Used if the destination was not specified in an Order Line.
Intermediate Destination

Location where all Load Units belonging to this Order must be transported to, before they arrive at the final destination.
Shipping Condition

For informational purposes (project specific use only).
Loading Sequence

Loading Sequence for all Load Units of the Order within the same Route into the shipping resource (e.g. truck). For a given Loading Sequences for a Route, shipment transports consider the Loading Sequence. Loading Sequence is the reverse of the externally defined Unloading Sequence of Route related Orders.
Host Shipping Point

For informational and organizational purposes to support the shipment operations. Set by the Host.

Use Modify Specific Settings button to change it.
Unload Point

For informational and organizational purposes to support shipment operations. Set by the Host.

Detail

Customer Order Reference

External Order Type

Documentation Details

Reason

Reason for the Order. For informational purposes (project specific use only).

Times

Execution
Earliest Start Time

Earliest time to start the order fulfillment operations (transports or picking). If scheduled automatic release of Orders is configured, should be the earliest possible time for Order release.
Latest Start Time

Latest time to start the order fulfillment operations (transports or picking). If scheduled automatic release of Orders is configured, should be the latest possible time for Order release.
Real Start Time

Actual start time of fulfillment operations.
Start Picking Deadline

Planned deadline for starting the picking (project specific use only).
Use Modify Specific Settings button to change it for multiple Orders simultaneously.
Pick Date

Planned date for picking fulfillment (project specific use only).
Earliest Staging Time

Earliest time to deliver to the staging area. (calculated as configured fix offset to the Latest Staging Time).
Use Modify Specific Settings button to change it for multiple Orders simultaneously.
Latest Staging Time

Latest time to deliver to the staging area. (If shipping transports are planned with "Pick-By-Time Requirements", is usually the Time after the required shipment is no longer possible). (Set by configured default (Control Data Parameter) if not specified directly or calculation possible with configured special "Staging Time Determination Policy" or calculated as configured fix offset to the target shipping time).
Use Modify Specific Settings button to change it for multiple Orders simultaneously.
Real Staging Time

Date when actual staging (marshalling) operations are confirmed as finished.

Shipping
Loading Time

Date when actual loading operations are confirmed as finished.
Finish Time

Actual time of finishing fulfillment operations.
Target Shipping Time

Planned date for shipping operations (calculated with configured "Shipping Time Determination Policy").
Use Modify Specific Settings button to change it for multiple Orders simultaneously.
Current Shipping Time

Date when actual shipping operations are confirmed as finished.
Delivery Time

Time the Order should arrive at the Customer's unloading address.

 INFO 

 This is the only time that must be set during manual Order creation. All other Times depend on the desired Delivery Time and are calculated due to global configurations or policies for Order Management.

Use Modify Specific Settings button to change it for multiple Orders simultaneously.

Palletizer

Last Pallet Planning Start

Time, when the last pallet planning run has been started for this order.

Last Pallet Planning End

Time, when the last pallet planning run has been ended for this order.

Pack Builder Error Reason

More detailed Error Reason that explains, why the Pack Builder could not build all pallets for this Order.

 INFO 

Only set, when the Service Qualifier of the Order is set to 'AMCAP'.

Additional Text

ID

Unique identifier of the text. Specifies the use of the text. Thus, the qualification of the type of text.

Text

Actual text of the Additional Text object. Can be project specific, e.g. for label printing.

Task

Orders are handled as tasks. The assigned Task leads the scheduling of all Tasks of this Order (see Tasks).
