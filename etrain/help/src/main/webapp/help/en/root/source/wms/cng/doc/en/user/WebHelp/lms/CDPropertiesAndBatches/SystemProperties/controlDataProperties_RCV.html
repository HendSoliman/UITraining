Configuration Parameters - Receiving

Purpose

This topic describes the configuration parameters of Receiving.

ReceivingGeneral.confirmationMode

DomainValue (AdviceConfirmationMode)

Client

PER_ADVICE

The advice confirmation mode defines at which time a confirmation for an advice shall be submitted. The possible values are:

  • PER_ADVICE (the Advice is confirmed, if an Advice has been completed)
  • PER_ADVICE_LINE (the Advice is confirmed, if an Advice Line has been completed)
  • PER_ADVICE_LINE_LU (the Advice is confirmed, if an Advice Line Load Unit has been completed)

 INFO 

Be aware that the confirmation mode affects the point in time where the inventory is set to 'free' as described below.

ReceivingGeneral.

confirmationOnStorage

boolean

Client

true

This parameter contains 'true', if inventory can be booked to receiving finished (availability status 'free') on storage of the inventory. Otherwise the inventory is free as soon as inventory is created.

 INFO 

The inventory booking depends on the confirmation mode as well. The availability status is set to 'free' not until the corresponding Advice Line Load Unit is confirmed or, in case of the 'confirmation on storage' is set to false, not until the receiving of the corresponding Advice Line Load Unit is completed. Depending on the confirmation mode this applies to either the Advice Line Load Unit or the Advice Line or the complete Advice.

Receivingprocessing.

boolean

 

true

This parameter contains 'true', if an advice line is automatically closed by the Full Pallet Receiving component, if the target quantity of the Advice Line has been reached.

Receivingprocessing.

calculateFifoDateUsingShelfLife

boolean

Client

false

This parameter contains 'true', if the FIFO date must be calculated using the current date and the SKU attribute shelf life, when no best before date is defined. The calculation of the best before date is defined as follows:

 

  • If the best before date is de-fined, use the best before date as FIFO date.
  • If this parameter is true, the FIFO date is set to the current day (the time is constantly 00:00) plus the shelf life of the SKU.
  • If this parameter is false, the FIFO date is set to the current day (the time is constantly 00:00).
