Manual Palletizing Operation

Purpose

The Manual Palletizing Dialog is used to cover the exceptional cases during the automatic palletizing process. This dialog provides all necessary functionalities to complete the current pallet and bring the system into a stabile state.

Detailed information about palletizing is provided in PalletizingResult, which is unique for each PickOrderLine

3D visualization Overview

The current package, which was notified by notifyItemArrival event is shown in red color and the last one in green. The last confirmed item is shown in blue. Already booked items are shown in grey and the cancelled items in black.
When using the slider the current selected item is shown in blue (only pos-sible to switch between packages, which are already confirmed on the pal-let).
In case of dialog start the next to palletize item is the active Pick Order Line with the lowest packSequenceNumber (packSeqNr. is available on the PalletizingResult).
This can be same with the package booked on the source location, but it can also differ.

  • red: package which is currently booked on the Pickstation (CCPA01PS01)
  • green: package which is booked on the Pickstation, but not longer physical there, e.g. the operator has taken this package from the con-veyor but not yet confirmed on the pallet.
  • blue: the latest confirmed package on the pallet, the Pick Order Line Status is complete
  • black: the Pick Order Line Status cancelled
  • grey: All other already confirmed packages the Pick Order Line Status is complete
      
Fields

Field

Description

Pick Order

The current pick order for manual palletizing, it's the id of a pick order

Sequence Number

The current picking package number, packages are picked by their package number. All packages are strictly in sequence

Load Unit Id

It's an id of  the pallet of Pick Container

Open Packages

It denote how many packages are not picked
