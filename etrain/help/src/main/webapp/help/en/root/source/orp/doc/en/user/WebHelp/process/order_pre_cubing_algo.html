DLU Quantity Estimation

Process: Order Pre-Cubing

Purpose

This describes how Dispatch Load Units and their content are calculated by default Order Pre-Cubing methods. Order Pre-Cubing is an Order planning step. The default Order Pre-Cubing method estimates the number of Dispatch Load Units per type.
The Order Pre-Cubing result can be used by an external Transport Management System (TMS) for planning transportation routes and truck capacities prior to order fulfillment.

 INFO 

The Order Pre-Cubing process can be triggered manually in the screen Order Pre-Cubing Results or automatically using "DLUQuantityEstimationDemon". This demon applies the Order Planning Status with a specific planning lifecycle which is configured in the configuration topic "orp-order:orderClientOrderType.planning.OrderPlanningStatusSequence". By default the demon performs the Pre-Cubing method "DLU Quantity Estimation" for Orders in planning status "pre-cubing". (If a failed operation, the status changes to "pre-cubing failed").
Overview

Order Pre-Cubing methods provided by default:

Method

Description

DLU Quantity Estimation

Estimates the number of Dispatch Load Units for each DLU type for a given Shipping Order. This is a step within Order planning, usually required to provide information on the type and number of Dispatch Load Units to a Transport Management System (TMS) for transport planning.

Further possible methods (examples for project specific enhancement only):

Method

Description

Volumetric

Cubing is based on volumetric considerations. In addition, item dimensions are considered for container size.

Exact

An exact pre-cubing is required which determines the individual Dispatch Load Units with their content. This Pre-cubing method can be used for Pick&Pack systems.

Exact Sequence

An exact pre-cubing is required including the exact sequence of items to be put on the Dispatch Load Unit for AMCAP-type palletizing.

Exact Grouping

An exact association of items to Dispatch Load Units is required. The sequencing of items is, however, only required to be exact on the level of different Weight Classes. For items of the same Weight Class, the sequence is arbitrary. This type of pre-cubing is suitable for ErgoPal-type palletizing.
     
DLU Quantity Estimation

Dispatch Load Unit Quantity Estimation is configured by the:

  • Fulfillment Zones.
  • empty weight, max weight, volume, target volume rate and target weight rate of the Dispatch Load Unit Type.
  • preferred fulfillment zone, preferred fulfillment handling type, total volume, and total weight of the SKU Quantity Unit.
  • packing group of the SKU.

 

DLU Quantity estimation consists of 2 steps:

  1. Calculation of a packaging unit list for all Order Lines of the Order.
  2. Assigning these packaging units to Dispatch Load Units.

 

Packaging Unit Calculation:

  • For each Order Line, packaging units are determined using the Sales and Pick Quantity Units of the SKU. SKU Quantity Units with higher factor to base are preferred.
  • Each packaging unit has a quantity, volume, weight, fulfillment zone, fulfillment handling type and packing group. If fulfillment zone and fulfillment handling type are not configured by the SKU Quantity Unit, the defaults are read from the configuration topics "cor-sku:Fulfillment.defaultFulfillmentZone" and "cor-sku:Fulfillment.defaultFulfillmentHandlingType".

 

Assigning Packaging Units to Dispatch Load Units:

  • For each fulfillment zone, fulfillment handling type and packing group combine a list of Dispatch Load Units, calculated based on the packaging units of step 1.
  • While the total packaging units to be assigned to Dispatch Load Units do not fit into the biggest Dispatch Load Unit Type of the current fulfillment zone, the biggest Dispatch Load Unit Type fulfillment zone is selected. If the smallest Dispatch Load Unit Type that can load all remaining packaging units for the fulfillment zone, the fulfillment handling type and packing group combination is selected.
  • The packing sequence type of the fulfillment zone defines the order that packaging unit are assigned to Dispatch Load Units.
  • A new Dispatch Load Unit Type must be selected when the maximum volume or weight of the current Dispatch Load Unit Type was reached. Target rates for volume and weight of the Dispatch Load Unit Type are considered as well.
