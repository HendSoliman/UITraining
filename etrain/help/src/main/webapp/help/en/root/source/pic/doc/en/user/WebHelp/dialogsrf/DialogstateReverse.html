Reverse

Brief Description

The dialog state "Reverse" appears after this function was selected by operator from dialog state "Picking".
Reverse Picking is only allowed in case the Source Load Unit is not stacked anyway and of the same type as the Pick Container.
It provides the functionality to confirm the operation of reverse picking with exchange of Source Load Unit and Pick Container with separate confirmation of exchange.

      
"Reverse" State Screen

      
Fields

Field

Description

SKU

The SKU id uniquely identifies a Stock Keeping Unit within a Client.

Quantity Target

The target quantity to be picked into the Pick Container.

Quantity Unit

The quantity unit of the Pick operation. This will be the "Pick Quantity Unit" in case of exactly one Default Pick Quantity Unit is set for the quantity units of the SKU. Otherwise the Base Quantity Unit is displayed.

Quantity Current

The picked quantity into the Pick Container with same Quantity Unit.

Put Location

The Alias represents the name of the Put Location on the Pick Cart for easy identification. The Alias is only textual information and will not be validated from the system.

Source Base Load Unit ID

The unique identifier of the current Source Base Load Unit to be exchanged with Pick Container.

Quantity Reverse

The picked and counted quantity for subsequent restoring with same Quantity Unit.
     
Special Buttons

Function/Button

Description

Enter "* Quantity Reverse"

With launching this dialog state the field "Quantity Reverse" is focused for entering the picked and counted quantity for subsequent restoring of changed Load Unit.

Change to "Short"

key "F2"

Dialog proceeds in state "Short" where operator is able to confirm inventory shortage due to several configured shortage reasons.

Details in Dialog State "Short".

With confirmation of this inventory shortage finally an additional exchange of Source Load Unit and Pick Container is booked by the system and must be performed manually by the operator. The Pick Container remains empty and will be subsequently deleted.

 INFO 

Confirmation of inventory shortage is impossible with previous entered Quantity Reverse >0 

Change to "Info"

key "F11"

Dialog proceeds in state "Pick Info" where operator is able to view detailed information about current Pick operation.

Details in Dialog State "Pick Info".

Cancel

key "ESC"

Cancel this dialog state and return to former state without confirming any changes.

Confirm as "ok" with "reverse"

key "Enter":

The requested Pick operation is confirmed with finally additional exchange of Source Load Unit and Pick Container.

The confirmed quantity of Inventory Unit of displayed Source Load Unit is booked into the Pick Container and the Pick is booked and confirmed with Code 'ok'.

The exchange is performed manually by the operator.

In case of Quantity Reverse = 0 the Pick Container remains empty and will be subsequently deleted.
