Exception

Purpose

This state allows confirmation of Picks with different Confirmation Codes. The RapidPick screen allows three Confirmation Codes:

  1. Short: Confirm inventory shortage due to several configured shortage reasons.
  2. Full: Confirm that not all of required quantity could be picked into the current Pick Container.
  3. Over: Confirm more than requested quantity due to several configured reasons for advised overdelivery.

Prerequisites

For confirmation with "SHORT" Confirmation Code:

  • No quantity set to a value greater than the requested target quantity.
  • Multiple quantities can be reduced to 0, but only one quantity can be reduced to a value greater 0.
  • One Confirmation Reason assigned to Function Code 'Short'.

For confirmation with "FULL" Confirmation Code:

  • No quantity set to a value greater than the requested target quantity.
  • Only one quantity was changed at all.
  • One Confirmation Reason assigned to Confirmation Code 'Full'.

For confirmation with "OVER" Confirmation Code:

  • No quantity set to a value lower than the requested target quantity.
  • Only one quantity was changed at all.
  • One Confirmation Reason assigned to Confirmation Code 'Over'.
      
Functions / Buttons
Quantity adjustment buttons

Button

 Description
Plus Button Increment the current quantity. The steps can be selected using the toggle button between 0.1, 1 and 10.
Calculator The calculator image opens a small dialog, which allows to enter an arbitrary quantity for confirmation.
Minus Button Decrement the current quantity. The steps can be selected using the toggle button between 0.1, 1 and 10.

 

Functions

Function

Description

Pick Confirmation with Confirmation Code different from "OK"

Press available Confirmation Reason

After quantity adjustment, Picks are confirmed with the selected Confirmation Code.

Short: Requested pick operation is confirmed with inventory shortage due to displayed Confirmation Reason.

 INFO 

If short, all active and not yet active Picks from the same Inventory Unit are confirmed or respectively cancelled. Due to quantity changes, some Picks are booked with OK and the next Pick, where the requested target quantity can't be fulfilled will be booked with Short. All additional Picks are cancelled and depending on the configuration, a replacement reservation may be initiated.

Full: Requested pick operation is confirmed with finishing the current Pick Container due to the displayed Confirmation Reason and selected quantity. After confirmation, the Pick Container is pushed away and a new Pick Container is required.

Over: Requested pick operation is confirmed with advised overdelivery due to the displayed Confirmation Reason.

 INFO 

Only available Confirmation Reasons,  assigned to exactly one Confirmation Code, display.
