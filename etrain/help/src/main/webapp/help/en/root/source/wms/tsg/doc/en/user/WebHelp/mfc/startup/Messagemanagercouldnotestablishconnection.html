Message manager could not establish connection

Symptom

In case of material moving jobs e.g. inbound or outbound delivery, no load units are moved on automatic devices e.g. conveyors.

The inbound advices or the outbound orders stay in status released but do not proceed to active. They are not assigned to an MFC transport order.

In GUI dialog Administration, Classic Platform, System Log the recurring information

msgmgr_conv_<name> new connection state waiting for connection confirm

msgmgr_conv_<name> new connection state connection established

msgmgr_conv_<name> new connection state connection disconnected

appears

 

      
Brief description

The msgmgr process could not connect via TCP/IP to its assigned PLC /Control

 

      
Categories
  • MFC
  • PLC/Control

 

      
Possible errors

A TCP/IP port on the host could not be connected or the assigned PLC/Control is down.

 

      
Trouble shooting
