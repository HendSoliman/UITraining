Order Allocation

Purpose

Before an order can be planned for palletizing the inventory of the order is needed in the Multishuttle buffer. The MS buffer contains of different aisles, whereby each aisle has exact one palletizer. The order allocation process assigns the order (or a part of it) to an aisle, where the order will be planned and palletized. During the assignment the palletizing groups are considered as well (see: Palletizing Group Assignment Dialog).

For the assignment the aisle are selected which fulfills following points:

  • The complete not yet assigned inventory of the current order can be stored in the aisle.
  • Use the aisle will least packages stored in it.
  • For SKUs of the order, where palletizing group is defined, use only aisles with this palletizing group assignment.
      
Demons, Cronjobs and Events

The trigger for aisle assignment is only throw a demon. Because the assignment of aisle to order is mandatory for all other following processes like order planning and palletizing, this demon should be activated.

Demon / Cronjob

 Description Configuation

AssignOrdersToAislesDemon

This demon looks at all orders with following condition:

  • The order is not paused.
  • The order status is released.
  • The service qualifier is RapidPal (or RapidPal Purge)
  • And there is no aisle assignment yet.
 No specific configuration needed.

 

No events are produced or handled during order allocation.

      
Facts

No facts are written during order allocation.

      
Alarms and Exceptions

Following alarms can be raised:

Alarm

 ID Description

AisleAssignmentToOrderFailedAlarm

PIC_PAL_0006

 This alarm raises, when there are order for allocation available, but no aisle for it is found.

 

Following exceptions can be thrown:

Exception

 ID Thrown when ... Handling

NoAisleForOrderFoundException

 PIC-PAL-0005

Either no aisles with enough free capacity is found or in case of a specific palletizing group no assignment for this to an aisle exists.

 Wait until there is enough space in the Multishuttle buffer or in case of using palletizing group make sure that the used palletizing groups have at least assignment to one aisle (see Palletizing Group Assignment Dialog for more details).
