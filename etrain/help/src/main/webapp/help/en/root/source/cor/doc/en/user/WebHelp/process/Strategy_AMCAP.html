AMCAP Strategy

Purpose

It is an important characteristic of the AMCAP concept that the distribution of products across the aisles of the Multishuttle buffer is done during the storage process in order to be able to exploit the full throughput capacity of individual aisles during output due to the lack of inter-aisle sequencing requirements.

The current concept on the inbound side is bases on the combined usage of pushers for distribution across two aisles as well as inter-aisle transfer for final distribution.

In AMCAP installation Multishuttle Flex is used. In this chapter only the putaway strategies are described. For detailed on usage of slots in MS Flex see: SlotDetailDialog.

      
General conditions and requirements
  • In an AMCAP system, items are stored without trays
  • Items have a large variety of dimensions, ranging
  • from 150mm x 100mm x 50mm
  • to    650mm x 650mm x 450mm
  • The Multishuttle Flex in an AMCAP installation is not used for storage but as an intermediate buffer for sequencing
  • Residence time of items in the MS Flex shelf is usually short, typically less than a day
  • Typically, all locations in a MS Flex shelf for AMCAP have equal height to allow for the largest flexibility.
  • All locations are considered equal with respect to dimensions
  • No weight restrictions need to be considered for location search
  • No other sorting conditions need to be considered for location search
  • No interdependencies between different products or product groups need to be considered

Location search

During storing items in Multishuttle aisles, the order-related demand is considered. Therefore the orders are assigned to one or many aisles in advance. For details see: Order Allocation.

Following further restrictions are considered in order to find the destination location:

  1. In order to optimize the lift operation the two items of a pair are directed to the same level.
  2. If the destination aisle requires inter-aisle transfer (iAT), levels are preferred having free iAT slots.
  3. In a pair requiring iAT, the direction of iAT is always different in order not to block iAT slots.
    This is avoided by storing one item in the arrival aisle whereas the other is stored in a different aisle.
    But if there is only need on one side, then both items are send to that.
  4. Palletizing groups are restricted to specific aisles, this restriction is considered when selecting an aisle.

In order to fulfill the requirement (1) PLC waits for pairing and afterwards MFC requests for destination for a pair of packages.

For the requirement (2) the strategy "Inter-aisle Transfer" is configured. Thereby the level with the less iAT is used, which ensure to have iAT slots available.

The requirement (3) is handled as follow:

  • It consider the order demands which are assigned to the aisles. Further it consider also the palletizing groups assignment, e.g. requirement (4).
  • First of all the current aisle is determine, e.g. as soon as an item or a pair of items requests for the destination, this is in front of a specific aisle (on the so-called pick station).
  • If the current item(s) are needed in the current aisle, then use a location in the current aisle.
  • If the current item(s) are needed by nor of the aisles, then also use the current aisle.
  • Demand is in another aisle(s):
  • Request for only one item (no pair), then use the nearest aisle with demand.
  • Request for a pair of items, then send one in the right and one in the left direction. 

How are the distance and direction (right/left) determined?

The virtual locations of the area have sequence numbers. These are configured from 0 to #Aisles-1 (from right to left). As larger the different between current and target sequence number is (absolute), as long is the distance. Further the greater sequences than of the current aisle means left direction and with lower sequences means right direction.
