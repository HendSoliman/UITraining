Goods-To-Person Picking: General Exceptions

Symptom

An active Order with included Goods-To-Person Picking operations (!) could not get completed because of an assigned Pick Order could not be correctly executed or completed.

 INFO 

Picking processes could be very complex and are mostly project specific with respect to the details. Thus only the main and most applicable issues are described below.
     
Brief Description

The further execution of an active Order with included picking operations (!) is not possible although the previous operational steps are already finished. The possible trouble shooting steps depend on various data sets in the Order Fulfillment as well as the Pick Controller arena.

Within this topic the following 3 main conditions which are characterized through the state of the related Pick Job(s) of the Order are distinguished and separately described.

The related Pick Job of the executing Pick Controller could either be

  1. not yet assigned to any Pickstation
  2. currently running at a Pickstation
  3. yet finished but (at least) one Pick Container is missing at it's destination (e.g. the shipping area).

 

Most important dialogs for analysis and trouble shooting support:

  • Dialog Retrieval Administration (primarily)
  • Dialog Pickstation Analysis.
     
Problem Analysis

The problem analysis is done by the analysis of the specific conditions of the involved data sets of the subsystems Order Fulfillment (ORF) and Pick Controller (PIC).
The following data sets are relevant within the described trouble shootings:

  • Pick Order (ORF) (with related Pick Container)
  • Picks
  • Pick Missions (with related Source Load Units)
  • Workstations

 

Step

Action

1.

Start the Dialog Retrieval Administration.

Use the superior filter and enter the Order ID that belongs to the active Order which should be analyzed.

2.

Take a detailed look at the different panels to identify the specific situation with focus on the following data.

3.

First panel (for further data description refer to Orders/ Pick Orders):

  • Pick Order Status
  • Pick Order Configuration Status (only in case of involved serial picking with a preceding Pick Zone) (use the context menu navigation to Dialog Pick Order).

4.

Third panel (for further data description refer to Picks / Pick Missions / Pick Walks):

  • Pick Status
  • Pick Mission Status
  • Mission Blocked
  • Job Blocked.

5.

Second panel (for further data description refer to Full Load Unit Retrievals / Pick Container / Dispatch Load Units)

Only in case of all Pick Orders are in status "finished":

  •  Pick Container Status.

6.

In case of a missing Source Load Unit at the Pickstation start the Dialog Pickstation Analysis.

Filter the appropriate Pickstation and analyze the Load Unit match in the lower grid.

Detailed Description

In case of the further execution of an active Order with included picking operations is not possible various data sets in the Order Fulfillment as well as the Pick Controller arena characterize the different reasons and required trouble shooting steps.

After first executing the described Problem Analysis the relevant conditions of the involved data sets are well known. Below the different conditions are described in general including only the main details.

  1. Pick Job is not yet assigned to any Pickstation

With the Order release the Inventory reservation respectively Load Unit allocation takes place. For the pick portion Pick Orders are created and directly released in case of the Release Mode of the related Task is configured as "automatically".
Otherwise the accordant and required preceding event or schedule plan needs to be analyzed separately. (Reason 1)

In case of involved serial picking with a preceding Pick Zone the release mode of the assigned Task is "triggered" and the pick operations of the preceding Pick Zone have not yet finished. In the related Dispo Zone of the preceding Pick Zone only aggregated reservations with not yet created Pick Orders exist. (This is only the case if the related Dispo Zone has a "two phase" reservation mode (e.g. for Multishuttle Retrieval Optimizations) otherwise the serial picking Tasks have the release mode "dependent (complete)" and will be automatically released when the picking Tasks of the preceding Pick Zone are confirmed.)
The preceding serial picking step will proceed by execution of the reservation of available Inventory Units and is done by a special running demon. (Reason 2)

In case of the allocated Source Load Unit of a Pick is not residing on a location within a configured Pick Controller or Picking Zone while the Pick is created the Pick remains in status "created". Not until the Load Unit arrives at a bin location within an appropriate Zone the Pick proceed it's life cycle. (Reason 3)

In case of the Pick Missions are assigned to a Pick Job only on demand of a Pickstation as late as possible ("late binding") a Pickstation needs to provide free capacity for further workload (free Pick Job Queue entry or free Put Location in case of configured "Continuous change of Pick Container"). The Pick Mission must match the allowed type of workload of the Pickstation (Service and Processing Qualifier). (Reason 4)

A Pick Mission (and thus the related Pick Job as well) is "blocked" in case of at least one required Source Load Unit is currently not stored. In case the Pickstations are configured to consider this blocking flag all Source Load Units of the analyzed and blocked Pick Mission need to get stored before this Pick Mission could proceed. (Reason 5)

 

  1. Pick Job is currently running at a Pickstation

Due to a required Load Unit sequence a specific Source Load Unit of a currently running Pick Job is expected at the Source Location of the Pickstation but not yet arrived and a different Load Unit was scanned and currently active in the Picking Dialog. Various exceptional cases of the transport execution are possible reasons for this. (Reason 1)

With the confirmation of each single pick operation in the Picking Dialog the related Pick is confirmed to the Order Fulfillment Subsystem. With the last Pick the Pick Mission proceed to status "complete". With confirmation of pushing away a completed Pick Container the Pick Mission proceed to status "finished". The Pick Job proceed accordingly with respect to all of it's Pick Missions. In case of any exception occur with one of this operations the Picker is informed directly in the Picking Dialog with an appropriate error message and is not able to continue his work. All known reasons for this exceptional situations based on an incorrect manual booking in the system. (Reason 2)

 

  1. Pick Job is yet finished but (at least) one Pick Container is missing at it's destination (e.g. the shipping area)

A Pick Order is finished at a Pickstation with confirmation of the last Pick of the last Pick Order Line. The Pick Container remains in status "picked" until it has arrived at the Destination Location of the related Pick Order Line. (Reason 1)

      
Trouble Shooting
  1. Pick Job is not yet assigned to any Pickstation

Solution for Reason 1: Pick Order is not yet released (no Picks or Pick Jobs exist)

Step

Action

1.

For the analysis of the failed precondition of the Pick Order release select the pending Pick Order in the Dialog Retrieval Administration and start the Dialog Pick Order (context menu).

In the Dialog Pick Order start the Dialog Task Administration (context menu).

2.

The executive Task dependencies are listed in the appropriate specific panel.

Perform a separate problem analysis for the reference of the previous dependant task.

In case there is no problem identifiable for the previous task no further trouble shooting is necessary and just the referenced Pick Order or respectively Full Load Unit Retrieval need to be executed.

3.

In case of the previous task represents a Pick Order and the problem of this Pick Order couldn't be solved the Pick Order or respectively the appropriate Pick Order Line(s) could be cancelled.
Switch to the Dialog Retrieval Administration and use the related button for the mentioned Pick Order or Pick Order Line respectively.

4.

In case of the previous task represents a Full Load Unit Retrieval and the related problem couldn't be solved the Full Load Unit Retrieval could be cancelled. Switch to the Dialog Retrieval Administration and use the related button for the mentioned Full Load Unit Retrieval. 

 

Solution for Reason 2: Pick Order is in configuration status "incomplete" (no Picks or Pick Jobs exist)

Step

Action

1.

The allocation of the Inventory Units (consuming the aggregated reservations of the Order) is done in the second phase of a "two phase reservation". A special "area related transport workload" demon controls when the workload of the related area falls below a configured limit and initiates further outbound transports by replacing the aggregated reservations to reservations of Inventory Units. 

Start the Cron Jobs Screen and ensure that the "FillAreaTransportWorkloadDemon" for the assigned area is up and running.

Relevant for the decision of assignment of new workload is the Control Data Parameter "OrderFulfillment.FillAreaTransportWorkloadDemonIdleThreshold".

2.

In case of the first step of the serial picking was clarified (and cancelled) anyway and the existing "incomplete" Pick Order still exists it must be deleted with use of the button "Remove (delete) the selected Pick Orders".

 

Solution for Reason 3: Picks exist in status "created" (no Pick Jobs exist)

Step

Action

1.

In the Dialog Retrieval Administration select the Pick in status "created" and start the Dialog Load Unit Administration (context menu).

2.

Clarify the Source Load Unit and initiate the inbound transport into the appropriate storage area.

Further trouble shooting hints for that are provided in the related trouble shooting guide for the Transport Execution Exceptions.

3.

In case of the Source Load Unit could not get clarified (e.g. because it was lost) the related Pick Order Line must be cancelled.
Switch to the Dialog Retrieval Administration and use the context menu (first entry "Show Pick Order Lines") of the yet selected Pick to switch to the Pick Order panel and automatically select the related Pick Order Lines.
Use the button "Cancel the selected Pick Order Lines".

 

Solution for Reason 4: Pick Missions exist in status "released" (no Pick Jobs exist)

Step

Action

1.

In the Dialog Retrieval Administration select the Pick Walk in status "released" and start the Pick Walks (context menu).

In case of "late binding" select the Pick Mission in status "released" and start the Dialog Pick Missions (context menu).

2.

Start the Workstations screen and ensure that at least one Pickstation in the mentioned Picking Zone exists with the following conditions:

  • Service Qualifier matches the condition of the Pick Job/Pick Mission (refer to Workstation Capabilities)
  • Processing Qualifier matches the condition of the Pick Job/Pick Mission
  • at least one Pick Job Queue entry in status "free" (refer to Dialog Workstation Queue Entries)
  • in case of the Pickstation is configured for "Continuous change of Pick Container" (refer to Dialog Workstation Locations) the number of currently "active" Pick Missions must be less than the number of "available" Locations of type "work" (i.e. at least one Pick Location is not occupied by a Pick Container).

3.

 The Load Unit Saturation Limit of the Pickstation must not be exceeded. That means the number of currently ordered Source Load Units must not exceed the "LU Saturation" of the Pickstation.
Start the Dialog Pickstation Analysis, execute a filter in the lower grid "Load Unit match" for the column "Status (Transport)" with the values "released" and "active" and sum up the loaded rows.

 

Solution for Reason 5: Blocking situation of the Pick Job respectively the related Pick Missions

Step

Action

1.

In the Dialog Retrieval Administration select the Pick Mission with enabled flag "Mission blocked" and start the Dialog Pick Missions (context menu).

All Source Load Units are listed in the lower Dialog Load Units and the currently not stored Load Units are listed at the top of the list. 

2.

Only in case of one of the currently not stored Load Units is in any exceptional situation (e.g. located at a Clarification Station) clarify it and initiate the inbound transport into the appropriate storage area.

Further trouble shooting hints for that are provided in the related trouble shooting guide for the Transport Execution Exceptions.

3.

In case of all assigned Source Load Units are stored and the "Mission blocked" flag is still enabled the button "Remove the block flag" could be executed for the selected Pick Mission.

 INFO 

Be aware of the possible extensive implications in case of a "Mission blocked" flag will be removed by mistake.

 

  1. Pick Job is currently running at a Pickstation

Solution for Reason 1: Missing Source Load Unit at the Pickstation

Step

Action

1.

Clarify the missing Source Load Unit.

Further trouble shooting hints for that are provided in the related trouble shooting guide for the Transport Execution Exceptions.  

2.

In case of the Source Load Unit is finally missing within the required sequence the Source Load Unit could be cancelled and skipped within this Pick Job.

This must be done by the Picker in the Picking Dialog with the appropriate button "Cancel Reservation" (in case the Picker has the appropriate access rights).

3.

In case of the picking operation could not executed anyway for a few Picks the related Pick Order Lines could be cancelled
with use of the button "Cancel the selected Pick Order Lines" in the Dialog Retrieval Administration.

 

Solution for Reason 2: Pick Mission (Pick Job) is executed completely but remains in status "active" or "complete"

Step

Action

1.

For any reason the Picker is informed directly in the Picking Dialog and is not able to continue his work.
The necessary trouble shooting steps of this situation strongly depends on the details of the error message in the dialog.

Possible steps are listed below and provided in the Dialog Retrieval Administration.

2.

Before any trouble shooting can be executed the specific reason mentioned in the error message must be analyzed and clarified. 

 INFO 

Because of this specific exceptional situations normally based on any (incorrect) manual booking in the system overall hints for analysis and trouble shooting could not provided here.

3.

In case of all Picks are normally executed but some were not booked
the button "Book Pick(s) with code OK" could be executed.

4.

In case of the Pick Container was pushed away successfully but was not booked
the button "Confirm Pick Container" could be executed.

5.

In case of the picking operation could not executed anyway the related Pick Order Lines could be cancelled
with use of the button "Cancel the selected Pick Order Lines".

6.

In case of the Pick Container arrived but not booked at the appropriate destination
the button "Confirm the transport of selected Load Units at the defined destination" could be executed.

7.

In case of various problems were solved anyway and the Pick Job was finished but not booked
the button "Remove the selected Pick Job from the Pick Job Queue" could be executed.

 

  1. Pick Job is yet finished but (at least) one Pick Container is missing at it's destination (e.g. the shipping area)

Solution for Reason 1: Pick Order is finished and (at least one) Pick Container is in status "picked"

Step

Action

1.

Clarify the Pick Container and continue the transport to it's destination.

Further trouble shooting hints for that are provided in the related trouble shooting guide for the Transport Execution Exceptions.  

2.

In case of the Pick Container has already arrived at the Destination Location of the related Pick Order Lines
the button "Confirm the transport of selected Load Units at the defined destination" in the Dialog Retrieval Administration could be executed.
