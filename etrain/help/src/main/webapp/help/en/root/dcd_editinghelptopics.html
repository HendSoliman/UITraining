Editing Help Topics

Purpose

This seems to be the first time you have used the <Edit> button to edit a help topic.

Here you find information how to prepare the infrastructure necessary for help authoring.

Prerequisites
  • An HTML editor must be installed on this computer.
    Recommended HTML editors are
  • Microsoft FrontPage
  • Microsoft SharePoint Designer
     
  • DocHTML shall be installed if extensive editing is intended.
    Minor corrections can also be done without this tool.
    DocHTML facilitates document editing conforming to Dematic iQ standards.
    DocHTML can be installed from here.

  • HTMLEdit must be installed.
    This little tool allows opening an HTML editor out of a browser window.
    HTMLEdit can be installed from here.

 

Infrastructure is installed. Do not show this hint again.

 

Background

Calling a program such as an HTML editor out of the browser is normally not possible because of security reasons.
Therefore, the installation of an additional tool on your computer is required to allow direct editing.

Dematic iQ Web Help uses the following procedure:

The edit button is linked with a URL htmledit:<document URL>.
The protocol htmledit is a protocol similar to other protocols such as http but specific to Dematic iQ Web Help.
It is not known to be used by any other program.

In order to launch an HTML editor on the local machine, it is necessary to register a protocol handler for the htmledit protocol, which specifies the program to be started.
Such protocol handlers may be defined manually through the Windows Explorer (Extras -> Options -> File Types) with file type URL: <protocol> without file extension.
Since the Web Browser transfers the whole URL (including "htmledit:") to the program specified in the protocol handler, the editor can not be launched directly with this URL but the URL of the document must be extracted first. This is done by a small command script using string functions of Windows Scripting Host.

The whole infrastructure can be set up automatically using the HTMLEdit installer.

This solution does not impose any security conflicts since the user himself determines how the specific protocol is handled.
There is no usage of ActiveX.
