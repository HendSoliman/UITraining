Location - Detail

Purpose

Only locations which are not of usage type "bin" could be managed with this dialog. If the location has the usage type "bin", only the viewing mode is available.

The creation of one Bin (Location of usage type Bin) is possible anyway.

Properties

Field

Description

Dispo Zone

Disposition Zone which the Location belongs to. (see Detail Dialog Dispo Zone).

Usage Type

Usage Type of Location in terms of storage or transport concerns:

  • facility (virtual coordinate representing a Facility)
  • area (virtual coordinate representing an Area)
  • group (virtual coordinate representing a Group (Aisle))
  • subgroup(virtual coordinate representing a Subgroup (Level))
  • bin (Storage Location for storage of Load Units)
  • transport point (Coordinate within the transportation system)
  • fork (Coordinate represents the fork of AS/RS or Fork Lift Trucks)
  • lift (Coordinate represents a place of lifts).

Operation Type

Operation Type of Location in terms of business related operations:

  • unpacking (Location for unpacking or repacking operations at a stationary Workstation)
  • qa decision point (Location where a specific quality assurance decision for further handling of a Load Unit is made)
  • unspecified (Location where no operation type is used).

Function Group

Assign dynamic runtime data to location:

  • Sort Lane
  • Sort lane left
  • Sort lane timeout
  • Inbound destination not reachable
  • Check cycle
  • Request Transport
  • Request Destination
  • Start next group
  • Request Aisle
  • Request Bin
  • Info LU
  • Start NextGroup
  • Request Destination or Transport
  • Request Destination and check cycle
  • Pusher
  • Drop Station LC11
  • AMCAP Conveyor Pusher
  • Rapid PAL Source
  • MFC FLS Occupation
  • Request Level
  • Video Recorder

Location Type

Physical attributes of the Location. (see Detail Dialog Location Type)

Height Classification

For easier processing, Location heights are divided in several project specific height classes.

 INFO 

Is not allowed to differ from the assigned Level.

Sequence Number

Order of Locations independent of the coordinates. May be used e.g. for Picking where the Locations are executed with respect to this sequence.

Check Digit

Allows verification of Locations, e.g. while in manual scan operation.

States

Field

Description

Occupancy

Occupied

If a transport point is occupied.

 INFO 

Only used for transport points. Not relevant for bins.

Availability Status

Extent to which the Location is available for the storage of additional Load Units. Accounts for reservations for Putaway and occupied positions.

  • free (not occupied and fully free)
  • partial (partially occupied by Load Unit(s))
  • full (fully occupied by Load Unit(s))

Reservation Status

Extent to which the Location is available for the storage of additional Load Units. Accounts for reservations for Putaway and occupied positions. Values are identical to Availability Status.

Availability Layout Status

The combination of this status with the Layout assigned via configuration "Location Type Load Unit Type" gives information about the current occupation and current blocking situation on the Location. Reservations on the Location are not considered in this status. (see also Detail Dialog Layout)

Reservation Layout Status

Combination of this status with the Layout assigned via configuration "Location Type Load Unit Type" gives information about the current occupation and current blocking situation on the Location. Reservations on the Location are not considered. (see also Detail Dialog Layout)

Error

Error Code

Last error that occurred at this Location:

  • OK (no error occurred)
  • place occupied (location (bin) is physically full, but logically free within the system)
  • place free (location (bin) is physically empty, but logically occupied within the system)
  • multiple codes set by the underlying transport system (MFC) if any error occurs during transport operation

Error Reason

Reason for the error.

Lock

Lock Status

If the Location was locked for specific operations. Meant as a long term lock set by an user.

  • available (all planning operations are allowed)
  • locked for input (inbound planning operations not assigned to the Location automatically)
  • locked for output (outbound planning operations not assigned to the Location automatically)
  • locked for input and output (inbound and outbound planning operations not assigned to the Location automatically)

Lock Status Reason

Reason for the lock.

Lock Status Change Date

Time of the last change of the Lock Status.

Lock Status Change User

User who made the last change of the Lock Status.

Technical Lock Status

If the Location was locked for specific operations for a short term by a underlying transport subsystem (project specific use only):

  • available (all transport operations will execute)
  • locked for input (inbound operations not assigned to the Location automatically)
  • locked for output (outbound operations not assigned to the Location automatically)
  • locked for input and output (inbound and outbound operations not assigned to the Location automatically)

Technical Lock Status Reason

Reason for the lock. Usually delivered from the underlying Transport System (MFC).

Technical Lock Status Change Date

Time of the last change of the Technical Lock Status.

Temporarily Blocked for Putaway

If the Location is temporarily blocked for Putaway.

Configuration Status

If Location configuration is complete and the Location can therefore be used by the system.

Edit all status values. Only allowed for users with specific access rights.

Handling

Field

Description

Type

Velocity Classification

What goods a Location can be used for (e.g. for fast-moving vs. slow-moving products).

Dedicated Pick Face

If the Location is a Dedicated Pick Face. (see also SKU Administration: Dedicated Pick Face Tab)

Dedicated Pick Face SKU

The Stock Keeping Unit that is assigned to this Dedicated Pick Face.

Hot

If the location belongs to the Hot Zone of an Area.

 INFO 

Make sure to define Hot Zone configuration for same area and Velocity Class.

Capacity

Available Capacity

Maximum number of Load Units which can still be stored at this Location. Maintained on each storage or retrieval operation on this Location.

Reservation Capacity

Maximum number of Load Units which can still be stored at this Location. Maintained on each disposition operation on this Location.

Transport Capacity

The transport capacity specifies the maximal number of load units which can be transported to the location. A value of 0 means that there a no capacity constraints.

 INFO 

Only used for transport points.

Last Stocktaking Date

Date when the last Stocktaking action took place at this Location.

Weight

Weight Limit

Maximum overall weight limit of all Load Units stored on this Location. Measuring unit defined on a project specific basis.

Weight Limiter

Weight limiter which this location belongs to. (see also Dialog Weight Limiter)

Reserved Weight

Overall weight of the Load Units that are stored or planned to be stored at this Location. Measuring unit defined on a project-specific basis.

Multishuttle

Bay Width

The width of a bay is configured in the Area and thus valid for all Locations of the Area but could be adapted individually here.

Right Split

If a slot should be split on the left or the right side. Supposed to be alternated on each split or merge operation of Slots in order to assure balanced load of shelf bays.

 INFO 

Only in case of dynamic slotting type of location management.

Transport Configuration

Default Destination

The transport point where a Load Unit is directed to in exceptional cases where a fix default destination is required.

 INFO 

If no value is configured for a Location the assigned default destination of the related Subgroup, Group, Area and last of the Facility is used instead.

Coordinate

The coordinate part of the Location ID is redundant and used for an efficient lookup by the MFC.

Save Sensor Data

Flag indicating whether sensor data being reported by the PLC should be updated in the Load Unit.

 INFO 

Normally, the sensor data are mirrored between PLC and MFC and thus should not be saved by default.

Save Ufo

If an UFO showing up at a Location needs to be saved in the database or not.
