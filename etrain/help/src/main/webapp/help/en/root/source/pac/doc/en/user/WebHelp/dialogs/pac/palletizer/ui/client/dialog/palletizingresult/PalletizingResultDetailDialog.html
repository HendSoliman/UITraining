Description

Pick Order Line

The primary key of the pick order line to which the current palletizing result is assigned to.

Palletizing

Pack Sequence Number

The sequence number calculated by the packbuilder. This is a strict sequence.

Pack Sequence Group

The sequence group calculated by the packbuilder. Within each group the sequence is changeable, e.g. for the palletizer it doesn't matter which one of a group arrives first.

Quantity Unit

The quantity unit with detailed packaging information. This quantity unit was used by packbuilder for planning.

Type

The type shows how this result is to use. There are currently two result types available:

  • Planning (Result of pallet planning from an order)
  • NumberOfPallet (Estimate the number of pallets; not in use)

Depended on the type different attributes of the result are filled.

Slipsheet

This flag specify, whether this object represents a slipsheet or not. Packbuilder sends item object for real items and for slipsheet. For differentiate this flag is used.

Case Alignment

This field is empty for slipsheets, because they are not pushed on the pallet. For all other cases the mapping of packbuilder value is as follow:

  • Positive (Packbuilder returns 5)
  • Negative (Packbuilder return 7)
  • NoAlign (For all other cases)

When the pallet is building from right to left, then packbuilder use 5 o'clock configuration. From left to right it use 7 o'clock.   

Positive Alignment
Negative Alignment

Case Orientation

Orientation of cases on the pallet. This value is used to orientate the case in the correct direction. For the orientation it is always common, that the x is length and this is always in the conveying direction.

But the direction of the palletizer cell is in another direction. Thus the orientations received from packbuilder have to be adjusted that the case can be turned correctly on the conveyer.

Following values / mapping for this field are possible:

  • LENGHT_WIDTH (Packbuilder=1; to conveyor = WIDTH_LENGTH)
  • WIDTH_LENGTH (Packbuilder=2; to conveyor = LENGTH_WIDTH)
  • LENGTH_HEIGHT (Packbuilder=3; to conveyor = HEIGHT_LENGTH)
  • HEIGHT_LENGTH (Packbuilder=4; to conveyor = LENGTH_HEIGHT)
  • WIDHT_HEIGHT (Packbuilder=5; to conveyor = HEIGHT_WIDTH)
  • HEIGHT_WIDTH (Packbuilder=6; to conveyor = WIDTH_HEIGHT)

Palletized Manually

If is set, then the case was palletized manually.

First Layer

This is set, when the case is direct on the pallet.

Slipsheet Position

If the current position is a slipsheet, then the packbuilder returns also a position where to put this slipsheet on the pallet.

  • Near (e.g. near to the x=0)
  • Rear (e.g. near to x=max)

Packbuilder creates positions for near and/or rear. If there are two slipsheets (on both positions) this will be combined before sending to PLC.

With this condition the position which are send to PLC have two further values:

  • - (no slipsheet)
  • B (on both positions)
  • N (Near)
  • R (Rear) 

Stretch Program

The stretch program is proposed by packbuilder and this is forward directly to PLC.

 INFO 

Currently no specific stretch program are used.

On Slider Plate

This is true, when packbuilder is configured to use slider plate and the current case is on slider plate.
