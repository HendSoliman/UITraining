Replenishment Planning Workflow

Description

Recurring checks for Replenishment are based a demon for Threshold-based Replenishments. The demon configurations is described here.

The started demon uses the configured Replenishment Thresholds. Calculation starts if appropriate entries (active and no replenishment running) exist.

In the calculation phase, threshold values are checked.

Threshold-based Replenishment are only created if the total Aggregated Inventory in the Disposition Zone or the Inventory at the Dedicated Pick Face fell below this threshold quantity.

Initiating a Replenishment occurs by creating Replenishment Orders containing the configured parameters from the Replenishment Threshold Configuration.

      
Triggers and Actors

This is started by the Control Center Supervisor when Threshold based Replenishment must be executed periodically.

      
Prerequisites
  • Replenishment Supervised Zones are configured
  • Replenishment Thresholds for the desired SKUs are set
  • Area Access entries with flag replenishment set are configured

 INFO 

The demon "ReplenishmentThresholdAdministrationDemon" creates automatically replenishment thresholds for SKUs that have dedicated pick faces in the "Replenishment Supervised Zone".
     
Required Resources
  • Replenishment Supervised Zones
  • Replenishment Thresholds
  • Area Access
  • Cron Jobs.
      
Work Flow for Demons and Batches

Step

Description

1.

Open Cron Jobs screen to activate Replenishment demon functionality.

2.

Select a Demon by filtering the "Demon Class Name" for "CheckReplenishmentThresholdDemon".

3.

Start the desired demon.
     
Result

Replenishment Orders are created for all SKUs where available quantities fell below the configured thresholds.
