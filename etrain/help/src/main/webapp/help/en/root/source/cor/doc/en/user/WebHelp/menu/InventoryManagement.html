Inventory Management

Overview

Warehouse Inventory, identified by Client, Stock Keeping Unit (SKU) and possibly extended batch, is distributed on the Load Units (LU's) in the system. For distinguishable stock on a Load Unit, an Inventory Unit (per Client, SKU, batch, etc.) is associated to the Load Unit.

Summarized information for the Inventory quantities of a certain Client and SKU (and sometimes batch) is provided by the screens Inventory Query and Aggregated Inventory.

Individual Inventory Units display in the Inventory Unit Administration screen. Here, it is also possible to administrate Inventory Holds as well.

 INFO 

Stock Balancing defines the specific request for providing a current inventory list for inventory reconciliation issues for an external Host system.
Any preparation of an Inventory Reconciliation List is done with use of the appropriate and project specifically adapted database view.
Creation of this list and preparation for any Host is done with use of the appropriate Cron Job "ReconciliationBatch".
