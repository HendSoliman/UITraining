Detail Dialog Dispo Zone

Purpose

This screen opens when creating, viewing, or modifying Disposition Zones.

 INFO 

After creating or deleting a Disposition zone, the Materialized View that relates Areas and Disposition zones must be refreshed.

 INFO 

For configuration purposes refer to the related topic in the Warehouse Administration Guide.
     
Screen

      
Fields

Field

Description

ID

Unique ID of the Dispo Zone for identification.

Description

Description of the Dispo Zone. For display purposes.

Area

If a Load Unit can be moved from this Disposition Zone to another Location using the Area Access mechanism. Area may also be virtual. (refer to Area Access)

Reservation Type

How Inventory is reserved on Inventory selection (disposition):

  • per inventory unit (selected Inventory Unit is reserved directly and immediately)
  • aggregate (selected Inventory is reserved on the level of Aggregated Inventory. The reservation on Aggregated Inventory is consumed directly.)
  • two-phase (selected Inventory is reserved on the level of Aggregated Inventory. Before it is consumed, the reservation is concretized to a Inventory Unit Reservation for later use).

 INFO 

The concrete reservation is done by a special demon (FillAreaTransportWorkloadDemon). If the configuration topic "orp-orderfulfillment:OrderFulfillmentGeneral.avoidParallelAccessInSecondPhase" is set, Orders that causes a concurrent access exception are skipped temporarily.

Full Load Unit Retrieval

If full load unit retrieval can be disposed in this Disposition Zone. Otherwise, Inventory is always picked.

Partial Excluded

If partial Load Units are to be excluded and local units must not be selected as Full Load Unit Retrievals.

Picking

If Picking is possible in this Disposition Zone.
