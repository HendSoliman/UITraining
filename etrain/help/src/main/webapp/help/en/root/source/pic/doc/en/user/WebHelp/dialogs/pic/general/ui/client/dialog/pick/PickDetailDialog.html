Overview

Status

Current execution state of the Pick:

  • created (Pick is currently not (yet) residing within a Picking Zone and thus not ready for release)
  • pending (Pick waiting for grouping to a Pick Mission)
  • released (Pick grouped within a Pick Mission)
  • active (Pick Mission execution started and all Picks of the Pick Mission proceed to this status)
  • complete (Pick executed by the Picker, but not yet confirmed to the Order Fulfillment System)
  • confirmed (Pick successfully confirmed to the Order Fulfillment System)
  • cancelled (Pick cancelled without the defined execution)
  • finished (Pick Mission finished and all Picks of the Pick Mission proceed to this status).

Urgency

 Set on behalf of the Pick Order. Defines the processing sequence. Lower value indicates a previous execution.

Sequence

Processing order of the Picks within the Pick Job.

Service

 Set on behalf the Pick Order to request a specific processing quality. May be used in a project specific way. Only Pickstations with the configured Service can execute these Picks.
  • default (No specific handling deviation is required)
  • rush-order (Rush Orders can be clustered in a specific way for rapid processing)
  • repacking (Repacking Orders are picked in a specific way)
  • block (Picks of different Orders are picked into the same Pick Container. Picks of multiple Pick Orders are clustered to a Pick Mission. This Service overrides the original one which could be launched through the Pick Order anyway.
    Be aware of a special demon (HandleDeferedPicksDemon) which groups these Picks to a Pick Mission separately)
  • RAPIDPAL (Orders for automated robot palletizing on specific AMCAP Station)
  • ERGOPAL (Orders for palletizing on specific ErgoPal Station)
  • AMCAP Depalletizing (Orders for automated robot depalletizing at specific Depalletizing Stations)
  • Manual Depalletizing (Orders for automated manual depalletizing at specific Manual Depalletizing Stations).

Pick Order Line

Confirmation
Code Specific processing of the Pick. Defines the main actions within the Order Fulfillment System after confirmation of the Pick, as configured in Confirmation Reason for the Confirmation Code.
  • Not Picked (Initial value: Pick was not yet executed)
  • OK (target quantity and required services were fully executed)
  • Short (target quantity was not fully executed due to inventory shortage)
  • Full (Pick was split because target quantity does not fit into the Pick Container)
  • Blocked (inventory of this Pick is currently blocked and could not be reached; the Pick was not processed further)
  • Skip (Pick is explicitly skipped temporarily and injected to the end of the current Pick Walk for later processing and confirmation)
  • Over (Explicit Picking of more than the target quantity, while observing overdelivery tolerance limits)
  • Under (Explicit Picking of less than the target quantity, while observing underdelivery tolerance limits)
  • Partial (Pick was successfully picked with a certain portion of the target quantity only. The remaining quantity will be executed and booked accordingly further).
Reason

The Reason of the Pick associated with the Code. Possible Reasons are configured in Confirmation Reason and define the main actions within the Order Fulfillment System.
