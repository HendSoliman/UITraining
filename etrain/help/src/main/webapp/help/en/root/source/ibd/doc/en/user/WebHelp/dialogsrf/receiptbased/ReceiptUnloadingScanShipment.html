Scan Shipment

Purpose

This step appears after a Dock has been scanned.
The operator is asked to scan the ID of the Shipment that he will unload.

      
Screen

      
Prerequisites
  • A Dock has been selected.
      
Fields

Field

Description

Shipment

The Id of the Shipment to be scanned.

 INFO 

If there is exactly one Shipment already assigned to the current Dock, the Id of this Shipment is proposed.

Status

The status of the scanned or proposed Shipment describes the progress of shipment processing:

  • created (The Shipment has been registered but not yet arrived at the warehouse. As long as a Shipment has not yet arrived at the warehouse it is not tracked by the system and can still be deleted)
  • pending (The Shipment has arrived at the warehouse and was registered)
  • released (The Shipment is authorized (released) for unloading)
  • at dock (The Shipment is confirmed at a Dock)
  • active (The first inventory has been received based on this Shipment)
  • complete (Receiving has been completed for this Shipment. A Shipment is complete, when all its Deliveries are complete)
  • finished (Receiving has been completed for this Shipment and a host confirmation has been sent in case of required. A Shipment is finished, when all its Deliveries are finished).

Carrier

Carrier that has delivered the Shipment. 

Truck

License plate of the truck of the Shipment.

Trailer

Identification of the trailer (e.g. the license plate of the trailer).

Seal

Seal number of the container used for shipment.

Dock (Shipment)

Assigned Dock where the scanned or proposed Shipment shall be unloaded.

Dock

The previously scanned and selected Dock the operator is currently working at.
     
Special Buttons

Button

Description

F2: This Dock

Shifts the Shipment to the current Dock.

Enter: OK

Shipment is assigned to the Dock and unloading process could be started.

ESC: Cancel

Cancel Unloading and switch to the main menu.
