Repacking

Purpose

Repacking screen supports spontaneous unpacking of Inventory Units onto other Load Units or Inserts. After Scan of Source Load Unit relevant data of all Inventory Units on it will be displayed respectively selectable. With scan of Destination Load Unit the destination of subsequent booking is defined. After selecting the desired Inventory Unit, entering the quantity and executing the "Unpack" function, the entered quantity of the selected Inventory Unit is booked onto the Destination Load Unit or selected Insert. Based on the configuration, the Load Units automatically receives an inbound transport.

      
Prerequisites
  • Screen must be executed at a configured Workstation
  • Source and Destination Load Unit must be available at the Workstation.
      
Procedure

Step

Description

1.

Start the Repacking screen.

Select Source

, if former and no more required Source Load Unit displays.

 INFO 

The Load Unit gets an inbound transport automatically if required depending on configuration topic "itl-unpacking:UnpackingWorkstation.automatedPutaway".

3.

Scan a Source Load Unit. (Confirmation occurs implicitly during scan or using "Enter" key.)

All Inventory Units located on scanned Load Unit load and the first display directly. All Inventory Units on the Source Load Unit also list, when Load Units are geometrically stacked.

4.

Select an Inventory Unit from the "Inventory Units" field.

4a.

If further stacked Load Units or Inserts are located on the scanned Source Load Unit, the Load Unit or Insert must be selected first (scanned Load Unit is pre-selected).

4b.

Navigate to the "Load Unit View" of Source (tab key / mouse click) and select the Load Unit or Insert (arrow keys / mouse click).

4c.

Selectable Inventory Units are adjusted to the stacked Load Unit or Insert. Select an Inventory Unit from the "Inventory Units" field.

Select Destination

5.

Press "F5" or click
, if former and no more required Destination Load Unit displays.

 INFO 

The Load Unit gets an inbound transport automatically if required depending on configuration topic "itl-unpacking:UnpackingWorkstation.automatedPutaway".

6.

Scan a Destination Load Unit (Confirmation occurs implicitly during scan or using "Enter" key.)

6a1.

If further stacked Load Units or Inserts are located on the  scanned Source Load Unit, the Load Unit or Insert must be selected first (scanned Load Unit is pre-selected).

6a2.

Navigate to the "Load Unit View" of Destination (tab key / mouse click) and select the desired Load Unit or Insert (arrow keys / mouse click).

6b1.

The scanned Destination Load Unit does not exist yet. A new Destination Load Unit should be created.

6b2.

Select a Load Unit Type for the new Destination Load Unit in the "Load Unit Type" field.

 INFO 

Unpacking of a Dispatch Load Unit requests the creation of a new Dispatch Load Unit. The type will be defined by the type of the new Load Unit. In case of multiple DLU Types exists for the used Load Unit Type the system applies any without providing a specific selection logic. This is done for example if a Pick Container that is represented as a DLU as well is manually repacked in a new Load Unit.

6b3.

Press "F3" or click
.

If the Target LU field is still empty, the system generates a new Load Unit ID and displays it.

 INFO 

This Load Unit is created on a Workstation's Location immediately by executing this step.

6b4.

If a Load Unit Label is required, press "F4" or click
.

Booking

7.

Enter the desired quantity for Repacking in terms of displayed Quantity Unit .

7a.

If an alternative Quantity Unit is desired, change the Quantity Unit in the "Unit" field and enter the adapted quantity.

8.

Press "F6" or click
.

The entered quantity for Repacking is booked from the selected Inventory Unit of Source Load Unit onto the Destination Load Unit or Insert.

 INFO 

If the corresponding zone can't be determined uniquely (i.e. there is more than one stock taking zone assigned to the station) the inventory date is set at the load unit. Also if the setting is location and the load unit is not stored.
