Purpose

This topic describes the configuration of storage-related dynamic domain values.

Load Unit Type

When an undefined Transport Unit Type shall be transported by the MFC, the following must occur:
1) The Transport Unit Type must be configured in the MFC.
2) A dynamic domain value must be created in the given domain value container for the new Transport Unit Type. The external name of the domain value must be set to the ID of the dynamic domain value.

HeightClassification

com.dematic.wcs.lms.
res.abstractions.storage.
domainvalues

Location

When predefined height classifications are insufficient, additional height classifications can be created by creating a new domain value in the given domain value container. The external name of the domain value must be the height classification  communicated by the MFC.
