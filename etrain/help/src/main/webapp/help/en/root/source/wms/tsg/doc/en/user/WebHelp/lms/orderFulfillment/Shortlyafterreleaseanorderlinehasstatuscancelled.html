Order Line cancelled with Release

Symptom

Shortly after releasing the corresponding task of a pending order the state of the state of one ore more Order Lines switches to CANCELLED. 

      
Brief Description

When the corresponding task of an order is released the system tries to satisfy the requested retrieval described via the given criteria. Depending on the current situation inside the warehouse it could be that the requested inventory respectively load unit isn't available. Hence the Order Line change it's status to CANCELLED. 

Most important dialogs for analysis and trouble shooting support: 

  • Dialog Retrieval Analysis.
      
Possible Reasons

Depending on the used criterion of the Order Line the following reasons are possible for the described symptom:

Criterion

Reason

Inventory

Insufficient available stock of the SKU inside the warehouse.

Parameter

No or not enough the parameters fitting load units are available inside the warehouse.

Load unit

The load unit is not available inside the warehouse.

Location

No load units are available for the given location.
     
Detail description

When a new order is submitted into the order management it is required to release the order for processing. A new order has status "PENDING". After an order is released the status of one or more Order Lines switches from RELEASED to CANCELLED. Maybe also the status of the order changes to FINISHED.

Depending on the used criterion the cause of the above mentioned reason is different.

Inventory criterion

If the given criterion of the Order Line is the inventory criteria then the available stock of the given SKU must fit to the specified quantity base relative to the given over- and underdelivery tolerance percentage. Because no picking is possible furthermore the available quantity of the specified stock must be distributed on fitting subsets of load units containing the asked SKU.

Causes for insufficient available stock of a SKU could be:

  • Not enough free quantity of the requested SKU/batch is inside the warehouse.
  • One or more locations storing load units with the requested stock have a long term lock for output and the remaining stock is to less or not in fitting subsets existent.
  • One or more aisles have a long term lock for output and the remaining stock is to less or not in fitting subsets existent.
  • There is no transport way to the given destination available for one or more load units containing a part of the required inventory and the remaining stock is too less or not in fitting subsets existent.
    Note: The transport way is relative to the current location of a load unit while the retrieval strategy is collecting the required stock. The load unit type must fit to the type of the destination location.
  • The SKU or a requested batch of the SKU is locked. (For example while quality assurance checks)
  • One or more inventory units have a lock and the remaining stock is too less or not in fitting subsets existent.

Parameter criterion

If the given criterion of the Order Line is a parameter criteria then the parameters of one or more available load units must match the specified parameters of the parameter criterion.

Causes for a system state when no or not enough the parameters fitting load units are available inside the warehouse could be:

  • No load unit inside the warehouse has the criterion matching parameters.
  • All locations storing load units with the requested parameters have a long term lock for output.
  • All aisles storing load units with the requested parameters have a long term lock for output.
  • There is no transport way to the given destination available for all load units fitting the required parameters.
    Note: The transport way is relative to the current location of a load unit while the retrieval strategy is collecting the required stock. The load unit type must fit to the type of the destination location.

 INFO 

If the Order Line specifies that all (and not only one) load units matching the given parameters are requested, then the retrieval strategy will collect all available load units matching the criteria. Matching load units in locked aisles or on locked locations are not collected and will stay on their current locations. The Order Line status will only switch into CANCELLED if less then one load unit was found.

Load unit criterion

If the given criterion of the Order Line is a load unit then causes for a system state when the load unit is not available inside the warehouse could be:

  • The requested load unit doesn't exist inside the warehouse.
  • The location storing the requested load units has a long term lock for output.
  • The aisle storing the requested load units has a long term lock for output.
  • There is no transport way to the given destination available for the load unit.
    Note: The transport way is relative to the current location of a load unit to the requested destination. The load unit type must fit to the type of the destination location.

Location criterion

If the given criterion of the Order Line is a location criterion then the resulting load units must be inside the requested location. Such a location can be a location (=bin), an aisle, an area or a facility.

Causes for a system state when no load units of the given location are available could be:

  • No load unit is inside the given location.
  • All locations (=bins) storing load units inside the given location (aisle, area or facility) of the criterion have a long term lock for output.
  • The location which is specified as criterion is long term locked for output.
  • There is no transport way to the given destination available for all load units which are stored in the given location.
    Note: The transport way is relative to the current location of a load unit while the retrieval strategy is collecting the required load units. The load unit type must fit to the type of the destination location.

 INFO 

The Order Line status will only switch into CANCELLED if less then one load unit in the specified location was found. 
     
Trouble Shooting

Determine what exactly happened

Step

Action

1

The status of the Order Line should be CANCELLED.

2

The cancel reason of the Order Line has a hint about cancellation because of the retrieval strategy.

 

Reason

Description

1a

If the status of the Order Line is not "CANCELLED" then the corresponding Order Line isn't cancelled.

2a

If the status is "CANCELLED" and the cancel reason has no hint about cancellation because of retrieval strategy
then the task of the order or the Order Line was cancelled via the GUI by an user.

2b

If the status is "CANCELLED" and the cancel reason has a hint about cancellation because of retrieval strategy
then one oft the above mentioned reasons are responsible for the cancellation.

 

Solution for Reason 1a:

Step

Action

1

Nothing to do.

 

Solution for Reason 2a:

Step

Action

1

Find out the user who cancelled the Order / Order Line and discuss a solution. However you have to create a new order for the cancelled Order Lines.

 

Solution for Reason 2b:

Step

Action

1

Depending on the used criterion the cause of the above mentioned reason is different. Find out the used criterion of the Order Line.

2

Check possible reasons for the criteria described in the "detail section" of this chapter and follow one ore more possibilities:

  • Unlock corresponding SKUs or batches .
  • Unlock corresponding locations (bins, aisles) with long term locks for output.
  • Insert missing area access entries for load unit types and areas.
  • Check available quantity of unreserved and unlocked inventory for the requested SKU (and Batch).
  • Check for existing load units matching the given parameters, location or load unit ID of the criteria.

3

Create a new order for the cancelled Order Lines and try again.
