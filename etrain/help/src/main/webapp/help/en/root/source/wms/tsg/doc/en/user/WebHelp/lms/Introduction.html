Logistics Manager (LMS)

Introduction

Logistics Manager (LMS) as a subsystem of the Dematic iQ® is responsible for nearly all business processes within a warehouse. It manages all key resources, such as Storage Locations, handling Areas, Load Units and Inventory.

Logistics Manager is also responsible for covering inbound operations (Receiving and Put-away), outbound operations (Order Fulfillment Planning and control, Full Load Unit Retrieval and Picking) and internal (optimization) operations (Replenishment, Housekeeping and Stocktaking).

Logistics Manager typically receives Advices and Shipping Orders from a host system. It does the process planning according to given specifications and restrictions and delegates all required fulfillment transactions to the underlying executing subsystem like Material Flow Controller, Pick Controller or Sortation Controller.

Logistics Manager keeps the data concerning Load Units (LU), Storage Locations and Stock Keeping Units (SKU). LMS can be run in two different modes: "Stock Keeping Mode" and "Parameter Mode". In Parameter Mode LMS does not manage inventory and inventory attributes are stored as generic parameters of Load Units. In Stock Keeping Mode Inventory Units are used for management of Inventory.Â They assign Inventory attributes like SKU, batch and quantity to a Location.

If Dematic Distribution Center Director (Dematic iQ®) keeps Inventory information of Load Units, it is capable to operate more independent of the Host and enables more sophisticated optimization strategies, which make use of this data.
