Load Detail

Purpose

This screen opens when creating, viewing or modifying a Load.

      
Screen

      
Fields

Field

Description

ID

Unique identification of the Load.

 INFO 

Project specific policies may be implemented in case the identifier shall have a certain semantic, e.g. depending on the day of the year.

Route

The Route the Load is assigned to.

Dock

The Dock the Load is assigned to.

Status

How far the Load was processed:

  • pending (Load has been created)
  • authorized for staging (Staging of Dispatch Load Units for the Load may start)
  • arrived (The truck has been arrived at the Dock)
  • authorized for loading (Loading of the Load may start at the assigned Dock)
  • loading (Loading is in progress)
  • loaded (Loading is finished)
  • closed (The Load has been closed and shipping documents have been printed)
  • departed (The truck has been departed)
  • finished (The Load is confirmed to the host).

Assembling Status

Defines whether or not all Dispatches and Dispatch Load Units of the Load are completely assembled:

  • pending (Assembling is not yet completed)
  • completed (Assembling is completed).

 INFO 

A status of "pending" does not prevent from loading. Loading of a Load can start even if assembling is not yet completed. However closing a Load requires that Assembling is "completed".

 INFO 

The "Assembling Status" of the Load changes (in the standard) automatically to "completed" when the first Dispatch Load Unit was loaded.

Truck License Plate

License plate of truck or other loading equipment.

Freight

Carrier

The assigned Carrier of the loading equipment wherein the Load was shipped.
Truck License Plate

Identification of the truck, if used (e.g. license plate).

Trailer Identification

Identification of the trailer, if used (e.g. license plate).

Serial Number of Container

Serial Number of the container wherein the Load is shipped.

Manifest

Identifier of the shipping manifest of the Load.

BOL

Identifier of the Bill of Loading of the Load.
