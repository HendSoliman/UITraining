Subgroup - Detail

Purpose

For an overview of all elements used to group locations in a warehouse, see the Warehouse Structure Description.

      
Screen

      
Fields

Field

Description

Facility

Selected Facility. Read only field.

Area

Selected Area. Read only field.

Group

All Groups of the selected Area. Each Subgroup (Level) must be assigned to a Group (Aisle).

ID

Unique ID of the Subgroup (Level).

Y-coordinate

Height coordinate (Y) of the level. Can only be defined while creating a new Level.

Subgroup Type

The subgroup represents a group of locations which are a further division of a group which could be:

  • Level (see Warehouse Structure Description)
  • Dummy (This type groups transport points usually with respect to the PLC in a project specific and particular way).
Height Classification

Height classification of all Locations of this Level.

 INFO 

Is not allowed to differ from the assigned Locations.

Configuration Status

If Level configuration is finished and the Level can therefore be used by the system.

Handling

Lock Status

If the Level was locked for specific operations. Meant as a long term lock set by an user.

 INFO 

With a lock of this type, all current Inventory reservations within this level will be cancelled with an immediate new reservation process.

 INFO 

This Lock Status can only be changed with use of the related function of the overview screen.
Values are identical to those of the Technical Lock Status.
Lock Status Reason

Reason for the lock.
Technical Lock Status

If the level was locked for specific operations, short term, by a underlying transport subsystem (project-specific use only):

  • available (all transport operations will execute)
  • locked for input (inbound operations not assigned to the Level automatically)
  • locked for output (outbound operations not assigned to the Level automatically)
  • locked for input and output (inbound and outbound operations not assigned to the Level automatically)

 INFO 

During inbound or outbound operations, technical lock status can be enabled or disabled within the strategies.
Technical Lock Status Reason

Reason for the lock.

Statistics

Capacity

Overall number of storable Load Units within this level. Calculated as sum of appropriate values of all Locations of the Level less the positions locked for input.
Reservation Capacity

Number of Load Units which can still be disposed within this level. Maintained on each disposition operation with this Level and calculated as a sum of the appropriate value of all Locations of the Level.
Fill Level

Used capacity of this Level in percent.

Inter Aisle Transfer

Minimum Reachable Aisle

The lowest Aisle (Sequence) which is reachable via Inter Aisle Transfer from this Level (locks considered).

Maximum Reachable Aisle

The highest Aisle (Sequence) which is reachable via Inter Aisle Transfer from this Level (locks considered).
