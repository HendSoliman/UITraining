Put new Pick Container at RapidPick Station

Description

If additional Pick Containers are required at a Pickstation, this must be performed.

      
Triggers and Actors

A picker must perform this after launching the RapidPick Screen when not all required Pick Containers are booked on Put Location, a Pick Container is sent away and a new one is required. 

      
Prerequisites
  • Active Pick Mission assigned to the Pickstation (see Workstation Queue Entries).
  • Additional Pick Container is required to process the one of the current Pick Missions.
  • Pickstation's Put Location is available in "free" status.
  • For automatic stations, an empty Load Unit is available in the system and can be transported to the station.
  • For manual stations, an empty new Load Unit is required at the station.
  • For manual stations, the system's internal DBTM (Database Transfer Message) connection must be established and running.
      
Required Resources
  • RapidPick Screen
  • Picking step
  • Create Pick Container (optional).
      
Work Flow

Step

Action

1.

Open RapidPick Screen.

2.

The screen Picking displays and indicates the put action for additional Pick Containers.

 INFO 

Indication is not available for automatic stations.
Automatic station
2a. Next Pick Container is automatically transported to the free Put Location.
Manual station
2b. Put a new, empty Load Unit on the free Put Location.
2c. A message (DBTM) is sent to the screen and a new Pick Container is created. Optional or exceptional case: Use the screen "Create Pick Container".
3. When new Pick Containers are required, proceed to step 2.
     
Result

All required (one or multiple) new Pick Containers are booked onto the displayed Put Locations.
