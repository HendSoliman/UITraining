Detail Dialog Stocktaking Zone Configuration

Purpose

This screen opens when creating, viewing, or modifying Stocktaking Zone Configurations.

      
Screen

Overview

Handling

      
Fields

Field

Description

Zone

Assigned Zone for which this Stocktaking Zone is responsible. Zones are administered through the Zones screen.

Description

Textual description of the configuration set for the related Stocktaking Zone.

Overview

Stocktaking Execution Type

Type of Stocktaking execution used in this Stocktaking Zone. Defines the basic fulfillment procedures for Counting Order execution.

  • Goods-to-Person (Counting is executed using the screen for Counting. Load Units will be requested to a Workstation)
  • Person-to-Goods Offline (Counting is executed using printed Counting Lists. Requires manual entry of counting results on the Counting Orders)
  • Person-to-Goods Online (Counting is executed using mobile terminals. Operator is guided to locations with requested counting operation).

Stocktaking Date Container

Stocktaking Objects (see Terms and Definitions) of this Stocktaking Zone contains the last stocktaking date.

  • location
  • load unit
  • inventory unit

 INFO 

If the corresponding zone can't be determined uniquely (i.e. there is more than one stock taking zone assigned to the station) the inventory date is set at the load unit. Also if the setting is location and the load unit is not stored.

 

Inventory Correction Type

How to correct counting differences with required bookings on Inventory Units:

  • stocktaking completion (Inventory correction bookings are done automatically at the end of Stocktaking with closure of the Counting Order)
  • no correction (no Inventory correction bookings are done automatically according to confirmed Inventory differences)
  • during counting (Inventory correction bookings are done automatically during confirmation of the single counting result, directly at execution of the Counting operation).

Handling

Maximum Number of Counting Objects

Limits the size of the Counting Orders during the creation process. If this is reached, an additional Counting Order is created for the same Stocktaking Zone.

Number of Recounts

Maximum number of required recounts to accept a counting deviation, if the configured "Recount Tolerance" is exceeded.

Zero Confirmation Required

Separate zero quantities confirmation is required during the Stocktaking process (confirmation of Counting Orders). If enabled, helps to avoid accidental Inventory Unit deletion with Inventory Correction Booking.

 INFO 

 Deleted Inventory Units cannot receive any quantity increase within Stocktaking processes. Inventory Unit creation is supported by Receiving processes only.

Automatic Creation of Recount Counting Order Lines

Whether to confirm Counting Order Lines that are out of tolerance. Counting Order Lines for recounting operation will generate automatically by the system if this is enabled. Otherwise, the operator must initiate the recounting operation manually.

Check Number of Load Units

During 'Person-To-Goods Online' counting a check for each location can be performed. If this flag is set the operator is requested to count the number of Load Units at each Location and perform a scan for each Load Unit.

SKU Scan Required

Whether the operator has to scan the inventory before counting.

Load Unit Scan

Scanning handling for Load Units during Person-to-Goods online counting process:

  • always (The system request the scan of each Load Unit)
  • more than one (The system required scan only if more than one load units are on a location)
  • never (No scanning of load units is required).

Tolerance

Non-Automatic Tolerance (%)

Percentage of deviation from the expected quantity counting result confirmation by the operator. Leads to not automatically accept. Forces the operator to accept explicitly.

Recount Tolerance (%)

Percentage of deviation from the expected quantity recounting result confirmation by the operator. Leads to not automatic accept. Forces the operator to accept explicitly. If 0, deviation is accepted at once after security call back.

Exceeding Tolerance Ignored for Booking

If enabled, can manually close the Counting Order even if existing Counting Order Lines are "out of tolerance". Otherwise, all these Counting Order Lines must be confirmed in particular.
