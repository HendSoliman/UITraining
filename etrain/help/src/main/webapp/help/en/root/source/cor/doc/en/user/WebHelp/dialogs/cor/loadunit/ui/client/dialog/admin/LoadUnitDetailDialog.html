Fields

Field

Description

ID

ID of the Load Unit for identification.

Overview

Owner

Owner of the Load Unit.

Alternative ID

Optional ID of the Load Unit (e.g. from an external system). SAP GUID, for example.

Current Location
Location

Current Location of the Load Unit including the Position inside the assigned Location which reflects the storage position of this Load Unit inside the Location. May be the real position or just a sequence.

 INFO 

The position must be interpreted with respect to the assigned layout. For details, see the Warehouse Administration Guide.

Location Arrival Date

Timestamp when the Load Unit arrives at the current Location.

Classification

Load Unit Type

Load Unit type, e.g. Europallet, Industry pallet, Tote, Tray.

Ufo

If the Load Unit must be manually or automatically created for clarification purposes. After the clarification was done the Load Unit should be deleted.

Virtual

LU is virtual and not a real.

Height Classification

For easier processing, Load Unit heights are divided in several project specific height classes. Both attributes, height and height classification should be maintained.

Quality

The quality specifies the quality of the loading device (e.g. euro-pallet).

Status

Status

Status of the Load Unit:

  • no TO (not currently stored without a current Transport Request)
  • building (currently used within a business process (e.g. Picking) and not yet ready)
  • scheduled (scheduled for a transport that has not yet started)
  • transport (currently in transport)
  • stored (stored without a current Transport Request).

 INFO

Field is disabled when the Load Unit has a Transport Request.

Status Reason

Reason for the current status.

Lock Status

If the Load Unit is locked for specific operations:

  • free (not locked)
  • locked for stocktaking (locked for a currently running stocktaking counting operation)
  • locked for clarification (locked for a currently running or planned clarification operation; set automatically by the system in exceptional cases and could be set/released by a separate dialog function.)
  • locked due to inventory missing (locked due to a detected inventory miss in any other process).

Reservation Status

Current reservation state of the Load Unit:

  • free (Load Unit is not reserved)
  • inventory reserved (Inventory Unit on the Load Unit is reserved for a specific business process (e.g. a Product Load Unit for picking))
  • children reserved (Inventory Units on children of the Load Unit are reserved)
  • Load Unit reserved (Load Unit is reserved for a specific business process (e.g. a Pick Container before the first Pick))
  • Load Unit picked (for Pick Container if a Load Unit reservation was set before)
  • Load Unit loaded (Load Unit was loaded while outbound Shipping process)
  • Load Unit shipped (Load Unit was shipped)
  • Load Unit confirmed (Load Unit was confirmed while outbound shipping process to the external host system).

Base Load Unit

If the Load Unit is a Base Load Unit which is the lowest Load Unit of the Load Unit stack, which contains this Load Unit. Used for transport issues.

Empty

If the Load Unit is empty. No Inventory Unit is located on the Load Unit or any stacked Load Unit on it.

QA Required

If a Quality Assurance check shall be initiated.

Error

Logistics Error Code

Errors related to logistic aspects of a Load Unit. Set while appropriate business processes and supports the clarification (at a Clarification Station).

  • picking clarification (Load Unit was sent to a Clarification Station by any picking process)
  • putaway clarification (Load Unit was sent to a Clarification Station by any putaway process)
  • qa clarification (Load Unit should be checked for quality assurance reasons at a QA Workstation)
  • receiving clarification (Load Unit was sent to a Clarification Station by any receiving process)
  • sortation clarification (Load Unit was sent to a Clarification Station by any Sortation process)
  • wrong inventory (Any problem with the inventory on the Load Unit and must be clarified (at Clarification Station))
  • wrong load unit (Load Unit was not the expected and must be clarified (at Clarification Station)).

Logistics Error Reason

Reason for the logistic error (code).

Inventory

Discriminating Key

For the Inventory Balancing in the warehouse, is the distinguishing aspect defined by the Inventory on the Load Unit. The discriminating Key of a Load Unit is set for single Inventory Units on Load Unit which is not yet "picked". For multiple Inventory Units on a Load Unit, this key is set to null. For order related Inventory (from status "picked"), this key is set and updated accordingly.

Mislocation Factor

Indicator of the optimal Location of a Load Unit. 0 means an optimal storage. Value greater than 0 means a suboptimal Location. The greater the value the worse the placement of the Load Unit. An undefined value indicates that the factor has not yet calculated.

Hot

If the Load Unit is stored or planned to be stored within a Hot Zone.

Inventory Directly Inside

If the Load Unit contains Inventory. If a Child Load Unit of the Load Unit contains Inventory this flag is not set.

Single SKU

If the Load Unit contains only inventory of a single SKU.

Partial

If Inventory was picked from this Load Unit. Otherwise, the Load Unit is a Full Load Unit.

Fill Level

Filling degree in percentages. Can be project specific.

Last Stocktaking Date

Date when the last Stocktaking action took place on this Load Unit.

Dimension & QA

Dimension

Height

Static LU dimension: length of vertical edge. Measured value or a height classification. Measuring unit is defined on a project specific base. 0 (zero) represents an unspecific height. Both attributes, height and height classification should be maintained.

Length

Static LU dimension: Length of longest edge different from height. Measuring unit is defined on a project base. 0 (zero) represents an unspecified width.

Width

Static LU dimension: Length of smallest edge different from height. Measuring unit is defined on a project base. 0 (zero) represents an unspecified width.

Case Orientation

The three dimensional orientation of the Load Unit. It is used for cases in Multishuttle Flex systems:

  • Length in Conveying Direction, Width Across
  • Length in Conveying Direction, Height Across
  • Width in Conveying Direction, Length Across
  • Width in Conveying Direction, Height Across
  • Height in Conveying Direction, Length Across
  • Height in Conveying Direction, Width Across.

Orientation

Last scanned orientation which depends on the specific scan which was confirmed by the Material Handling Equipment. The orientation defines the display orientation of the Inserts (compartments) of the scanned Load Unit in dialogs at the related Workstation.

x-Effective-Extension

The effective length of a Load Unit is its extension in x-direction of the Multishuttle rack under consideration of positional uncertainty due to previous handling by a Multishuttle Flex which has not been closed to the Load Unit´s x-Extension.

x-Extension

Dynamic dimension: LU extension along X-Axis of equipment system, usually in base of the measuring unit.

y-Extension

Dynamic dimension: LU extension along Y-Axis of equipment system, usually in base of the measuring unit.

z-Extension

Dynamic dimension: LU extension along Z-Axis of equipment system, usually in base of the measuring unit.

Tare Weight

Tare weight of the empty Load Unit (in base unit of measure).

Weight

Weight of the load Unit. Measured or calculated. Measuring unit is defined on a project specific base. 0 (zero) represents an unspecified weight.

Quality Assurance

QA Reason

Reason for the intended QA check is to be set by the operator when the QA Required flag is set.

Ignore Difference

If differences between actual and expected weight shall be ignored when the Load Unit passes a QA Decision Point.

Expected Weight

Calculated weight of the whole Load Unit including Inventory.

Checkpoint

Location where the actual weight of the Load Unit has been determined.

Original Destination

The original destination of the Load Unit prior to divert to a QA Destination.

Transport

Load Unit Request

Request Location

The Location where the Request will be fulfilled. If the Load Unit is booked on this Location the Load Unit Request will be deleted directly.

 INFO 

A Load Unit Request assigns the Load Unit to a specific business process until it is deleted.

The requesting business process will only proceed if the request will be fulfilled.

Request Type

The type of the Load Unit Request specifies the business process which currently requests the Load Unit for fulfillment:

  • Pick Container (Already confirmed Pick Container triggers the next process step)
  • Empty LU (Empty Load Unit was requested for specific Empty Load Unit Supply, normally for project specific use)
  • Product LU (Source Load Unit was requested for Goods-to-Person picking at a stationary Pickstation)
  • Putaway (Source Load Unit contains further Picks (for a different Pickstation) after push away pick operation and a further outbound process for picking must be triggered)
  • Product without LU (Inventory must be removed from Source Load Unit for later processing, e.g. in AMCAP operation where the cases are stored on trays which must be removed before palletizing)
  • Pick Container in Pick Zone (Pick Container is scheduled for serial picking an must be forwarded to the next Pick Zone).

Source Location

Source Location and Position of the Load Unit is set in case of a Transport is requested.

Intermediate Destination

Set if a Transport is requested. The Load Unit must be transported to the intermediate destination before it is transported to the destination.

Destination

Set if a Transport is requested.

Contour State

Byte pattern that is filled, when the Error Code is ME (Measurement Error).

Depalletizing Sequence

How often this pallet has been de-palletized (one digit 1-9, A-Z).

Case

Running number of depalletized cases (set to null, when the Depalletizing Sequence is incremented).

MFC Result

Error Code

Last transport error which occurred with this Load Unit.

Error Reason

Detail about the current error (code).

Statistics

Transport Counter

Count of all transports of this Load Unit. May be used for maintenance purposes.

Aisle Internal Relocation Counter

Number of aisle internal relocations of the Load Unit. After an aisle change, this counter will be initialized.

Reservation

Reservation Reason

Text reference to a specific reservation process.

Reservation Priority

Relative urgency of a reservation.

Reservation Client

Client the Load Unit was reserved for.

Order Type

If the order type ID, the ordering Client, the order ID and order Line ID of a Load Unit are set, the Load Unit is used for the specified Order and cannot be used for other Orders.

Order ID

Order ID that this Load Unit is exclusively used for.

Order Line ID

Order Line ID that this Load Unit is exclusively used for.

Composition

Base Load Unit

Lowest Load Unit of the Load Unit stack, which contains this Load Unit. Used for transport issues.

Stacked On

Load Unit on which this Load Unit is stacked.

Position On Parent

If the Load Unit is stacked the position reflects the alignment position inside or on the 'Stacked On' Load Unit.

Contains Stacked Load Units

If the Load Unit has other Load Units stacked onto it.
