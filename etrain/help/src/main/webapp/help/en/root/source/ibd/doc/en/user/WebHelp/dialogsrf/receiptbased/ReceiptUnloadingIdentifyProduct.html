Identify Product

Purpose

This step appears after a Load Unit and Type has been defined and no Receipt could be determined uniquely.
The operator scans a GS1 product barcode for selection of the Receipt.

 INFO

A GS1 barcode respectively its GTIN part is required.
The barcode may contain the batch and/or best before date which will be used as additional qualifier for identifying the appropriate Receipt.
     
Screen

 INFO

The scanned GTIN is a GTIN-14 where the prefix '01' is the barcode Application Identifier which identifies the GTIN.
     
Prerequisites
  • A Dock, Shipment, Load Unit and Load Unit Type have been selected.
      
Fields

Field

Description

Client

The Id of the Client of the inventory to be received.

Dock

The Id of the selected Dock the operator is currently working at.

Shipment

The Id of the selected Shipment.

SKU

The Id of the Stock Keeping Unit of the inventory to be received.

Batch

The batch of the inventory to be received.

Quantity

The total quantity to be received according to the Receipt.

Quantity Unit

The Base Quantity Unit of the inventory to be received.

Barcode

Input field for the GTIN Barcode to be scanned.

 INFO

A GS1 barcode respectively its GTIN part is required with optionally contained batch and quantity.
     
Special Buttons

Function/Button

Description

F2: SKU

Enter the Stock Keeping Unit.

F3: Batch

Enter the batch.

F4: Quantity

Enter the quantity.

Enter: OK

Select an appropriate Receipt with the matching data of the barcode or SKU and batch and quantity.
Switch to step Check Product.

ESC: Cancel

Cancel and switch back to step Check LU Type.
