Missing initial scripts

Symptom

Files for database initialization (LmsInitialConf.xml, LmsInitialConf.cmd) are not located at %WCS_HOME%/PostinstallWCS. 

      
Brief Description

The initialization procedure is needed to load default values, creating database indexes, database constraints and database views.  

      
Possible Reasons
  • Errors while installation process
  • User deletes files (LmsInitialConf.xml, LmsInitialConf.cmd) manually
      
Trouble Shooting

Determine what exactly happened

Step

Action

1

Look at folder %WCS_HOME%/PostinstallWCS

 

Reason

Description

1

The files LmsInitialConf.xml and LmsInitialConf.cmd were lost while installation process

2

The files have been deleted manually by a user

 

Reason 1/2:

Step

Action

1

Shutdown BEA Weblogic server.

2

Open folder %WCS%/runtime/lms/res/doc/scripts

3

Run WCS_LMS_Indexes.sql (e.g. type WCS_LMS_Indexes.sql | sqlplus <Username>/<Password>@<SID> )

4

Run WCS_LMS_Constraints.sql (e.g. type WCS_LMS_Constraints.sql | sqlplus <Username>/<Password>@<SID> )

5

Run WCS_LMS_Views.sql (e.g. type WCS_LMS_Views.sql | sqlplus <Username>/<Password>@<SID> )

6

Run WCS_LMS_InsertDefaultData_Base.sql (e.g. type WCS_LMS_InsertDefaultData_Base.sql | sqlplus <Username>/<Password>@<SID> )

7

Open folder %WCS%/runtime/lms/res/doc/scripts/installation

8

Run WCS_LMS_InsertDefaultData_MFC_mapping.sql (e.g. type WCS_LMS_InsertDefaultData_MFC_mapping.sql | sqlplus <Username>/<Password>@<SID> )

9

Run WCS_LMS_ExternalForwarder_Service_cfg.sql (e.g. type WCS_LMS_ExternalForwarder_Service_cfg.sql | sqlplus <Username>/<Password>@<SID> )

10

Run WCS_LMS_TransportExecutor_Service_cfg.sql (e.g. type WCS_LMS_TransportExecutor_Service_cfg.sql | sqlplus <Username>/<Password>@<SID> )
