Configuration Parameters - Inventory

Purpose

This topic describes the configuration parameters for Inventory.
Located in demo\target\configuration\cor-inventory.xml.

      
INV - Inventory

Key

Type

Dimension

Default

Description

Inventory.

onlyNotifyHoldChangesIfStockTypeChanges

boolean

Client

true

This parameter contains 'true', if Inventory Management shall only notify a host about hold changes, if the stock type for the inventory changes. For the hosts SAP and Siematic IT this property has to be true.

Inventory.

leaveConfirmedMeansShipped

boolean

Client

true

This parameter contains 'true', if the confirmation of the leave on an Inventory Unit means that the Inventory Unit is shipped.

Inventory.

LostAndFoundLocation

String

LostAndFound

Lost inventory is booked to this location and should be clarified. This parameter is the location id of the lost and found location.

Inventory.

inventoryMovementReasonMandatory

boolean

Client

true

This parameter contains 'true', if a host relevant inventory change that is not ordered by the host (that means it is initiated inside the LMS) requires the specification of an Inventory Movement Reason.

Inventory.

InventoryJournalRemainingTime

int

  

168 (7 Days)

Time span in hours after which Inventory Journal entries can be deleted after they their Inventory Units have left the warehouse.

Inventory.

notifyAddBookingsInitiatedByHost

boolean

Client

false

When this parameter is set to true, Inventory Unit add bookings are notified to host on the availability status transition from in receiving to free.

Inventory.

InventoryUnitSpecificationClassname

String

 

InventoryUnitSpecification.class.getName()

This parameter specifies the name of the class that RES-INV use as implementation of an Inventory Unit specification. This parameter allows to determine a project specific Inventory Unit specification in the inventory loader tool so that project specific Inventory Unit attributes can be set by the inventory loader.

Inventory.
InventoryMovementTypeBreakage

String

 

BREAKAGE

This parameter defines the movement type id for inventory breakage.
