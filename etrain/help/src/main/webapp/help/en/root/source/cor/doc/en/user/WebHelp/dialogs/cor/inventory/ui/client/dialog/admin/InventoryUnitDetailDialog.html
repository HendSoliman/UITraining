Detail Dialog Inventory Unit

The detail dialog is opened when viewing inventory units by applying the respective buttons in the overview window.

Overview

Status

Availability Status

Processing state for the Inventory Unit and its availability to satisfy Customer Orders:

  • in receiving (Inventory Unit was created as part of a Receiving process and the Receiving has not yet been confirmed to the host.)
  • free (Inventory Unit has no reservation and can be used to satisfy Customer Orders.)
  • reserved (Inventory Unit has a reservation and at least a part of the Inventory Unit cannot be used to satisfy Customer Orders. Part of the Inventory Unit may be available.)
  • picked (Inventory Unit is assigned to a Customer Order and was picked for the Customer Order. Not available for other Customer Orders.)
  • packed (Inventory Unit is assigned to a Customer Order and was packed for the Customer Order. Not available for other Customer Orders.)
  • loaded (Inventory Unit is loaded onto a truck for Shipping and has left the Shipping Area, but it has not been confirmed to the host that the inventory left the system. This means the inventory is still managed by the LMS.)
  • leave confirmed (Leave of inventory was confirmed to the host and the Inventory Unit is not part of the inventory managed by LMS, but the inventory unit has not left the Shipping Area yet.)
  • shipped (Leave of inventory was confirmed to the host and the Inventory Unit is not part of the inventory managed by LMS and it has left the Shipping Area. This Inventory Unit can be deleted.)
In Receiving

Active if the Inventory Unit was received and the Receiving was not yet confirmed to the host.
On Hold

Active when there is at least one Inventory Hold for the selected Inventory Unit.

Reservations

Inventory unit reservations reserve parts of an inventory unit for a certain order.

Quantity
Quantity

Total quantity of this Inventory Unit expressed in the Base Quantity Unit of the SKU.
Unit

Unit of the quantity of the SKU.
Free Quantity

Quantity available for disposition, provided the inventory is not on hold. Quantity is expressed in the Base Quantity Unit of the SKU.
Quantity Missing

Quantity identified as missing (e.g. by a near zero stocktaking during Picking). In some cases, a representation of missing quantity is required until all bookings connected to the missing quantity are finished. When reserved inventory is identified as missing, the missing quantity is managed in LMS until the Inventory Unit Reservations on the missing quantity are canceled.
Total Weight

Total weight of the Inventory Unit (in base unit of measure). This attribute is kept up to date when the weight is measured and the assigned SKU requires weight tracking.
Net Weight

Net weight of the Inventory Unit.

May be used to completely identify the Inventory Unit.

Part of the EPC code that identifies the Inventory Unit for a given SKU.

Outbound
Order ID

If the order type ID, ordering Client, order ID, and order line ID of an Inventory Unit are set, the Inventory Unit is used for this Order and cannot be used for other Orders. The availability status of the Inventory Unit is then at least 'picked' (it is not 'in receiving', 'free' or 'reserved').
Order Line ID

If the order type ID, ordering Client, order ID, and order line ID of an Inventory Unit are set, the Inventory unit is used for this Order and cannot be used for other Orders. The availability status of the Inventory Unit is then at least 'picked' (it is not 'in receiving', 'free' or 'reserved').
Order Type ID

If the order type ID, ordering Client, order ID, and order line ID of an Inventory Unit are set, the Inventory Unit is used for this Order and cannot be used for other Orders. The availability status of the Inventory Unit is then at least 'picked' (it is not 'in receiving', 'free' or 'reserved').
Ordered by Client

Client of the Order that uses this Inventory Unit. If the Order Type, ordering Client, Order, and Order Line of an Inventory Unit are set, the Inventory Unit is used for this Order and cannot be used for other Orders. The availability status of the Inventory Unit is then at least picked (it is not in receiving, free or reserved).

ERP
Host Notification Status

If a host notification (inventory add or remove) is currently required, not required, or scheduled in the future for this Inventory Unit:

  • not required (host notification is not required.)
  • scheduled (host notification is required and scheduled, but not be done yet.)
  • to be done (host notification must be performed now.)
  • done (host notification was performed.)
