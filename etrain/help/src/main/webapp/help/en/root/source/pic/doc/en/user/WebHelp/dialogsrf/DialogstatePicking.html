Picking

Purpose

"Picking" appears after scan of required Pick Face (Source Location) and provides the main instructions of the current pick operation like SKU quantity, Put Location and additional information like Source Load Unit, best before date, batch and service as well. In case scanning of product label is required further input field for label data is provided in separate dialog state before.
To confirm this state simply with key "OK" is the standard procedure with use of this screen. The requested Pick operation is confirmed without any variation regarding the displayed data.

Prerequisites

For entering the "Picking" state the following prerequisites have to be fulfilled:

  • All required Pick Containers are put on the correct Put Locations on the Pick Cart.
  • Required Pick Face (Source Location) is scanned.
  • Previous Pick is confirmed and further Pick on same Source Load Unit is available.
  • Display of "Reverse Pick recommended" needs the appropriate configuration of Pick Zone (see Pick Zone Configuration).
  • Display of additional field in further dialog state for scan of "Product Label" needs the appropriate configuration of Pick Zone (see Pick Zone Configuration).
      
Fields

Field

Description

SKU

The SKU id uniquely identifies a Stock Keeping Unit within a Client.

Quantity

The target quantity to be picked into the Pick Container.

Quantity  Unit

The quantity unit of the pick operation. This will be the "Pick Quantity Unit" In case of exactly one Default Pick Quantity Unit is set for the quantity units of the SKU. Otherwise the Base Quantity Unit is displayed.

Put Location

The alias represents the name of the Put Location on the Pick Cart for easy identification. The Alias is only textual information and will not be validated from the system.

Source Load Unit

The unique identifier of the current Source Load Unit.

Best before date

The best before date of the Inventory Unit to be picked.

Batch

The batch of the Inventory Unit to be picked.

Product Label

Only in case of additional scan of Product Label is required this further Input field is displayed.

 

Colored text "Reverse recommended" in case of recommended Reverse Picking.
     
Special Buttons

The available function in "Scan Product Label" is:

Function/Button

Description

Scan
"* Product Label"

Only in case of required scan of Product Label the field "Product Label" is displayed and focused for entering the scanned Pick Container with launching this dialog state. After Scan the Dialog proceeds to Picking. In case of invalid Scan an error message appears.

Change to "Short"

key "F2"

Dialog proceeds to Short, where the operator is able to confirm inventory shortage due to the required product is not available for scanning.

The available functions in "Picking" are:

Function/Button

Description

Confirm as "OK"

key "Enter":

The requested pick operation is confirmed without any variation regarding the displayed data.
The confirmed quantity of Inventory Unit of displayed Source Location is booked into the Pick Container and the Pick is booked and confirmed with Code 'ok'.

Change to "Info"

key "F11"

Dialog proceeds to Pick Info, where the operator is able to view detailed information about current pick operation.

Change to "Short"

key "F2"

Dialog proceeds to Short, where the operator is able to confirm inventory shortage due to several configured shortage reasons.

Change to "Over"

key "F3"

Dialog proceeds to Over, where the operator is able to confirm advised picking of more than requested quantity without violation of overdelivery tolerance. In case of overdelivery tolerance 0 this function is disabled.

Change to Skip

key "F4"

Dialog proceeds to Skip, where the operator is able to confirm the desired skip of current Pick and it's rescheduling to the end of current Pick Walk.

Change to Blocked

key "F5"

Dialog proceeds to Blocked, where the operator is able to confirm the skip of current Pick without appending it to the end of current Pick Walk due to Pick Face is permanently blocked.

Change to Full

key "F6"

Dialog proceeds to Full, where the operator is able to confirm that not all of required quantity could be picked into the current Pick Container.

Change to "Reverse"

key "F7"

Dialog proceeds to Reverse, where the operator is able to confirm the operation of reverse picking with exchange of Source Load Unit and Pick Container. In case of reverse picking is not allowed this function is disabled.
