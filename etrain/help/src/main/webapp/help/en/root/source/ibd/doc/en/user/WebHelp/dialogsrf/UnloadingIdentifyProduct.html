Identify Product

Purpose

The dialog step "Identify Product" is next after a Load Unit and its type has been defined and more than one valid Advice Line is available. In case several different products reside on the Load Unit the product code has to be scanned.

 INFO 

A GS1 barcode respectively its GTIN part is required.

The barcode may contain the batch and/or best before date which will be used as additional qualifier for identifying the appropriate Advice Line.
     
Screen

 INFO 

The scanned GTIN is a GTIN-14 where the prefix '01' is the barcode Application Identifier which identifies the GTIN.
     
Prerequisites
  • An open and appropriate Advice has been determined
  • A Load Unit and a Load Unit type have been selected
  • An Advice Line is not uniquely determined with the selected Load Unit.
      
Fields

Field

Description

Client

The Id of the previously selected Client.

Dock

The Id of the previously selected Dock.

Supplier

The Id of the (possibly) previously selected Supplier.

SKU

The Stock Keeping Unit to be received.

Batch

The batch of the product to be received.
     
Special Buttons

Button

Description

Barcode

Input field for the GTIN Barcode to be scanned.

 INFO 

A GS1 barcode respectively its GTIN part is required with optionally contained batch and BBD.

F2: SKU

With use of "F2" the Stock Keeping Unit may be entered.

F3: Batch

With use of "F3" the batch may be entered.

OK

Select an appropriate Advice Line with the matching data of the barcode or SKU and batch.

Cancel

Cancel and switch to "Check LU-Type".
