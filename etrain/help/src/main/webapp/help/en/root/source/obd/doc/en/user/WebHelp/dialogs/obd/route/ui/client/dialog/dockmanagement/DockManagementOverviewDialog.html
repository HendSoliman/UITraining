Dock Management

Purpose

The Dock Management dialog is used to assign a Route to a Dock or revoke an existing assignment.

 INFO 

It's not possible to have an interference between the time period limited by the expected loading and departure time of the selected unassigned Route and the already occupied time period of the Dock.

     
Screen

      
Fields
  • Dock
    This panel provides the selection of a Dock. If a new Dock is selected all data will be refreshed automatically
  • Departed Routes
    All assigned but already loaded or departed Routes (in status "loaded" or "finished") of the Dock are displayed.
  • Unassigned Routes
    All available and not yet assigned Routes are displayed.
  • Assigned Routes
    The already assigned Routes in status "planned" or "active" are displayed.

Field

Description

ID

Unique identification of the Route.

Expected Loading Time

The planned or expected loading time is used for slotting of the Docks. It is initialized automatically from the average loading time of the Itinerary and can be adjusted manually based on the number of DLUs.

Departure

The planned point in time for the departure of the Route.
     
Special Buttons

Button

Description

Refreshes the specific panel. The Refresh Button of the Dock Panel refreshes all data.

Assign the selected Route from the "Unassigned Routes" Panel to the selected Dock.

Revoke the Dock assignment of the selected Route from the "Assigned Routes" Panel.
