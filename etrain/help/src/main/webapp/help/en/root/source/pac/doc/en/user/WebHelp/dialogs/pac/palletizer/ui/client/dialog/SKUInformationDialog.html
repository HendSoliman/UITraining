SKU Information

Purpose

This dialog based on a very central view, which is used during the AMCAP process from various sub-processes. This view contains for each Stock Keeping Unit (SKU) the information about the need and availability in all available aisles. Thereby all SKU-QuantityUnits are considered which are qualified for AMCAP, e.g. their Preferred Fulfillment Handling Type is AMCAP.

The usage of the information from this view are used by:

  • Assigning orders to aisles (complete or partially using palletizing groups)
  •  Starting pallet planning for orders
  • During putaway in the MS-Flex
  • Starting inter aisle transfer (iAT) for balancing the inventories

Fields

Field

Description

Client

Owner of the Inventory of this Stock Keeping Unit.

SKU

Uniquely identifies a Stock Keeping Unit within a Client.

Description/SKU

Description of the Stock Keeping Unit.

Batch

Batch in which the inventory was produced.

ID/Aisle

Aisle for which the information about the Stock Keeping Unit is provided.

Factor To Base/Quantity Unit

Quantity conversion factor for this packaging level to the Base Quantity Unit of this SKU.

Quantity Available

Quantity stored in the Multishuttle buffer storage in Base Quantity Unit.

Quantity Available For Retrieval

Quantity available in the Multishuttle buffer storage in Base Quantity Unit.

Thereby the lock status of the location, level and aisle are considered as well.

Quantity Needed

Quantity needed for the assigned order lines in Base Quantity Unit.

Shipping Quantity Available

Same as Quantity Available but converted in the current SKU Quantity Unit quantity (quantityAvailable / factorToBase).

Shipping Quantity Needed

Same as Quantity Needed but converted in the current SKU Quantity Unit quantity (quantityNeeded / factorToBase).

Shipping Quantity Available For Retrieval

Same as Quantity Available for Retrieval but converted in the current SKU Quantity Unit quantity (quantityAvailableForRetrieval / factorToBase)

Quantity In Transport

Quantity on the way to the area. This includes the complete quantity of the current SKU with the current SKU-QuantityUnit in the Multishuttle buffer.

Quantity In Transport Aisle

Quantity on the way to the current aisle. Because the concrete aisle is selected in front of the aisle pair this information is available up to the pusher and for any inter aisle transfer.

Primary Key/SKU

Primary Key of the current SKU.

Primary Key/Quantity Unit

Primary Key of the current SKU-QuantityUnit.

Primary Key/Aisle

Primary Key of the current SKU aisle.
