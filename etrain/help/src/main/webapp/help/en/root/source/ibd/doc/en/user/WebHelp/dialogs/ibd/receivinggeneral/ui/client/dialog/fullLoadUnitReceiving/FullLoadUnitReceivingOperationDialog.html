Full Load Unit Receiving

Purpose

This screen provides the required functions for Receiving Full Load Units in automated or manual Areas ordered to given Advices and yet provide Advice Line Load Units. The operator can create and store advised Inventory by scanning advised Load Units via barcode. The advised Load Units are expected to contain only Inventory of one Stock Keeping Unit (SKU).

      
Functions
  • Confirm Receiving of load unit data and create the Load Unit and specified inventory in the system.
  • Complete Receiving process of the Load Unit related Advice - Advice status proceeds to "finished".
  • Complete Receiving process of the Load Unit related Advice Line - Advice Line status proceeds to "complete".
  • Reprint Advice Line LU Label.
  • Modify Load Unit Data (Load Unit Type, height classification, default destination and weight).
  • Send Load Unit to Clarification Station.

For a detailed description, see "Full Load Unit Receiving Use Case".

      
Overview

Frame

Description

Load Unit

Main data of scanned/entered Load Unit. Data can be changed on demand.

Load Unit Details

All data for current receiving step as Client, SKU, Quantity Unit, batch, best before date and delivery text.

Load Unit Overview

All data for current Receiving step as Advice ID, Advice Line ID, number of LUs in Advice, Supplier and Inventory Hold.

Actions

All available functions for current step.
     
Screen

      
Fields

Field

Description

Load Unit Panel

Barcode Scan

Barcode can be scanned or entered. Valid Load Unit ID must be included..

 INFO 

 Must comply with GS1 (Global Standard 1) standards. Values are project specific.

Load Unit Scan

Load Unit of the scanned barcode.

Load Unit Type

Load Unit Type of the announced Load Unit.

Default Destination

Default destination for announced Inventory or Load Units. Can be a Facility, Aisle, Zone or Location. Must be interpreted as a proposal. Can be manually changed according to the Area Accesses related to the Location of the current Workstation.

Height Classification

Information about Load Unit height and to confirm by the operator. If various height classes, operator can modify this information.

Weight

Weight of Load Unit, in kg.

Mix Load Unit

If Load Unit contains Inventory of more than one SKU.

 INFO 

 Only for information. Possible to receive multiple SKUs on one pallet while Full Pallet Receiving process as a Load Unit Stack. If Load Unit Stack includes only one SKU, this flag is also true.

Load Unit Overview

Client

Client within that new Inventory shall be managed.

Batch

Batch in which Inventory was produced.

Quantity

Quantity related to the Receiving Quantity Unit on the advised Inventory on the scanned Load Unit.

Receiving Quantity Unit

Quantity unit of the received quantity.

Best Before Date

Date before it is recommended to consume Inventory. Used as FIFO criterion, when goods are selected for an Order.

SKU

Stock Keeping Unit ID of announced Inventory, specified by the Advice Line.

Best Before Date

Best Before Date of the announced Inventory, specified by the Advice Line.

Description

Text description of the SKU.

Delivery Text

Usually contains handling instructions for announced Inventory. Delivery text of the Advice displays. If defined in the Advice Line, this text displays.

Load Unit Details

Advice

Identifies an Advice uniquely within a client and an Advice type.

Advice Line

Identifies the Advice Line uniquely within an Advice.

Number of Load Units

Number of pallets announced with this Advice. Calculated once, after the Advice is completely entered by a user or a host system.

Supplier

Supplier ID that delivered the goods.

Inventory Hold

If the advised inventory must be set on hold during creation and for what reason. If null, inventory does not need to be set on hold. A use case for this attribute is the SAP quality state.
     
Special Buttons

For a general operational flow, see the "Full Load Unit Receiving Use Case".

Button

Description

Key: Enter

Confirm current screen state. E.g. loading of Load Unit or Complete Receiving. In the Modify Data state, modified data is persisted.

Key: Escape

Cancel current screen state. E.g. loaded data is deleted and previous state displays again.

Key: F2

Toggle the mode for Scan between Load Unit and Barcode.

Key: F2

Complete receiving process of the Load Unit related Advice - Advice status proceeds to "finished".

Key: F3

Complete receiving process of the Load Unit related Advice Line - Advice Line status proceeds to "complete".

Key: F4

Release Load Unit data (displayed in upper frame "Load Unit") for modification. Changeable fields:

  • Load Unit Type
  • Default Destination
  • Height Classification
  • Weight (kg)

Key: F5

Change Load Unit Type by selecting a possible Load Unit Type.

Key: F6

Change default destination by selecting a possible Default Destination.

Key: F7

Change Height Classification by selecting a possible height classification.

Key: F8

Reject Load Unit. System creates a Transport Request to the Clarification Location and sends it to the underlying transport system (MFC).

Necessary if any problems with Inventory or Load Unit data occurred and must be clarified at the Clarification Station.

Key: F9

Reprint Advice Line LU Label.

 INFO 

Enabled only if the Product Code of the SKU is available.
