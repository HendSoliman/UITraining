Short

Purpose

"Short" appears after this function was selected by operator from dialog state "Picking".
It provides the functionality to confirm inventory shortage due to several configured shortage reasons.

 INFO 

In case of any shortage the shorted Pick will be re-planned by Order Fulfillment in the short term and, if possible, reassigned to the current Pick Walk.
Replenishment of the shorted Pick Face is done separately based on the appropriate Replenishment configuration.
     
Screen

      
Fields

Field

Description

SKU

The SKU id uniquely identifies a Stock Keeping Unit within a client.

Quantity Target

The target quantity to be picked into the Pick Container.

Quantity  Unit

The quantity unit of the pick operation. This will be the "Pick Quantity Unit" In case of exactly one Default Pick Quantity Unit is set for the Quantity Units of the SKU. Otherwise the Base Quantity Unit is displayed.

Put Location

The Alias represents the name of the Put Location on the Pick Cart for easy identification. The Alias is only textual information and will not be validated from the system.

Error Reason

The Confirmation Reason details the chosen function.

Quantity Current

The picked quantity into the Pick Container with same Quantity Unit.
     
Special Buttons

Function/Button

Description

Enter "* Quantity Current"

With launching this dialog state the field "Quantity Current" is focused for entering the picked quantity.

Change Confirmation Reason

key "F9"

Dialog for change of Confirmation Reason appears.

With cursor up and down in the reason list an alternative Confirmation Reason is selectable.

Change to dialog state "Info"

key "F11"

Dialog proceeds to "Pick Info", where the operator is able to view detailed information about current pick operation.

See details in Pick Info.

Cancel

key "ESC"

Cancel this dialog state and return to former state without confirming any changes.

Pick Confirmation as "short"

key "Enter":

The requested Pick operation is confirmed with inventory shortage due to displayed Confirmation Reason.

The confirmed quantity of Inventory Unit of required Source Load Unit is booked into the Pick Container and the Pick is booked and confirmed with Code 'short'.
