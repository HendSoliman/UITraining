Detail Dialog Order Line

Purpose

This screen opens when creating, viewing, or modifying an Order Line.

 INFO 

The displayed data on the first tab depends on the related Order Line criteria and could be either Inventory, Load Unit or Location.

Additional Text

      
Fields

Field

Description

Order ID

The Identification of the Order is only unique within a Client.

Order Line ID

The identification of the Order Line is unique within an Order and is usually represented by a running number.

Status

The status reflects the current execution state of the Order Line. Possible values are:

  • pending (The Order is waiting for release for fulfillment and all Order Lines proceed to this status)
  • released (The Order is released for fulfillment and all Order Lines proceed to this status)
  • active (The execution of the Order is already started; Full Load Unit Retrievals and/or Pick Orders are in progress and all Order Lines proceed to this status)
  • cancelled (The (single) Order Line has been cancelled without the defined execution)
  • complete (The execution of the single Order Line is finished and the Order Line is fully completed).

Criteria Used

Order Line criteria are important and relevant for inventory allocation processes using the Retrieval Strategies. Following criteria are possible to define the relevant objects in the allocation process:

  • Inventory Criteria (Requirements to Inventory aspects (e.g. SKU, batch, best before date, etc.) lead to selection of parts of Inventory Units or complete Load Units)
  • Load Unit Criteria (Requirements to specific and concrete Load Units, always lead to selection of complete Load Units)
  • Location Criteria (Requirements to specific and concrete Locations (e.g. the whole Facility, specific Areas, Aisles or concrete Storage Locations) always lead to selection of complete Load Units).

Inventory

SKU

SKU

Requested Stock Keeping Unit.

Description

Description of the SKU.
Batch

If required inventory must be of this specific Batch. If empty, it is not considered, i.e. can be any with respect to Retrieval Strategies.
Best Before Date

If required inventory must be of this specific Best Before Date. If empty, it is not considered (or by FiFo) i.e. can be any with respect to Retrieval Strategies.

Mark

Special Inventory Mark

If required inventory must be of this specific "Special Inventory Mark". If empty, only inventory of no "Special Inventory Mark" will be allocated. Predefined extendable values are:

  • inventory on commission
  • order inventory
Special Inventory Reference

Linked corresponding object Special Inventory Mark of the inventory. Conditions are logically related to the Special Inventory Mark.
Hold Reason ID

If the required inventory must be of this specific hold. If empty, only inventory of no holds will be allocated.

Quantity
Quantity Base Target

Requested quantity in base quantity units.
Base Unit Base quantity unit of the SKU. Represents the relevant quantity unit for the Quantity Base Target, Quantity Base Target Host, Quantity Base Dispo and Quantity Base Current.
Quantity Packaging

The Quantity of requested Packaging Units, if a specific Packaging Level is requested.
Packaging Level

The Requested Packaging Level. If not requested, any Packaging Level can be selected
Factor To Base

Factor from the SKU's base quantity unit the target quantity unit (informational purposes only. Set by the Host). Actual conversion factor defined by SKU Master data.
Quantity Base Dispo

Already allocated (disposed or reserved) quantity. Only direct allocation of concrete Inventory Units are considered by this quantity. Aggregated Inventory reservations are not totaled here. Directly relates to Inventory Unit reservations, mentioned in Dialog Inventory Unit Administration.
Quantity Base Current

Already fulfilled quantity and is set with status progress of the Order Line to "active".

Quantity Base To Be Cubed

This attribute is required for Order Cubing and represents the yet remaining quantity to be cubed for this Order Line.

Number of Load Units Base

The number of load units required for this inventory criteria.

 INFO 

Only used if the inventory rounding method is "number of load units" or "at most number of load units".

Number of Load Units Dispo

The number of load units currently reserved for this inventory criteria.

 INFO 

Only used if the inventory rounding method is "number of load units" or "at most number of load units".

Tolerance

Overdelivery Tolerance

Factor, in percent, up to which overdelivery is allowed.
Example: Required target quantity is 100. Overdelivery tolerance is 8. Delivery up to 108 is allowed.
Underdelivery Tolerance

Factor, in percent, up to which underdelivery is allowed. If set to 100%, an Order Line is set to completed even nothing was delivered.

 INFO 

If the underdelivery tolerance of any Order Line will be violated while Order Release the further actions on the Order will be configured by the configuration topic "orp-orderfulfillment:OrderFulfillmentClientOrderType.underdeliveryAction". Possible actions are:

  • DEFAULT (Cancel the underdelivered Order Lines only)
  • PAUSE (Set the Order to "paused" and stop further processing until either manual cancellation or further order release. Pause functionality is not supported during the replacement reservation process.)
  • CANCEL (Cancel the Order)
  • NONE (Proceed all actions on the Order Line with yet available quantity and ignore the underdelivered quantity. Further inventory reservation is possible until the status is not yet complete.)
Unlimited Overdelivery Allowed

If enabled, overdelivery is allowed without any limit. Useful if only Full Load Units are requested, regardless of quantity.
Request Whole LUs

If enabled, only Full Load Unit Retrievals are allowed for this Order Line. Picking operations are not initiated. If set, full load unit retrievals are selected from all disposition zones regardless of the flag full load unit retrieval in the disposition zone.
Rounding Method

The Inventory Rounding Method of the created Replenishment Order Lines. How Inventory Units will be rounded during reservation:

  • none (no special rounding method)
  • at least full inventory unit (if the quantity to be reserved is completely available on the first matching Inventory Unit, the complete free quantity on the Inventory Unit is reserved. If this would exceed the over delivery tolerance, the maximum quantity within the overdelivery tolerance is reserved)
  • complete packaging unit (only complete packaging units are reserved within the boundaries of overdelivery and underdelivery tolerance. The quantity to be reserved on Inventory Units with a packaging level is rounded up to an integer quantity related to the packaging level. The quantity to be reserved on Inventory Units without a packaging level is rounded up to an integer quantity related to the Base Quantity Unit of the SKU. If the rounded up quantity exceeds over delivery tolerance, there is no rounding)
  • whole layers (only complete layer (on the pallet) are reserved within the boundaries of overdelivery and underdelivery tolerance. The quantity to be reserved on Inventory Units is rounded up to the next full number that fills a full layer of the SKU. If the rounded up quantity exceeds over delivery tolerance, there is no rounding)
  • number of load units (if possible, exactly the specified number of load units are reserved)
  • at most number of load units (the specified quantity base target is reserved but not more than the the specified number of load units).
Minimum Quantity Per LU The minimum quantity of the specified inventory that must be reserved per load unit. Inventory is not selected, if there is less available quantity on one base load unit.
Minimum Rest Quantity The Minimum Rest Quantity of the created Replenishment Order Lines. If less than the Minimum Rest Quantity would remain free after the reservation on a source inventory unit the complete free quantity is reserved within the boundaries of over delivery and under delivery tolerance.

Load Unit

Load Unit ID

Unique ID of the requested Load Unit.

Location

Unique ID of the requested Location. Can be a virtual Location representing the Facility, an Area, Aisle or Level or a concrete Bin.

Order Line

Start Time

Actual start time of fulfillment operations.

Finish Time

Actual finish time of fulfillment operations.

Hold Option

Set by the Host. For informational purposes only. (can be project specific)

  • no hold
  • hold when processing (Quality Hold is set within the Host while processing the Order)
  • hold before processing (Quality Hold is set within the Host before processing the Order)

Aisle for Palletizing

The Multishuttle Buffer Aisle to which the Order Line is assigned for the palletizing process (AMCAP).

Pallet Planning Running

This field is checked, if pallet planning for this order line is active.

Variation
Cancel Reason

Detailed reason for Order Line cancellation:

  • none (not cancelled)
  • underdelivery tolerance (cancelled due to violated underdelivery tolerances)
  • overdelivery tolerance (cancelled due to violated overdelivery tolerances)
  • operator (cancelled manually by an operator)
  • split lines cancelled (cancelled because related split Order Lines were cancelled)
  • no destination (cancelled because no improved destination was found) (project specific use only)
  • by host (cancelled by the Host)
  • no reservation made (canceled because no reservation has been made)
  • fulfillment canceled (canceled because fulfillment has been canceled).

ERP
Host Order Type

Related Order Type within the Host. Relevant for this Order Line. Set by the Host:

  • customer order.
Host Move Type

Movement type to be used by Logistics Manager. Defined by the Host. Values restricted to those configured in the screen Inventory Movement Types.
Host Move Reason

Related reason of the movement within the Host. Set by the Host. Values restricted to those configured in the screen Inventory Movement Reasons.

Additional Text

ID

Unique identifier of the text. Specifies the use of the text. Thus, qualification of the type of text.

Text

Actual text of the Additional Text object. Can be project specific, e.g. label printing.
