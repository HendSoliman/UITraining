Putaway Configuration

Introduction

The Putaway component provides an overall default Putaway strategy with three levels of selectors which are affected according to the configuration.

  • Default Putaway Strategy - Basic Conditions
  • Default Putaway Strategy - Basic Process Flow
  • Default Area Selector
  • Default Aisle Selector
  • Default Bin Selector.

 INFO 

For specific strategy usage, see Process of Strategy Selection. 

 INFO 

For Strategy descriptions and use of specific global configuration topics, see the Warehouse Administration Guide.

 INFO 

For Multishuttle Areas specific default strategies are affected. The decision for the appropriate strategy depends on the configuration topic "cor-strategy:Putaway.isStrategyMultishuttle" for each Area.

 INFO 

A Maximum Fill Level of Aisles/Levels can be set via the configuration topic "cor-putaway:Putaway.maximumFillLevel" per Area. The default is set to 98%. The Default Strategies for choosing an aisle in classic Warehouses and the default strategies for choosing a bin in Multishuttles will then ignore aisles/levels with a fill Level higher or equal to this value, if the current Location of the given load unit is not within this aisle/level.

 INFO 

To optimize bin usage in double deep Areas, use "SwitchDoubleDeepPutawayStrategyDemon". This does the following for every area with enabled configuration topics "cor-putaway:StrategySwitch.isPreferPartialBins", ".thresholdPreferPartialBins" and ".thresholdPreferEmptyBins":

  • If the number of empty bins in an aisle is below ThresholdPreferPartialBins, the strategy is changed to "Coordinates Ascending", so that partially occupied Bins are chosen first.
  • If the number of empty bins in an aisle is above ThresholdPreferEmptyBins, the strategy is changed to "Coordinates Descending, so that empty Bins are chosen first.

This is valid for Multishuttle areas as well!
     
Default Putaway Strategy - Basic Conditions

The default Putaway Strategy ensures that a Destination Location is only selected, if all of the following basic conditions are fulfilled:

  • Aisle, Level or Location is not locked for input.
    • Except: Shuffle Move Putaway is allowed in levels locked for input since the Load Unit is just located inside the level. Find lock status details in Subgroup Detail Dialog.
  • Facility, Area, Aisle and Location are completely configured.
  • Area Access entry exists from the current Area of the Load Unit to the Area of the Destination Location for the Load Unit Type of the Load Unit to be stored.
  • Height classification of the Load Unit is smaller or equal than height classification of the Location.
  • Load Unit to be stored would not exceed the weight limit of the Destination Location.
  • Load Unit to be stored would not exceed the limits defined by the Weight Limiter defined for the Location.
  • Location has at least one free position for the appropriate Load Unit Type.
  • In case of dynamic slotting an appropriate Slot is available (see Slotting Definitions).

 INFO 

A short lock (technical lock) of a Location is not considered, but consideration could be enabled through the configuration topic "cor-putaway:Putaway.considerShortLockOnDisposition".
     
Default Putaway Strategy - Basic Process Flow

The default Putaway Strategy performs the following steps:

  1. Dedicated Pick Face Selection (only for classic strategies)
    Dedicated Pick Faces are Locations configured to be used for Picking the SKU assigned to the Location. If the Load Unit to be stored contains only one SKU and the SKU is assigned to one or more Dedicated Pick Faces and one of the Dedicated Pick Faces can be selected according to the basic conditions (see above), a position on the Dedicated Pick Faces is selected and reserved. The configuration of Dedicated Pick Faces could be done with Dialog Dedicated Pick Faces.
    The configuration topic "cor-putaway:Putaway.useDedicatedPickFaces" must be enabled.
  2. Preferred Destination Selection (only for classic strategies)
    If a matching Preferred Destination exists and can be selected according to the basic conditions (see above), a position on this Location is selected and reserved. The configuration of Preferred Destinations could be done with Dialog Preferred Destination Administration.
    The configuration topic "cor-putaway:Putaway.usePreferredDestination" must be enabled.
  3. Intermediate or Destination Location selection using default Area, Aisle or Bin Selector
    The type of Selector to use is derived from the current destination (as set within the active Transport Request) of the Load Unit:

     INFO 

    For details, see Process of Strategy Selection.
  4. If an Area was chosen and before an aisle is selected, it checks if a Hot Zone Configuration exists. If yes, the number of Load Units on Hot Zone Locations in this Area with the same Discriminating Key is checked against the Load Unit Limit of the Hot Zone Configuration.
    If the number is smaller than the limit, the current Load Unit gets the flag 'Hot'. This means it shall be stored on a Hot Zone Location.

     INFO 

    Putaway Strategies that must be active to take effect:

    Classic: Putaway Bin Strategy 'HOT_ZONE'

    Multishuttle: Aisle Strategy 'VELOCITY_CLASSIFICATION', Level Strategy 'VELOCITY_CLASSIFICATION'
     
Default Area Selector

As described above in the basic process flow, the configured Area Selection executes as follows, if required:

  • Appropriate database execution selects with only use of configured Putaway Area Strategies, which are related to Areas or Aisles only.
  • For each Aisle of the result set, (ordered by the configured statement) a possible Location is selected.
  • If the required granularity is bin, this selection is ordered as configured with consideration of the strategies related to bins only. The result is the preferred bin.
  • If the required granularity is not bin, no sorting occurs and only the existence of a possible Location is checked. The result is the preferred Area.
  • Possible Locations are checked against the possible local weight limits of the Location.

Putaway Area strategies can be assigned per Facility. The configured sequence provides the priority/sorting criteria of the applied strategies.

Default Aisle Selector

As described above in the basic process flow, the configured Aisle Selection executes as follows, if required:

  • Appropriate database execution selects with only use of configured Putaway Aisle Strategies, which are related to Aisles only.
  • If the Load Unit is already in an Aisle that within the Destination Area and the Source Area is a different Area than the Destination Area (no Area internal relocation) and the granularity of the request is bin, this Aisle is selected.
  • All aisles, but the aisle of the Load Unit´s current Location, having a fill level higher than or equal to the configuration topic "cor-putaway:Putaway.maxFillLevel" is ignored. The default value of this parameter is 98%.
  • For each Aisle of the result set, (ordered by the configured statement) a possible Location is selected
  • If the required granularity is bin, this selection is ordered as configured with consideration of the strategies related to bins only. The result is the preferred bin.
  • If the required granularity is not bin, no sorting occurs and only the existence of a possible Location is checked. The result is the preferred Aisle.
  • Possible Locations are checked against the possible local weight limits of the Location.

Putaway Aisle Strategies can be assigned per Area. The configured sequence provides the priority/sorting criteria of the applied strategies.

Default Bin Selector

As described above in the basic process flow, the configured Bin Selection executes as follows, if required:

  • Appropriate database execution selects for configured Putaway Bin Strategies within a given Aisle (ordered by the configured statement)
  • Possible Locations are checked against the possible local weight limits of the Location.

Putaway Bin Strategies can be assigned per Area. The configured sequence provides the priority/sorting criteria of the applied strategies.

Process of Strategy Selection

The following preconditions exist for the respective strategies:

  • If the granularity of the current destination is Aisle or bin and the granularity of the request is Aisle, set the current destination back to the Destination Area and use the Aisle Selector.
  • When the current destination of the Load Unit to store is not defined yet or is the Facility, the Area Selector is used to determine a virtual intermediate destination. If a valid Area is found, it is set for the intermediate destination of the Load Unit to store.
  • When an Area is the current destination of a Load Unit to store, the Aisle Selector is used next to determine the next virtual intermediate destination. Once a valid intermediate destination is found - in this case an Aisle -, it is set as the virtual intermediate destination of the Load Unit to store.
  • When an Aisle is the current destination of the Load Unit to store, the Bin Selector is used to determine a (final) bin. If a valid destination bin was found, it is set as current destination of the Load Unit to store.
