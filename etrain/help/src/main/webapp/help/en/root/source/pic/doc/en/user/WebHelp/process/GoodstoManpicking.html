Basic Processes Goods-to-Person Picking

Introduction

The Pick Controller for Goods-to-Person Picking includes a specific collaboration with transport management for supply with source and empty pallets and pallet release at a stationary Pickstation when a pick is complete. Using containers, cases or cartons instead of pallets differs mainly with respect to the transport handling. Picking is done at Pickstations which are Workstations for picking purposes that contain all properties to fulfill Goods-to-Person requirements and represent their own workload of assigned Pick Missions.

      
Configuration

In addition to the basic properties of a Workstation the following are of specific relevance for picking purposes of a Pickstation:

  • Capabilities
  • Zones
  • Locations.

For details on creating a full Pickstation configuration, see the Workstations screen.

      
Workload Management

The collection of assigned and sequenced workload of a Pickstation is represented by the Workstation Queue Entries which are occupied by Pick Missions.
The number of Queue Entries is defined by the configured queue limit of the Workstation and will be adapted automatically when changed.

The assignment of Pick Missions is done in defined groups up to a configured limit (for details see the relevant attributes in the Detail Dialog Workstation). The number of Pick Missions within a group will not exceed the number of available and enabled Put Locations.

An overview of feeding the Pickstations by populating the Queue is described in the use case Pick Controller Workload. The donor request (outbound transport of the Source Load Units) is done if the configured number of assigned entries are available. All required donors will be requested in a single sequenced transport group. If the Request Threshold Counter of the Workstation has reached the limit for failed attempts (configuration topic "cor-workstation:Workstation.requestSizeThreshold") the request is initiated anyway for less then configured entries.

If Block Management is enabled and an already assigned Pick Mission gets blocked all blocked Pick Missions will be immediately removed from the Queue and directly replaced if possible.

      
Picking Operation

The picking operation is done at stationary Pickstations with specific operational screens dependant on the physical type of the classic Pickstation or the special Rapid Pick stations. The handling is described in detail in the different use case descriptions for Goods-to-Person Picking.

The picking screens provide all functionality for picking purposes at a Goods-to-Person Pickstation for all types of Load Units. Features include detailed instructions, pick operation information and different types of pick confirmation. Reverse picking execution and the splitting of a pick operation are also provided. It also supports external trigger for automatic process flow with signaling of the conveyed Load Units and external signals for pick confirmations (e.g. by additional Put-to-Light application).
