Business Journal Detail Screen

Fields

Field

Description
Primary Key Primary Key for Business Journal entry.

Object

Key of the object, e.g. LoadUnit Id, SKU Id.
The key is the unique key of the object carrying semantics.

Message
Message Key Key of the message which uniquely identifies a message with-in a bundle.
All keys within a bundle have the same prefix (4 characters) and a 4 digits numeric value, separated by a hyphen (e.g. INVJ-1234 or APPL-0600)

Text

User language dependant message of the Business Journal entry.

Journal Category

Identify the category which refers to the specialization of the Business Journal entry.
For example Transport Journal, Inventory Journal, Receiving Journal, or other.

Log Type

The LogType of journal entry.
The LogType is used to distinguish the different business processes or activities for which journal entries are created.

Object Type Optional type of object to which the entry refers.
Types may be: LoadUnit, TransportOrder, BatchProcess, or other.
Typically, the object type is determined by the type of the journal entry.
Types are typically entity names.
Within the UID, internationalization is required for the object type.
Task Id Task to which the entry is related.
Parent Id of the parent journal entry.
Finish Time Finish Time of business process to which the entry refers.
This is not the time when the journal entry was made but the completion time of the business process to which the entry refers!
The time shall be specified as precise as possible.
Originator Class Name of the class which has implemented the business process to which the entry refers.

Terminal Id

The number of the terminal at which the revision was created.

Trace Pointer

The Trace Pointer is an (embedded) data structure which allows an easy identification of the trace file location which corresponds to a certain business transaction.
It encompasses the following elements:
 - the sifting key of the trace file which determines the trace subdirectory
 - the incident id which can be used to locate the trace output of a business transaction within a trace file
 - the server name being relevant in a MSE
