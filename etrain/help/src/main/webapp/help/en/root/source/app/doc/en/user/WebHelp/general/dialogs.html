Dialogs and Sub dialogs

Purpose

This chapter describes the general structure and layout of grid based dialog and sub dialogs. In addition to grid based dialogs, Dematic iQ® also encompasses graphical dialogs which are not grid based. These dialog are described in the context of the business use case the refer to.

Definition of notions:

  • Dialog
    A dialog is a screen window which resides in the Dialog Area of UI frame. In general, dialogs are opened via the Accordion Menu. Often the central layout element of a dialog is one or more tabular presentation of datasets.
    Dialogs are non-modal in the sense that there can be more than one open dialog in the dialog area at the same time.
  • Sub dialogs
    Sub dialogs are pop-up screen windows which are open from dialogs or by means of navigation. They consist of several form based panels which presents all fields of a dataset.
    Sub dialogs are always modal, which means that the user can only perform actions which are offered by the sub dialog itself while it is open. The DC Director UI does not perform any non-modal sub dialogs.

Standard buttons

The standard buttons offers are a set of actions which apply to all data grids. Depending on business context the exact set of standard actions may vary:

  • Refresh ()
    A refresh triggers new retrieval of data from the database. If filters are active, they are applied when retrieving the data.
    Dialogs do not refresh them automatically. This has to be done explicitly by the user.
    The pull down menu of the refresh button gives access to predefined filter sets and the filter history.
  • Printing / Export ()
    The printing of datasets is achieved by generating a pdf file which is subsequently downloaded to the browser. Hereby, the "What You See Is What You Get" (WYSIWYG) principle is applied, i.e. the current column layout is used.
    This pdf file can then be printed by making use of standard browser functionality. The exact procedure how to do this depends on the browser being used and the individual settings of the user. INFO Please ensure that there is no popup-blocker active for Dematic iQ® URL. Otherwise the downloaded pdf file will not show up.
    Before the pdf file is generated the user can decide whether he wants to print the data as they are now being displayed in the data grid or whether he wants have all data being printed. In the latter case a new retrieval of data from the database is necessary which might yield a different result compared to what is currently displayed.
    A special form of printing is the export of data into an external files. The following formats are supported: Excel, csv, ODF, XML, RTF. This functionality gives the user the possibility to process data outside Dematic iQ®.
    Layout oriented formats (RTF, PDF) create pages in landscape format with additional headers and footers, where all column widths display according to the relative size in the table. Data centric formats (XLS, ODT, XML, CSV, HTML) export only the column headers and the data.
  • Modify () / View ()
    A subdialog opens which displays the field values of the selected dataset in a tabular form. Depending on his access rights, the user has the possibility to modify the values or just to view them.
  • Create ()
    A sub dialog opens which allows the user to fill a tabular form with the field value of new dataset. This action requires no dataset to be selected.
  • Duplicate ()
    Similar to the new action a sub dialog open. The entry fields of the form are already populated with the values of the selected data set. This gives the user the advantage that he has to modify only these values which differ from the selected data set.
  • Delete ()
    Deletes the selected datasets.
  • TracePointer ()
    Open filtered Trace Location in browser.

Dialog specific buttons

A dialog may also offer functionality which is specific for its purpose. In the example shown above, the assignment from user to groups is such a dialog specific action.

Dialog specific buttons can be displayed as:

  • icons

  • icons with text

  • pure text

In case when there is not enough space available, logically related text button are group together in a pull down menu. 

Status of the tabular data
  • Available rows displays the number of rows which are currently available in the gird.
    A > in the value indicates that there are more data available in the database which will be displayed when the user scrolls down. A click on the number determines the full number of rows in the database.
    To limit resource consumption, at most 1000 rows are displayed. The value 1001 indicates that there are still data in the database which cannot be displayed anymore due to resource limits. In this case it is recommended to add a new filter criterion to reduce the number of selected rows.
  • The last refresh indicates the age of the date being currently displayed.  

 

In general, a dialog is initially filled with data when it is open. However, especially when there is a huge amount of data in a table it makes more sense to first specify certain filter criteria which limit the data set before retrieving data. In this case, the message "Set filter and click refresh button!" and the refresh icon () is displayed in the grid.

Heading The heading gives a short description of the purpose of the modal dialog.
At the right side there buttons to close () the sub dialog and to open the help page () explaining the sub dialog. The sub dialog can also be closed by pressing the ESC key.

Id

Shows the key fields of the displayed dataset.

Tabbed Pane

The tabbed panes shows the fields of the dataset. If a data set encompasses a lot of fields, several tabs are used. Each of the them groups logically related fields together. The exact number of tabs being shown depends of the concrete dataset.

The last tab is always entitled "History". It contains information when and by whom the dataset was created and lastly modified.

Buttonbar

 On the right hand side there is "OK" button which triggers the action (e.g. storing modified data). Similarity, the "Cancel" button aborts the operation. All modifications are discarded and the dialog is closed.
On the left hand side there is a dropdown menu, which encompasses all actions of the toolbar of the main dialog which are applicable to the currently displayed dataset. This allows the triggering of action directly from the sub dialog, i.e. without having to go back main dialog. This is especially useful if the sub dialog has been open by means of navigation where there is no direct correlation to a main dialog.

 

Mandatory Fields

Mandatory fields are fields which must be filled in before the dataset can be stored.

Mandatory fields which still needs to be filled in are marked with an orange asterisk () in front of the label. After data has been entered the color of the asterisk changes to black.

The same indication is contained in the heading of tabbed pane which contains mandatory fields. If at least one mandatory field is encompassed in the tab which has not yet been filled in, an orange asterisk is shown.

 

Picker

For simple data fields (e.g. text of numeric) values can be typed in directly. For more complex data fields "picker" assist the user enter valid data efficiently.

The picker is a small symbol at the right hand side of an entry field. The following variants are supported:

  • ComboBoxes (): a combo box opens from which the user can select a single value.
  • Date/time fields (): a calendar control opens from which the user can choose date and/or time.
  • References to datasets (): a reference sub dialog opens which displays a table having the (logical) key fields and the primary key as columns. The user can select a value by selecting a row from the grid. Filter criteria can be used to reduce the displayed amount of data sets as usual.

The shape of the picker may vary dynamically depending on the amount of datasets can choose of. For a small amount of data a combo box is more efficient, whereas a for a large set an reference sub dialog is more appropriate.
