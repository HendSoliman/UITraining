Housekeeping Configuration

Overview

Housekeeping is used to optimize storage distribution for Load Units. The Housekeeping Configuration defines mislocation issues to be improved and/or eliminated by housekeeping processes for the desired warehouse Areas. A mislocation issue specifies where a Load Unit is not optimally stored (e.g. a low Load Unit on a high storage bin or not matching velocity classifications between Load Unit and Storage Location).

For housekeeping purposes, the sum of mislocation levels that apply to a load unit is set in the Load Unit. Load Units with the highest values are optimized first (i.e. relocated).

 INFO 

For configuration purposes refer to the related topic in the Warehouse Administration Guide.

Predefined Mislocation Issue Views

Mislocation Issue View

Description

COR_OPT_HSK_HEIGHT_V

Load Units with a "lower" height classification are located on "higher" Locations.

COR_OPT_HSK_SKU_V

Load Units of inventory with any velocity classification are located on "faster" Locations.

COR_OPT_HSK_SKU_M_V

Load Units of inventory with velocity classification "B" are located on "faster" Locations.

COR_OPT_HSK_SKU_S_V

Load Units of inventory with velocity classification "C" are located on "faster" Locations.

COR_OPT_HSK_BIN_M_V

Locations of velocity classification "B" are occupied with Load Units having Inventory with a "faster" velocity classification.

COR_OPT_HSK_BIN_S_V

Locations of velocity classification "C" are occupied with Load Units having Inventory with a "faster" velocity classification.

 

 INFO 

 For Load Units with multiple SKUs, the velocity classification of the "slowest" SKU  is used to calculate the mislocation level of the Load Unit.
