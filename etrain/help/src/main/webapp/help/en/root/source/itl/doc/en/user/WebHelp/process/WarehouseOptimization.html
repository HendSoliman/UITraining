Warehouse Optimization

Overview

The Logistics Manager supports different warehouse optimizations and Housekeeping functionalities. There are two main fields for optimization support which are provided by different components:

  • Housekeeping (Optimization): Optimizes storage distribution for Load Units that are not optimally stored (e.g. regarding height or velocity classifications).
  • Velocity Class Analysis: Calculates Stock Keeping Unit velocity classes based on each single SKU access with various configuration parameters.
      
Housekeeping

The Housekeeping Configuration defines mislocation issues to be improved and/or eliminated by housekeeping processes for desired Areas of the warehouse.
A mislocation issue specifies if a Load Unit is not optimally stored (e.g. low Load Unit on a high storage bin or not matching velocity classifications between Load Unit and Storage Location).
The sum of mislocation levels that apply to a Load Unit is set in the Load Unit. Load Units with the highest values are optimized first (i.e. relocated).
The calculation of the mislocation level of all stored Load Units occurs periodically using an appropriate batch program only. It is not calculated directly with the storage operation of a Load Unit.

For required relocation transports, Housekeeping periodically creates Transport Requests for the Load Units with the highest mislocation levels using an appropriate batch program. Transport Request execution can be started automatically or by manual release based on the related configuration.

 INFO 

Creating and executing these Housekeeping transports is done without usage of the Order Management and relatged Housekeeping Orders.

Configurations for execution are described in the Execution of Housekeeping.
