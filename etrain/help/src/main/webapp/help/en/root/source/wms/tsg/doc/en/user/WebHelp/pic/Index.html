Introduction

The Pick Controller subsystem (PIC) is the central task execution subsystem within the DCD concerning all picking processes. The communication with Logistics Manager proceeds in general through the following two channels:

  • LMS-ORF (Order Fulfillment) provides Pick Requests to the Pick Controller
  • The Pick Controller provides Pick Confirmations and other process information to LMS-ORF.

The architecture allows not only for a single Pick Controller but for multiple Pick Controllers to be used in parallel, depending on specific demands. Dependant on the Picking Zone the Order Fulfillment subsystem (ORF) will address the adequate Pick Controller and submit the provided picking workload as Pick Orders to it.

Two main different Pick Controller Types are supported:

  • Person-to-Goods Picking (Route based Picking) and
  • Goods-to-Person Picking.

 

A short list of the basic Pick Controller Responsibilities could be given as follows:

  • Planning, Optimization and Execution of Picking
  • Equipment Configuration and Communication with Picking equipment
  • Signaling of Replenishment Requests and execution of Perpetual Stocktaking for Pick Faces or Source Load Units
  • Handling and delivery of Pick Container

 

For a very first introduction into Pick Controller arena refer to "Overview of Subsystem PIC".
