Inventory Hold Assignment

Purpose

On this screen, operators can manage locks on Inventory Units for:

  • Single Inventory
  • Whole Inventory of a SKU
  • Whole Inventory of a SKU and Batch

Text displays in red when a hold is available. Holds and comments only display when an option is selected.

If an assignment is changed, other options are no longer available (Radio buttons are disabled).

 INFO 

For configuration purposes refer to the related topic in the Warehouse Administration Guide.
     
Screen

      
Fields

Field

Description

Hold Level

Hold levels:

  • Only inventory unit > Only the selected Inventory Unit gets the selected hold.
  • SKU > All Inventory Units of the selected SKU gets the selected hold.
  • SKU and batch > All Inventory Units of the selected SKU and batch gets the selected hold.

All hold levels can be assigned concurrently and will not affect each other if releasing one hold.

SKU

SKU of the selected Inventory Unit.

Batch

Batch of the selected Inventory Unit.

Comment

Hold comment. Comments cannot be edited after a hold is saved.

Hold Reasons

Available Holds

All available holds, except assigned holds.

 INFO 

Hold Reasons must be applicable by user.

Assigned Holds

 Already assigned holds.
