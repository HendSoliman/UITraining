Detail Dialog Full Load Unit Retrieval / Pick Container / Dispatch Load Unit

Purpose

This screen opens when viewing the selected data.

      
Screen

Reference

Load

      
Fields

For field descriptions on the "Reference" tab, see Detail Dialog Order Line / Pick Order Line.

Field

Description

Full Load Unit Retrieval 
Load Unit ID

Unique ID of the Load Unit to be moved by this Full Load Unit Retrieval. 
Load Unit Type

Load Unit Type of the related Load Unit. 
Status

Current execution state of the Full Load Unit Retrieval:

  • pending (Full Load Unit Retrieval is awaiting release for fulfillment)
  • released (Related Transport Request of the Full Load Unit Retrieval was released)
  • complete (Related Transport Request of the Full Load Unit Retrieval was finished and the Load Unit was delivered to the required destination)
  • cancelled (Full Load Unit Retrieval was cancelled without the defined execution and the Load Unit is no longer requested).
Source Location

Source Location of the related transport. 
Destination Location

Final Destination Location of the related transport. 
Completion Time 

Time when the Full Load Unit Retrieval status switched to completed or cancelled. 

Pick Container 
Load Unit ID

Unique Load Unit ID which is related to this Pick Container. Also represents the Pick Container ID. 
Load Unit Type

Load Unit Type of the related Load Unit. 

Status 

Current execution state of the Pick Container:

  • no pick order (Pick Container is not assigned to a Pick Order yet) (project specific use only)
  • picking (Pick Container is currently used for Picking processes)
  • picked (Pick operations are finished and the Pick Container is scheduled for transport to the Destination Location)
  • complete (Pick Container arrived at the Destination Location of the related Pick Order Line).

Dispatch Load Unit 
Load Unit ID

Unique Load Unit ID which contains Order related Inventory. A Dispatch Load Unit is created during a Shipping Order Fulfillment process; usually when the Pick Container is created or during creation of Full Load Unit Retrievals and is also related to an existing Load Unit.
Dispatch Load Unit Type

Type of Dispatch Load Unit. See Dispatch Load Unit Type.
Status

Current execution state of the Dispatch Load Unit:

  • planned (Dispatch Load Unit is only planned and not yet used for the Order Line)
  • available (Dispatch Load Unit is just created and available but yet unused)
  • used (Dispatch Load Unit is used and assigned to an Order)
  • picked (Picking into the Dispatch Load Unit is finished)
  • packed (Packing of the Dispatch Load Unit is finished)
  • staged (Dispatch Load Unit has been staged)
  • loaded (Dispatch Load Unit has been loaded)
  • shipped (Shipment of the Dispatch Load Unit is started)
  • completion intended (Completion of the Dispatch Load Unit during Order Cubing intended)
  • completion required (Completion of the Dispatch Load Unit during Order Cubing required)
  • cancelled (The related Order has been cancelled. Prevents loading).

Task

Full Load Unit Retrievals are handled as Tasks. Multiple Full Load Unit Retrievals could be grouped together by a Task (see Tasks).
