AMCAP Inventory Balancing

Purpose

Due to different reasons the initial distribution of products across aisles cannot exactly cover the required demand in these aisles. The reasons for this may be:

  • Excess quantities during replenishment which exceeded the actual order demand and could not be directly stored in aisles associated with an order.
  • Late order change or order cancellation
  • Consideration of urgent orders
  • Re-allocation of orders to other aisles because of blocked resources
  • Collection of items in an aisle for being palletized in order to purge the Multishuttle

In this case, the required stock distribution across the aisles can be archieved by moving from aisles having excess quantities to aisles with missing quantities.

Because the previous sub-process of replenishment is only considering the total orders demand and the total available stock in Multishuttle. Thus the items with excess quantities in another aisle than the assigned one is very important.

      
Demon and View

Demon

 Description Configuration

AMCAPInventoryBalancingDemon

Look over the all aisles of the Multishuttle buffer and use items where quantity excess in their aisle and send them to the aisle where these are needed.

 No specific configuration

There are various views, which provides information about the current need and availability of packages in Multishuttle:

  • PAC_PAL_AISLE_WITH_MORE_NEED_V
    This view provides the aisles, where a specific package is needed. Thereby it considered the transports in the aisles. This view is used at the putaway step and determine all groups where the current package is needed. The concrete aisle where it is finally stored is selected by putaway. Thereby the aisle close to the current position is selected.
  • PAC_PAL_MORE_NEED_V
    This view provides a list of packages, which are still needed in any aisles. Thereby the transports to the area MS (includes also the transports to specific aisles) are considered as well.
  • PAC_PAL_MORE_AVAILABLE_V
    This view provides all packages, which are stored in Multishuttle but not needed by any order.

AMCAPInventoryBalancingDemon use the last two mentioned views and created inter aisle transfers based on the data provided by these views.

      
Workflow

Step

 Description
1 Find out which packages are still missing in specific aisles, e.g. have more need.
2 If there is no more need, stop immediately.
3 Otherwise find out which packages are free and can be used for entries found in step 1, e.g. are more available.
4 If there is no more available, stop immediately.
5 If there are more need and more available, then create transports from more available aisles to more need aisles.
