Receiving

Purpose

This screen provides all functions for quantity based receiving in automated or manual Areas ordered to given Advices. Receiving can be processed in single steps according to multiple Advices and into multiple inserts of one or several Load Units. To use existing Load Units for adding newly received Inventory, these Load Units can be received from the warehouse to the current Workstation. It is also possible to print EAN-Labels.

      
Functions
  • Quantity based receiving in automated Areas ordered to given and chosen Advice
  • Print EAN-Labels during Receiving process
  • Initiate Outbound Transport Requests of Load Units for additional deposit of received Inventory on Load Units stored yet
  • Final storage of Load Units of goods receipt.

For a detailed description see, "Quantity based Receiving".

      
Overview

Frame

Description

Advice

Main data of scanned/entered Barcode or Advice.

Receiving

All data for current Receiving step as remaining quantity, quantity of receipt Inventory, Load Unit and Destination Area for subsequent Putaway.

Load Unit View

Destination Load Unit (with or without inserts). Insert for current deposit is marked in red. Empty inserts are green. Yet occupied inserts are yellow. Only possible to choose empty inserts.

 INFO 

Inserts are geometrically stacked Load Units as multiple partitions on the Base Load Unit. Colors are described in Brief Description of Load Unit Management

Actions

All functions for current step.
     
Screen

      
Fields

Field

Description

Advice

See Detail Dialog Advice and Detail Dialog Advice Line.

Barcode

Barcode can be scanned or entered. Product code, best before date and batch are read from the barcode. Product code is used to get the SKU Quantity Unit. In conjunction ,the SKU and Advice can be affected.

 INFO 

If multiple SKU Quantity Units exist, the product affects the SKU Quantity Unit to be used.

 INFO 

 Must comply with GS1 (Global Standard 1) standards. Values are project specific. For example, Product Code type could be included as GTIN-13.

Advice

Directly enter an Advice and Advice Line ID or select from a list.

Advice Line

Directly enter an Advice Line ID or select from a list.

 INFO 

If only one Advice Line for the current Advice exists, this will load automatically.
Receiving

Quantity Remaining

Currently outstanding quantity of Advice Line.

Unit

Quantity Unit of Advice Line.

Quantity Receiving

Quantity to be deposited with current Receiving step.

Quantity Target (Total)

Total quantity of Advice Line.

Number of Load Units

Number of already existing Advice Line Load Units.

Destination

Desired Destination Area for subsequent Inbound transport (Putaway) of received Destination Load Unit. Initialized with the accordant value of the Advice of the displayed Advice Line and can be changed after Load Unit entry regarding reachable destinations of current Workstation due to entered Load Unit Type.

Load Unit

Entered (scanned) Load Unit for deposit of received Inventory.

Load Unit Type

Must be chosen from a list if the Load Unit doesn't exist yet. Specific selection screen appears automatically.
     
Special Buttons

The operational flow in principle is described in use case "Quantity based Receiving".

Button

Description
Key: Enter

Confirm current screen state, e.g. loading of Advice data, Advice Line data and Load Unit. In Modify Data state, modified data is saved.

Key: Escape

Cancel current screen state, e.g. loaded data is deleted and previous state re-displays.

Key: F2

Toggle the mode for Scan between Advice and Barcode.

Key: F3

Only Advices in released and active statuses display. The selected Advice loads automatically and provides selection list for the Advice Lines subsequently.

Key: F2

Current Advice Line will be confirmed with entered Quantity. Load Unit stays on dedicated Location for further Receiving process.

Key: F5

SKU Labels are printed. Opens the Print Labels dialog.

 INFO 

Enabled only if the Product Code of the SKU is available.

Key: F6

For additional deposit on yet stored Load Units. Load Unit selection is restricted with the following constraints:

  • Load Unit is stored.
  • Reservation Status of the Base Load Unit is free.
  • Lock Status of the Location and the Aisle is available.
  • Load Unit is empty (or at least one Insert on the Load Unit).
With multi-selection, any entry can be selected. All selected Load Units directly get an Operator Retrieval Order to the current Workstation by clicking "OK".

Key: F7

For Return Advices only if deposit the same inventory (SKU and Batch) on Load Units stored yet is desired. Load Unit selection is restricted:

  • Load Unit is stored.
  • Reservation Status of the Base Load Unit is free.
  • Lock Status of the Location and the Aisle is available.
  • Load Unit contains the same SKU and batch (of current Advice Line).
By clicking "OK", a new Advice Line Load Unit in status pending is created (for the complete remaining quantity). With this step, receiving will only be allowed into this new assigned Load Unit.

Key: F9

Change screen to "Change Quantity" state; allows to change the receiving quantity.

 INFO 

Button is only available when no Advice Line Load Unit is assigned to the current Advice Line. 

Key: F4

Advice Line data (displayed in upper frame "Advice Data") is released for changing. Changeable fields are:

  • Batch
  • Best Before Date
  • Destination.

Key: F9

Change Destination Location for subsequent inbound transport (Putaway) of received Destination Load Unit after a Load Unit was entered (scanned). The following Locations display:

  • Default Destination of the Advice.
  • All reachable destinations (Area) of current Workstation due to entered Load Unit Type.

Key: F11

Change best before date. Only possible using this date/time chooser.

Key: F12

Cancel process and go to initial state.
