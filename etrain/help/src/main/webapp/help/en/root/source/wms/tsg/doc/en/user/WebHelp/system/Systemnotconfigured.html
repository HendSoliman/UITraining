System not configured

Symptom

Error messages on server side log.

ServiceNotConnectedException, InvalidConfigurationException, ServiceNotNeededException

      
Brief description

The system configuration (trader) ensures that provided and requested services are assigned to each other. This means that no service in the system can be called without an initial trader run. Whenever provided or requested services change the trader has to be run again.

      
Possible reasons

In case of a failure, the following reasons are possible:

  • new components deployed for the first time
  • the server having adopted versions with extended service configuration
      
Detail description

The task of the trader is to create or update the system configuration and to detect possible inconsistencies.

A component provides services that it implements (and events that it fires) through its component interface. Furthermore, it declares services that it requests itself (and events that it is interested in) at this interface. Using this information the system configuration is capable of initializing the system explicitly according to the requirements.

Some example error messages:

  • ServiceNotConnectedException
    There is no provider configured for service 'XYZ'.
  • InvalidConfigurationException
    There is a problem due to system components.
  • ServiceNotNeededException
    Component 'XYZ' is not configured to use service 'ABC'.
      
Trouble shooting

There are different possibilities to perform the system configuration of the system:

  • using the GUI
  • modify some database table

Step

Action - using the GUI

1

Open the dialog 'System Configuration | Requested Services' or 'System Configuration | Provided Services'

2

Press the button Configure System

3

Choose an option for configuration

  • Initialization only
  • Configuration only
  • Completely

4

Press the OK button to start the configuration

Step

Action - modify specific database tables

1

Shutdown the application server if still running.

2

Open sqldeveloper.

3

Truncate the following three tables:

- PROVIDEDSERVICE

- REQUESTEDSERVICE

- RECENTSERVICECONFIG

4

Restart the application server.
