Overview

Person-to-Goods Picking

Brief Description

Picking in Person-to-Goods Pick Zones occurs using the screen RF-Picking which provides all functionality for Picking purposes with Person-to-Goods picking strategies. Single and multi-Order Picking are supported. For multi-Order Picking, it is assumed that Pick Containers are transported on a special Pick Cart.

The Pick operation with all available confirmation codes, Reverse Picking execution, and Pick operation splitting are supported by detailed instructions and information by separate screen states for each single operational step.

Use Case

Description

New Pick Container

If additional Pick Containers are required to process the current Pick Walk, this must be performed to supply all needed Pick Container. This is the standard opening when the RF-Picking screen starts.

Pick Operation

A Source Location must be scanned to indicate the right location for picking after arriving at the next required Pick Face. The subsequent pick operation includes the standard case for single pick confirmation:

  • without any variation
  • with handling of inventory shortage
  • with further split Pick operation
  • with advised overdelivery
  • with blocked Pick Face
  • with Reverse Picking functionality.

Perform Zero-Cross Check

This must be performed if Zero-Cross or Nearly Zero-Cross Check of Source Load Units Inventory Unit is required because the remaining quantity fell below configured limit of SKU.

Drop Pick Container

This must be performed if all Pick Containers of a current Pick Walk finished due to finishing the last Pick of a related Pick Walk.
