Destination Load Unit

Unique ID of the destination Load Unit. This is constructed with following rule: <sourceId><depalSequence>. The depalSequence is a parameter of the Load Unit and here the Load Unit object of the source pallet is used. This parameter is incremented at every arrival of the source pallet to any De-Palletizer station (automatic or manual).

Extra

Extra Function:

  • .. (Normal Pick: defines the picking of a known item type)
  • TP (Test Pick: the operator defines the pick program for a new and unknown item type). The De-Palletizer tries to de-palletize the first layer with the given depalletizer program. A explicit confirmation from user is required to complete this process. The default depalletizer program is set during Teach-In.

Empty Pallet

If the Mission is for an empty pallet the appropriate default values will be set automatically, e.g. if LoadUnit.empty is true.

Process Details

Process Status

Status of the Mission:

  • active (available for processing)
  • send (in process and already transmitted to the De-Palletizer)
  • finished (De-Palletizer robot completed the processing)
  • rejected (Mission cant be processed due to various possible errors).

Event Code

Event Code (Status confirmation by the De-Palletizer robot):

  • .. (DEMI accepted)
  • OK (OK)
  • TE (Unexpected empty; The confirmation in LMS with this code will automatically cancel the replenishment order and send the pallet to clarification)
  • TR (Job cancelled; The confirmation in LMS with this code will automatically cancel the replenishment order and send the pallet to clarification)
  • DO (Robot stopped; The confirmation to LMS is to do manually. LMS will send the pallet to another available De-Palletizer station. If no other station is available the pallet will be stored.)
  • DH (Height exceeded; The confirmation to LMS for this code is to do manually. LMS will cancel the replenishment order and send the pallet to clarification)
  • DI (Data invalid source; The confirmation in LMS with this code will automatically cancel the replenishment order and send the pallet to clarification)
  • DD (Data invalid destination; The confirmation in LMS with this code will automatically cancel the replenishment order and send the pallet to clarification)
  • DP (Data invalid destination position; The confirmation in LMS with this code will automatically cancel the replenishment order and send the pallet to clarification)
  • PG (Data invalid pick program; The confirmation in LMS with this code will automatically cancel the replenishment order and send the pallet to clarification)
  • PL (Data invalid layers to pick; The confirmation in LMS with this code will automatically cancel the replenishment order and send the pallet to clarification)
  • LH (Data invalid low height; The confirmation in LMS with this code will automatically cancel the replenishment order and send the pallet to clarification)
  • HH (Data invalid high height; The confirmation in LMS with this code will automatically cancel the replenishment order and send the pallet to clarification)
  • PP (Pallet position off; No automatic confirmation to LMS. Correction on PLC side needed.)
  • DE (Destination position empty; No automatic confirmation. Just wait until the position is available.)
  • PI (Picking in progress; Nothing is done in this case)
  • NA (De-Palletize manually; In this case the pallet is send to a manual de-palletizing station)

Priority

The priority of the Mission defines the first processing order criteria. The urgency of the Pick Mission is used here.

Depalletizer Subsystem ID

Unique ID of the De-Palletizer subsystem (set automatically).

Fifo No

The Fifo number defines the second processing criteria.

Fifo Date

The Fifo Date defines the third processing order criteria.

Pick Configuration

Layers

Layers On Pallet

Total number of layers on the pick/source pallet before the picking operation starts.

This is determined with the configured SKULUCapacity and the available Inventory Units of the source Load Unit.

Items Per Layer

Number of items in each of the pallet layers.

Determined from the SKULUCapacity.

Layers To Pick

Number of layers to be picked.

Depended on the quantity of the current Pick for which this DEMI is created.

Layers picked

Number of already picked layers.

This value is always set to 0.

Pick Program

Pick Program One Layer

Pick program for a single layer at the LPF.

Depended on the type of the source load unit (pallet), the de-palletizing program is determined from the PalletizingQuantityUnit.

 INFO 

Because we have only one source location for de-palletizing, only this field is used. Pick Programm 2 to 4 are ignored.

Pick Program Two Layers

Pick program for simultaneously picking of 2 layers at the LPF-Robot.

Pick Program Three Layers

Pick program for simultaneously picking of 3 layers at the LPF-Robot.

Pick Program Four Layers

Pick program for simultaneously picking of 4 layers at the LPF-Robot.

Height

Pick Pallet Height

Total height of the pick pallet including the base pallet in mm.

The Load Unit height is used here.

Base Pallet Height

Height of the base pallet in mm.

The height from the Load Unit Type.

After Handling Acceleration

After handling acceleration/deceleration in percentages (e.g. for the Stripper Plate, Descrambler etc.) (20-100%).
