RF-Picking

Purpose

The RF-Picking dialog covers the user interface for operational picking purposes in Person-to-Goods Pick Zones with support of RF-Terminal.

The pick operation with all available confirmation codes and execution of reverse picking is supported by detailed instructions and information by separate dialog states for each single operational step.

 INFO 

General guiding and basic introduction for Pick Controller (PIC) is given in "Overview Pick Controller (PIC)".
     
Main Functions

The RF-Picking dialog is used to perform the following tasks:

  • Picking in a Person-to-Goods Picking Zone with support of RF-Terminal with Web User Interface
  • And in more detail:
  • Assignment of Pick Container to related Pick Mission of current Pick Walk and positioning on Pick Cart
  • Pick operation with confirmation without any variation, handling of inventory shortage, reverse picking, advised overdelivery and blocked pick faces
  • Delivery of all Pick Containers at once at the end of the Pick Walk
  • Perform nearly zero-cross check
  • Information screen with main data of current pick operation as Pick Walk, Source Location, Source Load Unit, Client, SKU with description, batch, best before date, pick quantity, Pick Container with correspondent Location Alias, Customer, required Service, Destination ,free text of related Order Line, Number of finished picks, Number of outstanding picks and the map between put location of pick cart and pick container.
      
Main dialog overview

Screen

Description

Pick Info

This dialog state provides all relevant data of the current Pick operation for information purposes and can be called any time within the other dialog states.

Put Pick Container

Appears in case of further Pick Container is required for processing the current Pick Walk. The Pick Container is assigned to the next free Put Location on operators Pick Cart. In this case the Put Locations are ascending numbered starting from 1 and not especially identifiable. In case scanning of Put Location is required a further input field for especially identifiable location data is provided.

Scan Source Location

Arriving at the next required Pick Face the Source Location has to be scanned to verify the Source Location for Picking.

Picking

Appears after scan of Source Location and provides the relevant main instructions of the current Pick operation. This function is the standard procedure with use of this dialog. The requested Pick operation is confirmed without any variation regarding the displayed data.
In case scanning of product label is required further input field for label data is provided.

Full

appears after this function was selected by operator from dialog state "Picking".
It provides the functionality to confirm that not all of required quantity could be picked into the current Pick Container.

Short

Appears after this function was selected by operator from dialog state "Picking".
It provides the functionality to confirm inventory shortage due to several configured shortage reasons.

Reverse

Appears after this function was selected by operator from dialog state "Picking".
It provides the functionality to confirm the operation of Reverse Picking with exchange of Source Load Unit and Pick Container.

Over

Appears after this function was selected by operator from dialog state "Picking".
It provides the functionality to confirm advised Picking of more than requested quantity without violation of overdelivery tolerance.

Skip

Appears after this function was selected by operator from dialog state "Picking".
It provides the functionality to confirm skipping the current Pick and append it to the end of current Pick Walk due to temporarily blocked Pick Face.

Blocked

Appears after this function was selected by operator from dialog state "Picking".
It provides the functionality to confirm skipping the current Pick without appending it to the end of current Pick Walk due to Pick Face is permanently blocked.

Verify Pick Container

If scan of Pick Container is required to indicate the use of correct Pick Container this dialog state appears after confirmation of single pick operation. Put location scan required can be configured in the related Picking Zone Configuration.

Zero Cross Check

Appears in case of zero-cross or nearly zero-cross check of Source Load Units Inventory Unit is required because remaining quantity falls below configured limit of Stock Keeping Unit's Cycle Counting Threshold and zero cross check is allowed in pick zone config.

Drop Pick Container

Appears in case of all Pick Containers of current Pick Walk have finished because of finishing the last Pick of related Pick Mission.

Next Pick Walk

After finishing a Pick Walk the start of the next Pick Walk could be confirmed or cancelled explicitly with this dialog state.
