Restacking

Purpose

The Restacking screen supports spontaneous stacking of whole Load Units onto other Load Units and De-Stacking. After a Source Load Unit scan, the appropriate data displays. With a Destination Load Unit scan, the destination of subsequent booking is defined. If one of these two Load Units contained Inserts or further stacked Load Units, it is possible to deal with a single Insert/Load Unit. The selected Source Load Unit is booked onto the selected Destination Load Unit or directly on the Workstation's Location with possible stacking of the whole Load Unit stack at once.

      
Screen

      
Fields

Field

Description

Source

Source Load Unit

Scan field for the Source Load Unit. Left section displays Source Load Unit data.

Stacked Load Unit

List of all stacked Load Units or Inserts of scanned Source Load Unit. Selection of a Load Unit to be (de)stacked can be done from of this list or from the Load Unit View. Multi-Selection is only possible if no position must be entered for the Destination Load Unit (see Field Position).

 INFO 

If no further stacked Load Units or Inserts are available on Source the Source Load Unit itself is listed.

Load Unit View

Graphical view of Inserts or stacked Load Units on Source Load Unit only displays if the related Load Unit Type is compartmentalized, i.e. configured with a layout for geometrical Stacking (LU type with grid sizes greater than 0). Load Units are depicted as follows:

- white: not yet stacked LU on raster

- red: LU is selected

- green: free and not selected

- yellow: occupied and not selected.

Target (Destination)

Target Load Unit

Scan field for the Destination Load Unit. Middle section displays Destination Load Unit data.

Load Unit Type

Enter/Select from drop down list the Load Unit Type of current Destination Load Unit.

Position

Enter position of a new Insert to be stacked onto Destination Load Unit only if Load Unit Type of Destination Load Unit is compartmentalized, i.e. configured with a layout for geometrical Stacking (LU type with grid sizes greater than 0).

 INFO 

Before stacking the dialog validates the entered position, e.g. overlapping of already available inserts are avoided.

If the operator don't provide the position (or it is set to 0), the next free position will be used.

Load Unit View

Graphical view of Inserts or stacked Load Units on Destination Load Unit is only displays if the related Load Unit Type is compartmentalized, i.e. configured with a layout for geometrical Stacking (LU type with grid sizes greater than 0). Load Units are depicted as follows:

- white: not yet stacked LU on raster

- red: LU is selected

- green: free and not selected

- yellow: occupied and not selected
     
Special Buttons

Button

Description

Source

F2: Finish

Remove Source Load Unit from the display. A new Source Load Unit can be scanned for the next Stack action.

 INFO 

The Load Unit gets an inbound transport automatically if required depending on configuration topic "itl-unpacking:UnpackingWorkstation.automatedPutaway".

Target

F3: New

Create a new Destination Load Unit of selected Load Unit Type. Only activated if Load Unit Type is selected from list.

The Load Unit is created on Workstation's Location immediately.

Target

F4: Print

Print a Load Unit Label for the Destination Load Unit.

Target

F5: Finish

Remove Destination Load Unit from the display. A new Destination Load Unit can be scanned for the next Stack action.

 INFO 

The Load Unit gets an inbound transport automatically if required depending on configuration topic "itl-unpacking:UnpackingWorkstation.automatedPutaway".

Load Unit based

F6: Stack

Stack and book the selected Load Unit(s) in the displayed list onto the Destination Load Unit.

 INFO 

All Load Units that are stacked on the selected Load Unit will be booked automatically as well.

Load Unit based

F7: Destack

Destack and book the selected Load Unit(s) in the displayed list directly on the Workstation's Location.
