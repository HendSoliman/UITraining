Depalletizer Missions

Purpose

A De-Palletizer Mission (DEMI) represents the single de-palletizing tasks of a pallet at an automatically operated Depalletizer Station.

The De-Palletizing process to any De-Palletizer is done with a specific DCI specification. For the details on the De-Palletizing process and the DCI specification see De-Palletizing.

This dialog shows all missions which are either already sent to the De-Palletizers or which are ready to send. Further the rejected are available for analyzing, correction and re-send. In case of a failure and manual correction the quantity for each layer can be adjusted in the detail dialog.

The De-Palletizing process allows the sending of only one DEMI at a time to one De-Palletizer. In the AMCAP process the DEMIs are created as soon the source pallet has arrived at the De-Palletizer (created with process status 'active'). A background process looks for each De-Palletizer at active DEMIs and send one of them to the De-Palletizer. As soon the sent DEMI is processed it will be removed and the next active DEMI is send to the De-Palletizer.

On the LMS side the De-Palletizing process is represented with Picks. Thereby each Pick Mission represents the source pallet to be de-palletize and for different SKUs on the pallet one Pick is created (e.g. for rainbow pallets). The Picks are sorted, e.g. the sequence is strict in layers order. For homogenous pallets only one Pick per Pick Mission is created.
A DEMI is created for exact one Pick. As mentioned above with the arrival of the pallet at the De-Palletizer station the first DEMI with the first Pick (sequence=0) is created. With the confirmation of the DEMI the Pick is booked and if it is finished and there is another Pick within the current Pick Mission, a new DEMI for this is created.

Special Buttons

Button

Description

Opens a new screen to create a new De-Palletizer Mission with process status 'active'. The background process will send it to the De-Palletizer if no other DEMI is status send exists.

Deletes the selected Mission and sends a message to the De-Palletizer for cancellation.

Resend the selected Mission. Thereby the status is set to active and the background process send it once again to the De-Palletizer.

This function is to use, when the De-Palletizer finished the de-palletizing but the confirmation from De-Palletizer was missing. Thereby the selected DEMI is removed without any confirmation to the De-Palletizer. But a confirmation to the overlaying system (e.g. LMS) is initiated. Hereby the current de-palletized quantity from the DEMI is used for informing the overlaying system.
