Book Breakage

Purpose

An operator can book Inventory as damaged by indicating a reason to the damaged quantity. The quantity is expressed in the same SKU Quantity Unit as used in the Confirm step. The quantity is predefined with the remaining total quantity of the Source Load Unit and can be changed.
Upon confirmation, the specified quantity is booked accordingly.

 INFO 

 Inventory Movement Reasons are defined by assignments to the Inventory Movement Type defined in the configuration topic "cor-inventory:InventoryUnit.inventoryMovementTypeBreakage".
     
Screen

      
Fields

Field

Description

Quantity Damaged

Damaged quantity on the Source Load Unit. Predefined with the currently remaining quantity on the Source Load Unit.

Inventory Movement Reason

Inventory Movement Reason ID. Confirmed with the configured Inventory Movement Type with this booking.
     
Special Buttons

Button

Description

Related key: "ENTER"

Book specified quantity with the given reason, damaged.

Related key: "ESC"

Return to Confirm step without any bookings.
