Configuration Parameters - Transport Request

Purpose

This topic describes the configuration parameters of Transport Request.
Located in demo\target\configuration\cor-transportrequest.xml.

      
TRQ - Transport Request

Key

Type

Dimension

Default

Description

TransportRequest.
MaxAisleInternalRelocations

int

Area (prefix “AREA:”)

2

This parameter defines the maximal number of Aisle internal relocations.

TransportRequest.
ShuffleMoveOnSequenceViolation

boolean

 

false

This parameter defines if a shuffle move should be done if Load Units have to be retrieved from the front and the rear positions of a bin and the transports belong to the same Transport Group, but the sequence number of the transport of the rear position is lower. The default behavior is that the MFC corrects the sequence after retrieval by overtaking.

TransportRequest.LoadUnitForwardPolicies

String

 

com.dematic.wms.cor.depalletizing.control.DepalletizingLoadUnitForwardPolicy

This parameter contains a comma separated list of class names of LoadUnitForwardPolicy implementations. The policies are executed in the order defined by the comma separated list for a Load Unit to be forwarded. When all policies return null the Load Unit is stored.
