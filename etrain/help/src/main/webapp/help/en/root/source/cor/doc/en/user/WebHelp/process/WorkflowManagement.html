Workflow Management

Overview

Workflow Management is a basic and important component within the Logistics Manager LMS. It provides basic components which are responsible for a common management of workflow in a warehouse. In a warehouse a lot of several requests exist - e.g. Advices for Receiving processes, Orders for different outbound processes like Full Load Unit Retrievals or Picking. As far as it is reasonable these all are handled in a unified way as Tasks. The Workflow Management allows to define dependencies between the Tasks and the overall control management of the task execution.

A Task is an entity which allows supporting a unified workflow of fulfillment tasks (Transport Requests, Advises, Orders, Pick Orders, etc.). Tasks can be associated with resources. Resources can be of any type, e.g. human or technical. Optionally a task can be bound to a Facility or an Area. Since Tasks can have children, it is possible to build hierarchies of Tasks (e.g. Order to Pick Order to Transport Request). Furthermore it is possible to define chains of dependent Tasks. So the start of a Task can depend on the start of Tasks this Task depends on.

Tasks can be released manually, automatically, triggered, or dependent on previous tasks in a chain. Triggered Tasks are released by a mechanism specific to a task type, e.g. by a demon.

A Worker is a special resource which represents an employee. In order to operate with the system a User must always be assigned to a Worker who is member of one Worker Group which defines the type of allowed workload.
