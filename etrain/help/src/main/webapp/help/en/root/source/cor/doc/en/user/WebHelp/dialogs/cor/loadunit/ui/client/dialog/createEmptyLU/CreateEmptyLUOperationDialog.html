Create Empty Load Units

Purpose

This dialog is used for the creation of empty Load Units with arbitrary Compartments (Inserts). The allowed stackable Load Unit Types (of the Compartment) for the scanned Load Unit are provided by the dialog as a list and the user can easily put these onto the Base Load Unit using Drag & Drop functionality. Further any empty compartment can be deleted within this dialog and the store functionality allows to initiate a putaway transport for the Base Load Unit.

 INFO 

The dialog displays only the Load Unit Types that are yet stackable on the Base Load Unit. Hence it is possible, that some types are not available although they are defined as allowed stackable Load Unit Types.
     
Functions
  • Create an empty Load Unit (with no inventory booked on)
  • Load any yet existing Load Unit
  • Stack Load Units (put compartments on the Base Load Unit) using
  • Drag & Drop functionality (for a single put)
  • Fill Up mechanism (for automatic multiple puts)
  • Delete any empty compartment (also the empty Base Load Unit can be deleted)
  • Store the Base Load Unit with all its compartments.
     
Overview

Frame

Description

Attributes

All relevant Base Load Unit related data is located in this frame. A scan field for the Base Load Unit and allowed Load Unit Types for creating compartments are main fields in this frame. Further the Load Unit Type of the Base Load Unit is shown in a separate field.

Load Unit Widget

The current structure of the Base Load Unit is shown in this frame.
The Base Load Unit respectively the Compartments will be depicted as follows:

  • white: not yet stacked Compartments on the raster of the Base Load Unit
  • red: Base Load Unit or Compartment is currently selected
  • green: occupied with a not selected empty Compartment which is still removable
  • yellow: occupied with a not empty (and not selected) Compartment (inventory inside!) and thus not removable.

Actions

Only the currently allowed functions are enabled in the appropriate state of the dialog operation. The other buttons are disabled.

 INFO 

Further details on Load Units are provided in the description of the Load Unit Detail Dialog.

Fields

Field

Description

ID

Scan or Input field for the (new) Base Load Unit. After the successful scan the Load Unit is loaded to the dialog and its structure is shown in the Load Unit Widget frame.

Load Unit Type

The Load Unit Type of the scanned Base Load Unit is displayed for information only.

Stackable Load Unit Types

All Load Unit Types that are allowed to be stacked on the scanned Base Load Unit are listed. The allowed types of the Compartments must be set in the "allowed Stack Load Unit Type" of the Base Load Unit Type. All Load Unit Types which are not any longer stackable on the Base Load Unit due to the already occupied raster will be removed from the list.

 INFO 

The Load Unit Types are displayed as a linked string of
"Load Unit Type Name" - "Raster Size x" - "Raster Size y".
     
Special Buttons

Button

Description

This function is only available in the initial state. It is used for loading any yet existing Load Unit or creating a new Load Unit. If a new Load Unit should be created the operator is asked for the used Load Unit Type. After selection a new Load Unit with the scanned ID, the selected Load Unit Type and the Location of the assigned workstation is created.

Each scanned but already existent Load Unit will be booked onto this location immediately.

 INFO 

If the dialog is not running at a specific workstation the new created Load Unit has no location set. Before the initiation of any transport operation the location must be set anyway.

This function removes the selected empty Base Load Unit or selected empty Compartment from the system. Only empty Load Units are allowed to be deleted. If the Base Load Unit is deleted the dialog proceeds with the initial state.

Puts the selected Load Unit Type onto the next possible free position of the Base Load Unit.

The Base Load Unit will be completely filled up with the selected Load Unit Type.

This function removes all empty Compartments from the Base Load Unit.

With this function a Load Unit Label is printed for the Base Load Unit.

Initiation of an Inbound Transport of the Base Load Unit into the Warehouse.

The dialog proceeds with the initial state of the dialog.

 INFO 

If the dialog is not running at a specific workstation the new created Load Unit has no location set. Before the initiation of any transport operation the location must be set anyway.

In case of storage of the Base Load Unit is not required this function can be used to go back to the initial state for the execution of another Base Load Unit.
