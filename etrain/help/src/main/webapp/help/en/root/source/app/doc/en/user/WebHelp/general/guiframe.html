Application Frame

Purpose

This chapter describes the structure and the individual elements of the Dematic iQ® Application (UI) Frame.

      
Overview

The following screen shot shows the major elements of the UI frame.

 

The UI Frame contains the following topics:

Topic

Description

Title bar

The title bar shows the name of the installation site and both the customer and Dematic Logo. It has no further functionality.

Toolbar

The toolbar encompass some general purpose buttons and dropdown menu.

Accordion menu

The accordion menu in combination with the tree view can be used to open new dialogs
Dialog area The dialog area encompasses all open dialogs in a MDI (Multiple Document Interface) metaphor.
Message area The message area displays all active alarms which are relevant for the current user and the terminal.
Status bar The status bar contains a feedback area and displays some general information of the main frame.

 

Sliders can be used to adjust the size of the accordion menu and the message area.

      
Toolbar

The toolbar contains the following elements from left to right:

Element

Description

Toggle button to show or hide the Message Area.

Hiding the message area increases the vertical space available for the UI frame. If the message area is hidden, all currently active alarm notifications are no longer visible. However, there is still a summarized view of the alarms in the Status Bar.

Toggle button to show or hide the Accordion Menu.

Hiding the Accordion Menu increases the horizontal space available for the UI frame. In this case, dialogs can still be opened efficiently by using the Dialog Search.
Settings menu

This dropdown menu allows to change properties of the UI frame and of general settings which affects all dialogs.

  • The "Language ..." submenu offers the possibility to change the language in which the UI is presented to the user online. In this context, online means without fetching new data from the server.
  • The "User ..." menu entry offers the possibility to change settings which apply to the currently logged on user. Click here for further details.
  • The "Desktop Layout ..." submenu allows to personalize the way the UI frame shows up after an initial log-on.
  • The "Show Title Bar" toggle menu can be used to show or hide the title bar.
  • The "Show Status Bar" toggle menu can be used to show or hide the Status Bar.
  • The "Full Screen" toggle menu shows or hides all frame elements beside the tool bar and the dialog area. Please be aware that, the browser decoration still remains visible. In order to get rid of the latter, one has to set the browser in full screen mode as well. In general, this is done by pressing F11 on the browser.
  • The "Show Online Help external" controls whether help pages shall be opened in a new window or as a new (internal) dialog. Please be aware, that it depends on the browser being used and the  browser-settings made by the user whether "new window" means really a new external window or just a new tab within the already existing browser window.
Window menu This drop down menu encompasses one entry for each open window in the order in which the dialogs have been opened
When selecting one menu entry, the corresponding dialog will be brought to the foreground. Doing this one can switch between dialogs.
In addition to that, one can use the "Close all" menu entry to close all active dialogs.
Help menu
  • Opens the online help system.
  • Shows the "About dialog" containing Copyright and versioning information.
Triggers an Object Search across the whole Dematic iQ® database.
Triggers a logout from the UI.

 

Besides the Dialog Area, the tool bar is the one and only element which cannot be hidden dynamically. All potentially hidden elements of the UI frame can be made visible again by using functionality of the toolbar.

 

      
Accordion Menu

The accordion menu contains the dialog tree view and a vertically stacked list of buttons below. Each accordion button consists of an icon and a label. Each button corresponds to a major functional area of the Dematic iQ®. The exact layout may slightly vary for a concrete Dematic iQ® installation.

Procedure:

Element

Description

1.

Click on the accordion button of the functional area the dialog belongs to. The corresponding dialog tree is opened in the tree view.

2.

Navigate to the dialog entry in the tree view.

3.

Open the dialog by clicking on the corresponding note. If the dialog is already opened in the Dialog Area the dialog is brought to the foreground (if it is not pinned).

A dialog has a pin status pinned () or not pinned (). This pin status of a dialog controls what happens if the user tries to open a dialog which has already an open dialog window:

  • If the dialog is not pinned (default), the dialog is just brought to the foreground and no additional dialog window is opened.
  • If the dialog is pinned, an new dialog window is opened.

 

A click on one of the labels opens a dialog tree in the tree view above the accordion menu. This dialog tree encompasses all dialogs which belong the selected functional area. Thus, the procedure to open a dialog is:

Step

Description

1.

Click on the accordion label of the functional area the dialog belongs to. The corresponding dialog tree is opened in the tree view.

2.

Navigate to the dialog entry in the tree view.

3.

Open the dialog by clicking on the corresponding note. If the dialog is already opened in the Dialog Area the dialog is brought to the foreground (if it is not pinned).

By using the context menu of a dialog node, a dialog can be added to the Favorites.

 

The accordion may be resized to grant more space the tree view. In this case, the buttons are are reduced to their icons at the bottom of the accordion. If there is not enough space to display all icons, a small drop down menu appears which gives access to no longer visible accordion entries.

The dialog search can be used if the exact position of a dialog in the accordion menu and the tree view is not known to the user. Dialog Area

The dialog area acts as a desktop for all open dialogs. The following actions can be taken on a dialog:

Action

Description

Drag of the title bar

The dialog is brought to the foreground and moved to another position on the dialog area. Dialogs cannot be moved outside the dialog area.

Maximize () or double click on title bar

The dialog is maximized to cover the complete dialog area.
Minimize () The dialog is minimized to an icon in the lower left corner of the dialog area.
Restore () or double click on title bar For a maximized or minimized dialog the original size of the dialog is restored.
Close () The dialog is closed.
Pin ( / ) This toggle button controls and visualizes the pin status of the dialog. Pinned dialogs get an additional number appended to the dialog title in order to make them distinguishable from non-pinned dialogs. Pinned dialogs cannot be un-pinned. They can only be closed.
Add to Favorites () The dialog is added to the Favorites list.

 

      
Message Area

The message area displays all active alarms which the current user and/or the terminal is subscribed for. Only a limited amount of active Alarms is displayed. The list is sorted by Alarm category, Creation date and Alarm Status, so the higher priority Alarms are listed from top down. See Alarm Monitor for a more detailed description of the alarming functionality and its configuration mechanisms.

The Alarm grid of the Message Area offers the following functionality:

Action

Description

Double click on message line

 Opens a sub dialog which displays detailed information on the selected alarm

Context menu of a message line

 Offers standard actions like acknowledgement on the selected alarm.

 

If available, the overview map of the visualization is displayed at the left hand side of the message area.

      
Status Bar

The status bar on bottom of the UI frame displays the following information (from left to right):

Element

Description

Feedback area

The feedback area shows positive completion messages originating from actions triggered by the UI, e.g. 'The user 'yxz' has been created successfully'. This form of feedback gives the user a decent information about the result of an action he has originated (without bothering him with popping-up message boxes).

Please be aware, that negative messages or fault indications which might require immediate user attention are never display as feedback in the status bar. Such type of messages are displayed in normal message boxes which must be explicitly confirmed by the user.

Alarm Summary

The alarm summary counts the number of alarms () and warnings ( )which are currently active for the user and terminal. This gives the user the possibility to get a summarized overview of the alarm status relevant for him even if the message area is temporarily hidden.

A click on one of the symbols opens the Alarm Monitor TODO link with the corresponding category as predefined filter. Test.

Server name ()

The server name the UI frame is connected to. This field can be used to distinguish between UI frames running in the same browser but originating from different servers (e.g. productive server and test server).
User name () Name of the user currently logged on to the UI frame.
Terminal () The terminal name the UI is running on. If not explicitly started with a terminal parameter, this field shows the hostname of the machine the browser is running on.
