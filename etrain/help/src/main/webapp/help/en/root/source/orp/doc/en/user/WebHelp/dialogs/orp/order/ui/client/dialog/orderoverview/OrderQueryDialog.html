Purpose

This screen shows Order Lines and the most important data from their Order as well.

Description

Load Unit ID Unique ID of the requested Load Unit.
Location Unique ID of the requested Location. Can be a virtual location representing the facility, an area, aisle or level, or a concrete bin.
Order Status

Current execution state of the Order:

  • created (Order was created and not yet ready for pending, but ready for preparation phase (project specific use))
  • prepared (Order preparation phase is finished, but not ready for pending (project specific use))
  • pending (Order is awaiting release for fulfillment)
  • released (Order is released for fulfillment)
  • active (Order execution already started; Full Load Unit Retrievals and/or Pick Orders are in progress)
  • complete (Order execution is finished and Order is fully completed; all Order Lines are completed or cancelled)
  • to confirm (Order is ready for confirmation message to the host system)
  • cancelled (The Order is successfully canceled and waiting for confirmation to be sent to host)
  • finished (All required tasks for the Order are finished. If required, Order was confirmed to the host).
Order Line Status

Current execution state of the Order Line:

  • pending (Order is awaiting release for fulfillment and all Order Lines proceed to this status)
  • released (Order is released for fulfillment and all Order Lines proceed to this status)
  • active (Order execution already started; Full Load Unit Retrievals and/or Pick Orders are in progress and all Order Lines proceed to this status)
  • cancelled (single) Order Line was cancelled without the defined execution)
  • complete (single Order Line execution is finished and the Order Line is fully completed).
Order ID

Only unique within a Client.
Order Type

Controls principle order planning and fulfillment processes. See Order Types in Order Management Overview for a listing.
Order Line ID

The identification of the Order Line is unique within an Order and is usually represented by a running number.
Batch If the required inventory must be of this specific Batch. If empty, it is not considered, i.e. can be any with respect to the Retrieval Strategies.
Best Before Date If the required inventory must of this specific Best Before Date. If empty, it is not considered (or by FiFo), i.e. could be any with respect to the Retrieval Strategies.
Criteria Used Required objects to retrieve with this Order Line. With Order Release, Retrieval Strategies are used to allocate the optimal required Inventory for the given Order Line criteria.
  • Inventory Criteria
  • Load Unit Criteria
  • Location Criteria.
Wave Groups Orders for collective disposition and execution for picking and shipping purposes.
Route Order could be grouped to routes, which define the same transport for collective shipping. The Route is a simple grouping property without its own features.

Client

Owner of the Order. Thus, owner of the Inventory to allocate with this Order.
External Order Type Order types used by the customer.
Service Qualifier

Influences concrete fulfillment of the Order within specific order planning or execution processes. May be project specific. Can limit the list of available service qualifiers using the configuration topic "orp-order:OrderOrderType.allowedServiceQualifiers".
